在区块链 TechCrunch 上建立社交媒体平台的竞赛

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/25/the-race-to-build-a-social-media-platform-on-the-blockchain/

人们很容易对大型社交媒体巨头持怀疑态度。他们通常向广告商出售用户数据，并因其内容审核方法而陷入争议。Web3 的支持者认为他们可以建立一个新的系统，完全避开平台的集中力量——一个分散的社交媒体(DeSo)生态系统，在这个系统中，用户理想地对他们创建的内容拥有更多所有权。

建立在 Dfinity 的互联网计算机协议基础上的区块链社交网络 DSCVR 已经加入了建立可扩展 DeSo 平台的竞赛，由 Polychain Capital 领投了 900 万美元的种子资金。据该公司称，这轮融资的其他参与者包括advanced Ventures、Tomahawk VC、Fyrfly Venture Partners、Shima Capital 和贝塔斯曼数字媒体投资公司(BDMI)。

这是一个竞争激烈的领域，许多初创公司和大公司竞相建立一个为用户提供公用设施的网络。本月早些时候，前比特币基地员工丹·罗梅罗(Dan Romero)获得了由 a16z 牵头的 3000 万美元，用于开发 Farcaster ，这是一个 DeSo 协议，允许用户在不同的应用程序之间移动他们的社交身份。TechCrunch 报道了另一家种子阶段的初创公司 Primitives，该公司在 5 月份为其自己的 Solana DeSo 网络筹集了 400 万美元。大型科技公司也在这场游戏中——Twitter 资助了其服务的一个分支，名为 BlueSky ，这是一个开源的 DeSo 项目，成立于 2019 年，尚未上线，但正在公开试验其开发过程。

就 DSCVR 而言，它拥有称为“门户”的令牌控制社区，用户必须拥有一定的 NFT 才能参与其中。联合创始人兼首席执行官里克·波特在接受 TechCrunch 采访时表示，这些社区通过向 DSCVR 用户空投数字资产来帮助 NFT 项目进行宣传。

“人们需要为他们的 NFT 分配。他们希望为他们的非功能性金融工具实现流动性，他们需要真实的人，而不是机器人，来拥有这些非功能性金融工具，并有可能由我们自己来持有它们。所以我们成为了 NFTs 的分销渠道，这样(寻求分销的人)就有了 10，000 个实体收藏，他们可能会向我们的顶级用户投放 1，000 个，”波特说。

DSCVR 上的主画面图像图像来源: DSCVR

根据波特的说法，DSCVR 已经拥有超过 10 万名用户，并通过空投奖励机制为这些用户创造了“数百万美元”的 NFT 奖励。波特说，它与其他 DeSo 平台的区别在于它的目标是在 Web 2.0 和 web3 之间建立一座桥梁。

“我们如何让一个可能不习惯 web3 的典型 Web 2.0 用户，向他们介绍 web3 技术，并在 web3 中给他们绝对真实和真正的本地体验，而不做出重大牺牲来弥合这一差距？波特说:“这是我们正在寻找的差异之一，我们试图增加 web3 体验的视角或保真度。”。

为此，波特指出，通过 MetaMask 或其他提供商将外部钱包链接到社交档案可能会很复杂，尤其是对那些不太熟悉 web3 技术的人来说。DSCVR 的答案在于其原生钱包，波特表示，该平台上的用户可以使用该钱包，并且不需要他们交出对自己钥匙的控制权。

他补充说，DSCVR 试图回答的问题是，如何以“很少甚至没有牺牲”的方式给用户一个 web3 原生的、分散的体验有趣的是，该公司选择建立在互联网计算机协议上，而不是最流行的智能合同链以太坊。波特说，他是在与 Dfinity 的总裁兼首席科学家 Dominic Williams 会面后做出这一决定的，当时波特正在波士顿咨询公司担任顾问，从事一个位于区块链的智能合同项目。

在参与了 Dfinity 的协议开发后，几年后，Porter 决定离开他在 BCG 的职位，开始全职为 Dfinity 工作。他的第一个项目是建立一个关于该协议的留言板，他说这让他了解到对 web3 社区的需求，并激励他决定建立一个社交网络。

互联网计算机生态系统也有自己的争议。Dfinity 正面临一场集体诉讼,指控其将代币作为未注册证券非法出售。该诉讼还将 Polychain 和 a16z 列为被告，因为这两家公司都是 Dfinity 的早期支持者。上个月，Dfinity 发现自己站在了等式的另一边，它对《纽约时报》的两名记者 T2·安德鲁·罗斯·索尔金和 T4·伊弗拉特·利夫尼提起诉讼，指控他们诽谤 Dfinity。

当被问及为什么波特认为互联网计算机是建立他的社交平台的最佳选择时，他说:“感觉我是在 AWS 之类的东西上开发，但它是完全去中心化的，对吗？我有可升级的智能合同，所以我可以实际进行更改并逐步升级我的应用程序，同时仍然让一切都得到验证、在线和可跟踪。”

不过，波特说，在构建 DeSo 平台时，直接与客户沟通是必要的。

“难的是在 web3 中，你得不到很多关于用户的分析。你必须真正与你的用户群交流，真正了解他们面临的问题是什么？又有哪些摩擦点？如果一些用户面临问题，我们不会收到警报，”波特说。“我们希望，要么他们在我们的网站上发帖，要么他们有朋友在我们的网站上发帖，告诉我们他们面临的问题，然后我们通常可以跳上去解决这些问题。”
