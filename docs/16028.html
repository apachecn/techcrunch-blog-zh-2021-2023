据报道，苹果正在试验语言生成人工智能

如果不是上周硅谷银行(SVB)倒闭，科技领域的几乎所有对话似乎都围绕着人工智能和聊天机器人。在过去的几天里，微软支持的 OpenAI 发布了一个新的语言模型，叫做 GPT-4 。其竞争对手 Anthropic 发布了 Claude 聊天机器人。谷歌表示，它正在将人工智能集成到其工作空间工具中，如 Gmail 和 Docs。微软必应凭借聊天机器人功能的搜索引起了人们的注意。行动中少了一个名字？苹果。

上个月，总部位于库比蒂诺的公司举行了一次内部活动，重点是人工智能和大型语言模型。根据《纽约时报》的报道，包括 Siri 开发人员在内的许多团队正在定期测试“语言生成概念”。另外， 9to5Mac 报道称，苹果在 tvOS 16.4 中引入了一个新的“Siri 自然语言生成”框架。

人们抱怨 Siri 不理解查询(包括我的)已经很久了。Siri(以及 Alexa 和谷歌助手等其他助手)无法理解生活在世界不同地区的人的不同口音和语音，即使他们说的是同一种语言。

ChatGPT 和基于文本的搜索的新名声使人们更容易与不同的人工智能模型进行交互。但目前，与苹果人工智能助手 Siri 聊天的唯一方式是在辅助功能设置下启用一项功能。

在接受 NYT 采访时，曾为 Siri 工作的前苹果工程师约翰·伯克(John Burke)表示，苹果的助手由于“笨拙的代码”而发展缓慢，这使得即使是基本的功能更新也变得更加困难。他还提到 Siri 有一个包含大量单词的大数据库。因此，当工程师需要添加特征或短语时，必须重建数据库——据称这一过程需要长达六周的时间。

NYT 的报告没有具体说明苹果是在建立自己的语言模型，还是想要采用现有的模型。但就像谷歌和微软一样，蒂姆·库克领导的公司不想局限于提供 Siri 驱动的聊天机器人。由于苹果长期以来一直以自己是艺术家和创作者的盟友而自豪，它应该会希望将语言模型的进步应用到这些领域。

该公司已经使用人工智能功能有一段时间了，即使它们起初并不明显。这包括键盘上更好的建议、摄影中的处理、遮罩与 Face ID 解锁、跨系统将物体与背景分离、Apple Watch 上的洗手和碰撞检测，以及最近 Apple Music 上的卡拉 ok 功能。但是他们中没有一个会像聊天机器人那样露脸。

苹果对其人工智能努力一直保持沉默。但在一月份，该公司启动了一个项目，为作者提供人工智能驱动的旁白服务，将他们的书变成有声读物。这表明 iPhone 制造商已经在考虑生成性人工智能的用例。如果我们在几个月后的全球开发者大会(WWDC)上听到更多关于该公司在这些领域的努力，我不会感到惊讶。
