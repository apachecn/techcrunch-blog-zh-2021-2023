Hasura 的新 SDK 现在允许开发者将任何数据源转换成 GraphQL API

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/28/hasura-now-lets-developers-turn-any-data-source-into-a-graphql-api/amp/

图片来源:爱德华·穆热夫斯基/科学图片库/盖蒂图片社

自几年前发布以来，GraphQL 平台 Hasura 简化了将数据源连接到应用程序，然后通过 GraphQL 访问它们的过程，使得开发人员更容易使用 GraphQL。现在，在其 virtual HasuraCon '22 开发者大会上，Hasura 通过推出其 GraphQL 数据连接器(GDC)扩展了这一功能，这是一个新的 SDK，允许开发人员将几乎任何数据源引入 Hasura，以便将其作为 GraphQL API 公开。

值得注意的是，作为一个开源项目，Hasura 已经具备了其中的一些功能，但正如该公司的首席执行官兼联合创始人 Tanmai Gopal 所指出的那样，新的 SDK 现在使这一点在网络上变得更加容易访问和可用——开发者也可以使用他们选择的语言来编写这些代理。

图片来源: Hasura

“对于企业中的人们来说，这变得非常有价值，因为有时他们拥有所有这些任务关键型数据和这些遗留资源，但没有人再对其进行创新，”Gopal 说。“或者，您希望使用这些即将到来的利基数据源或特殊的工作负载，这些工作负载虽然还为时过早，但您已经准备好以关键的方式使用它们。现在，我们能做的是——与这些贡献者、数据库供应商、Hasura 团队本身一起——我们可以合作并创建这些代理，这样，我们就可以开始逐步开放对所有这些资源的访问。”

随着今天的发布，Hasura 还宣布了其他一些更新。例如，Hasura Cloud 和 Cloud Enterprise 现在可以在 Google Cloud 上使用。鉴于 Hasura 在 PostgreSQL 数据库方面的工作背景，Google Cloud 的新 AlloyDB 服务非常适合这里。

“组织正在迅速采用 GraphQL，我们很高兴能够帮助这些组织实现其应用程序的现代化和数字化转型。Google Cloud India 董事总经理比克拉姆·辛格·贝迪表示:“通过在 Google Cloud 上使用 Hasura 的 GraphQL 平台，世界各地的开发人员可以以更低的成本大规模构建下一代应用。

Hasura 的联合创始人 Rajoshi Ghosh 告诉我，虽然该公司两年前在 AWS 上推出了云服务，但该团队一直知道它必须在客户所在的地方。“因为你的数据在哪里，我们就在哪里，这就需要我们与所有的云提供商合作，”她说。“在测试项目期间，我们看到了谷歌的大量应用。[ … ]您的数据无处不在。它在成倍增长。它在成长。无论您是否知道，您的数据正在某个地方迅速增长。就像这样。因此，数据在不同的数据源和不同的云上迅速增长。然后，Hasura 成为最终的数据 API，去掉了基础设施层。”

有了这个，Hasura 现在支持 AWS 和 Google Cloud，尽管团队告诉我 Azure 支持也在工作中。该服务已经支持微软的 SQL Server，随着今天的发布，它正在向其平台添加事件触发支持。

同样新的是用于构建实时应用的流订阅 API，以及对 Hasura 云和企业中 OpenTelemetry Traces 的支持。
