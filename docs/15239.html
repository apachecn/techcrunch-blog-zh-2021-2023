web3 安全公司 Stelo Labs 在 Andreessen Horowitz 领导的种子轮融资中筹集了 600 万美元，该公司独家告诉 TechCrunch。

该公司由首席执行官 Ben Scharfstein 和首席技术官 Aman Dhesi 共同创建。两人都有产品管理背景——Scharfstein 曾在谷歌工作，Dhesi 曾在脸书、DoorDash 和 Square 任职。

Scharfstein 说，Stelo Labs 的目标是使 web3“对每个人来说都是安全和可理解的”。“当我们想到 web3 可以成为什么和能够实现什么时，如果事情不安全，它就会被切断。”

Dhesi 说，虽然有许多 web3 安全公司，但 Stelo Labs 专注于帮助防止基于 T4 以太坊的用户进行 T2 恶意交易、网络钓鱼和社会工程。“随着人们不断创新和创造新的方案，这些攻击的范围越来越大，人们可以通过更多的方式对您进行网络钓鱼。”

这家初创公司在 9 月份推出了 Stelo 扩展，以帮助用户保护他们的钱包免受诈骗和钓鱼攻击。开源扩展就像任何交易和用户钱包之间的防火墙，但永远无法访问个人的私有种子短语或密钥。

“我们意识到 web3 中的安全问题是产品问题、界面问题以及数据问题和安全问题，”Dhesi 说。“但在 web3 中，责任往往落在用户身上。

Dhesi 说:“没有中介机构在那里提供帮助。“这实际上是为用户打造体验和产品，让他们有安全感。”

沙夫斯坦说，自 9 月份以来，它已经保护了数千个钱包，总价值超过 1 亿美元。

Dhesi 说，该服务通过其引擎运行最流行的签名类型或交易协议，以帮助确定交易是低风险、中等风险还是高风险，并提供上下文，以便人们可以理解他们正在签署的内容，而不是告诉他们该做什么。

今天，它发布了其开发者 API，因此独立的 dApps 和钱包可以使用其 Stelo 交易引擎。联合创始人说，它还推出了令牌批准体验,通过向第三方建议撤销批准来帮助用户了解他们钱包的“健康分数”并保持其安全。

Scharfstein 说，这笔资金将用于扩大团队及其产品，使其超越谷歌 Chrome 扩展，并“在用户所在的地方与他们见面”。

随着时间的推移，Stelo 计划开发其 API 的嵌入式版本，并找到一种方法使以太坊交易“人类可读”，因为目前有许多细微差别，Dhesi 说。“我们不希望每个开发人员都重新发明轮子，所以我们将把 Stelo 用户界面打包到一个库中，这样他们就可以把它放进去，进行定制……这是我们将在未来几个月投资的事情。”

联合创始人说，从长远来看，Stelo 希望建立一个用户界面，通过其服务进行每一笔交易，无论是通过加密钱包、面向消费者的扩展还是其他途径。

Scharfstein 说，虽然这项服务目前专注于以太坊，但 Stelo 希望在未来将它带到更多兼容以太坊虚拟机( EVM )的连锁店。“我们希望确保每笔交易都经过我们的风险延伸。”
