微软终于修复了 Windows 零日漏洞

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/15/follina-windows-zero-day/?&web_view=true

微软终于修复了被政府支持的黑客利用的 Windows 零日漏洞

微软终于发布了“Follina”的修复程序，这是 Windows 中的一个零日漏洞，正被国家支持的黑客积极利用。

作为微软每月发布的安全补丁的一部分，这个高严重性漏洞的修补程序(被跟踪为 CVE-2022-30190 )已经发布。但正如网络安全公司 Sophos 所指出的，该修复不在此次发布的补丁列表中——尽管它已证实 Follina 现已得到缓解。

“微软强烈建议客户安装更新，以全面保护漏洞，”微软在 6 月 14 日对其原始公告的更新中说。

攻击者利用 Follina 缺陷，在打开或预览恶意 Office 文档时，通过 Microsoft 诊断工具(MSDT)执行恶意 PowerShell 命令，即使宏已被禁用。该漏洞影响所有仍在接收安全更新的 Windows 版本，包括 Windows 11 ，并使威胁参与者能够在受损系统上查看或删除数据、安装程序和创建新帐户。

网络安全研究人员在 4 月份首次观察到黑客利用该漏洞针对俄罗斯和白俄罗斯用户，企业安全公司 Proofpoint 上个月表示，一个中国政府支持的黑客组织正在利用零日攻击国际藏人社区。Follina 现在还被一个标记为 TA570 的中国威胁组织滥用，在持续的网络钓鱼活动中让受害者感染 Qbot 银行特洛伊木马，以及针对美国和欧洲政府机构的网络钓鱼攻击。

Follina zero-day 最初是在 4 月 12 日标记给微软的。然而，一位被称为 Crazyman 的安全研究人员在一条推特中说微软最初将该漏洞标记为“与安全无关的问题”,他被认为是第一个报告该漏洞的人。

网络安全公司 Tenable 的高级研究工程师克莱尔·蒂尔斯(Claire Tills)告诉 TechCrunch，“鉴于微软最初对该漏洞的否认以及该漏洞在公开披露后的几周内被广泛利用，在周二发布补丁之前，人们有很大的猜测，即微软是否会发布补丁。”他指出，这正成为一种“令人担忧的趋势。”

“Tenable 发现并披露了微软 Azure Synapse Analytics 的两个漏洞，其中一个已经被修补，另一个没有，”她补充道。这两个漏洞都没有被分配 CVE 编号，也没有记录在微软 6 月份的安全更新指南中。"

除了缓解 Follina，微软还修复了三个“关键的”远程代码执行(RCE)缺陷。然而，这些都还没有被积极开发。
