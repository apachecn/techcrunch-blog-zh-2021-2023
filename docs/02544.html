code 公证人筹集 1250 万美元的 B 轮融资以保护软件供应链 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/01/24/codenotary-raises-12-5m-series-b-to-secure-software-supply-chains/

code 公证人(code 公证人)今天宣布，它已经从 Bluwat、Elaia 和其他新老投资者那里筹集了 1250 万美元的 B 轮投资。code 公证人是一项服务，它让开发团队更容易建立透明的软件供应链(也是流行的开源软件 immudb 不可变数据库背后的公司)。新一轮融资使该公司的总融资额达到 1800 万美元，包括去年 7 月 550 万美元的 A 轮融资。

code 公证人由首席执行官 Moshe Bar 和首席技术官 Dennis Zimmer 共同创建，他们帮助人们识别和跟踪 DevOps 周期中的所有组件。这意味着，当供应链受到攻击或出现 Log4j 这样的漏洞时，公司可以更容易地找出这些库在哪里被使用，并减轻潜在的爆炸半径。因为所有这些都位于 immudb 之上，immudb 是一个提供显而易见的篡改历史系统(没有任何区块链巫术)的分类账数据库，所以用户应该能够完全信任这些信息。将 code 公证人添加到他们的软件供应链后，该服务会根据所看到的内容自动创建物料清单。

图片来源:公证人

“我们的任务是确保我们可以信任我们在任何组织的应用程序开发中使用的工件，无论是开源组织还是内部企业组织，”Bar 说。“当我们创办这家公司时，我们一直在寻找方法来确保我们存储的信息(关于谁处理了哪个工件、何时、如何以及他们对工件做了什么)不会被篡改。”由于当时没有满足 code 公证人要求的可用数据库，该团队编写了自己的数据库。Bar 指出，immudb 提供了与区块链相同的加密验证，但采用了性能更高的数据库形式。

瑞士 Bluwat AG 高级合伙人 Pascal Blum 是code 公证人的早期投资者，他说:“code 公证人提供了一种解决方案，使组织能够快速识别和跟踪其 DevOps 周期中的所有组件，从而恢复其无数应用程序中的信任和完整性。“结合code 公证人领先的不可变数据库 immudb ，该公司在这一新市场中取得了领先地位。”

这项服务目前有 100 多个客户，虽然它不能透露其中大多数的名字，但该小组指出，它包括一些世界上最大的银行。

根据 Zimmer 的说法，code 公证人的大多数客户首先在软件管道中实现该服务，以便能够建立他们的软件从源代码到产品的出处。他还指出，这些客户群从小型软件开发公司到大型 ERP 公司都有，例如，他们经常使用这项服务来披露他们在新版本中投入的质量保证工作，并为使用他们软件的外部客户提供材料清单。Bar 补充道，通常是金融组织和政府机构站在思考这些问题的最前沿。

code 公证人计划使用新的资金来加速其产品开发，并扩大其全球营销和销售。
