Dozer 退出 stealth，帮助任何开发人员在几分钟内构建实时数据应用程序

原文：https://web.archive.org/web/https://techcrunch.com/2023/04/03/dozer-exits-stealth-to-help-any-developer-build-real-time-data-apps-in-minutes/

数据已经成为世界上最大的资源之一，支撑着从 T2 的视频推荐引擎和 T4 的数字银行到蓬勃发展的人工智能革命。但是，在一个数据越来越多地分布在不同地点的世界中，从数据库到数据仓库到数据湖等等，将所有数据组合成一种兼容的格式用于实时场景可能是一项艰巨的任务。

对于上下文，不需要即时、实时数据访问的应用程序可以简单地以固定的时间间隔批量组合和处理数据。这种所谓的“批量数据处理”对于处理月销售数据等事情很有用。但通常情况下，一家公司会需要实时访问其创建的数据，例如，这对于依赖于每笔销售的当前信息的客户支持软件来说可能是至关重要的。

在其他地方，打车应用也需要处理各种数据点，以便将乘客和司机联系起来——这不是一件可以等几天的事情。这类场景需要所谓的“流数据处理”，即收集和组合数据以供实时访问，这要复杂得多。

这就是 Dozer 着手解决的问题，通过即插即用的数据基础设施后端，直接从任何来源提供快速、只读的 API。

Dozer 是 Vivek Gudapuri 和 Matteo Pelati 的杰作，他们大约一年前在新加坡的基地创建了这家公司。二人组在亚洲和东欧建立了一个由 10 人组成的分布式团队，准备扩展产品的现有来源(即不完全开放的来源)，成为一个完全货币化的产品。

Dozer 一直在与一些未公开的设计合作伙伴一起测试其产品，今天它正从秘密状态中浮出水面，可供任何开发者访问。该公司还透露，它已经从红杉资本的印度分支机构(通过其在 T2 的 Surge 项目)、谷歌的 Gradient Ventures 和一月资本筹集了 300 万美元的种子资金。

Dozer 联合创始人 Matteo Pelati 和 Vivek Gudapuri。图像来源:推土机

分布的

已经有无数的工具被设计来转换、集成和利用分布式数据，包括流数据库和 ETL(提取、转换、加载)工具，如 Apache Flink 、 Airbyte 和five tran；用于临时数据存储的缓存层，例如 Redis；由类似 Hasura 或 Supabase 支持的即时 API 在系统间传输数据。

就 Dozer 而言，它在所有这些不同的类别中工作，采用它认为最好的部分，并消除构建支持实时数据应用程序的基础设施和管道的摩擦。

用户将 Dozer 插入到他们现有的数据堆栈中，其中可能包括数据库、数据仓库和数据湖，Dozer 负责实时数据提取、缓存和索引，并通过低延迟 API 将其呈现出来。因此，虽然 Airbyte 或 Fivetran 等工具有助于将数据放入数据仓库，但 Dozer 专注于另一方面:“以最有效的方式访问这些数据，”Gudapuri 向 TechCrunch 解释道。

Gudapuri 说 Dozer“采取了一种固执己见的方法”，处理非常具体的问题，仅此而已。例如，现有的流数据库解决了许多问题，远远超出了 Dozer 所提供的，即在单个产品中提供实时数据更新和 API。

Gudapuri 说:“我们在每个类别中解决了适量的问题，为开发者提供了快速的构建体验，以及随时可用的性能。"开发人员(目前)不得不集成几个工具来达到同样的目的."

举例来说，现有的流式数据库可能会尝试向用户呈现整个数据库体验，其中充满了查询引擎、数据探索、OLAP ( )在线分析处理等等。Dozer 故意不提供这些东西，而是专注于 Pelati 所说的使用 SQL、Python 和 JavaScript 的“预先计算的视图”，所有这些都可以通过低延迟 gRPC 和RESTAPI 访问。

Pelati 说，正是因为这个原因，Dozer 可以保证更好的数据查询延迟。

Pelati 说:“由于这些设计选择，Dozer 提供了优越得多的查询延迟，这是面向客户的应用程序所必需的。“一个开发人员可以在几分钟内启动整个数据应用程序；这通常需要几个月的努力。一个团队不必构建和维护几个集成，节省了时间和金钱。”

(不完全)开源因素

虽然 Dozer 被吹捧为“开源”平台，但快速浏览一下 GitHub 上的许可证就会发现，它使用的是Elastic license 2.0(El v2)，这是企业搜索公司 Elastic 两年前作为其从真正开源过渡的一部分而采用的同一许可证。事实上，弹性许可并不被认为是开源的，因为它阻止了第三方获取软件并以托管或管理服务的形式提供。

更准确地说，ELv2 可以被称为“源代码可用”许可证，这实际上意味着它提供了更宽松的开源许可证如 MIT 的许多好处，包括代码库透明性，扩展 Dozer 功能的能力，或微调功能和修复错误。光是这一点就足以赢得各种规模的企业的心，只要不是 AWS 或其他云计算巨头想在 Dozer 上直接赚钱。

然而，该公司表示，它确实打算“很快”转向双许可证，核心推土机项目中的一切都将获得麻省理工的许可，除了“一个核心模块”此外，该公司很快强调其所有的客户端库都已经获得了麻省理工学院的许可，包括 Python 、 React 和 JavaScript 。

值得注意的是，一些公司已经创建了内部工具来解决类似于推土机正在解决的问题，包括几年前制造推土机的网飞公司。值得注意的是，《推土机》的主要创作者之一， Ioannis Papapanagiotou ，现在是《推土机》的顾问。

对于 Dozer 来说，现在仍处于早期阶段，但有了来自许多高调支持者的 300 万美元银行存款，该公司在推进商业化的过程中资金相当充足，这将包括推出一个充满一堆附加功能的托管 SaaS 版本。Gudapuri 表示，预计这将在未来几个月内上线。

“托管服务将负责自动扩展、即时部署、安全性、合规性、速率限制和一些附加功能，”Gudapuri 说。
