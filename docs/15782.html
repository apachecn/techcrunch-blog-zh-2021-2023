安全消息应用排队警告英国在线安全法案风险网络安全| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/10/uk-osb-e2ee-warning/

安全消息应用正在排队反对英国政府的在线安全法案 (OSB)中的措施，他们认为这些措施会破坏网络用户赖以保护其通信的强大加密，从而与促进在线安全背道而驰。

Meta 拥有的 WhatsApp 是 Signal 应用程序背后的非营利信号基金会，Element 运营着非中心化的 Matrix 协议，以及质子(ProtonMail)等端到端加密电子邮件提供商警告说，立法草案包含的措施可能会危及强大加密的安全性——他们表示，这项技术对确保网络用户的通信安全至关重要。

昨天，WhatsApp 的负责人威尔·卡斯卡特在接受BBC和卫报采访时，将 OSB 描述为西方世界最令人担忧的立法。他还暗示，该平台将不会遵守英国的法律要求，削弱其为用户提供的加密级别，而是宁愿被英国当局屏蔽。

已经在上议院进入委员会审议阶段的立法草案，对数字服务施加了保护用户免受一系列伤害的责任，并包含英国互联网监管机构 Ofcom 在严重侵权的情况下阻止不合规服务的权力——除了高额罚款(高达全球营业额的 10%)的制度；甚至根据最近扩大的刑事责任，违法的高级管理人员面临坐牢的威胁。

“没有办法只在世界的一个地方改变(WhatsApp)，”卡斯卡特告诉该报。“一些国家选择了封锁；这就是运送安全产品的现实。例如，我们最近在伊朗受阻。但我们从未见过一个自由民主国家这样做。

“事实是，我们在世界各地的用户都想要安全。我们 98%的用户都不在英国。他们不希望我们降低产品的安全性，简单来说，我们选择降低产品的安全性是一个奇怪的选择，会影响到这 98%的用户。”

信号基金会本周也对这项立法发出了另一个警告，主席梅雷迪思·惠特克在一篇博客文章中警告说，该平台将“坚决反对对私人和安全通信的威胁”——重申了她上个月给BBC的一条信息，当时她说非营利组织将“100%”离开英国，而不是削弱其用户的安全和隐私。

“正如所写的那样，在线安全法案中的条款准备好了清除隐私，同时为威胁英国每个人的安全和保障的剥削开辟新的载体。作为第一个这样的法案，它也可能创造一个肯定会被专制政府复制的模板。我们反对目前形式的法案，并认为关键条款需要从根本上重新考虑，”她昨日写道。

“当伊朗政府屏蔽信号时，我们认识到，这个威权国家并不代表伊朗需要隐私的人民，我们与我们的社区合作，设立代理和其他手段，确保伊朗人能够获得信号。正如在伊朗一样，我们将继续尽我们所能确保英国人民能够获得信号和私人通信。但我们不会破坏或损害我们对英国和世界其他地方人民的隐私和安全承诺。”

上个月，Element 在一篇 的博客文章中也提出了担忧，Element 是 E2EE 矩阵协议背后的英国初创公司，该协议用于分散式消息传递，呼吁政府将整个事情推倒重来——将当前的草案称为“对加密的攻击”

“本来可能是一项建设性的立法，最终却成了一项臃肿而不切实际的提案，起草时几乎没有任何技术实力。该法案目前削弱了英国的数字安全，威胁到基本隐私，阻碍了英国的科技产业，并带来了不断蔓延的审查和全面监控的前景，”联合创始人马修·霍奇森在博客中写道 OSB 认为英国提议进行国家监控和审查，而不是为世界其他地区树立一个原则性的榜样。与欧洲或美国相比，它更接近俄罗斯和中国政权的做法。”

霍奇森今天通过电子邮件与 TechCrunch 讨论了他的担忧，他还警告说，如果 OSB 以目前的形式获得通过，并且没有修改以解决加密威胁，Element 将被迫停止向英国用户提供服务。

“我们不能将客户端监控/内容扫描添加到我们的应用程序中，因为它从根本上破坏了加密，并让坏人更容易访问加密内容，”他告诉我们，强调了它的红线。“我们不能在我们的应用程序中插入第三方专有代码，即使它是特殊的‘政府认可’代码。”

他说，如果英国法律最终走向这个黑暗的地方，Element 也将别无选择，只能“被迫停止向英国公民提供服务”——这意味着它可能会将其应用从英国应用商店中移除，将英国托管的面向公众的服务器转移到其他国家，并阻止英国的 IP 地址连接到这些服务器。

“如果被迫的话，我们会将总部迁到另一个国家，并关闭我们在英国的法人实体(我们已经有了美国、法国和德国的实体)，”他还告诉我们，并警告说:“这将从根本上减少守法的英国公民的隐私(并灾难性地损害 Element 作为英国领先的加密通信初创公司之一，反过来损害 Element 的付费客户，包括英国政府)。"

我们联系了科学、创新和技术部，寻求对消息应用程序对 OSB 对 E2EE 的影响的担忧做出回应，但在撰写本报告时，该部门尚未发出回应。如果我们得到政府的回应，我们会更新这份报告。

更新:政府发言人表示:

“作为一项技术中立的法案，在线安全法案不禁止或破坏端到端加密。政府完全支持用户的隐私权，包括通过加密。平台将理所当然地需要采取相称的措施来保护儿童免受虐待，这包括私人信息服务。

“昨天，技术部长米歇尔·多纳兰(Michelle Donelan)与 Whatsapp 会面，讨论了这个问题，我们将继续与更广泛的行业合作，开发保护公共安全和隐私的创新解决方案。”

OSB 是英国在历届保守党首相的领导下，多年来为制定旨在处理非法(或有害)在线内容的规则而进行的一次尝试。它涉及数字活动的许多方面，旨在应对一系列风险和危害——从恐怖主义和儿童性虐待材料(CSAM)，到仇恨言论、在线性攻击和网络欺凌，甚至是诈骗广告(仅举几例)——包括网络用户相互交流时可能产生的危害。

部长们普遍认为，这一领域的立法是必要的，因为主要的社交媒体平台未能坚持他们声称在自己的 T & Cs 中保持的安全标准，尽管起草的法案将适用于许多数字服务，而不仅仅是社交媒体巨头。

政府声称，为什么需要一部法律来迫使网络公司清理他们的行为，儿童保护是主要的动机。然而，该法案不仅规定了范围内服务应如何调节公共对话的规则，还要求用户对用户服务监管私人言论，以确保其中不包含恐怖主义或 CSAM。而正是这些条款，尤其是，法律专家曾警告威胁端到端加密(E2EE)。

即使该法案没有明确要求提供商放弃 E2EE，但令人担忧的是，该制度将迫使他们致命地削弱他们为用户提供的安全和隐私水平，以避免遭受重大处罚的风险。

数字权利组织很早就对该法案对加密的潜在影响发出了警告——随着该法案在议会的进展，他们的警告越来越强烈。在上个月发布的政策简报中，开放权利组织警告说，它所谓的“T4 形式的聊天监控”正通过立法中的“后门措施”溜进来。报纸继续呼吁将 E2EE prT6】私人信息服务完全排除在法案范围之外。

该法案给加密带来的风险看起来是双重的:要么(1)服务提供商可能会被迫删除(或实际上永远不会实施)E2EE——通过保留加密密钥来降低向用户提供的安全级别，以便它可以解密(和扫描)内容，从而遵守法律对他们提出的检查用户通信非法活动的要求。

或者(E2EE 服务可能会感到被迫(或者实际上是被 Ofcom 命令)实施客户端扫描以遵守法律——这意味着在数据被加密之前，用户的消息内容将在他们的设备上被扫描。

近年来，客户端扫描一直是英国政府感兴趣的领域——英国政府一直积极鼓励开发扫描技术,可用于扫描 E2EE 消息，而无需在技术上破解加密。(DSIT 证实，最近的“安全技术挑战基金”促成了五种正在开发的“概念验证”工具，它声称这些工具展示了潜力，但承认需要进一步开发。)

然后，去年，部长们提交了一份 OSB 修正案，提议给予 Ofcom 新的权力来迫使服务提供商实施内容扫描技术，即使他们的平台是 E2EE——提出了一个平台可能被合法强制进行客户端扫描的场景。

这一举动大大加剧了人们对该法案对网络安全影响的担忧。

虽然客户端扫描技术本身通常不被认为是加密中的后门，但它仍然存在巨大的争议——因为它需要对邮件内容进行全面扫描(这对隐私来说很可怕，而且有可能造成新的大规模“误报”伤害——例如，如果父母分享的孩子洗澡时间的照片被扫描算法错误地识别为 CSAM，而没有做错任何事情的护理人员会将他们的私人通信交给警方)；还因为它在 E2EE 组合中引入了额外的技术，这可能会在代表当前加密“黄金标准”的系统中产生新的漏洞。

隐私和安全专家仍然一致警告客户端扫描带来的风险——例如，他们指出，这种技术过度滥用，而且尚未被证明达到这种敏感用例应该要求的强大准确性水平。他们还警告说，沿着这条路走下去的西方民主国家可能会帮助并鼓励专制政权通过扫描政治反对派并进一步寻求压制自由表达的技术版本。

到目前为止，这些担忧都没有阻止立法者通过抓住一种侵入性的和未经证实的技术来推动一个受欢迎的政策目标(儿童安全)——这种技术提供了一种廉价而简单的方式来声称他们正在为一种类型的虐待做些什么，这种虐待是从复杂的社会问题中出现的，包括社会和公共服务的资源不足，这是他们似乎不太有兴趣试图“解决”的事情(另见:欧盟关于 CSAM 缓解的提案——也希望推动平台采用扫描技术——尽管在实践中，鉴于欧盟禁止一般监控义务，那里的立法者可能很难实施这样的制度。)

虽然电子邮件服务没有明确纳入 OSB 的范围，但对英国旅行方向的关注程度如此之高，以至于 E2EE 电子邮件提供商也加入了公众对该法案的关注。

E2EE 电子邮件提供商质子 称 OSB 是“误导和危险的”和“除了名义上禁止端到端加密”——最近呼吁修改草案，以保护它所说的互联网运行所依赖的加密。“削弱端到端加密将降低每个人的网络安全，包括该法案试图保护的儿童，”它进一步警告说。"如果没有强有力的加密，数百万人的敏感数据将面临风险."

在另一次公开干预中，总部位于欧洲大陆的 E2EE 电子邮件提供商 Tutanota 公开质疑英国是否会步俄罗斯和伊朗的后尘，封锁加密技术？“曾经是世界上最大的民主国家的英国正在发生的事情真的令人担忧，”联合创始人马蒂亚斯·普法乌建议道。“英国政府仍然相信他们可以有一把‘神奇的钥匙’来访问加密通信——完全无视技术背景和密码学专家一再说过的话:你不能后门加密，并确保这个后门不会被恶意行为者滥用。”

虽然自 2019 年的“在线危害”白皮书以来，以及自 2021 年的法案初稿以来，OSB 已经采取了许多额外的措施，但政府声称所有这些措施都加强了该提案，因为它是对儿童安全组织正在进行的活动的回应，这些活动敦促部长们进一步解决安全风险。 该法案的最新修订实际上撤回了针对合法但有害言论的条款——在一些议员提出对言论自由的影响的担忧后——表明部长们也不能幸免于对他们做法的批评。

尽管选民在不久的将来会读到指责政府导致 WhatsApp 在英国陷入黑暗的头条新闻，这一前景是否足以让部长们重新考虑他们对强加密的立场仍有待观察。

无论 WhatsApp 发生什么，该法案似乎不太可能停止使用所有拒绝部署客户端扫描的 E2EE 消息应用程序，如果这真的是部长们的目标的话。

很像 Signal 的建议，如果它被迫正式关闭英国服务，英国用户可以求助于代理服务器来保留对其应用程序的访问，Element 的霍奇森指出，如果它不得不退出英国，这也不一定意味着它开发的技术在当地的使用结束——因为它的应用程序和 Matrix 协议本身是开源的，“任何人都可以运行自己的部署。”

“安全的端到端加密已经在公共领域存在了 30 多年，攻击端到端加密服务提供商不会使其消失，因为任何人都可以运行自己的服务，”他建议道，并补充说:“我们还有一个长期运行的项目，以对等(P2P)矩阵的形式使服务器在矩阵中可选。一旦 P2P Matrix 或类似技术在未来几个月流行起来，用户就不再需要服务器了——每个应用实际上都是自己的服务。因此，不管 OSB 对加密的攻击如何，用户将能够安全地通信，而不依赖于任何服务提供商。”

如果英国政府真的想解决虐待儿童的问题，霍奇森建议它雇佣更多的警察——以“专注于传统的调查和渗透，并用工具和培训武装他们，以打击网上的不良行为者。”

“强迫加密服务加入大规模监控，是试图将治安工作廉价外包给服务提供商，”他补充道——在网上，这相当于强迫在每个人的卧室里安装闭路电视摄像头，以捕捉“可能的滥用行为。”
