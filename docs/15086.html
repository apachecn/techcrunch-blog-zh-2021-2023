GitHub 裁员 10%并完全远程化| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/02/09/github-lays-off-10-and-goes-fully-remote/

技术裁员还在继续。微软旗下的 GitHub 今天宣布，将在公司财年结束前裁员 10%。在《财富》杂志首次报道这一消息之前，GitHub 大约有 3000 名员工。该公司还将关闭所有租约到期的办公室，部分原因是它们的利用率低，并转向远程优先的文化。

GitHub 还将继续冻结招聘，这是它在 1 月份首次宣布的，同时还将进行一些其他内部变革，以“保护其业务的短期健康”。

“我们宣布了一系列艰难但必要的决策和预算调整，以保护我们业务的短期健康发展，并赋予我们投资于长期发展战略的能力。一位公司发言人告诉我们:“你可以在下面查看我们的 CEO 给员工的完整信息，以及这些变化的更多细节。”。

对于一家以保持独立于其公司所有者而自豪的公司来说，GitHub 也在为其视频会议需求转向团队，这一举动有点非正统。作为削减成本的另一个标志，它将其笔记本电脑更新周期从三年延长到四年。

“虽然我们的整个领导团队已经仔细考虑了这一步骤并达成一致，但最终，作为首席执行官，决定权在我。GitHub 首席执行官托马斯·多姆克(Thomas Dohmke)今天在给公司员工的一封电子邮件中写道:“我知道这对你们所有人来说都很困难，我们将以对每一位哈伯的最大尊重来度过这一时期。”。

他指出，他希望公司成为“面向未来世界的开发者至上的工程系统”，重点关注人工智能。鉴于 GitHub 最近对其 Copilot 的关注以及微软向人工智能的全面转变，这可能并不意外。

以下是多姆克今天早些时候发给该公司的电子邮件全文:

GitHub 首席执行官 2023 年 2 月 9 日给员工的信息

今天，我们宣布了一系列艰难的决定，包括告别一些人和制定新的预算调整，旨在保护我们业务的短期健康，同时也让我们有能力投资于我们的长期战略。怀着对 Hubbers 的极大敬意，我首先想弄清楚我们为什么要做这些决定，以及这对 GitHub 的未来意味着什么。

持续增长对每个企业都很重要。对于 GitHub 来说，这意味着我们将继续在全球范围内提高开发人员的工作效率，并随着我们帮助客户的机会的变化而快速行动。今天，我们是 1 亿开发者的家园，我们必须成为面向未来世界的开发者至上的工程系统。人工智能时代已经开始，我们一直在通过 GitHub Copilot 引领这一变革，这是我们迄今为止最成功的产品发布。我们有一个巨大的机会来紧急构建一个集成的、人工智能驱动的 GitHub。我们必须继续帮助我们的客户利用 GitHub 发展壮大，加速并简化他们的云采用之旅，同时每天为他们提供支持。这将需要我们高度重视并改变我们如何以及在哪里投资有限的资源。

首先，我们将把我们的工作与我们能够最好地影响这些目标和我们所有产品的客户需求的领域结合起来。不幸的是，这将包括导致 GitHub 在 2023 财年结束前裁员 10%的变动。一些 Hubbers 将在今天收到通知，其他人将紧随其后，因为我们将在 2013 财年结束前重新调整业务。我在 1 月 18 日宣布的暂停招聘仍然有效。

虽然我们的整个领导团队已经仔细考虑了这一步骤并达成一致，但最终，作为首席执行官，决定权在我。我认识到这对你们所有人来说都是困难的，我们将以对每一个哈伯的最大尊重来接近这个时期。我们将与受影响的 Hubbers 交谈，以便他们了解将提供的过渡补偿和 COBRA/COBRA 等价物(美国以外)。还将提供职业过渡服务援助福利。

此外，我们一直致力于提高我们的运营效率和业务规模。我们的决定之一是转向完全远程的 GitHub。我们在世界各地的办公室的利用率都非常低，这一决定证明了我们长期以来“远程优先”文化的成功。我们不会立即腾出办公室，但会在租约到期或我们有能力这么做时关闭我们所有的办公室。我们将与您分享最终确定的更多工作场所细节和过渡计划。

我们正在考虑进一步降低运营成本。我们将在未来几个月与您分享详细信息和过渡计划，但我想与您分享两个决定:I)立即生效，我们将笔记本电脑更新从三年期改为四年期。ii)我们将转向微软团队，其唯一目的是视频会议，从而节省大量成本并简化跨公司和客户对话。这一行动将于 2023 年 9 月 1 日完成。我们将继续把 Slack 作为我们的日常协作工具。

最后，我想向每一位 Hubber 和他们不可思议的才华致以最深切的感谢，是他们帮助 GitHub 发展到今天的位置。你所做的每一个承诺和你工作的每一天都帮助 GitHub 成为最大和最重要的软件开发平台。感谢你们的奉献、韧性和热情，让全球数百万软件开发人员受益。
