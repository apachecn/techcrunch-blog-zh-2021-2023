我在亚马逊网络服务公司担任了几年的总经理，我的团队推出了两项一级服务:亚马逊云搜索和亚马逊开放搜索。

像 AWS 的每一款产品一样，这些产品都是根据产品导向的增长(PLG)市场运动进行调整的。没有封闭功能或订阅层可供选择。相反，基于使用量的定价(UBP)模式一直被用来根据消耗量收费，与交付给用户的价值直接相关。

AWS 在 2011 年上市时，通过提供完全按需付费的全套产品，开创了产品主导的运动。开发者可以立即开始使用免费层中的服务来实现价值，并且没有强制捆绑或看门的客户经理。2011 年，AWS 远远走在了时代的前面。直到现在，我们才看到各种规模的公司转向以产品为导向的运动。

基于使用的定价是任何 PLG 战略的重要组成部分。事实上，我认为没有 UBP 就没有真正的 PLG。要真正以产品为导向，在采用产品和实现价值时不应有任何摩擦。门应该是敞开的，新用户应该能够进来并使用这些工具。

任何以产品为导向的战略要想取得成功，就必须实时了解和深入了解用户对产品的实际使用情况、正在使用哪些功能以及价值是如何实现的。以下步骤参考了我们在 AWS 推出和扩展服务的早期所遵循的流程。这一过程允许您在定价和产品决策中消除情绪和直觉，同时允许客户使用和消费来引导您的决策，并帮助您扩大业务规模。

当需要对产品包装和定价做出决策时，首先应该考虑历史使用数据的计量管道。

步骤 1:投资使用仪器

使用数据是任何以产品为导向的行动的基础。使用提供了驱动所有其他功能的智能，从定价和打包到销售、支持服务甚至产品路线图开发。这些数据显示了哪些功能正在推动流量和采用，以及您需要在哪些方面加大努力来继续满足用户需求。

大多数组织无法轻松地大规模实现使用工具，并且现有的监控和可观察性工具集无法满足总体准确性和可审计性的要求。计量解决方案诞生于对一种新技术的需求，这种新技术可以实时准确地跟踪任何规模的任何资源的使用情况，并使这些数据可用于分析和报告。

随着新功能和产品的开发，继续对其进行检测，以收集详尽的使用数据集，供您分析和制定业务决策。

第 2 步:让使用数据在整个企业中可用

持续评估并确保每个部门都能获得工作所需的洞察力。在以产品为导向的运动中，产品是你与客户互动的主要工具；其他走向市场行动旨在提供支持。组织中的角色可以通过正确的工具和策略有效地利用使用数据。

确保您的电表数据管道是使用和消费数据的唯一真实来源。您应该注意确定组织中每个角色需要哪些产品和使用数据才能成功。数据应该被编入索引，并可供有权限的用户访问。
