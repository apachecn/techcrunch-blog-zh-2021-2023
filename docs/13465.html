GitHub 为所有公共回购带来免费的秘密扫描

每个开发人员都知道将安全凭证硬编码到源代码中不是一个好主意。然而它还是发生了，一旦发生，后果可能是可怕的。到目前为止，GitHub 只向购买了 GitHub Advanced Security 的付费企业用户提供秘密扫描服务，但从今天开始，微软旗下的公司将免费向所有公开的 GitHub 回购提供秘密扫描服务。

仅在 2022 年，该公司就在其秘密扫描合作伙伴计划中通知合作伙伴，超过 170 万个潜在秘密暴露在公共存储库中。该服务扫描超过 200 种已知令牌格式的存储库，然后警告合作伙伴潜在的泄漏——您也可以定义自己的正则表达式模式。

图片来源: GitHub

“通过秘密扫描，我们发现了大量需要解决的重要问题，”Postmates 的安全工程师戴维·罗斯说。“在 AppSec 方面，这通常是我们了解代码中问题的最佳方式。”

现在，如果你在 GitHub 上托管你的代码，公司会自动直接通知你你的源代码中泄露的秘密。这也意味着你会收到没有伙伴通知的秘密警报(例如，可能因为你自己管理你的 HashiCorp 金库)。

要开始使用该服务，您必须在 GitHub 安全设置中启用该功能。然而，该服务的推出将是渐进的，直到 2023 年 1 月底才会对所有用户开放。

当然，GitHub 自己的工具并不是唯一扫描泄露机密的服务。还有像 Gitleaks (可以与 GitHub actions 集成)这样的开源工具，以及像nightness和 check pointSpectral这样的大量安全公司，尽管他们的服务往往远远超出秘密扫描的范围，通常面向企业。
