Sublime 凭借基于集体众包规则的反网络钓鱼电子邮件安全平台赢得 980 万美元| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/02/21/sublime-nabs-9-8m-for-an-anti-phishing-email-security-platform-built-on-collective-crowdsourced-rules/

恶意黑客越来越有创意地使用他们用来侵入网络窃取数据和造成破坏的技术，但他们打开那扇门的主要途径仍然相当一致。电子邮件是迄今为止最受欢迎的设置和执行网络钓鱼、勒索软件和其他攻击载体的入口，根据联邦调查局去年的一份报告，仅在美国，2021 年商业电子邮件交互就导致约 24 亿美元的损失。

今天，一家名为的初创公司 Sublime Security 正在悄然兴起，它采用了一种新颖的集体方法来解决这个问题:它为研究人员和安全运营人员(那些捍卫网络的人)建立了一个平台和特定领域语言(DSL ),以编写、运行和相互共享规则，来检测和阻止最常见(和最不常见)的通过电子邮件传递的各种威胁。

这家总部位于 DC 华盛顿州的初创公司已经进行了一年多的私人测试，在此期间，它已经获得了许多大型跨国客户，从政府机构到 Spotify 这样的公司，还有 2500 家其他公司。现在，随着它进入全面上市阶段，它还宣布投资 980 万美元。

分贝领投，Slow Ventures 和网络安全领域的一些个人参与，包括 Sounil Yu(网络防御矩阵和 DIE Triad 创造者)；Snort 和 Sourcefire 的创造者马丁·罗施；老将西索斯杰里佩鲁洛和迈克尔萨顿；德米斯图拉创始人 Rishi Bhargava 和 Slavik MarkovichLookout 创始人凯文·帕特里克·马哈菲；幻影网络和盘古公司的创始人奥利弗·弗里德里希斯。

Sublime 涵盖了恶意软件、勒索软件、凭据网络钓鱼、VIP 冒充和回拨网络钓鱼等载体。它的代码可以应用于微软 365 和谷歌 Workspace 企业邮件系统，也可以通过 IMAP 在个人账户上运行。除了最基本的用途(入站电子邮件安全)之外，Sublime 还可用于收集和分析组织面临的威胁趋势、阻止整个域、运行合规性和培训方面的安全练习，等等。

核心产品在自托管时可以免费使用。托管版 Sublime Cloud，前 10 个邮箱后收费。当企业客户自行托管但需要支持和监控服务时，他们也需要付费。

与伊恩·泰尔(Ian Thiel)共同创立 Sublime 的约书亚·卡姆乔(Joshua Kamdjou)在接受采访时表示，他最初是在为国防部工作时产生了创业想法，当时他还在上高中，就开始作为一名“白帽”黑客工作。

在那里，他非常熟悉恶意黑客对网络钓鱼电子邮件使用的技术。

“攻击者不断想出绕过防御的新方法，”他说，问题是大多数防御都是基于单个安全供应商设置的安全参数，用他的话说就是“黑箱”方法。当黑客应用新技术时，供应商有责任向他们的系统发布补丁和更新来解决这些问题。

但是新的技术会不断出现，如此等等，造成保护的滞后和空白。“供应商是瓶颈，”他说。在他自己的测试中，Kamdjou 会在一个月后应用网络钓鱼技术，然后在一个月后返回，“问题仍然存在。”

Kamdjou 看到了通过开发人员的集体知识和工作实践来构建解决方案的机会。他来自黑客和编码领域，使用 GitHub 这样的服务来跟踪和参与项目是他的天性。他将众包模型应用于 Sublime 如何跟踪和发展自己的威胁向量和方法数据库。

需要明确的是，Sublime 不是“开源”的，Thiel 和 Kamdjou 说他们仍然在考虑哪些方面，如果有的话，他们可能会开源。但它确实借鉴了一些这种精神。Thiel 说，Sublime 团队已经编写了 Sublime 数据库中大约三分之二的规则，其中三分之一由社区贡献。

各个组织随后会自行决定如何定制自己的电子邮件安全性，应用哪些规则，将哪些规则搁置一旁，从而将更多的权力交到客户手中。到目前为止，这是它的卖点之一。

分贝公司的合伙人 Dan Nguyen-Huu 在接受采访时说:“Sublime 给了检测团队重新控制电子邮件收件箱的机会。”“社区供电的 DSL 意味着它的所有客户都在说同一种语言，共享规则，并能够更好地进行补救，”他说。"这意味着他们可以团结起来对抗共同的敌人。"他补充称，该公司采取的方法在市场上是独一无二的。

“捍卫者比任何人都了解他们的网络，但我们没有把他们作为一个群体来武装，”Kamdjou 说。这也是与电子邮件无关的其他安全产品的数量。Kamdjou 列举了一些例子，如二进制的 YARA，日志的适马/EQL，网络的 Snort/Suricata，端点的 osquery/EDR，静态分析的 Semgrep。

有趣的是，到目前为止，Sublime 的贡献者数量只占其目前用户总数的一小部分。

“这有点像 Twitter，”Kamdjou 说。“大多数人不发推特，只是阅读，看起来我们的模式将是相似的，只有一小部分人写规则，其余的人发现那些有用的规则。”

Twitter 是一个恰当的类比，还有另一个原因:Thiel 说 Sublime 很大程度上是通过口碑增长的，很多这样的话已经在这个特定的社交平台上交换了。“信息安全生活在推特上，”他说。

随着像生成式人工智能这样的新工具代表着增加更复杂和更有说服力的电子邮件数量的潜在方法，你可以看到为什么和在哪里加快最终用户自己识别和应对这些威胁是有意义的。随着时间的推移，这可能会导致更多的贡献者和更崇高的使用；有趣的是观察人工智能模型如何以及是否开始应用于更多防御的产生。
