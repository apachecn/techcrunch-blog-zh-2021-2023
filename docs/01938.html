FTC 警告对未能修补 Log4j 漏洞的组织采取法律行动

原文：https://web.archive.org/web/https://techcrunch.com/2022/01/05/ftc-legal-action-log4j/

FTC 警告对未能修补 Log4j 漏洞的组织采取法律行动

美国联邦贸易委员会( FTC )警告称，未能针对广泛使用的 Log4j Java 日志库中的零日漏洞 Log4Shell 保护客户数据的美国组织可能面临法律后果。

在本周的警报中，消费者保护机构警告称，在 12 月首次发现的这一“严重”缺陷正被越来越多的攻击者利用，对数百万消费者产品构成“严重风险”。公开信敦促组织减轻漏洞，以减少对消费者造成伤害的可能性，并避免潜在的法律诉讼。

该机构表示:“当漏洞被发现和利用时，它会带来个人信息丢失或泄露、财务损失和其他不可逆转的伤害的风险。”“采取合理措施缓解已知软件漏洞的责任涉及法律，包括联邦贸易委员会法案和 Gramm Leach Bliley 法案。依赖 Log4j 的公司和他们的供应商现在就采取行动，以减少对消费者造成伤害的可能性，并避免 FTC 的法律诉讼，这一点至关重要。”

FTC 强调了 Equifax 的案例，该公司未能在 2017 年修补一个已知的 Apache Struts 漏洞，导致 1.47 亿消费者的敏感信息泄露。信用报告机构随后同意支付 7 亿美元与该机构和各州达成和解。

“联邦贸易委员会打算利用其全部法律权力，追究那些未能采取合理措施保护消费者数据免受 Log4j 影响的公司，”联邦贸易委员会表示，并补充说，它计划在“未来类似的已知漏洞”情况下，运用其法律权力保护消费者

对于热衷于逃避潜在的数百万美元罚款的组织，FTC 鼓励他们遵循美国网络安全和基础设施安全局(CISA)发布的指南。这促使企业将 Log4j 软件包更新到最新版本，采取措施缓解漏洞，并将有关漏洞的信息分发给可能易受攻击的第三方和消费者。

在微软本周警告log4 shell 漏洞对公司来说仍然是一个“复杂和高风险”的情况后，FTC 发出了警告，并补充说“在 12 月的最后几周，利用尝试和测试仍然很高”，低技能的攻击者和民族国家行为者都在利用这一缺陷。

“在这个节骨眼上，客户应该假设漏洞代码和扫描能力的广泛可用性是对其环境的真实和现实的危险，”它补充说。由于受影响的软件和服务数量众多，而且更新速度很快，因此预计补救的时间会很长，需要持续不断的警惕
