Nabla 为患者参与开放了一个医疗技术堆栈

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/02/nabla-health-tech-stack-api/amp/

图片来源:纳布拉

法国初创公司 Nabla 去年在推出了自己的女性健康诊所应用程序，开始从内部研究数字医疗保健后，正在执行转向 b2b 计划的下一步——今天宣布，它向其他数字健康企业和医疗保健提供商开放了机器学习技术栈，以便他们可以提供它所谓的“个性化医疗”。

Nabla 的人工智能支持的患者通信和参与/保留平台旨在支持临床医生提供更持续的数据驱动型服务，无论客户是提供实时远程医疗咨询还是通过异步、基于文本的消息传递向患者提供服务。

Nabla 的消息和远程咨询通信模块位于客户医疗保健服务之上，接收和结构化患者数据，其机器学习软件支持临床医生进行实时提示和可视化，并提供持续的患者外展功能以扩展服务供应。

这家初创公司认为，它的方法可以通过支持医疗保健专业人员在咨询期间提出相关问题来改善医疗效果，这是基于人工智能聚合患者活动和表面上下文相关数据的能力，以及之后的自动转录和建议临床医生对患者医疗文件进行更新等功能。

它将该平台的功能比作拥有一位真正细心的家庭医生，这位医生了解患者的全部病史和情况，并且对所有细节都有无错记忆。但是这项技术可以超越即使是一位伟大的医生所能提供的，因为它使医疗保健提供者能够通过持续的异步扩展来补充面对面的咨询，以提供一层连续的护理——例如通过后续的预定消息(例如，提供治疗提醒或询问患者的进展等)。当然，即使是最好的人类医生也无法在两次就诊之间为患者提供那种程度的检查和关注。

因此，Nabla 的前提是，将数字交付的同步(人类)护理与数据驱动的(人工智能)支持和异步跟进相结合可以实现双赢:对于患者来说，他们可以获得比传统医疗服务交付更多的持续(和潜在的整体)护理；对于数字健康企业来说，由于其平台提供的智能、个性化的帮助和外联，这些企业能够提高客户参与度和忠诚度。

纳布拉认为，客户保留已成为数字医疗服务提供商面临的一个紧迫问题——他指出，在疫情期间对该领域的兴趣激增之后，随着患者注意力的分散，以及更广泛的全球经济衰退使通过融资扩大规模的任务变得复杂，许多这些业务可能会突然回到现实。

“医疗科技当然受到经济衰退的很大影响，”联合创始人兼首席执行官 Alexandre Lebrun 说。“在我们周围，我们看到许多医疗科技初创公司……由于新冠肺炎危机，他们自动拥有大量患者……对他们来说，获得大量患者和参与非常容易。现在新冠肺炎已经结束了——再加上现金不再免费——他们发现自己面临着与电子商务公司相同的问题——我必须照顾我的客户，我必须努力留住他们，我必须让他们开心。不仅仅是自动的。”

Nabla(目前)不从事自动化医疗保健业务；相反，其平台提供实时临床医生支持和经临床医生批准的患者外展服务，这意味着，至关重要的是，一名合格的人类医生始终处于循环中，并负责患者护理决策。所以它的产品本身并不是医疗设备——尽管 Lebrun 可以设想在这个方向上采取进一步的措施。

他告诉 TechCrunch:“我们的长期目标是利用这些数据，不仅仅是为了一个病人的利益，而是学习和汇总所有这些数据，例如，尝试预测病人接下来会发生什么，或者更快地进行诊断。”“当然，我们拥有的数据对研究来说非常有价值，因为我们有关于患者的非常非常详细的信息，而不仅仅是典型的医院记录……(我们有关于)他们吃什么、他们如何生活、他们的社会环境、家庭环境的数据——我们知道这对健康非常重要，但这些信息在现有的医疗记录中找不到。但是我们有一部分。因此，这对未来的学术研究非常有价值——当我们问我们的用户，你是否同意为医学研究分享这些数据……大多数人说当然同意，如果他们理解我们分享的范围的话。”

Lebrun 最初从事聊天机器人方面的技术工作，显然非常了解这项技术的局限性。在将之前的人工智能初创公司( Wit.ai )出售给脸书之后，他留在了这家科技巨头，致力于开发其混合通用人工智能礼宾服务(又名“M”)——脸书最终决定不能为其用户群进行扩展。但是 Lebrun 已经看到了将人与人工智能结合起来进行决策支持的潜力，并决定回到创业领域，在效用看起来更容易磨练的更窄的医疗保健领域应用类似的混合方法。

Nabla 的创始人随后决定建立和运营一家女性健康诊所，以了解他们希望用机器学习软件提供和支持的行业的需求——在 2021 年 4 月日推出他们的诊所作为一款应用程序。Nabla 表示，该应用目前将继续运营(尽管它不再是他们的主要关注点或产品)，迄今为止已经积累了约 25，000 名患者。

这种方法意味着 Nabla 的技术处于相对新颖的位置——当然与历史上的一般健康产品相比——在开发过程中，主要通过女性的经验获得信息。它的联合创始人认为，这导致了一种产品，无论用户的性别如何，这种产品都比替代品更专注，而且作为一种医疗保健工具更有用。所以另一个双赢，正如他们所说的。

通过诊所直接接触患者和医生，为 Nabla 提供了与核心用户、数据和专业知识的联系，这是它开发机器学习健康堆栈产品所需的，该产品目前正在寻求货币化。尽管它强调患者数据保密要求意味着总是在严格限制数据访问的情况下工作——例如，其工程师不能直接访问用户的医疗信息(包括在开发人工智能技术栈期间)。

“我认为选择女性健康的结果是，我们非常重视移情护理。Lebrun 建议说:“在连续和多学科的护理方面，现有的医疗保健系统完全忘记了这一点。“对我来说，说好吧，我要开一家女性健康诊所，这是一个艰难的决定。我开始整天跟妇科医生学习……如果我的联合创始人不是女性，我就不会有信心开办一家女性健康诊所。所以很棒。他补充道:“我们一起创办 Nabla 的时候并没有计划过，但这很好。”。

“这让我们能够更多地关注这些事情。对女性来说，同理心、提供者方面的多学科能力以及对患者的整体看法比男性更重要。我是一个男人，我的胳膊有问题，我去看医生，十分钟后我知道该做什么。这就解决了——但这不是女人需要的。而这并不是现有系统所能提供的。因此，我们很快学会了通过混合异步和同步护理来提供这种护理。有趣的是，我们现在意识到，这种护理实际上对每个人都更好。即使是对男人来说。”

“我们真正想要建立的是一个患者参与堆栈，”联合创始人兼首席运营官戴尔芬·格罗尔补充道。“当我们做一些研究时，我们做了很多应用程序的测试版，我们发现女性在远程护理方面更投入。由于我们的 ML 模型，我们的重点是推动参与度，我认为这是拥有这种人群的最佳选择，这样我们就可以从他们那里了解很多见解，然后尽我们所能在参与度和保留率方面积累经验，这是医疗保健公司目前面临的主要挑战。所以我认为这不是唯一的原因，而是我们选择关注女性健康的原因之一。”

Nabla 的通信模块连接到其机器学习驱动的医生控制台，在过去的三个月里，已经通过 API 和 SDK 向第三方提供了这些模块——它说迄今为止已经注册了大约 10 个客户——但它今天宣布正式开放。

早期客户——横跨欧洲和非洲的美国、英国和法国等一系列市场——包括数字健康初创公司，如 Resilience、Cardiologs、Aura Fertility、Omena、Umana、Jeen 和 Tchak 以及老牌医疗机构，如 AP-HP，该公司称其为欧洲最大的医院集团。这个想法是让 b2b 业务从一开始就国际化，per Groll。

在讨论竞争格局时，Lebrun 称其最接近的竞争对手是美国公司【Canvas Medical】、Seqster 和 Zus Health——他确认 Nabla 对美国市场有很强的设计，因为它在数字医疗行动中占了很大比重。

“最接近的公司都在美国，我认为，他们很快就明白，将会出现一个新的医疗保健技术堆栈——就像 20 年前电子商务领域发生的那样——电子商务的每一部分都由少数公司之一管理，然后你组装这些砖块。他认为，同样的事情也开始在医疗保健领域发生，”他还将 Nabla 为临床医生提供支持的方法比作 GitHub 的“人工智能配对程序员”软件 Copilot。

“毫无疑问，我们很快就会竞争。当然，美国的需求略有不同。这些竞争对手——我认为我们有相同的理念，让医疗服务提供商建立他们想要的体验，让医疗服务提供商和医生的生活更轻松。但是我认为我们更关注异步护理——用同步护理构建异步护理是多么有趣。

“机器学习当然可以帮助做到这一点，我们在机器学习(空间)领域有 20 年的经验，之前有三家公司，我认为我们有丰富的现实世界经验……我们知道什么是可能的，什么是不可能的，什么是可取的或不可取的，我们正在努力应用这些经验。”

格罗尔认为，Nabla 相对于竞争对手的优势在于，它不仅专注于提取患者告诉医生的信息，还专注于构建这些信息，以便通过提出个性化后续建议，改善为他们提供的医疗保健服务，如果患者同意，还可能用于研究目的。(使用人工智能服务需要患者同意其数据被处理；另外，这家初创公司证实，为了研究目的，任何更广泛的聚合和匿名数据共享。)

“当我们与潜在客户交谈时，他们最终真正喜欢的是我们从所有通信中提取数据——特别是消息传递和远程咨询——我们不仅提取数据……我们还对其进行结构化和规范化，以便它可以成为他们的强大资产，”她说，并补充说:“所以我认为我们的区别在于……我们使他们能够拥有一个通信模块，而且能够利用他们在这些通信中拥有的数据。”

在研究方面，Lebrun 建议，这种方法——如果 Nabla 扩大使用范围——可能需要其医疗保健提供商合作伙伴和公共研究机构之间的合作，这些合作伙伴和公共研究机构将根据服务提供商提供的聚合匿名数据在特定领域开展研究。

“研究最好由公共机构或学术界来进行，”他认为。“所以我认为这将是一个三方合作，Nabla 的提供商同意贡献他们的数据。他们当然会征得病人的同意来分享他们的数据。然后学术合作伙伴或公共机构进行实际研究，但你可以将 Nabla 视为一个医疗保健提供商网络，他们可以通过一种简单的方式(因为它已经结构化)为研究提供数据。”
