在当今的数字世界中，要确保公司的安全，需要一个村庄，特别是公司的所有员工，他们必须时刻保持警惕，以避免网络钓鱼、信用填充和其他类型的常见攻击媒介和技术，这些都会导致他们的数据和业务受到威胁。一家名为 Push Security 的伦敦初创公司认为，它可以在这方面提供帮助——不是通过阻止在线活动和应用使用，而是通过监控用户何时对基于网络的应用做出不确定的选择，并展示如何修复它们。今天，它宣布了 1500 万美元的早期资金，以扩大这一努力。

A 轮由 GV (Google Ventures)领投，分贝筹和一些天使投资人参与。(这些人包括 Duo Security 的联合创始人 Dug Song 和 Jon Oberheide。)

这笔资金来自 400 万美元的种子资金和一些明显的早期迹象。这家初创公司表示，自 2022 年 7 月推出以来，其工具已被“数百”个团队和约 5 万名用户采用，客户包括 Reachdesk、Upvest 和 Tray.io(其创始人兼首席执行官 Rich Waldron 也是本轮天使投资人)。

Push Security 的联合创始人兼首席执行官亚当·贝特曼(Adam Bateman)表示，他作为一名道德黑客工作了多年，观察了许多员工中最常见的错误和做法，之后产生了 Push 的想法。

一个反复出现的事实是，无论一家公司的安全政策有多强，也无论它在防火墙、终端解决方案和其他方面投入了多少资金，人类围绕错误密码选择采取的行动，不经意地点击不可靠的链接，以及不知不觉地分享他们不应该分享的东西，往往被证明是第一个漏洞。

Push 的出发点是接受某些行为将成为人们的第二天性:也就是说，他们希望在工作中使用基于网络的应用程序来帮助他们更好地工作，即使这些工具不是由 IT 部门提供的。这已经成为一种趋势，尤其是在过去几年，越来越多的人远程工作，基于云的架构成为他们的标准。

Push 的方法遵循几个不同的轨道:它观察这些应用程序是如何使用的，然后当它发现员工以不太安全的方式使用这些应用程序时，它会自动向员工“推送”建议——比如，通过选择容易猜到的密码；它将通知“推”给安全和 it 团队，给他们活动的摘要，以便他们保持在循环中；然后，它会将该应用程序添加到仪表板上，以便这些团队进行监控，并在这些应用程序造成危险时进行标记，因为它们本身可能存在安全问题，并阻止那些可能非常危险的应用程序。

关键是 Push 试图通过不阻止活动来避免摩擦，但它通过向每个人推送更好的建议来实现更好的实践。

贝特曼将其方法比作语法方法。他在一次采访中说，“你不必这样做，但这可以防止你犯错。”。“跟我们一样。推让你安全。我们在人的层面上做的很多工作不是要成为执行者，是要成为护栏而不是大门。”

因此，尽管市场上有数百家公司提供密码管理、应用程序管理、桌面管理、防火墙、黑名单和白名单等等，但这里吸引投资者注意的是一种让人们继续按原样工作的工具的想法。

GV 的普通合伙人 Karim Faris 在一份声明中表示:“全球员工正通过 SaaS 应用程序走向更大的自由和灵活性，这带来了新的安全复杂性和挑战。”。“这一趋势表明，迫切需要更好、更简单的工具来吸引员工，并减轻集中化 IT 管理 SaaS 蔓延的负担。GV 很高兴与 Push 团队合作，帮助现代安全团队应对不断变化的网络安全威胁。”

贝特曼说，目前所有这些都是为那些已经使用 Office 365 或谷歌工作空间电子邮件来管理他们在工作中使用的应用程序的登录的员工提供的。如果有人试图使用非工作邮件注册工作应用，也会被标记出来。

推送安全功能仅适用于台式机和笔记本电脑，不适用于移动设备。这是因为移动设备管理，通常包括应用程序和其他移动应用，已经是一个非常广泛的领域。更令人费解的是人们可以很容易地从网上下载 SaaS 的台式电脑。

SaaS 的发展为工人们打开了一个生产力的新世界，但也打开了一个安全隐患的罐子。鉴于许多应用程序要求“访问你的联系人”和其他数据，以便最有效地工作(这就是为什么这么多应用程序首先使用他们的工作凭据登录)，如果这些帐户随后没有得到负责任的管理，就会造成潜在的数据泄露。

在 Push 的研究中，它发现通过其平台监控的 23%的微软集成和 17%的谷歌集成“被授权访问高风险资产和数据，如电子邮件、日历和共享驱动器”，并且作为人们使用这些凭据的数量的衡量标准，在微软应用集成中，只有三分之一通过 OAuth 获得 it 批准。(Push 说，另外三分之二是“由员工直接提供的，没有 IT 监督或可见性。”)它没有为基于谷歌的应用集成提供相应的数据。

Push 表示，自不到一年前推出以来，它已经在 it 仪表盘上添加了近 500 款 SaaS 应用。换句话说，除了其客户使用的最受欢迎或批准的应用程序，该客户群中的员工还集体获得了另外 500 个应用程序，这代表着一场潜在的泄露噩梦。
