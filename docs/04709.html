无代码/低代码空间在过去几年中迅速扩大。正如我们从本月早些时候在对活跃在该领域的投资者的最后一次调查中了解到的那样，该技术使现代软件开发的途径民主化，但仍有一些问题需要解决。然而，大规模采用仍然受到阻碍:许多组织更喜欢从零开始构建，完整的端到端解决方案仍然无处可寻。

为了更深入地了解这个领域的技术方面，我们决定与一些引领无代码/低代码革命的技术专家交谈。

首先，似乎无代码/低代码工具对从事 it 工作的人数没有太大影响。Quickbase 的首席信息官 Deb Gildersleeve 表示，无代码/低代码的传播将有助于它专注于更高要求的任务。

“我们认为，IT 需要花更多时间来思考技术如何影响人们。消除琐碎耗时任务的工具有助于节省时间和精力，专注于更大的问题，让人们的生活更轻松，”她说。

无代码/低代码在一定程度上招致了技术债务，这已经成为一个主要的讨论点。Retool 的创始人兼首席执行官许豪恩认为，目前与其说是消除技术债务，不如说是选择在什么情况下债务是可接受的后果。

“我们能做的是决定哪些技术债务值得低代码带来的灵活性，哪些技术债务达不到这个门槛。例如，从我们的角度来看，让非技术人员能够设计和定义他们自己的界面是非常值得的，”他说。“另一方面，我们发现让非技术开发人员管理集成、数据流、业务逻辑和 CRON 工作，而没有某种程度的技术监督或防护，不值得承担技术债务。”

在本次调查中，我们与高管们讨论了他们最喜欢的无代码/低代码工具，这些开发套件对 IT 就业市场的不同影响，以及如何确保最小的技术债务，等等。

OutSystems 首席技术官帕特里克·简

目前你管理的工作有多少是通过无代码/低代码完成的？2031 年，还会要求开发者学习如何编码吗？

作为一个 20 年前开创了这一类别的低代码平台的 CTO，我所做的一切都与低代码以及该工具如何帮助商业领袖和开发人员构建他们所需的严肃应用程序有关。事实上，我们使用我们的低代码平台构建了尽可能多的自己的堆栈——对于我们的 UI 工具，我们有一些基本的高代码组件，其余的大部分 OutSystems UI 平台是用低代码构建的。

展望未来，总是需要具有高级代码专业知识的开发人员。不要认为这些工具消除了学习如何编码的需要，它们应该被认为是一种消除应用程序开发中存在的长期、无差别的维护工作负担的方法。低代码应用程序开发平台将处理这种无差别的工作，开发人员不必担心这一点。

你最喜欢的无代码/低代码工具有哪些？

有大量的无代码/低代码工具可以满足一系列开发人员的需求。这一类别中的许多工具解决的问题范围很窄，当它们需要扩展或随着时间的推移而发展时，往往会遇到障碍。

以我的经验来看，企业需要的是一个结合了敏捷性、性能和规模的平台，以实现高质量和安全的应用。一个包含开发人员的高表达能力和高生产力，并提供完整的精英 CI/CD 功能。

只要你有软件，就永远需要有能力从头开始设计软件的人。 Deb Gildersleeve，首席信息官，Quickbase

公司应该寻求企业级低代码工具，使他们能够构建关键应用，解决严重的业务挑战，同时优化安全性、合规性和规模，并消除遗留代码和集成等问题。

无代码/低代码的兴起是否影响了在 IT 行业工作的人数？

无代码/低代码工具不会影响在其中工作的人数。相反，他们优化了 IT 的角色，帮助使遗留系统现代化，消除技术债务，并使他们能够快速构建应用程序。

它帮助 It 专业人员授权他们自己的团队构建他们需要的应用程序，而不是依赖现成的选项，并允许团队和开发人员专注于更有意义的创造性工作，而不是维护过时的后端系统或做琐碎的任务。

无代码/低代码工具的一个区别在于，它们是否能够以适当的治理和法规遵从性来体现 CI/CD 过程，从而确保公司将对不同生产和非生产环境的特权访问分开。

随着越来越多的企业采用低代码平台，我们将看到 IT 部门变得越来越重要，因为他们通过定制应用程序以更高的速度和敏捷性增加了更大的价值。这一领域正在迅速发展，有助于缩小我们面临的巨大发展人才缺口。

你认为除了无代码/低代码之外，还有哪些服务可以提供，让它成为更有吸引力的应用开发包？

我们看到的一个最大趋势是，需要构建可以快速扩展到数十万甚至数百万用户的严肃应用。许多开发人员面临的问题是，这样做需要开发在互联网规模的云中运行的应用程序，使用现代云架构和技术的最佳实践，这可能非常复杂和昂贵。
