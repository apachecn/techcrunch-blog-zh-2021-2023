NeuraLegion 成为 Bright Security 并融资 2000 万美元 A 轮 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/02/dynamic-application-security-testing-service-neuralegion-raises-20m-series-a-round/

专注于动态应用安全测试和识别业务逻辑问题的初创公司 NeuraLegion ，今天宣布更名为光明安全。此外，该公司还表示，已在由 Evolution Equity Partners 牵头的 A 轮融资中融资 2000 万美元。之前的投资者 DNX 风险投资公司、J-Ventures、Fusion Fund 和孵化基金也参与了这轮投资。

当 NeuraLegion/Bright Security 在 2018 年推出时，该公司的重点是一种人工智能驱动的 fuzzer ，它将帮助开发人员发现并减轻他们代码中潜在的安全漏洞。

虽然该团队为这项服务找到了一些潜在客户，但它几乎太好了。“喜欢它的人真的很喜欢它，” Gadi Bashvitz 解释道，他于今年一月成为 Bright Security 的首席执行官。“但很少有人能真正实施它，因为它是一种非常有害的产品，本质上，因为它期待零天，但却发现零天，所以造成了灾难。它会使任何运行它的目标崩溃，这意味着如果您在生产或接近生产的环境中运行它，您将不得不一次又一次地重新创建该目标。”这需要不同团队之间的大量协调——这不是大多数企业擅长的——因此该公司的许多潜在用户无法适应。

当该团队在 2020 年筹集到 470 万美元的种子资金时，它已经将重点转移到了动态应用安全测试上；即通过模拟外部用户的攻击来发现 web 应用程序和 API 中的漏洞。这仍然是 Bright Security 今天关注的重点，正如 Bashvitz 所指出的，该公司无意扩展到相关领域，如静态应用程序安全测试。该公司表示，现在有超过 4000 个组织在使用它的产品。

图片来源:光明保安

随着开发人员越来越多地承担安全测试的任务(除了 DevOps 模型带来的其他一切之外)，Bright 安全团队将重点放在使其工具尽可能易于使用和无摩擦上。它与现有的 CI/CD 管道集成，可以扫描从 web 应用程序到 Rest、SOAP 和 GraphQL APIs 的任何东西。该团队以避免误报而自豪，这经常导致开发人员不必要的浪费时间兔子洞。

至于更名，巴什维茨指出，有很多公司都有类似的名字，而“Bright”更好地代表了该公司的业务。他说:“我们试图为所有这些团队提供启发——既让他们看到漏洞，又试图为如何正确地进行(动态应用程序安全测试)提供指导。”。他还指出，DAST 近年来已经有点失宠(他称之为“”c . y . a .”工具)，尽管需要这些工具。

Evolution Equity Partners 的合伙人 Karthik Subramanian 表示:“应用安全已经准备好进行彻底改革，以满足当前市场的需求。“如果开发人员和 DevOps 团队无法使用专门为 AppSec 团队构建的工具，这些工具就已经过时了，我们的工具必须不断发展，以确保安全性不是一个团队的工作，而是一项共同分担的共同任务。”Subramanian 将加入光明保安公司的董事会。
