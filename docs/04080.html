遇见指定，一家为 Figma 和 GitHub 创造共同语言的创业公司。指定充当设计令牌和资产的中央存储库和 API。换句话说，设计者可以更新规范的 Figma 文件，更改将反映在 GitHub 存储库中。

这家初创公司筹集了 460 万美元(€400 万美元)的种子资金，由 Eurazeo 牵头。Bpifrance 的数字风险基金、360 Capital 和 Seedcamp 也参与其中。一些商业天使也投资了该公司，如 Clément Vouillon 和 Didier Forest。

当组织开始认真对待设计时，他们希望创建一个按钮、图标、字体、徽标、颜色等具有统一风格的设计系统。例如，在脸书、Twitter、Gmail 或 Pinterest 上，登录页面看起来完全不同。

然而，对于设计人员和开发人员来说，这通常仍然是一个手动过程。设计者用融合或概念中的设计符号和资产创建文档页面。然后，开发人员必须手动检查文档，确保他们使用的是最新的元素。

图片来源:指定

指定充当设计令牌和资源的中央存储库。您首先将 Specify 与一个或多个源以及一个或多个目标连接。

例如，您可以直接从 Figma 文件中获取信息和数据。设计者可以在 Figma 中更新一些东西，更改将反映在指定的存储库中。指定行为作为真理的唯一来源。

但是更改也可以更快地出现在您的应用程序中。当有更新时，Specify 可以在 GitHub 上自动创建一个 pull 请求——还有一个命令行界面。开发人员可以一键接受更改。通过这种方式，无需任何人工操作即可更新颜色、徽标、字体等。

Specify 不想将其产品局限于 Figma 和 GitHub。未来会有更多的数据源，比如 Dropbox 和 Google Drive。而 Specify 将能够更新更多目的地，比如 idea。将一个设计变更推向多个目的地的能力可能特别有用。

产品愿景很明确。Specify 希望成为设计团队的粘合剂。“通过我们的方法，我们认为我们的产品非常类似于细分市场，但对于设计来说，”联合创始人兼首席执行官 Valentin Chrétien 告诉我。
