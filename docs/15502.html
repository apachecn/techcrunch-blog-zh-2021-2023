OpenAI 推出用于语音到文本转录和翻译的 Whisper API | TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/01/openai-debuts-whisper-api-for-text-to-speech-transcription-and-translation/

为了配合 ChatGPT API 的推出，OpenAI 今天推出了 Whisper API，这是该公司 9 月份发布的开源 Whisper 语音到文本模型的托管版本。

Whisper 的价格为每分钟 0.006 美元，是一个自动语音识别系统，OpenAI 声称它可以在多种语言中实现“强大的”转录，以及从这些语言翻译成英语。它接受各种格式的文件，包括 M4A，MP3，MP4，MPEG，MPGA，WAV 和 WEBM。

无数组织已经开发了非常强大的语音识别系统，这些系统位于谷歌、亚马逊和 Meta 等科技巨头的软件和服务的核心。但 OpenAI 总裁兼董事长格雷格·布罗克曼(Greg Brockman)表示，Whisper 的不同之处在于，它是根据从网络上收集的 68 万小时多语言和“多任务”数据进行训练的，这导致对独特口音、背景噪音和技术术语的识别能力有所提高。

“我们发布了一个模型，但这实际上不足以让整个开发者生态系统围绕它建立起来，”布罗克曼昨天下午在与 TechCrunch 的视频通话中说。“Whisper API 是开源的大型模型，但我们已经优化到了极致。这要快得多，也极其方便。”

布罗克曼认为，当企业采用声音转录技术时，有很多障碍。根据 2020 年 Statista 的一项调查，公司将准确性、口音或方言相关的识别问题和成本列为他们尚未采用技术转语音等技术的首要原因。

然而，Whisper 也有其局限性——尤其是在“下一个单词”预测方面。由于该系统是在大量嘈杂的数据上训练的，OpenAI 警告说，Whisper 可能会在它的转录中包含实际上没有说出的单词——可能是因为它既试图预测音频中的下一个单词，又试图转录音频录音本身。此外，Whisper 在不同语言之间的表现并不相同，当涉及到在训练数据中没有得到很好代表的语言时，错误率更高。

不幸的是，最后一点对语音识别领域来说并不新鲜。偏见长期以来一直困扰着最好的系统，2020 年斯坦福大学的一项研究发现，亚马逊、苹果、谷歌、IBM 和微软的系统在白人用户比黑人用户的情况下犯的错误要少得多，约为 19%。

尽管如此，OpenAI 认为 Whisper 的转录能力正被用于改善现有的应用程序、服务、产品和工具。人工智能语言学习应用 Speak 已经在使用 Whisper API 来支持一种新的应用内虚拟说话伴侣。

如果 OpenAI 能够以一种主要的方式进入语音到文本的市场，这对微软支持的公司来说可能是相当有利可图的。根据一份报告，到 2026 年，这部分市场价值将从 2021 年的 22 亿美元上升到 54 亿美元。

“我们的情况是，我们真的想成为这种普遍智能，”布罗克曼说。“我们真的希望，非常灵活地，能够接收你拥有的任何类型的数据——无论你想完成什么类型的任务——并成为这种关注的力量倍增器。”
