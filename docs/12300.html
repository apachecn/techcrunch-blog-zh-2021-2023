微软的 SQL Server 2022 完全是关于 Azure | TechCrunch 的

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/16/microsofts-sql-server-2022-is-all-about-azure/

微软今天发布了 SQL Server 2022(T1)，这是其数据库软件的最新版本，最初于 33 年前推出。微软将这一版本描述为“迄今为止最支持 Azure 的 SQL Server 版本”，并连接到 Azure Synapse Link 以实现对数据库的实时分析，Azure without以实现数据治理和灾难恢复。在 Azure SQL Managed Instance 的帮助下，这一版本在许多方面都是该团队几年前开始的云连接基础工作的高潮。

“从一开始,[ SQL Server]的愿景就是——数据库非常复杂——如何让它变得极其简单？从很多方面来说，我认为这是它持续这么久以及我们如何发展它的关键原因，”微软 Azure Data 公司副总裁罗汉·库马尔告诉我。“对于 SQL Server 2022，我考虑的一件大事是，我们已经将它完全云连接到 Azure。”

他指出，虽然内部工作负载的迁移正在发生，但微软的客户都在以非常不同的速度迁移，出于多种原因，有些客户可能永远不会迁移到云。他认为，这就是为什么该公司总是押注于混合方法，但这也是为什么许多客户开始询问他们如何才能获得云的价值，而不必实际上将所有数据都转移到云中。“这真的是为什么我们投资把它做成一个云版本的关键论点，”Kumar 说。

图片来源:微软

一个很好的例子是新的灾难恢复功能，它允许用户在 Azure 上的 SQL 托管实例中复制他们的数据，并将其用作他们的主要本地 SQL 服务器的备份，这应该可以在主服务器停机时轻松地故障转移到主服务器。

Kumar 还指出，借助 Synapse Link，SQL Server 用户现在可以对其数据库进行实时分析，而无需设置复杂的基础设施。“你只需要勾选一个框，然后说:‘近乎实时地复制这些数据。’它将数据导入 Synapse，你可以让你的 Power BI 报告读取这些数据，整个管道就是为你构建的，”他说。

对于确实有混合设置的公司，对权限数据治理服务的支持现在使他们能够设置他们的策略，无论数据是驻留在云中的 SQL Server 中还是本地。

除了致力于新的云连接功能，该团队当然还致力于提高数据库的整体性能、稳定性和安全性。这项工作的核心是数据库的智能查询处理引擎，至少在这个版本中是这样，例如，它现在可以在许多更复杂的场景中优化查询。

同样有趣的是，通过 Azure Arc 为 SQL server 提供了一种新的按需付费计费模式，Azure Arc 是微软管理云和内部资源的平台。使用到 Azure Arc 的连接，这是 SQL Server 2022 安装过程的一部分，本地用户现在还可以选择支持云的计费来管理消费高峰或临时用例。

正如 Kumar 所指出的，尽管目前存在各种竞争，但 SQL Server 的使用仍在继续增长(尽管在某种程度上，这是由现有客户扩大其使用范围所驱动的)。

SQL Server 的新版本现已正式发布，包括免费开发版和速成版。
