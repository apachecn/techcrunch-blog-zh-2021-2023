当 GGV 资本伸出手来，说它已经建立了一个 API 优先公司的指数，我很好奇。创业公司和其他公司的风险指数近年来被证明是有用的工具，由 Bessemer 的云指数引领，它现在也是纳斯达克的可交易 ETF。

但 GGV 的想法是专注于创业，这意味着它比贝塞麦的构想更符合 TechCrunch 的胃口，所以我和投资团队切尔希·泰勒、蒂芙尼·勒克和杰夫·理查兹通了电话。不过，首先让我们谈谈定义。

什么是 API 优先的创业？

对于什么是 API 优先，或者说 API 主导的创业，你会有自己的定义。对于我们今天的工作，它是任何一家通过 API 交付其主要价值主张的初创公司，比如 Twilio，或者是为了使用 API 来促进特定的数据传输而构建的，比如 AgentSync 等。这是我们能做到的最简单的事了。

在我们分析GGV 为了坐在 API 主导的创业热潮之上而建立的之前，请记住软件市场正在慢慢超越 SaaS 。软件即服务本身就是软件商业模式的重组，这导致几乎所有的初创公司都将代码作为托管服务来销售，而不是单一的销售产品。但是，尽管 SaaS 已经成为今天创业公司事实上的典范，但通过按需定价提供软件交付服务的推动力越来越大。或者，我们可以称之为“正是你所需要的”定价。

Twilio 已经证明这种模式很受欢迎，适用于 API 优先的初创公司，类似于 Salesforce 为 SaaS 证明的模式。无论如何，如果软件市场继续转向——请注意，是缓慢地——按需定价，而不是传统的 SaaS 定价，API 优先的初创公司可能会站在商业模式历史的正确一边。

GGV 建造了什么？

API 优先的上市公司数量不多，这意味着关于这一群体的有用数据有些匮乏。坦率地说，SaaS 的一个好处是，由于它的成熟，有许多公司使用的商业模式已经跨越了公私界限。并不是说每个 API 优先的初创公司都不按照 SaaS 线定价，但是有一个宽松的定价划分，这意味着我们不能使用 SaaS 指标作为 API 主导的初创公司的一般比较。

即使我们可以，我们也不想。根据 Taylor 和 Luck 的说法，API 优先的创业公司与 SaaS 的创业公司有不同的“DNA 类型”，因为它们通常以社区为中心，并出售给开发者而不是公司。

SaaS 和 API 优先的创业公司在获取客户方法上的差异不仅仅是风格上的差异。通过让开发者加入，API 优先的创业公司可以在内部减少销售，从成本角度改变他们的运营模式。GGV 三人组指出，API 优先的初创公司往往会随着客户的增长而增长，由于使用率的有机增长，这可以导致有效的追加销售。

但是没有那么多的上市公司，我们不得不依靠私有市场的数据。因此，GGV 编制了一份以 API 为导向的私人公司公开名单，追踪它们的风险投资业绩，并打算每季度更新一次。随着时间的推移，越来越多的名字会被加入，最初大约有 30 家风险投资公司。泰勒说，风险投资小组然后“根据我们从公开渠道收集的融资总额对公司进行排名”。

今天API 优先列表读起来像一个草稿，一本将包含更多条目的分类账。TechCrunch 希望 GGV 利用其风险投资者的地位，礼貌地从其名单上的初创公司中提取数据，允许用更多的指标来扩大其数据足迹。

为什么是 GGV，而不是 TechCrunch？零初创公司将与我们分享他们最近一个季度的净美元留存。但也许是和 GGV？如果匿名的话？坦率地说，我愿意牺牲一只小动物来换取这些数据，所以我们希望 GGV 能够收集大量的数据，与公众分享。

今天，该榜单指出，GGV 跟踪的几十家 API 优先的公司作为一个整体已经筹集了 120 亿美元，其中大约 50 亿美元将在 2021 年筹集；当然，整个风险市场加速发展，但这仍然是去年总量的一大块。GGV 还指出，其名单中有 40%的公司专注于金融科技。在分析了我们上面的 API 创业摘要后，这个数字感觉很低。

今后

为什么要关心 API 主导的创业公司？除了越来越多接触 TechCrunch 的早期公司正在追求这种模式，所以如果你关心早期技术，它的成员很重要。但更重要的是，如果我们能够建设一个美好的未来。

无代码和低代码服务的增长让我开始思考。当我们可以将全球 API 创建的繁荣融合到无代码服务中时，会发生什么？多久我才能使用无代码工具和构建我自己的应用程序，通过 API 带来大量简洁的信息和功能，而无需启动 IDE？

当那个未来到来时，你卑微的抄写员将会忙于创建糟糕的混搭，也许会将死亡金属巡演日期与食品交付订单量混合起来，以跟踪重金属粉丝在表演后喜欢吃什么，一旦威士忌完全站稳脚跟。你可以想出自己的组合。

不管怎样，今天我们有了更多关于 API 主导的创业公司的数据。为更多的时间干杯。
