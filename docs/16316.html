GitHub 的 Copilot 不仅仅是代码完成，还增加了聊天模式等等

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/22/githubs-copilot-goes-beyond-code-completion-adds-a-chat-mode-and-more/

GitHub 今天宣布了 Copilot X 计划，这是其流行的 Copilot 代码完成工具的扩展，该工具最初于 2021 年推出预览版。据此，微软旗下的公司为 Copilot 推出了一个以代码为中心的聊天模式，帮助开发人员编写和调试他们的代码，以及用于拉式请求、人工智能生成的关于文档的答案等的 Copilot。不出所料，这些新功能由 OpenAI 的 GPT-4 提供支持，尽管值得注意的是，主要由于延迟原因，代码完成工具仍然基于 GitHub 的 Codex 模型，该模型源自 GPT-3。

“随着新模式的上线，我们问自己:下一步是什么？副驾驶的下一步是什么？GitHub 首席执行官托马斯·多姆克(Thomas Dohmke)告诉我:“我们相信，对于自动完成，我们已经确定了这种情况。

图片来源: GitHub

GitHub 的下一步自然是将聊天添加到开发者体验中，这并不令人惊讶。Copilot Chat 今天在预览版中发布，将在 Visual Studio 和 VS 代码中可用，稍后将支持其他 ide。

正如 Dohmke 指出的，这不仅将 ChatGPT 引入了 IDE，还将 IDE 的上下文引入了 ChatGPT。例如，这意味着您可以在编辑器中突出显示代码，然后向 Copilot 询问这些代码片段，使用它来重构这些代码，并在必要时请求它帮助您调试代码(因为它也知道终端中的输出)。GitHub 通过一些聪明的提示工程实现了这一点，这些提示工程尽可能多地填充了上下文。

当然，你也可以让 Chat 帮你写代码。例如，在一个演示中，GitHub 展示了如何让它构建经典的贪吃蛇游戏。

图片来源: GitHub

“我是一名开发人员，但我也是首席执行官。所以我真正开发代码的时间相当有限。前几天晚上我一直在玩它，它真的很有趣。它将乐趣带回了编程，因为你不必去找教程，也不必找出哪一个是正确的教程，你只需继续前进，”多姆克说。“我敢肯定，当你给学生和孩子们提供访问权限时，他们一定会大吃一惊。它变成了这个让人上瘾的东西，因为你有了可以不断提问的东西。”

不出所料，Copilot Chat 将只关注编程问题。它将很快关闭其他任何问题。

Dohmke 认为，虽然 autocomplete 已经让开发人员大大提高了他们的工作效率，但这个新的 Copilot 将使开发人员的工作效率提高 10 倍。

图片来源: GitHub

作为 Copilot X 计划的一部分，GitHub 也将 Copilot 引入命令行。与 Warp 对其终端所做的类似，GitHub 现在也通过帮助开发人员编写命令和循环来帮助他们使用命令行——并找到那些通常深埋在手册中的模糊功能标志(或至少一两个堆栈溢出搜索)。

还有更多。开发人员现在还可以注册一个新的 Copilot 功能的技术预览，该功能可以生成拉请求的描述，并自动标记它们。“没有人喜欢样板文件，”多姆克说。“无论是在代码层，使用 Copilot，还是在拉请求层，这都是枯燥和琐碎的工作。”他指出，这仅仅是开始。因为该工具已经扫描了代码，所以它还可以自动找到缺失的测试并生成它们，Dohmke 承诺我们将来会看到更多这样的测试。

图片来源: GitHub

Copilot for docs 也是新的，它丰富了 ChatGPT 现有的数据库，例如，你已经可以查询关于框架和语言的文档。但是 ChatGPT 的截止日期是 2021 年，这意味着这些信息并不总是最新的，所以 GitHub 增加了更多的最新数据(类似于微软的 Bing 用 Prometheus 模型为搜索做的事情)。GitHub 从 React 的文档开始，从 Azure Docs 和 Mozilla 的 MDN 开始，开发者将能够在类似 ChatGPT 的界面中查询这些文档。然而，更有趣的是，GitHub 计划允许组织在他们的存储库和内部文档上使用这个工具。不过，这还不适合公众消费。

“一旦您在每个应用程序中都有了一个 Copilot，无论是在 Outlook、Word 还是您的代码编辑器中，您都可以利用您在这些环境中编写的东西，而且您还可以利用来自您组织中其他员工的信息，无论是其他开发人员还是营销人员，他们现在可以更轻松地撰写公告博客帖子，因为他们已经在 Copilot 中看到了开发人员放入 git repo 中的发布说明。[……]它消除了公司中的孤岛，我认为这就是我们可能会将生产力提高 10 倍以上的地方。”
