CloudQuery 筹集了 1500 万美元来揭开云基础设施设置的神秘面纱

CloudQuery 首席执行官兼联合创始人 Yevgeny Pats 帮助启动了这家初创公司，因为他需要一种工具来帮助他了解他的云基础架构资源，而他在公开市场上找不到这种工具。他基于数据工程最佳实践，构建了自己的基于 SQL 的工具来帮助准确理解他正在使用的资源。

在他发布开源解决方案后，他发现他遇到的问题也是大型组织面临的问题。Pats 表示，如今他们有 20 家公司至少有 1000 名员工在使用该产品。投资者注意到了这一点，奖励这家初创公司 1500 万美元的首轮融资。

Pats 认为，从数据的角度来看，云基础设施被锁定在过去，他希望通过 CloudQuery 将其推向现代。“从数据工程的角度和最佳实践来看，云基础架构世界有点像 2011 年。因此，我们将这一点引入云安全、合规性和成本跟踪，并使用所有数据工程最佳实践来构建这些用例，”Pats 告诉我。

随着该产品获得越来越多的关注，该公司一直在努力让人们了解三大云基础设施供应商——亚马逊、微软和谷歌——的所有服务。这不是一件容易的事情，因为他说，仅 AWS 就有大约 350 项服务需要该产品使用。

此外，他们正在对不同的相邻 SaaS 工具进行分层访问，如用于监控的 Datadog 和用于单点登录的 Okta。利用来自这些工具的数据，用户可以基于特定条件设置合规性或安全性规则。他看到客户使用该产品来确定一些事情，例如他们是否过度供应云基础设施资源，找到不属于某个项目但公司仍在付费的“僵尸”资源，并根据 CloudQuery 数据帮助建立安全和合规性规则。

作为一个开放平台，公司或外部开发人员可以在他们认为合适的时候增加新功能，并且每个人都可以使用这些功能。虽然有一些商业公司正在研究可见性问题的某些方面，但 Pats 表示，作为一种开源产品，CloudQuery 在解决这一问题方面处于独特的地位。

“用户很快就会意识到，没有哪家供应商能够真正支持所有这些 API。每个供应商都缺少一些他们需要的 API 和数据。因此，实现这一目标的唯一方法是真正采用开源方式，他们可以通过编写自己的集成来为我们的官方集成做出贡献，”Pats 解释道。

该公司有 15 名员工计划至少在明年保持精益，也许会增加几名工程师，因为他们围绕产品建立社区。Pats 设想在某个时候推出该产品的 SaaS 版本，但这笔资金给他至少三年的时间，所以有足够的时间来实现收入。

Tiger Global 领投了该公司 1500 万美元的首轮融资，现有投资者 Boldstart、Haystack、Mango 和 Work-Bench 也参与了融资。该公司去年筹集了 350 万美元的种子资金，迄今总共筹集了 1850 万美元。
