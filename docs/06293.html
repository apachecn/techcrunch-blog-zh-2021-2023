Slai 做出了明智的选择

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/10/slai-seed-round/

Slai 为开发人员围绕机器学习设置做出了明智的选择

在一个有 Twilio 存在的世界里，你不会梦想在 193 个国家建立自己的短信堆栈，天知道有多少电信运营商。机器学习(ML)的情况并非完全不同；除非 ML 是你的软件的核心——很可能不是——为什么你要浪费时间组装整个基础设施。为了解决这个问题， Slai 正在为机器学习建立一个开发者至上的平台。它为开发者提供了快速发布机器学习应用的工具。

“今天，机器学习仍然是一门研究学科，开发人员仍然很难构建自己的机器学习应用程序，”Slai 的联合创始人兼首席执行官 Eli Mernit 分享道。“我们希望开发者能够构建最先进的机器学习模型。”

该公司今天宣布，它筹集了 350 万美元的种子轮投资，由 Tiger Global 牵头，Y Combinator，Charge Ventures，Uncorrelated Ventures，Twenty Twenty Ventures 和 Soma Capital 提供额外投资，天使投资人包括 Guy Podjarny 和 Jason Warner。

该公司的产品专注于让开发人员专注于机器学习模型，而不是占用大量时间但对应用程序本身没有直接贡献的所有周围混乱。

Slai.ai 在行动中的截图。图片来源: Slai

“该产品允许您连接数据源。这可能是您的数据库或 S3 桶，其中包含您想要发送到机器学习模型的数据。然后机器学习模型——只是一些 Python 代码——在数据中找到预测。我们已经将其封装在一个 API 中，该 API 对用户传入的输入进行验证，或者在输出返回给用户之前对其进行一些处理，”Mernit 解释道。“那些组件构成了机器学习应用。所以通常，如果有人手工做这些事情，他们必须自己建立一个网络服务器。他们必须建立一些版本控制系统，他们必须建立一些监控模型的方法。所有这些都意味着大量的无用功。我们所做的一切都是为了用户。他们需要关注的是他们的数据来自哪里，以及他们使用的是什么类型的模型。剩下的就交给他们了。简而言之，我们消除了机器学习开发过程中的所有粘合剂。”

该平台认为自己是 ML 的 GitHub 并使其易于将现有的机器学习方法应用到应用程序中。
