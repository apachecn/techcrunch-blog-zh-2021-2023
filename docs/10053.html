Dig 挖到 3400 万美元来解决云数据安全的碎片化世界| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/09/14/dig-scoops-up-34m-to-tackle-the-fragmented-world-of-cloud-data-security/

2022 年，存储在云中的企业数据量(相对于本地服务器)将达到 60% ，这是企业 IT 世界发展的一个信号。但是，如果你认为云=现代=更高效，当涉及到安全时，情况会更复杂:根据 IBM 的说法，多个孤岛意味着对可见性的多重挑战，造成安全漏洞，并导致去年一半的数据泄露发生在云中。

现在，一家名为 Dig Security 的初创公司正在开发专门解决这种复杂性的安全工具，该公司已经筹集了与其看到的需求相符的投资。

该公司评估并实时监控客户在基于多种云的环境中的数据资产，已经筹集了 3400 万美元的 A 轮投资，将用于继续扩展其平台。

首席执行官兼联合创始人丹·本杰明(Dan Benjamin)表示，这家初创公司如今的重点是公共云中的数据，而不是私有云。它与所有主要提供商(Azure、AWS、Google Cloud)以及数据仓库领域的知名公司(如 Snowflake 和 Databricks)相集成，它提供的服务包括数据安全状态管理(DSPM)、数据丢失预防(DLP)以及数据检测和响应(DDR)。

这项投资由 SignalFire 牵头，Felicis Ventures、Okta Ventures 以及之前的支持者网络安全专家孵化器和投资者团队 8 也参与了投资。Dig 在今年五月在秘密出现并宣布其 1100 万美元的种子期。Benjamin 说，迅速跟进的原因是种子轮实际上在宣布之前已经结束了一段时间；还因为 stealth 的出现让该公司从客户和投资者那里获得了大量的内部联系。

身份和访问管理巨头的战略投资部门 Okta Ventures 就属于这两种类型:Dig 与 Okta 的登录管理产品紧密集成，既可以监控潜在的恶意活动，也可以帮助建立更强的保护措施，在不良行为者控制之前阻止他们。

Dig 的创始人——本杰明、伊多·阿兹兰(R&D 副总裁)和加德·阿库卡首席技术官——都是重复创始人，他们在微软、谷歌、Mimecast、SAP 等公司都有过记录，正如本杰明解释的那样，有一个强大的第三方公司为云数据提供安全服务的论点，只是因为它已经成长为一个巨大的、但非常分散的市场，没有专有的“所有者”:由于微服务的流行，组织很少只使用一种服务，如 AWS 或 Azure 数据在多个实例上运行。

“一个典型的组织有 30 种不同类型的数据存储，分布在数万个实例中，”Benjamin 说。即使是较小的组织也可能有 10 种不同类型的数据存储。

这意味着，虽然云公司可能会开发强大的云安全产品，但无论数据在哪里，我们离这些产品有效地跨数据工作还有一段距离。

我还应该指出，Benjamin 还预测这种方法可能会随着时间的推移而改变:他曾在微软从事企业开发工作，知道这些大公司的兴趣是看看云数据安全公司(如 Dig)在未来几年将如何发展，并有可能抢购一个来增加这种功能。

然而，就目前而言，像 Dig 这样的公司显然有机会在市场上获得一些牵引力。

SignalFire 的风险合伙人 Nir Polak 表示:“Dig 在帮助 DDR 成为数据安全标准方面处于独特的地位。”作为 Exabeam 的创始人和董事长，他本人在网络方面有着丰富的经验。“远程工作的增加和云攻击的风险越来越大，要求实时数据安全功能，太多的组织仍然面临公共云背后隐藏的风险。SignalFire 很高兴支持 Dig 成为云数据安全市场上唯一一家跨任何云和任何数据存储提供实时数据保护的供应商。”

它不是唯一的参与者:许多其他人也在抓住机会，提供更全面的安全方法来弥补大多数企业环境的分散性。其他已经筹集到大笔资金的公司包括层流、 HYCU 、 vArmour 和木星龙。更大的科技公司正在进行收购，以将更多的云计算管理和安全功能引入他们的平台，包括谷歌收购 Siemplify ，以及 F5 收购 NGINX 。
