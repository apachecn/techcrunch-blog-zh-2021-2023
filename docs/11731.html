阿斯利康密码失误暴露患者数据| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/03/astrazeneca-passwords-exposed-patient-data/

制药巨头阿斯利康指责“用户错误”将一份凭据列表留在网上一年多，暴露了敏感患者数据的访问权限。

网络安全初创公司 SpiderSilk 的首席安全官 Mossab Hussein 告诉 TechCrunch，一名开发人员于 2021 年在代码共享网站 GitHub 上留下了阿斯利康内部服务器的凭据。侯赛因说，这些证书允许进入一个测试 Salesforce 云环境，通常由企业用来管理他们的客户，但测试环境包含一些患者数据。

一些数据与 AZ&ME 应用程序有关，该应用程序为需要药物治疗的患者提供折扣。

TechCrunch 向阿斯利康提供了暴露的凭据的细节，几个小时后，包含凭据的 GitHub 存储库就无法访问了。

在一份声明中，阿斯利康发言人 Patrick Barth 告诉 TechCrunch:“保护个人数据对我们来说极其重要，我们努力达到最高标准，并遵守所有适用的规则和法律。由于[原文如此]用户错误，一些数据记录在开发者平台上暂时可用。我们接到通知后，立即停止了对这些数据的访问。我们正在调查根本原因，并评估我们的监管义务。”

巴斯拒绝透露患者数据存储在测试环境中的原因，以及阿斯利康是否有技术手段，如日志，来确定是否有人访问过这些数据，以及哪些数据被泄露(如果有的话)。

对于像 SpiderSilk 的 Hussein 这样的安全研究人员来说，暴露或无意中发布到 GitHub 这样的网站上的用户名和密码等凭据是越来越常见的发现。在过去的几年里，这家初创公司发现了属于三星、有争议的面部识别初创公司 Clearview AI 的曝光数据；以及重启后的电影订阅服务 MoviePass。八月份，侯赛因发现了属于微软员工的凭证，这些凭证被无意中发布到了微软拥有的 GitHub 上。

侯赛因告诉 TechCrunch:“这不是我们第一次遇到工程师因人为错误在 Github 上泄露凭据的情况，这种情况一直在发生。”。“这些意外泄漏的风险在于它们随机发生，并且利用途径通常很简单(即，使威胁参与者的工作更容易)。"
