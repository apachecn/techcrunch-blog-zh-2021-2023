应对零日攻击的 CISO 剧本

乔纳森·特鲁尔撰稿人

Jonathan Trull 是的客户解决方案、架构和工程 SVP

Qualys

，他在网络安全领域拥有超过 18 年的经验，包括在工业界、学术界、美国海军预备役担任领导职务，以及担任科罗拉多州、Optiv 和 Qualys 的 CISO。

网络安全管理软件产品，殖民管道，MSFT 交易所——这些名字已经成为臭名昭著的网络安全事件的代名词。我们一直把每一个新的零日漏洞称为“叫醒电话”，但我们所做的只是集体按下贪睡按钮。

但是最新的广泛传播的关键漏洞 Log4Shell 的发现破坏了行业的假日季节。这是近年来出现的最大的网络安全威胁，这要归功于 Java 在 web 应用程序中几乎无处不在以及 Log4j 库的流行。由于其前所未有的规模，加上不容易发现的事实，从您的 it 环境中消除这一缺陷不是一件“一劳永逸”的事情。

全球各地的安全团队再次竞相修复软件漏洞，尽管攻击者已经开始以每分钟 100 次的速度瞄准易得之果——公共网络服务器。在被发现后仅仅七天，超过 180 万次攻击已经被检测到，这些攻击针对所有公司网络的一半。

你现在醒了吗？

我参加过许多 Qualys 客户(包括全球 19，000 多家企业，福布斯全球 100 强中的 64%)的紧急 Log4Shell 简报，很明显，处理持续不断的零日漏洞是当今安全团队面临的最大挑战之一。

就像清点一样，收集和分析威胁情报对于为安全团队采取有计划、有目的的措施提供必要的基础至关重要。

在应对像 Log4Shell 这样的零日漏洞时，确定修复和补丁的优先顺序可能会非常困难。以下是我们多年来学习和整理的应对安全威胁的一些步骤:

建立标准操作程序

创建详细的标准操作程序，其中包括针对漏洞类型量身定制的分步活动。

对于零日响应，必须包括以下信息:

  • 响应的流程。如果您需要帮助，美国网络安全和基础设施安全局(CISA)已经编写了一份非常好的指南。
  • 按照类型、严重性和所需的响应时间对漏洞进行分类。关键零日漏洞应有特定类别。
  • 为每个响应团队预先确定的服务级别协议。
  • 宣布和传达事故的程序(可参考事故响应标准操作程序)。
  • 跟踪、报告和结束事故并恢复正常运行的步骤。
