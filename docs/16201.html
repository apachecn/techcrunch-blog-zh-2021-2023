Twitter 测试基于政府身份的验证，新截图显示

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/20/twitter-testing-government-id-based-verification-new-screenshots-show/

Twitter 似乎正在为 Twitter Blue 用户测试一种新的验证流程，包括提交一个政府 ID。代码级洞察揭示了一个发送用户 ID 照片的过程，包括正面和背面，以及一张自拍照片以验证他们的 Twitter 帐户。该功能与其他仅向 Twitter Blue 订户提供的功能并列，如支持编辑推文、上传较长的视频、用文件夹组织书签和其他付费订阅特权。

产品情报公司watchly . ai，上周在 Twitter 的代码中发现了 ID 上传功能，但目前还不清楚它是否正在进行外部测试。该公司告诉 TechCrunch，它认为该功能正在美国进行测试，在 Android 版本的 Twitter 应用程序中发现了该功能。然而，它不知道有多少(或者是否有)Twitter 用户实际上看到了这个功能。

你可能还记得，在埃隆·马斯克(Elon Musk)的领导下，Twitter 对其验证流程进行了有争议的修改，从一个旧系统(如果用户是某种知名人士，如名人、政治家或其他公众人物，就要对其进行验证)转向一个用户只需支付验证勾号的系统。

该系统在首次推出时遇到了一些困难，因为用户验证了自己，然后开始假冒其他知名人士甚至公司，导致了混乱。Twitter 随后不得不暂停系统，重组系统，然后重新启动，增加保护。它还开辟了一条让企业验证自己的途径，将获得一个金色勾号，并表示将为一些高调账户贴上“官方”徽章。

尽管如此，即使改进后的系统需要一个电话号码才能被验证，但它已被证明容易受到假冒的威胁。正如《华盛顿邮报》今年早些时候报道的那样，Twitter 的系统在验证时没有要求提供带照片的身份证，这使得记者可以将经过验证的蓝色徽章添加到一个声称是美国参议员的虚假账户中。

如果 Twitter Blue 的验证过程中增加照片 ID 和自拍要求，将有助于打击假冒行为，如果该功能得到更广泛的推广。

图片鸣谢:Twitter 应用的 Watchful.ai 截图

在 Watchful.ai 提供给 TechCrunch 的截图中，Twitter 通知用户，新的验证过程将需要大约三分钟才能完成，他们的信息和图像将与第三方共享，以确认他们的身份。这表明 Twitter 本身并不处理验证过程，而是与一家提供商合作来完成这一重任。

尽管许多人仍然认为验证应该是提供给社区的一项服务，而不是一项付费服务，Twitter 将追逐影响力转变为付费功能的举措后来被 Meta 采用，因为它在广告之外寻求新的收入来源。上周， Meta 在美国的脸书和 Instagram 上推出了付费验证，此前在澳洲和纽西兰也有推出。它的系统允许用户按月付费购买它的蓝色勾号。然而，在 Meta 的情况下，验证还提供了假冒保护和直接访问客户支持，创作者和企业可能会发现这是值得的。

如果公开推出，基于政府身份的验证将是 Twitter 身份验证系统的一个显著变化，该系统如今更专注于提高 Twitter 蓝色订户在平台上的可见性，他们的推文在通知时间线中优先显示。但是，尽管 Twitter 现在可以验证某人是有真实电话号码的人，但这并不一定表明他们就是他们所说的那个人，正如《华盛顿邮报》的测试所表明的那样。
