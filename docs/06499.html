Storyblok 筹集了 4700 万美元来构建它的无头 CMS

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/17/storyblok-raises-47m-to-build-out-its-headless-cms-aimed-at-non-technical-users-like-marketers/

web 开发的世界继续变得越来越民主化——也越来越有创造性——这要归功于“无头”系统的创新，这些系统为网站的外观和功能提供了更多的灵活性:一个基于 API 的中间地带，介于使用刚性模板和从头开始构建和维护 web 技术堆栈的每一个组件之间。今天，一家专门为内容管理开发 headless 工具的初创公司宣布，在赢得一些重要客户后，将获得一大笔资金。

Storyblok 是一家成立于奥地利林茨的初创公司，它建立了一个无头 CMS，专为营销人员等技术和非技术用户设计，用于管理出现在网站、应用程序和其他数字界面上的内容，面向教育、商业、游戏和其他类型的出版商。它已经筹集了 4700 万美元，将用于继续扩展其 CMS 平台，使其具有更多功能。该公司的工具已经被其他 74000 家公司使用，包括网飞、阿迪达斯、T-Mobile、Happy Socks 和 Deliveroo，这些公司总共在它的基础上建立了大约 12 万个项目，包括需要定期更新的网站内容、游戏中的支线任务、应用程序中的广告功能等等。

B 轮由 Mubadala Capital 和 HV Capital 领投，3VC 和 firstminute Capital 也参与其中。在此之前，穆巴达拉于 2021 年 2 月发起了一系列 850 万美元的融资。Storyblok 迄今为止已经筹集了 5800 万美元，它没有披露其估值。

当今世界上有许多无头 CMS 提供商——像 Contentful 、 Prismic 、 Contentstack 、 Strapi 等公司——与 Storyblok 竞争，但首席执行官兼联合创始人张秀坤·安格雷尔认为他的公司代表了网络开发的新一波创新。

几年前，像 WordPress、Squarespace 和 Wix 这样的公司在用户构建网站时如何从动态模板范围中进行选择方面开辟了新天地。最近，一批新的创业公司利用围绕 API 的创新来插入复杂的技术流程，创造了一种新的更灵活的网站建设方法(“headless”一词是由 Commercetools 的创始人创造的，顾名思义，它最初将这一思想应用于电子商务网站的建设)。但这些仍然代表了更广泛的组织的一个痛点:在这个意义上，headless 可能消除了需要构建非常技术性和复杂的管理支付和数据库的后端组件，但没有构建或填充网站前端的技术细节。

这就是童话镇进入故事的地方。它的系统代表了新一代的无头开发，其平台也可以由营销人员等非技术人员使用，通过预先构建的模块作为“模块”来操作，这些模块可以更新，也可以由他们来操作。安格雷尔和他的联合创始人亚历山大·费格斯托弗(Alexander Feiglstorfer)在 2017 年提出了这个想法，当时他们看到当时使用的系统仍然需要开发人员建立和维护内容，最终仍然给组织带来了很多运营压力。模块是他们的解决方案:模块最初由开发人员编程，但随后可以被更新，并被任何人操纵。(开发初始模块的开发者可能来自相关公司，也可能来自第三方机构。Storyblok 本身在一个市场中提供了一个由 1000 个第三方组成的网络。)

Storyblok 吸引了很多人，部分原因是它解决了许多组织中存在的重大差距:最需要接触网站内容的人通常不是技术团队，而是管理内容的人。随着数字内容扩散到更广泛的格式和屏幕中——游戏和应用程序、智能手机和手表，这要求技术水平较低的团队提供更多投入，并给开发团队带来更多压力。

这反过来又给了设计人员和开发人员更多的时间来关注他们最擅长开发的组件。

“从第一天起，我们就是 Storyblok 的忠实信徒，自从我们的首轮投资以来，该公司成功扩大规模的速度令人瞩目。Mubadala Capital Ventures 的合伙人 Fatou Bintou Sagnang 在一份声明中说:“Storyblok 强大的有机牵引力是张秀坤和亚历克斯所打造的产品质量的真实证明，我们很高兴能继续与 Storyblok 团队合作。”

该公司并不认为自己是低代码和无代码工具浪潮的一部分:开发人员仍然需要参与构建营销人员可以更新的初始模块。

他说，安格雷尔认为开发无头结构并专注于这些模块是“其核心功能”的一部分。"所以转向构建无代码工具会感觉是对这个使命的背叛."然而，第三方已经插入到 Storyblok 系统中来创建这些工具，供其他人使用。(他说，一个例子是 Instant Commerce，它在 Storyblok 的基础上建立了一个集成，将 Storyblok 的内容管理功能的设计组件与 Shopify 的电子商务功能相结合。)
