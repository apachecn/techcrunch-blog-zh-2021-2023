GitHub 推出新的 Copilot 功能，让开发者用声音编码

GitHub 正在开发一种新工具，允许开发者用他们的声音编码。

在昨天的 GitHub Universe 年度大会上宣布，这项实验性功能与 Copilot 协同工作， GitHub 有争议的人工智能驱动的配对程序员，通过建议功能或代码行与软件开发人员合作——有点像 Gmail 的智能合成。

Copilot 于 6 月正式面向所有人推出，60 天免费试用后每月 10 美元或每年 100 美元。

GitHub 正在通过一个等待列表提供对新语音功能的访问，该列表现在对感兴趣的开发者开放，但本质上它将允许开发者通过“嘿，GitHub ”唤醒词激活 Copilot 的耳朵。它目前的范围有限，因为它只能与微软的源代码编辑器 VS Code 一起工作，但显然它正在努力扩展未来的东西。

据 GitHub 称，其新的语音助手可以理解 Copilot 的自然语言请求，以建议代码片段，或总结特定代码段的作用。但是，即使开发人员不想要任何代码建议，它也可以服务于其他实际用例，例如通过说类似“嘿 GitHub，转到第 34 行”的话来帮助他们导航代码库，或者甚至通过切换到 zen 模式来控制 IDE。

GitHub copilot 语音助手在行动。 图片来源: GitHub

虽然这仍然是由一个名为 GitHub Next 的研发团队开发的早期实验，但从可访问性的角度来看，它可能会产生重大影响，因为它减少了鼠标和键盘所需的交互量。

还不清楚 Copilot 是否能够与开发者对话，但根据 GitHub 发布的初步演示，似乎不能。不过，双向对话可能会很有用，例如，如果开发人员想要一段代码做什么的快速音频摘要，或者如果 Copilot 需要澄清用户提出的特定请求。

昨天在 GitHub Universe 的其他地方，微软旗下的公司也透露，它将很快在企业中针对 Copilot，推出一项新计划，允许企业在座位级别购买许可证——这将意味着额外的管理控制，以便公司可以管理和控制他们在整个组织中的 Copilot 部署。
