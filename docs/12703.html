AWS 为 Amazon Connect 添加了自动化代理监控

几年前，AWS 推出了面向客户服务的产品 Amazon Connect，将其放在企业应用程序的中间。这也将该公司置于与 Salesforce 和其他 SaaS 知名企业供应商直接竞争的位置。

当你在这个领域竞争时，你需要一些强大的功能，今天在拉斯维加斯举行的 AWS re:invent 上，AWS 首席执行官 Adam Selipsky 介绍了几个新功能，以帮助实现更多的自动化连接。

首先，该公司正在为 Amazon Connect 引入隐形眼镜下的新绩效管理功能，旨在帮助经理识别有问题的 CsA。该解决方案结合使用绩效评估表和机器学习驱动的语音分析来评估工作绩效。

它应该有助于识别可能需要额外培训或指导的代理。“这减少了联络中心经理识别绩效问题和帮助指导代理的时间，”Selipsky 今天解释说。员工可能会有不同的看法(机器人说我没有回答正确)。

在某种程度上，AWS 还引入了一项新功能，通过客户交互来指导代理，以便他们可以更快地以更一致的方式解决问题。这应该有助于减少错误的数量，以及对先前特性的需求(至少在理论上)。

图片来源: AWS

该公司还宣布了亚马逊连接预测，容量规划和调度的全面可用性，最初是在今年 3 月宣布的。它旨在帮助联络中心经理优化代理时间表，并确保他们有合适的人可用。

“Connect 是一个很好的例子，说明云如何消除限制，重新想象提供更好的客户服务等业务挑战，”Selipsky 说，这是 SaaS 公司一直都知道的事情，但对于倾向于专注于基础设施和平台的 AWS 来说，这是一种不同的方法。
