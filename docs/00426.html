GitHub 迎来了新的首席执行官

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/03/github-gets-a-new-ceo/

GitHub 首席执行官 Nat Friedman 将于 11 月 15 日卸任，成为微软旗下服务的名誉主席。最近刚刚成为 GitHub 首席产品官的 Thomas Dohmke 将接任首席执行官一职。

当微软在 2018 年收购 GitHub 时，开发者社区有相当多的担忧，认为这将是一个霸道的存在，并将代码共享和协作服务变成微软第一的平台。由于弗里德曼的开发者和开源背景，他在接受这份工作时带来了很多社区善意，GitHub 在他的三年任期内保持了独立性和平台中立性。

正如多姆克告诉我的，这在他的领导下不会改变。正如他在今天宣布之前的一次采访中指出的，弗里德曼曾要求他在收购后加入董事会。出生于德国的多姆克最出名的可能是 HockeyApp 的联合创始人兼首席执行官，该公司于 2015 年被微软收购。

“2018 年，Nat 从微软方面挑选我领导 GitHub 交易，”自收购 HockeyApp 以来在微软开发者部门排名上升的多姆克说。“对我来说，这真是一段激动人心的时光。这有点像回到了我的首席执行官时代，因为我领导着所有不同的交易职能。GitHub 在产品和工程世界之外还有很多方面需要解决。然后 Nat 邀请我和他一起加入 GitHub，我欣然接受了。我在 GitHub 负责战略项目和特殊项目。”

新任 GitHub 首席执行官托马斯·多姆克

他的第一个项目之一是在 2019 年初让私人回购对所有开发商免费。虽然大多数开发人员可能都意识到了这一点，但多姆克自己也承认，他没有弗里德曼担任首席执行官时的公众形象，但他强调，他作为开发人员和开源倡导者的背景符合过去 GitHub 首席执行官的总体路线。

“我希望人们在回顾过去几年 Nat 作为 GitHub 领导者的时候，会对所有让 GitHub 成为软件开发更好地方的东西感到兴奋和感激，”Dohmke 说。“我希望他们在回顾这个时代时，把它视为 GitHub 的伟大时代之一。在与微软交易期间，人们担心这对 GitHub 的独立性意味着什么。我们离微软越来越近了吗？我认为在过去的几年里，我们已经证明了我们将保持独立，我们将保持云中立，我们为开发者做正确的事情，我们将开发者放在第一位。希望他们将这种转变视为 GitHub 首席执行官作为开发者的传统的延续。以前的 CEO 们，我们都是开发者，Nat 是开发者，我也是开发者。我希望每个人都对我们公司的发展方向以及我们将如何创新和提高开发人员的工作效率感到兴奋。”

不过，他确实指出，他的整体风格有点不同，他更注重执行力，而不一定是一个有远见的人。

“在过去的几年里，我们一直密切合作，我们的许多风格都保持一致，我们几乎每天都有非常密切的对话。我认为 Nat 更有远见，能看到未来——我更有执行力，专注于现在必须发布的内容。但我认为我们都非常重视顾客，”多姆克解释道。

然而，展望未来，仍然喜欢在业余时间从事一些以移动为中心的私人编码项目的多姆克并不打算马上做出任何重大改变。相反，他描述的很多内容是 GitHub 已经走上的轨迹的延续——继续让 1 亿开发者进入其平台。在他看来，这有四个支柱。

首先，该公司将继续打造像副驾驶这样的人工智能项目。正如 Dohmke 指出的，在过去的几年里，GitHub 在软件开发的 CI/CD 方面做了很多工作——在 pull 请求之后发生的所有事情。他说，“有了人工智能和 Copilot，我们将把它带入内循环，带入开发者在笔记本电脑上做的事情，以及他们在编辑器中做的事情。”

他描述的第二个支柱是由 Visual Studio 代码支持的 GitHub 基于云的开发环境 Codespaces 。他说，这里的想法是确保 GitHub 可以满足开发人员的需求——让他们不必为每个新项目建立复杂的开发环境。

不出所料，第三个支柱是 GitHub 社区。“如果你想到今天的创作者社区，我们会想到抖音、YouTube 和 Substack，”他说。“但实际上，开源是最早的创意社区之一，可以追溯到 90 年代。如果你想想开源的早期，Linux 的早期——这都是关于创造东西和与世界分享东西，以及彼此合作来创造这个软件世界。因此，我们认为 GitHub 越来越成为一个创意社区，或者，如果你愿意，当它在 2008 年开始时，它是最初的创作者社区之一。"

在 Dohmke 看来，安全性是 GitHub 构建的第四个支柱——这显然是该公司一段时间以来关注的焦点。多姆克本人帮助领导了该领域的一些收购，包括 2019 年对dependent bot和 Semmle 的收购。

Dohmke 在 8 月份才成为 GitHub 的 CPO，看起来他没有想到会这么快就进入这个新角色。

“在 8 月份，我不知道我已经成为了首席执行官，但从作为朋友与 Nat 的交谈中，从作为 GitHub 合伙人与 Nat 的交谈中，我知道在某些时候他的心在别处，他想回到他的创业根源，更多地探索创业场景，”多姆克告诉我。“所以这已经有一段时间了，我们正在为有序的过渡做准备。”作为 GitHub 领导团队的一员，他已经参与了过去三年的所有产品评审。

弗里德曼证实，他想回到创业现场。他在今天的公告中写道:“考虑到我们已经取得的所有成就，以及我在微软的五年多的美好时光，我决定是时候回到我的创业根基了。”“驱动我的是让建筑商创造未来。我喜欢与开发人员一起工作并向他们学习，他们开发新工具和新项目，解决棘手的问题，用代码创造奇迹。这就是为什么我要继续我的下一个冒险:支持、建议和投资那些用技术创造未来并抓住当今一些最大机遇的创始人和开发者。”

很难责怪他。他在 2016 年将 Xamarin 出售给了微软，并停留了比大多数人预期的更长的时间。在某种程度上，这肯定是因为他被邀请担任 GitHub 的首席执行官。我们将密切注意他下一步的行动。毕竟，很有可能会有一只猴子作为吉祥物。
