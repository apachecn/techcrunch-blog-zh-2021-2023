Daily.dev 就像 Reddit 遇到堆栈溢出| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/23/daily-dev-is-like-reddit-meets-stack-overflow/

如果 Reddit 和栈溢出曾经创造了一个后代，它可能看起来有点像 Daily.dev ，也许在基因构成的某个地方还带有一点点黑客新闻。

Daily.dev 成立于特拉维夫，希望帮助志同道合的软件开发人员联系、分享知识并讨论他们的生态系统中发生的一切。这家成立三年的初创公司今天宣布了第一笔机构融资，从一系列支持者那里获得了 1100 万美元的种子资金，其中包括主要投资者 Aleph ，这是一家早期风险投资公司，此前曾投资过 WeWork 、柠檬水和 Houseparty 。

每日摘要

Daily.dev 可以通过多个渠道访问，包括一个 web 应用、 Android 、一个渐进式 web 应用( PWA )和一个浏览器 扩展。在最初的入职流程中，用户首先被要求选择他们感兴趣的主题，如数据库、开发工具或数据科学，然后他们使用自己的电子邮件地址或与脸书、谷歌、GitHub 或苹果相关的凭据进行注册。

Daily.dev:选择话题。图片鸣谢: TechCrunch /截图

然后，用户可以根据他们的偏好看到个性化的提要，其中可能包括操作指南、新闻文章、问题和一般性讨论，或者他们可以根据最受欢迎的帖子进行过滤。任何人都可以自由地给帖子添加书签，投票支持评论帖子，并发起他们自己的对话。

Daily.dev“最高票数”图片鸣谢: TechCrunch /截图

随着时间的推移，用户会根据他们在更广泛的 Daily.dev 社区中的投票积累一个“声誉分数。虽然这可以作为一个一眼的信任指标，但随着时间的推移，它也将被用来确定获得特殊特权，尽管该公司尚未宣布这将包括哪些类型的津贴。

图片来源: Daily.dev

虽然这种投票系统肯定会让人想起 Reddit 或 Stack Overflow 之类的东西，但 Daily.dev 值得注意的是，它没有向下投票按钮，这一举动部分是为了避免困扰其他类似在线社区的毒性和负面影响。

“建立一个健康和可持续的社区与管理情绪有很大关系，特别是消极情绪，”Daily.dev 联合创始人兼首席执行官 Nimrod Kramer 向 TechCrunch 解释道。"到目前为止，我们还没有建立一个向下投票的功能，主要是由于这个逻辑."

然而，值得注意的是，用户仍然会失去信誉点，例如，如果他们的一个帖子被其他用户举报，并且该帖子被平台版主禁止。克莱默还说，他们正在考虑其他方式来接受向下投票按钮的精神，而不损害他们试图培养的“健康”氛围。

“我们正在考虑无毒的方法来实施一个新的否决机制，让我们更好地个性化平台，也让开发者表达批评，而没有潜在的冒犯性影响，”他说。

不可否认的事实是，从 Discord 和 Hacker News 到 GitHub 和 Stack Overflow，网络上有很多以开发人员为中心的社区。那么，Daily.dev 到底希望在这个组合中加入什么呢？克莱默认为，大多数现任者要么过于笼统，要么过于专注于具体问题。

克雷默说:“有一个缺口——当开发者聚集在 Reddit、Discord 和 Twitter 等通用社交平台周围时，他们获得的专业内容与通用内容混合在一起，边缘化了为他们提供动力的特定领域和高度专业的见解。”"当他们在 GitHub 和 Stack Overflow 等开发者平台上合作时，他们获得的社交体验大多局限于特定的存储库和 Q & As。通过 Daily.dev，开发人员可以获得一个根据他们非常具体的偏好、价值观和语言构建的平台，并为学习、社区和协作提供广泛的机会。”

作为奖励，Daily.dev 已经在 GitHub 上提供了其平台的大部分内容，这是许多希望迎合软件开发竞争的公司的常见举动。开发人员喜欢开源工具，因为它允许他们修补底层技术，甚至自己贡献新代码。

“我们战略性地决定开放我们代码库的大部分源代码，以便与我们的全球开发者社区建立信任和透明度，”Kramer 说。“这是一个双赢的局面，因为我们的用户是开发者。开源可以让用户提供反馈、贡献功能和修复错误。”

生计

在商业模式方面，克雷默表示，Daily.dev 目前通过科技公司对开发者可能感兴趣的倡议进行付费推广来赚钱，目前转化为这样的推广讨论:

Daily.dev 推广帖子和讨论。图片来源: Daily.dev

至于它的用户群，Kramer 说 Daily.dev 有大约 100，000 名活跃的日常用户，包括来自所有大型科技公司的开发者，如苹果、谷歌、亚马逊、微软和 Meta。根据 Kramer 的说法，这些用户中约 90%也使用浏览器扩展，这意味着每当他们启动一个新标签时，他们都会看到 Daily.dev。

虽然 Daily.dev 在过去几年中依靠 bootstrapping 实现了增长，但其 1100 万美元的新现金注入将使其能够通过新功能来支持该平台。这包括一些被称为“Daily.dev squads”的东西，这实际上是开发者团体联合的私人空间。

克雷默说:“这项功能目前正在与我们数百名最活跃的高级用户进行封闭测试，并将于今年晚些时候公开发布。”。

在未来，Daily.dev 还可能通过寻求与开发者建立关系的公司来赚钱，例如，可能作为征求新产品反馈的门户，甚至可能是招聘。

“我们看到 B2B 领域有巨大的潜力，可以在公司和开发者之间进行有意义的、相关的交流，”Kramer 说。“科技公司渴望与开发人员特定的受众进行互动，而开发人员则在寻找新的解决方案，并以适当和尊重的方式提供给他们。”

尽管 Daily.dev 的三位联合创始人(克莱默、伊多·沙蒙和察希·马茨利亚)都不在以色列，但该公司称自己是“遥远的本土人”，16 名员工分布在 7 个国家。

除了首席投资者 Aleph 之外，Daily.dev 种子轮的其他支持者包括 Jibe Ventures、Secret Chord Ventures 和一些天使投资人。
