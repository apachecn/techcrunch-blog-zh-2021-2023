在亚马逊网络服务公司(Amazon Web Services)多年来发布的 300 多项服务中，没有一项因定价不当而被取消。

这不是偶然的。

基于使用的定价(UBP)带来了非常高的收入增长和产品采用率。任何在云中运营的公司都可以(也应该)采用基于使用的定价模式，这种模式由优化提供给客户的价值的最佳实践和框架支持。

我在 AWS 担任了几年的总经理，在那里我亲自监督了亚马逊云搜索和一些附属服务，因为它们在基于使用的定价模式的支持下，收入超过了 10 亿美元。后来，当我们推出 Amazon OpenSearch 时，同样的模式促成了它的巨大成功和采用。

基于使用的模型是唯一在云中有意义的模型。考虑到底层基础设施的弹性，任何置于顶层的东西都需要同样灵活，这包括定价。下面，我列出了在您的企业中开始使用基于使用的定价模型的六个步骤。

计量作为一种独立的产品存在的一个最重要的原因是，它被专门设计用来跟踪使用情况。

步骤 1:实施使用计量

许多公司犯了这样的错误:从定价模式开始，然后试图倒退到衡量使用情况。

这是错误的方法。第一步需要测量你所有的技术产品。如果你是一个白手起家的初创企业，在一开始就实施计量将会给你带来巨大的优势。

了解谁在使用什么、何时、何地以及使用多少，将有助于您了解所有职能部门和团队的宝贵见解，并使定价变得更加简单明了。

寻求专门构建的计量服务

不要陷入只计量定价计划规定的项目的陷阱。全面衡量您的技术堆栈。你应该先考虑计量，然后再考虑定价和计费，而不是从定价和计费倒退到计量。

这里有一个内化这一点的好方法:

假设您知道想要对 API 调用收费，并且您开始考虑一个按计数的 API 调用的分级定价模型。从这一点回溯到计量，您将得出这样的结论:您需要计量 API 调用的数量。

与向前计量方法形成对比。首先，您确定您需要使用工具 API 调用，因为这是您的客户如何使用您的产品的核心特性之一。然后，你问自己，什么是使用 API 调用的整体方法？
