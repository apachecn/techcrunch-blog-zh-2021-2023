五年后，谷歌仍然全力支持 Kotlin | TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/08/18/five-years-later-google-is-still-all-in-on-kotlin/

自从谷歌首次宣布将制作由 JetBrains 首次开发的用于 Java 虚拟机的静态类型语言 Kotlin ，这是在谷歌 I/O 2017 上编写 Android 应用程序的一流语言以来，已经过去了五年多。自那以后，谷歌进一步将 Kotlin 作为 2019 年编写 Android 应用程序的首选语言——尽管许多开发人员仍在使用 Java，但 Kotlin 正迅速成为为谷歌移动操作系统构建应用程序的默认方式。早在 2018 年，谷歌和 JetBrains 还联手推出了科特林基金会。

本周早些时候，我采访了谷歌的 Kotlin 产品经理詹姆斯·沃德，讨论了这种语言在 Android 生态系统及其他领域的作用，以及谷歌未来的计划。

毫不奇怪，谷歌希望随着时间的推移，所有 Android 开发者都将转向 Kotlin。“Android 上仍有相当多的 Java 程序在运行，”沃德说。“我们知道开发人员通常对 Kotlin 比对 Java 更满意。我们知道他们的工作效率更高，应用程序的质量也更高，因此让更多的人迁移他们的代码一直是我们关注的焦点。Kotlin […]与 Java 的互操作性使得人们可以逐渐地移动代码库，这将是一件很棒的事情，在这条路上，一切都是 Kotlin。”

然而，我们还没有完全到达那里，部分原因是 Java 生态系统太大了，它仍然有很大的引力。由于 Kotlin 与 Java 的互操作性，开发人员可以混合和匹配库，但要获得 Kotlin 的所有好处，开发人员必须留在 Kotlin 生态系统中。尽管所有的焦点都集中在 Kotlin 上，但值得注意的是，Android 平台的核心及其 API 仍然构建在 Java 之上。现在有一些 Android 库是用 Kotlin 编写的，但这显然只是整个平台的一小部分。

但是今天的 Kotlin 也不仅仅是 Android。在这一点上，服务器端 Kotlin 在 Google 变得非常普遍，到目前为止，Google 的内部代码库中已经有超过 850 万行 Kotlin 代码。该公司表示，这个数字现在每年翻一番。

事实证明，谷歌和 JetBrains 近年来一直在努力从零开始重写 Kotlin 编译器。这种新的编译器承诺会更快，并提供更好的接口，允许 ide 改进一些特性，如更好的代码提示和静态代码分析，现在它处于测试阶段，可能会在明年发布。由于该公司在这个项目上投入了相当多的资源——并且目前正在并行维护两个编译器——这意味着该语言本身的开发速度有所放缓。

“我们有意放慢了语言的变化速度，因为我们让两个编译器并行运行，”沃德说。“一旦我们发布了新的编译器，并且每个人都开始使用它，那么我们就能够在新的语言功能上投入更多。”

这些新特性中有上下文接收器，它目前仍然在一个标志后面。这将允许开发人员将参数传递给一个函数，例如，只编写一次连接到数据库的一些代码和值，然后在每次需要再次进行连接时再次使用相同的上下文。

至于科特林基金会，值得注意的是，直到现在，只有谷歌和 JetBrains 是这里的成员，这两家公司利用该基金会协调他们对科特林的投资。但沃德解释说，这两家公司正在寻求扩大新成员的基础。“我们有一个扩展到更大范围的计划，”他说。“这是发展 Kotlin 生态系统的核心:让 Kotlin 基金会的发展超越两家创始成员公司。”他指出，这两家公司还没有理由将该基金会置于像 Linux 基金会这样的组织之下，部分原因是因为如果只有两个成员，那就太过分了，但随着谷歌和 JetBrains 随着时间的推移带来更多成员，这种情况可能会改变。
