Grace 首次为 iOS 设备推出以隐私为中心的家长控制，该设备采用苹果的屏幕时间 API | TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/17/grace-debuts-privacy-focused-parental-controls-for-ios-devices-built-with-apples-screen-time-api/

一家名为 Grace 的新创公司正在启动，旨在让父母更容易监控和管理孩子在 iOS 设备上的屏幕时间和应用程序使用。虽然苹果提供内置的家长控制，但许多家长更喜欢基于应用程序的解决方案，而不是必须在苹果工具的设置中挖掘。此外，Grace 还为孩子们的屏幕时间安排提供了更多定制服务。例如，通过苹果的控制，父母只能为“停机时间”配置开始和停止时间，而不能设置应用使用应该受到限制的其他时间，如上学时间、家庭晚餐时间、家庭作业时间等。

Grace 也是首批使用苹果屏幕时间 API 的厂商之一，在去年的苹果全球开发者大会上介绍了。新的 API 允许开发者创建一个与苹果内置工具协同工作的界面，以扩展他们的功能。

要使用 Grace，父母需要在自己的设备和孩子的 iPhones 或 iPads 上安装该应用程序。然后，他们可以在一个直观的界面上远程配置各种屏幕时间控制，包括每日屏幕时间限制、应用时间限制、应用内购买限制等，并可以阻止浪费时间的应用等。有益的是，他们还可以创建屏幕时间表，更好地反映家庭自己关于孩子应该如何以及何时使用他们的设备的规则。

图片来源:优雅

例如，如果你想在睡觉时间和上学时间屏蔽抖音和 Snapchat，现在只使用苹果自己的工具是不可能做到的。相反，苹果的屏幕时间控制让父母可以设置基于时间的应用程序限制，或“停机时间”——在这段时间里，孩子只能使用父母批准的应用程序，并通过电话和短信与你允许的人交流，如妈妈或爸爸。在实践中，这意味着父母将停机时间配置为从就寝时间开始，并延长到晚上，以限制孩子熬夜使用他们的设备，或者他们可以选择仅在上学期间启用它。无论哪种方式，这种解决方案都不能完全满足家庭的需求。

与此同时，Grace 允许父母为他们的孩子创建所有不同类型的时间表，而不仅仅是一个停机时间表。这意味着父母可以在不同时间限制不同类型应用程序的屏幕时间。在学校，孩子们可以被允许访问教育应用程序和网络浏览器，但娱乐应用程序、社交媒体或游戏可能不允许。在家庭晚餐期间，所有的干扰都可以被关掉。父母可能还想配置与他们自己的家庭规则相关的其他屏幕时间表，如就寝时间、无屏幕周末、假期或他们选择的任何其他内容。

图片来源: Grace

Grace 的另一个好处是，它提供了内置过滤器的增强网站阻止功能，可以自动阻止超过 50，000 个成人、赌博、武器、酒精和毒品相关的网站，并且该列表会定期更新。苹果自己的屏幕时间工具只专注于阻止成人网站。

Grace 还增加了快速操作按钮，让父母暂停限制(“暂停”)或阻止所有应用(“锁定手机”)，而不必完全禁用屏幕时间设置。任何使用苹果屏幕时间的父母都会明白这里的好处。通常，家庭处于常规规则被暂停的位置——比如当孩子在通宵派对上熬夜时，当学校放假时，当他们生病在家时，或者当他们通过某种津贴制度获得额外的屏幕时间时，等等。但在其他时候，父母可能希望通过限制孩子使用他们的设备来让孩子接地，但实际上并不想把手机拿走，因为这也是他们在紧急情况下沟通的一种方式。然而，将他们锁在所有应用程序之外会起到惩罚的作用。

格雷斯的应用程序是由联合创始人莉安娜·卡诺瓦(产品和设计)和萨拉瓦特·卡诺夫(软件开发)创建的，他们都为他们的新项目带来了相关的经验。卡诺瓦之前在她的初创公司工作，专注于通过让儿童参与使用 AR 的交互式儿童书籍来减少儿童的手机成瘾。Khanov 曾开发过广告拦截器和隐私工具 1Blocker。他们指出，他们的新应用是引导式的，完全独立。

该团队希望苹果将更新 API，使他们在内置屏幕时间工具方面更具竞争力。他们说，今天的 API 缺乏向用户显示他们还剩多少时间和他们已经用了多少时间的能力。他们也希望看到一个改进的应用程序选择器可用——就像苹果提供的有搜索栏的那个。他们希望 API 也能像苹果自己的工具一样，包含设置通信限制的功能。

“尽管有这些限制，但我仍然认为与基于 MDM 的应用程序相比，这种解决方案更适合家长控制，”Khanova 在解释为什么 Grace 比使用 MDM 解决方案的现有应用程序更好时说道，MDM 解决方案是第三方屏幕时间工具缺乏 API 访问的早期解决方案。“[MDM apps]服务器可能会遭到黑客攻击，敏感数据可能会泄露，比如您孩子的电话号码、访问过的网站、搜索请求、安装的应用程序等。攻击者还可以远程删除设备，”她说。此外，“这些公司可能会收集大量数据，然后卖给广告商，”卡诺瓦警告说。

iOS 的屏幕时间管理工具有着复杂的历史。

2018 年 iOS 12 发布后不久，苹果推出了自己的内置屏幕时间跟踪工具和控件。但这家科技巨头随后开始大范围打击已经实现自己屏幕时间系统的第三方应用程序，称这些应用程序已经以危及用户隐私的方式这样做了。早期的应用程序提出了创造性的解决方案来解决用户对家长控制的需求，如使用 VPN 或移动设备管理(MDM)技术，后者是为企业使用而设计的，而不是面向消费者的应用程序。基于 MDM 的工具可以访问设备的位置，控制应用程序的使用，并设置各种权限——所有这些在锁定员工设备方面都是有意义的。但苹果辩称，同样的工具对儿童的隐私有风险。虽然这是正确的，但它也采取了不公平的行动，在它有自己的第一方解决方案时立即阻止这些企业继续运营，而不是允许它们过渡到更安全的解决方案，如 API。

更不用说，苹果多年来一直允许这些相同的屏幕时间应用程序及其后续更新，允许开发者建立业务，然后被苹果的政策变化摧毁。在 2020 年 7 月举行的反垄断国会听证会上，苹果首席执行官蒂姆·库克甚至被问及这一决定，高管再次为这一决定辩护，称其与消费者隐私有关。

苹果的解决方案是最终在 2021 年发布一个屏幕时间 API，它将允许开发者以一种安全、无 MDM 的方式在苹果现有功能的基础上构建。可以说，这种技术应该与政策的实施一起引入，而不是几年后。

然而，现在它已经可以使用了，像 Grace 这样的应用程序可以在没有被删除的威胁的情况下存在。

此外，在最近的苹果全球开发者大会上，该公司推出了几个新的屏幕时间 API 功能，包括以更保护隐私的方式显示应用程序和设备使用数据的能力，Grace 表示，它将用于绘制图表和显示类似于苹果自己的屏幕时间系统的活动。现在不仅可以在亲子关系中管理屏幕时间和限制，也可以为自己管理屏幕时间和限制——这是 Grace 打算在今年秋天晚些时候采用的另一个功能。

Grace 稍后还将推出锁屏小工具，以查看孩子们使用了多少屏幕时间，用于快速操作的新应用程序快捷方式和对 Mac 的支持。

Grace 目前是免费下载的，通过 19.99 美元/年的订阅可以获得高级功能。
