在第三方开发者越来越多的担忧中，Twitter 关闭了工具箱和其他项目

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/09/amid-growing-concerns-from-third-party-developers-twitter-shuts-down-toolbox-and-other-projects/amp/

图片来源:布莱斯·德宾/TechCrunch

从历史上看，Twitter 与第三方开发者社区的关系一直很混乱。但是在过去的几年里，该公司试图通过一些项目来安抚开发者社区，比如一个新的 API 版本和“Twitter 工具箱”集合来突出一些项目。

然而，在埃隆·马斯克的管理下，其中一些程序正在关闭——从 Twitter 工具箱开始。据知情的多名开发者称，Twitter 本周早些时候发送了一封电子邮件，称工具箱程序将于下周关闭 TechCrunch 已经看到了一份副本。

“感谢您成为 Twitter 工具箱试点计划的一部分。邮件中写道:“我们做出了一个艰难的决定，从 2022 年 12 月 15 日起停止使用 Twitter 工具箱。”。

Twitter 今年早些时候首次推出该程序，在相关时刻直接在用户的 feeds 上推广某些应用。这些应用包括调度和线程生成器/阅读器应用，如类型化、 Chirr 应用、线程阅读器和缓冲区；阻车器和保镖等安全工具；以及像 ilo 、 Blackmagic.so 、 Direcon Inc 这样的测量和分析工具。，随动杆和 Tweepsmap 。

当时 Twitter 的开发者平台产品负责人 Amir Shevat 也告诉 TechCrunch，这个社交网络也在考虑为第三方应用建立一个应用商店。

Twitter 磁贴试点关闭

今年 8 月，Twitter 宣布对新版 Twitter 卡“tiles”进行测试，允许出版商和开发者在推文中包含定制格式。这个想法是，当有人发布一个 URL 时，像图像、文本、视频或按钮这样的格式会自动呈现出来——就像 Twitter 卡片一样。

该公司也暂停了那个项目。根据一封发给开发者的电子邮件，Twitter 可能会考虑明年重新审视这个概念。

“我们已经得出结论，此时最好的办法是从这个试点项目中撤出，暂停与推文磁贴相关的更广泛的实验工作。我们仍然相信这个产品概念能够为我们的用户和像您这样的开发者提供价值。该公司在一封电子邮件中表示:“在我们进行 2023 规划时，我们将继续考虑这项工作，并可能选择明年重新审视它。”。

Twitter 开发者内幕计划

Twitter 去年引入了开发者内幕计划，以收集关于产品和 API 的直接反馈。虽然这个程序还没有被废弃，但是参与其中的人已经有一段时间没有收到 Twitter 的消息了。

参与该项目的开发人员特伦斯·伊登(Terence Eden)表示，内部人员参与了许多项目，包括标记有用的机器人和测试对话 API。他说，开发团队正在努力改进搜索并扩大 OAuth 的范围，这样开发者就可以更好地访问 Twitter 的某些部分。

Eden 和另一名开发人员 Somraj Saha 告诉 TechCrunch，在 Musk 接管后，随着项目经理开始离开公司，开发人员内部 Slack 小组的活动大幅减少。萨哈说，自 11 月的第三周以来，该组织就没有看到过来自 Twitter 用户的更新。因此，尽管 Twitter 没有发布官方声明，但该程序在目前的虚拟形象下已经寿终正寝。

API 和开发者关系的未来

马斯克接手几天后，Twitter 取消了其备受期待的 Chirp 会议，该会议计划在近十年后举行。

从那以后，开发人员就再也听不到 Twitter 团队的消息了。许多围绕 Twitter 构建的工具已经在扩展对 LinkedIn 和 Mastodon 等不同平台的支持。像 Tweetbot 和 Aviary 这样的第三方 Twitter 客户端的开发者正在开发乳齿象客户端，以迎合网络的日益流行。

由 Ev Williams 支持的 Twitter 线程制作应用typely ly告诉 TechCrunch，它正计划将 LinkedIn 支持扩展到其用户。该公司最近推出了一款人工智能机器人，可以帮助你编写更好的线程。它还计划改进机器人，使其能够更好地发布 LinkedIn 帖子。

另一个日程安排程序 Chirr App，最近提到它计划继续支持 Twitter，并在此过程中添加功能。但它对增加更多平台的连接持开放态度，并且已经在进行乳齿象集成。保镖，一个内容审核工具，说他们正在寻找添加集成到抖音和不和谐很快。该公司的联合创始人查尔斯·科恩(Charles Cohen)也告诉 TechCrunch，它正在考虑缩减与 Twitter 相关的开发。

TechCrunch 采访的开发者对 Twitter 的开发者关系及其 API 的未来有不同的看法和担忧。他们中的大多数人——包括 Twitter 开发者论坛上的那些人——目前担心公司与第三方开发者相关的项目没有一个清晰的路线图。他们中的很多人告诉我们，由于开发者关系团队失去了很多成员，联系 Twitter 关于开发相关的问题变得很难。

虽然马斯克强调了 Twitter 的工程主导方法，但他对开发者生态系统大多保持沉默。如果该公司过去几年来修复与开发商关系的努力付诸东流，那将是一种耻辱。
