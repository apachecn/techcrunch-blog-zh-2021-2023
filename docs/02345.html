免费工作的开源开发者发现他们有力量

原文：https://web.archive.org/web/https://techcrunch.com/2022/01/18/open-source-developers-who-work-for-free-are-discovering-they-have-power/

大多数人都没有意识到这一点，但是你每天使用的许多设备和应用程序都是建立在开源软件之上的，由一两个无偿的开发人员维护，他们修补错误并改进代码以回馈社区或作为一个激情项目。

以 cURL 为例，这个库使得软件访问另一个系统中的数据变得简单，比如在一个 API 中。该库几乎用于所有现代联网设备，从 iPhone 到汽车、智能冰箱和电视，以及 ，但它基本上由一个开发人员 【丹尼尔·斯坦伯格】免费维护了近三十年。

尽管许多开源项目包含在营利性软件和设备中，通常除了简单的确认之外没有任何补偿，但该系统基本上工作可靠。一些开源开发者能够通过像 GitHub 赞助商 和 给我买杯咖啡 这样的程序，与公司签订维护合同，或者在一家付钱给他们维护他们的库的公司工作，来成功地支持他们的工作——但这远非常态。

当出现广泛的安全漏洞时，该系统的不公平性往往会暴露出来， 例如 2021 年 12 月 Log4j Java 库中出现的 Log4shell 漏洞 ，引发了一系列严重的安全漏洞公告，影响了世界上一些最大的公司。

受影响的库的开发人员被迫夜以继日地工作，没有任何补偿，也不承认他们的作品最初是免费使用的。CURL 的 开发人员经历了类似的行为 ，依赖他的项目的公司要求他在他们的代码遇到麻烦时飞去帮助他们，尽管没有为他的服务付费。

因此，一些开源开发者开始意识到他们拥有巨大的权力，尽管他们的工作报酬很少，这并不奇怪，因为他们的项目被世界上一些最大、最赚钱的公司所使用。

例如，在 1 月初，两个流行的 npm 软件包“colors”和“faker”的开发者 Marak Squires， 故意对他们的代码进行更改 ，破坏了他们的功能，任何人使用它们， 输出“LIBERTY LIBERTY”然后是乱码和使用时的无限循环。

虽然斯奎尔斯没有评论做出这些改变的原因，但他曾在 GitHub 上说过“我将不再用我的免费作品来支持财富 500 强(和其他小公司)。”

Squires 的改变打破了其他受欢迎的项目，包括 亚马逊的云开发套件 ，因为他的库每周在 npm 上安装近 2000 万次，数千个项目直接依赖于它们。在几个小时内，npm 回滚了流氓发布，GitHub 暂停了开发者的帐户作为回应。

虽然 npm 的反应是意料之中的，在之前的事件中，恶意代码被添加到库中，并最终被回滚以限制损害，但 GitHub 是一个新的事件:代码托管平台关闭了 Squires 的整个帐户，尽管他是代码的所有者，他有权随心所欲地更改代码。

这也不是开发者第一次撤回他们的代码以示抗议。“left-pad”的开发者在 2016 年从 npm 中取出了他的代码 ，在与 Kik messenger 就他拥有的另一个开源项目的命名发生争执后，破坏了数万个依赖于它的网站。

令人惊讶的是，尽管偶尔会有高调的图书馆抗议行业的工作方式，但这种类型的事件并不常见:开源开发者继续免费工作，尽可能地维护他们的项目，即使数百万美元的产品是在他们的工作基础上创建的。

在 Log4J 事件后，甚至白宫在与业界的一次会议后也承认了开源对技术行业的重要性， 在 2022 年 1 月表示 “开源软件带来了独特的价值，也带来了独特的安全挑战，因为它的广泛使用和负责其持续安全维护的志愿者数量。”

然而，尽管有这样的声明，广受欢迎的开源软件仍然资金不足——至少在它获得关注之前是如此。在 Heartbleed 漏洞 将更广泛的互联网置于风险之前，受影响的开源项目 OpenSSL、 每年仅收到 2000 美元的捐款 在问题曝光后增长到 9000 美元。

OpenSSL 背后的团队， (几乎每一个现代网络设备 )当时写道，“这里应该有至少六个全职的 OpenSSL 团队成员，而不仅仅是一个，能够专注于 OpenSSL 的维护和发展。”相反，项目团队继续寻找承包工作来支付维护项目的成本。

开发者可以改变他们的开源许可，将他们的工作转化为产品，或者争取更多的赞助商，但是 没有一个适合每个项目的通用解决方案 。直到业界找到更好的方法来资助所有这些免费工作——似乎没有人愿意这样做——我们应该期待更多的开源开发者表现出不服从的行为，故意破坏他们的工作来照亮他们所做的贡献。

从长远来看，这是不可持续的——但不清楚我们将如何摆脱这一困境，因为开源气球在今天生产的每一款软件和连接设备中的使用，但继续依赖于少数开源开发者没有糟糕的一天并决定打破一切。

如果像 cURL 这样被数百万设备使用的库被包含在从你的洗衣机到你的汽车的所有东西中，但它的创造者厌倦了支持它，并决定向世界发送一个信息，那会怎样？我们在过去很幸运，损失可以逆转，但我们在未来可能不会这么幸运。
