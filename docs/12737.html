软件供应链安全比网络安全管理软件产品和 Log4J 更广泛

丹·洛伦茨是

Chainguard

。此前，他是谷歌开源安全团队(GOSST)的软件工程师和负责人。他创立了 Minikube、Skaffold、TektonCD 和 Sigstore 等项目。

网络安全管理软件产品和 Log4j 让软件供应链安全问题成为企业和政府都非常感兴趣和关注的话题。

网络安全管理软件产品是一个可怕的例子，说明软件构建系统的完整性会出什么问题:俄罗斯情报部门劫持了网络安全管理软件产品软件公司的软件构建系统，偷偷给一个软件添加后门，并搭便车进入成千上万客户的计算机网络。Log4J 是开源软件垃圾输入、垃圾输出问题的缩影:如果你从互联网上获取无担保代码，就会有 bug，其中一些 bug 是可利用的。

然而，很少被提及的是，这些攻击仅仅代表了可能的不同类型的软件供应链危害的一小部分。

让我们来看看一些鲜为人知但同样严重的软件供应链攻击类型。

未经授权的行为

这类攻击描述了未经授权的用户破坏开发人员的笔记本电脑或源代码管理系统(例如 GitHub ),然后推送代码。

一个特别著名的例子是，一名攻击者入侵了托管 PHP 编程语言的服务器，并在编程语言中插入了恶意代码。虽然发现得很快，但如果不加以纠正，这些代码将会在互联网的大范围内造成广泛的未经授权的访问。

安全厂商的前景是出售一个白日梦，即“扫描仪”和“软件组成分析”软件可以检测到软件工件级别的所有关键漏洞。他们没有。

幸运的是，最近开发的工具，如 Sigstore 和 gitsign 降低了这种类型的攻击的可能性，以及这种攻击发生时的损害。

发布服务器受损

最近，一名攻击者，可能是中国情报部门，入侵了发布中国消息应用咪咪的服务器，用一个恶意版本替换了正常的聊天应用。该恶意软件允许攻击者远程监控聊天软件。

这种攻击源于这样一个事实，即软件行业未能像对待生产环境和网络边界一样对待软件供应链中的关键点(如发布服务器或构建系统)。

开源软件包存储库攻击

从存放 Python 包的 Python 包索引，到 npm ，世界软件现在确实依赖于巨大的软件包商店，开源软件程序员的苹果应用商店。
