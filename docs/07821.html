Copilot 发布后，开源开发者被敦促放弃 GitHub

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/01/open-source-developers-urged-to-ditch-github-following-copilot-launch/

为开源软件项目提供支持和法律服务的非营利组织软件自由保护协会(Software Freedom Conservancy)在退出代码托管和协作平台后，呼吁开源社区放弃 GitHub。

一周前，微软旗下的 GitHub 推出了商业版的 Copilot ，这是一款人工智能驱动的双人程序员，通过在软件开发人员键入时建议他们的行或功能来与他们合作。这有点像 Gmail 的智能撰写功能，它通过使用上下文提示来建议邮件中的下一段文本，从而努力加快您的电子邮件写作速度。

软件自由保护协会(Software Freedom Conservancy)由许多知名公司(如 Google、Red Hat 和 Mozilla)提供资金支持，其成员涵盖超过 40 个项目，包括 Git(GitHub 非常依赖的项目)、Selenium(项目)和 Godot(项目)。

GitHub 与开源

虽然软件自由保护协会对 GitHub 的不满在一定程度上早于 Copilot，但 GitHub 的最新发布似乎是最后一根稻草。问题的关键，也是自去年首次亮相以来软件开发领域争论的焦点，是 Copilot 是建立在开源社区辛勤工作之上的专有服务。事实上，Copilot 是与 OpenAI 合作开发的，OpenAI 是一家人工智能研究组织，微软早在 2019 年就向投资了 10 亿美元，并在很大程度上依赖 OpenAI Codex，后者是在大量公共源代码和自然语言模型上训练出来的。

副驾驶围绕提出了许多重要的问题，谁实际上创作了一个软件。“开源”并不意味着完全免费，仍然需要满足许可要求并包括属性，所以如果 Copilot 从一个项目“借用”代码并向另一个项目的作者建议，这是否会打开版权侵权诉讼的闸门？围绕什么是合理使用，以及 Copilot 提出的缺乏透明度的问题，也有一大堆讨论，软件自由保护协会的 Bradley M. Kuhn 去年写了一篇深度文章，题为“如果软件是我的 Copilot，谁编写了我的软件？”。

鉴于开源软件的精神集中在每个人为了更大的利益而一起工作的概念上，没有哪一方比另一方受益更多，GitHub 决定以每月 10 美元的价格推出 Copilot(尽管对一些开发人员来说是免费的),并对用于训练系统的具体数据进行最少的了解，这导致了 Software Freedom Conservancy 采取了立场。这意味着该组织本身正在内部结束自己对 GitHub 的使用，并引入一个程序来帮助其成员项目从 GitHub 过渡。最重要的是，它表示不会接受没有明确计划从 GitHub 迁移开源项目的新成员。

“我们自己已经考虑这个行动有一段时间了，但上周的事件(Copilot 的发布)表明这个行动早就应该了，”该组织在一篇博客文章中写道。

虽然社区中的许多人可能不同意 GitHub 通过专有产品将开源开发人员的劳动货币化的最新方法，但现实情况是 GitHub 是全球软件合作者的事实上的平台，因此这场运动可能很难获得任何真正的动力。当然，还有其他选择，比如 GitLab 的自托管社区版，但 GitHub 已经做了出色的工作，使自己成为全世界数百万开发者的“粘性”命题。

同样值得注意的是，微软的老对手亚马逊最近推出了自己的副驾驶化身，名为 CodeWhisperer，上周在预览中推出。从它的推出可以明显看出，亚马逊正试图解决 Copilot 引起的一些版权问题——例如，如果 CodeWhisperer 生成一个与在其训练数据中发现的现有片段相似的代码建议，它将突出显示与原始功能相关联的许可证。然后由开发人员决定是否使用这些代码。

因此，虽然软件自由保护协会的活动在让人们放弃 GitHub 方面可能会有成效，但结合亚马逊产品的竞争，它可能会对微软施加足够的压力，以改变 Copilot 的运营方式，从而使其代码建议的来源更加透明。

TechCrunch 昨日联系 GitHub 寻求置评，但截至发稿时，尚未收到回复。
