Knative 成为 CNCF 项目 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/02/knative-becomes-a-cncf-project/

云本地计算基金会(CNCF)是许多最重要的现代开源项目的所在地，包括 Kubernetes。今天，CNCF 技术监督委员会宣布，它现在也接受了作为 CNCF 孵化项目的 Knative。

CNCF 首席技术官 Chris Aniszczyk 说:“Knative 是一项强大的技术，它与云原生生态系统很好地集成在一起，使得在 Kubernetes 上运行无服务器容器变得更加容易。“我们认为该项目将极大地受益于基金会下的开放治理模式，通过接触新的贡献者和最终用户，允许它进一步发展。我们期待与 Knative 社区合作，并欢迎该团队的贡献。”

Knative(发音为 kay-nay-tiv)是由谷歌在 2018 年创建的，但从那以后，包括 IBM、红帽、VMware、TriggerMesh 和 SAP 在内的许多其他行业巨头也在研究它。这个项目背后的想法是让用户更容易在 Kubernetes 上构建、部署和管理无服务器和事件驱动的应用。当许多企业开发新应用程序或更新现有应用程序作为其数字化转型项目的一部分时，这也是他们目前的发展方向。尽管 Knative 仍然是一个非常年轻的项目，但彭博、阿里云、彭博、IBM 和 VMware 已经在生产中使用它，而谷歌使用 Knative 来支持谷歌云的无服务器计算平台。

该项目在去年 11 月达到了其 1.0 版本的里程碑，不久之后，谷歌宣布已经将该项目提交给 CNCF 进行考虑。现在这一过程已经完成，谷歌将捐赠 Knative 商标，知识产权和代码给 CNCF。

Knative 指导委员会和 DOCS-UX 领导卡洛斯·辛坦拿表示:“Knative 1.0 最近取得了稳定的成就，我们相信将该项目捐赠给一个供应商中立的家庭是实现项目增长和社区自治的下一步。”“我们相信, CNCF 是厂商中立的基础，希望 Knative 的接受将鼓励更多的公司采用、贡献和宣传该项目。它还将使 Knative 社区更接近生态系统中的其他云原生项目，包括它所构建的所有项目，从而帮助建立反馈和功能的良性循环。”
