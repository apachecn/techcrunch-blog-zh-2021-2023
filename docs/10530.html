Cloudflare 瞄准 AWS，承诺向使用其自有平台的初创公司提供 12.5 亿美元 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/09/27/cloudflare-takes-aim-at-aws-with-promise-of-1-25-billion-to-startups-that-use-its-own-platform/

三年前上市的安全、性能和可靠性公司 Cloudflare 今天早上表示，它将帮助使用其无服务器计算平台的初创公司与数十家风险公司建立联系，这些公司已集体提出从其现有基金中向这些公司投资高达 12.5 亿美元。

这是一个聪明、引人注目的激励措施，可以吸引更多的初创公司使用这款已有五年历史的产品，根据 Cloudflare 的说法，这款产品使开发者能够在不配置或维护基础设施的情况下构建或增强应用。Cloudflare 在一份相关的新闻稿中指出，初创公司可以利用 Cloudflare 去年收购的平台快速扩展:Zaraz，这是一家承诺用一行代码提高网站性能的初创公司。(Cloudflare 没有承诺收购其他初创公司，但这个建议正在酝酿中。)

事实上，据我们所知，这个资助计划实际上是 Cloudflare 瞄准了 AWS、Azure 和 Google Cloud 等利润丰厚的产品。为此，我们在周末询问了 Cloudflare 首席执行官 Matthew Prince，为什么 Cloudflare 认为它可以从这些大得多的公司那里窃取市场份额。

“我不会把它描述为从任何人那里‘窃取’市场份额，”他说。“这是赢得市场份额的问题，而赢得市场份额的方法就是以更实惠的价格提供更好的产品。”

当被问及便宜多少时，他只是说，因为它的构建方式，它“比传统公共云便宜得多”。普林斯解释说，现代浏览器“几乎在今天打开的每一个网页上都会遇到新的、不可信的代码。他们需要一种快速、安全地执行代码的方法(并使用一种叫做隔离的技术)来实现这一点。”Cloudflare Workers 是该平台的名称，“采用了受浏览器启发的隔离技术，并将其作为一个开发平台。”

这个想法似乎引起了程序员的共鸣。Cloudflare 的股票在 2022 年与其他科技股一起受到重创，但相对而言表现良好，它没有按产品类别分列收入。但它表示，Cloudflare 的合同客户中有 20%使用 Cloudflare 工作人员。它还表示，到今年年底，将有超过 100 万名开发人员使用 Cloudflare Workers 平台。

图片来源: Cloudflare

这种新策略是这种情况的一部分。

Prince 表示，将平台上的初创公司与风险投资联系起来的想法来自于它与风险投资公司的现有关系，这些风险投资公司开始注意到他们的投资组合公司越来越多地使用 Cloudflare Workers 作为他们的开发平台。“当他们进行尽职调查时，”普林斯说，风投们会“敦促[创始人]‘为什么是 Cloudflare，而不是 AWS 这样的平台’，[而且]一个又一个初创公司给出的答案是，Cloudflare 的员工规模更大，性能更好，运营成本更低。”

“如果你是一名风险投资人，你会多次听到最有前途的初创公司这样的回答，这会引起你的注意，”他补充道。

许多公司已经同意支持开发人员使用 Cloudflare Workers 构建他们的应用程序，包括:Altimeter Capital、Altos Ventures、Amplify Partners、Bain Capital Ventures、Bessemer Venture Partners、boldstart ventures、Cowboy Ventures、db、Emergence Capital、FirstMark、Greylock Partners、IVP、Lightspeed、Meritech、New Enterprise Associates (NEA)、Norwest Venture Partners、Pear、Root Ventures、Scale Venture Partners、Signal Peak、Silverton Partners、StepStone Group、Threshold Ventures、U.S. Venture Partners、

它明确表示，Cloudflare 不提供任何资金，也不做出任何资金决策。不涉及任何谨慎的新基金。所有资助决定将由参与公司做出。

此外，除了这些风投的潜在投资外，申请资助计划的初创公司将有资格获得许多其他 Cloudflare 计划，这些计划旨在加速初创公司的发展，并获得“独家”活动。
