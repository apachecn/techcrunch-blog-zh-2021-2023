Hygraph API 帮助开发者联合来自多个来源的内容| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/09/13/new-hygraph-api-helps-developers-federate-content-from-multiple-sources/

在内容管理系统的早期，你有一个创建和存储内容的工具，通常是一个网站，仅此而已。随着时间的推移，随着更多的输出选项出现在网站之外，无头内容管理系统的发展使得从单一来源生成多种格式的内容变得更加容易。

德国初创公司 Hygraph 更进一步，今天宣布了一个 API，使开发人员能够从各种来源提取内容，并将这些内容用于多种格式甚至以内容为中心的应用程序。Hygraph 的首席执行官兼联合创始人 Michael Lukaszczyk 说，他的公司以前叫做 GraphCMS，称这种方法为联合内容平台。

“联合内容平台是一个内容平台，它不仅是前端不可知的，也是后端不可知的，”他说。

他认为这种方法完全改变了 CMS 的概念。“无头 CMS 允许 CMS 平台和目的地之间的一对多关系。我们正在通过多对多关系扭转局面。他说:“我们的解决方案让你可以利用你所有的内容来源，你在企业技术上的所有投资，我们用一个通用的内容 API 将它们聚合起来，同时允许你以一种极其灵活的方式将内容分发到任何渠道。”。

左边是一个传统的 CMS(一对一)，中间是一个无头 CMS(一对多)，右边是一个像 Hygraph 的联合内容平台(多对多)。图片来源: Hygraph

系统基于 GraphQL API，占了公司原来的名字。他举例说，内容可以来自电子商务系统、客户关系管理系统或几乎任何你可以通过 API 访问的系统。内容通过 GraphQL API 推送，开发人员可以定义任何输出，如手机、手表或笔记本电脑。

“您可以使用我们的联合机制在 API 层集成所有这些系统。开发人员和实现应用程序只需要使用一个通用的 GraphQL API，就可以为所有第三方系统的所有内容和数据提供服务，”Lukaszczyk 解释道。

根据 Crunchbase 的数据，这家总部位于柏林的初创公司于 2017 年启动，已经筹集了约 1400 万美元。该公司目前有 65 名员工，当他着眼于建立一个多元化的组织时，他说多元化有许多不同的载体。

“首先，我们关注性别多样性。因此，我们的负责人是女性，我们正在建立一个主要由女性组成的人力资源团队，以吸引更多的女性人才。我们也在寻找更多的女性候选人来填补一些领导职位，”他说。除此之外，该公司在建立全球员工队伍时，还会考虑文化和年龄多样性等因素。

有了新产品，这家初创公司将根据开发者插入 API 的数据源数量收费。内容联盟 API 在今年夏天早些时候发布了软测试版，并从今天开始正式发布。
