Petals 正在为运行文本生成人工智能 TechCrunch 创建一个免费的分布式网络

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/20/petals-is-creating-a-free-distributed-network-for-running-text-generating-ai/amp/

图片来源: natrot / Getty Images

BigScience 是一个由初创公司humping Face支持的社区项目，其目标是让生成文本的人工智能广泛可用，该项目正在开发一个名为 Petals 的系统，通过加入互联网上人们的资源，该系统可以像 ChatGPT 一样运行人工智能。有了 Petals，志愿者可以贡献他们的硬件能力来处理一部分文本生成工作，并与其他人合作完成更大的任务，类似于 Folding@home 和其他分布式计算设置。

“Petals 是拥抱脸、Yandex Research 和华盛顿大学的研究人员正在进行的合作项目，”Petals 的主要开发者、Yandex 的研究工程师 Alexander Borzunov 在电子邮件采访中告诉 TechCrunch。“与通常不太灵活的 API 不同，Petals 完全是开源的，因此研究人员可以集成 API 中尚未提供的最新文本生成和系统适应方法，或者访问系统的内部状态来研究其功能。”

开源，但不是免费的

尽管有的缺点，像 ChatGPT 这样的文本生成人工智能还是很有用的——至少如果社交媒体上的病毒式演示是可靠的话。ChatGPT 及其同类产品承诺，通过大规模生成类似人类的代码、文本和公式，将一些通常会困扰程序员、作家甚至数据科学家的平凡工作自动化。

但是运行起来很昂贵。根据一项估计，ChatGPT 每天花费它的开发者 open ai 10 万美元，相当于每月 300 万美元。

运行尖端文本生成人工智能所涉及的成本使它一直被归入初创公司和拥有大量资金支持的人工智能实验室。提供一些更有能力的文本生成系统技术的公司，包括 AI21 实验室、 Cohere 和前面提到的 OpenAI，已经从风投那里筹集了数亿美元的资金，这并不是巧合。

但是花瓣使事物民主化——在理论上。受 Borzunov 早期专注于通过互联网训练人工智能系统的工作的启发，Petals 旨在大幅降低运行文本生成人工智能的成本。

“Petals是实现机器学习模型真正协作和持续改进的第一步，”拥抱脸的教员研究员科林·拉弗尔通过电子邮件告诉 TechCrunch。“它……标志着一种正在进行的转变，即从主要局限于超级计算机的大型模型向更广泛的东西转变。”

Raffel 提到了过去一年开源文本生成社区中发生的各种淘金热。由于志愿者的努力和科技巨头研究实验室的慷慨，这种尖端的文本生成人工智能一度超出了小开发人员的能力范围，突然变得可用，经过培训并准备部署。

BigScience 推出了 Bloom ，这是一个在许多方面与 OpenAI 的GPT-3(ChatGPT 的前身)不相上下的语言模型，而 Meta 开源了一个相当强大的人工智能系统，名为 OPT。与此同时，微软和英伟达合作开发了有史以来最大的语言系统之一，MT-NLG。

但是所有这些系统都需要强大的硬件来使用。例如，在本地机器上运行 Bloom 需要几百到几千美元的 GPU 零售价。Borzunov 声称，一旦达到足够的容量，Petals 网络将强大到足以运行和微调聊天机器人和其他“互动”应用程序的人工智能系统。要使用 Petals，用户需要安装一个开源库，并访问一个提供连接 Petals 网络说明的网站。连接后，他们可以从运行在 Petals 上的 Bloom 生成文本，或者创建一个 Petals 服务器将计算反馈给网络。

服务器越多，网络越稳定。如果一台服务器出现故障，Petals 会尝试自动寻找替代服务器。虽然服务器在大约 1.5 秒不活动后断开连接以节省资源，但 Borzunov 表示，Petals 足够智能，可以快速恢复会话，只给最终用户带来轻微的延迟。

测试运行在 Petals 网络上的 Bloom 文本生成 AI 系统。图片来源:凯尔·威格斯/ TechCrunch

在我的测试中，使用 Petals 生成文本对于基本提示(例如“将‘猫’翻译成西班牙语”)需要几秒钟，对于更复杂的请求(例如“以狄德罗的风格写一篇关于宇宙本质的文章”)需要 20 多秒钟。一个提示(“解释生命的意义”)花了将近三分钟，但公平地说，我指示系统用比前几个更冗长的答案(大约 75 个词)来回应。

图片来源:凯尔·威格斯/ TechCrunch

这明显比 ChatGPT 慢，但也是免费的。虽然 ChatGPT 现在不需要任何成本，但也不能保证将来会是这样。

Borzunov 没有透露 Petals 网络目前的规模，只是说自 12 月初推出以来，已经有“多个”拥有“不同容量的 GPU”的用户加入了该网络。目标是最终引入一个奖励系统来激励人们捐赠他们的计算机；Borzunov 说，捐赠者将获得“Bloom points ”,他们可以花在“更高的优先级或增加的安全保证”上，或者可能换取其他奖励。

分布式计算的局限性

Petals 承诺提供一种低成本的，如果不是完全免费的，替代 OpenAI 等供应商提供的付费文本生成服务。但是主要的技术问题还有待解决。

最令人担忧的是安全漏洞。Petals 项目的 GitHub 页面指出，由于 Petals 的工作方式，服务器有可能恢复输入文本——包括本应保密的文本——并以恶意的方式记录和修改它。这可能需要与网络中的其他用户共享敏感数据，如姓名和电话号码，或者调整生成的代码，使其被故意破坏。

Petals 也没有解决当今领先的文本生成系统固有的任何缺陷，比如它们倾向于生成有毒和有偏见的文本(参见拥抱脸知识库上的 Bloom 条目中的“限制”部分)。在一次电子邮件采访中，Yandex Research 的高级研究科学家 Max Ryabinin 明确表示，Petals 旨在用于研究和学术用途——至少目前是这样。

Ryabinin 说:“Petals 通过公共网络发送中间数据，所以我们要求不要将它用于敏感数据，因为其他对等点可能(理论上)从中间表示中恢复它们。”“我们建议那些想使用 Petals 存储敏感数据的人建立他们自己的私人群组，由组织和他们信任的授权处理这些数据的人托管。例如，几个小型创业公司和实验室可能会合作并建立一个私人群体，以保护他们的数据免受其他人的影响，同时仍然可以获得使用 Petals 的好处。”

与任何分布式系统一样，Petals 也可能被最终用户滥用，要么是希望生成有毒文本(例如仇恨言论)的不良行为者，要么是拥有特别资源密集型应用程序的开发者。Raffel 承认 Petals 在开始时将不可避免地“面临一些问题”。但是他相信这个任务——降低运行文本生成系统的障碍——最初的颠簸是值得的。

“鉴于最近许多社区组织的机器学习努力取得了成功，我们认为继续开发这些工具很重要，并希望 Petals 将激励其他分散的深度学习项目，”Raffel 说。
