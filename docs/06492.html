Koyeb 是一个无服务器平台，可以与您的 GitHub 库集成

自从我第一次报道这家初创公司以来，Koyeb 已经发展了很多。该公司仍然专注于无服务器基础设施。但是现在它提供了一个通用的无服务器平台，你可以通过一个简单的“git push”命令或者使用 Docker 容器来配置它。

该公司的无服务器平台现在可以作为公共预览使用，有一个免费层可以开始并试用该服务——免费层可以让你在平台上运行两个 nano 应用程序。在私人测试阶段，已经有 10，000 名开发人员对其进行了测试。目前有 3000 个应用程序运行在 Koyeb 的基础设施上。

Koyeb 希望尽可能抽象您的服务器基础设施，以便您可以专注于开发而不是系统管理。您可以使用它来托管 web 应用、API 或事件驱动的工作负载。

在幕后，创业公司不使用 Kubernetes。取而代之的是，它已经基于《爆竹微卷》、《流浪者》和《库马》建立了自己的定制栈。它运行在采用最新英特尔和 AMD 芯片的裸机服务器上。

有两种方法可以将您的应用程序部署到 Koyeb。您可以从您的 git 存储库(目前仅限于 GitHub 存储库)或任何公共或私有容器注册中心进行部署。Koyeb 有一个 web 界面，但也提供了命令行界面和 API。

当您部署新的应用程序时，Koyeb 会为您的应用程序提供一个“. koyeb.app”子域，并自动使用 TLS 保护应用程序。您也可以配置自己的域名。

如果你需要更多的资源，你可以很容易地从滑块缩放你的应用程序。在这种情况下，Koyeb 会在几个新的实例上启动你的应用，流量会自动在这些实例之间进行负载平衡。

所有这些对于开发团队来说都是透明的。每次有新的 git 提交时，Koyeb 都会自动开始构建和部署您的应用程序。

Koyeb 计划提供一个全球边缘网络。该服务目前在巴黎的一个核心位置和 250 个边缘位置运行，用于本地负载平衡、TLS 加密和类似 CDN 的缓存。到今年年底，你的应用将同时部署到全球 10 个核心地点。

很明显，Koyeb 仍然是一个正在进行中的工作。但是对于不想花太多时间管理云基础设施的精益开发团队来说，这听起来是一个有希望的开始。
