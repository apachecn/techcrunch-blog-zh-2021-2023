谷歌云跨平台数据存储引擎

原文：https://web.archive.org/web/https://techcrunch.com/2022/04/05/google-cloud-launches-biglake-a-new-cross-platform-data-storage-engine/

在其云数据峰会上，谷歌今天宣布预览推出 BigLake，这是一个新的数据湖存储引擎，使企业更容易分析其数据仓库和数据湖中的数据。

这个想法的核心是利用谷歌在运行和管理其 BigQuery 数据仓库方面的经验，并将其扩展到谷歌云存储上的数据湖，将数据湖和仓库的最佳功能结合到一个服务中，抽象出底层的存储格式和系统。

值得注意的是，这些数据可以位于 BigQuery 中，也可以位于 AWS S3 和 Azure 数据湖存储 Gen2 中。通过 BigLake，开发人员将能够访问一个统一的存储引擎，并能够通过单个系统查询底层数据存储，而无需移动或复制数据。

Google Cloud 数据库、数据分析和商业智能副总裁兼总经理 Gerrit Kazmaier 在今天的公告中指出:“跨不同的湖泊和仓库管理数据会产生孤岛，并增加风险和成本，尤其是在需要移动数据时。”。 BigLake 允许公司统一他们的数据仓库和湖来分析数据，而无需担心底层存储格式或系统，这消除了从源复制或移动数据的需要，并降低了成本和效率。”

图片来源:谷歌

使用策略标签，BigLake 允许管理员在表、行和列级别配置他们的安全策略。这包括存储在谷歌云存储中的数据，以及两个受支持的第三方系统，其中谷歌的云计算分析服务 BigQuery Omni 实现了这些安全控制。这些安全控制还确保只有正确的数据流入 Spark、Presto、Trino 和 TensorFlow 等工具。该服务还集成了谷歌的 Dataplex 工具，以提供额外的数据管理功能。

谷歌指出，BigLake 将提供细粒度的访问控制，其 API 将跨越谷歌云，以及像面向开放列的 Apache Parquet 和 Apache Spark 这样的开源处理引擎这样的文件格式。

 

图片来源:谷歌

“组织必须管理和分析的宝贵数据量正在以令人难以置信的速度增长，”谷歌云软件工程师 Justin Levandoski 和产品经理 Gaurav Saxena 在今天的公告中解释道。“这些数据越来越多地分布在许多位置，包括数据仓库、数据湖和 NoSQL 商店。随着组织的数据变得越来越复杂，并在不同的数据环境中激增，出现了孤岛，从而增加了风险和成本，尤其是在需要移动数据时。我们的客户已经说得很清楚了；他们需要帮助。”

除了 BigLake，谷歌今天还宣布，其全球分布式 SQL 数据库 Spanner 将很快获得一个名为“更改流”的新功能。有了这些，用户可以很容易地实时跟踪数据库的任何变化，无论是插入、更新还是删除。“这确保了客户始终能够访问最新的数据，因为他们可以轻松地复制从 Spanner 到 BigQuery 的更改以进行实时分析，使用 Pub/Sub 触发下游应用程序行为，或将更改存储在 Google 云存储(GCS)中以实现合规性，”Kazmaier 解释道。

谷歌云今天还推出了 Vertex AI Workbench ，这是一款用于管理数据科学项目整个生命周期的工具，已经退出测试并正式推出，并推出了 Looker 的 Connected Sheets，以及在其 Data Studio BI 工具中访问 Looker 数据模型的能力。
