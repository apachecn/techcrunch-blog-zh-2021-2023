Tweetbot 和 Twitterrific 要求客户拒绝退款，这在罕见的应用商店中是个例外

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/01/twitter-apps-tweetbot-and-twitterific-ask-customers-to-decline-their-refunds-in-rare-app-store-exception/?utm_source=Telegram&utm_medium=iXNews&utm_campaign=PerspectiveIX.com

Twitter 上个月正式禁止第三方客户端，为热门应用画上了一个的句号，包括 Tweetbot 、twit terrious等。现在，事情发生了不寻常的变化，两名开发者本周为他们关闭的应用程序更新了新功能:他们要求他们的订户通过点击他们非功能性应用程序中的新“我不需要退款”按钮来拒绝接收退款。此外，在 Tapbots 的 Tweetbot 应用程序的情况下，用户可以选择将他们的订阅转移到该公司的最新应用程序——其乳齿象客户端象牙。这些选项允许同情这些独立开发者困境的订户通过不要求退款来提供支持。

退一步说，这是一种前所未有的情况，也是大多数基于订阅的 iOS 应用永远不会面临的情况。

在大多数其他情况下，一家公司决定终止 API 访问，就像 Twitter 所做的那样，会被提前告知。这将允许依赖 API 功能的企业与他们的客户就变更进行沟通，并准备采取下一步措施。然而，第三方 Twitter 客户端没有任何警告。

Twitter 在其 API 变化之前没有进行沟通，甚至没有承认在用户和开发人员的强烈反对下所做的事情。在最初无视这些投诉后，该公司后来在推特上表示，它只是在“执行其长期存在的 API 规则。”但是这些规则直到 API 访问被取消后才被记录在的开发者条款中。Twitter 所指的“长期”规则仍不清楚。(也许是那一次，12 年前，当公司告诉开发商不要试图通过建立客户来竞争？谁知道呢！)

虽然 Twitter 可能从未完全接受有应用程序提供替代体验的想法，但它最终决定放过这些特定的第三方应用程序，即使是在打击其他 API 用法的时。Twitter 似乎不情愿地承认了应用程序对生态系统做出积极贡献的悠久历史。此外，他们支持 Twitter 的一些最忠实的用户。

尽管与 Twitter 的官方应用相比，这些应用各自的用户群较小，但它们足以帮助维持独立开发者的业务。像大多数订阅应用一样，他们也通过 App Store 上的月订阅和年订阅来赚钱。这意味着当 Twitter 关闭时，这些公司陷入了一个不幸的境地，他们的预期收入几乎立即停止。对于提前支付一年服务费用的年度用户来说，他们很快将不得不发出现金退款。

根据他们的应用商店页面，这两款应用——tap bots 的 Tweetbot 和 Iconfactory 的 twit terrorism——都在周一推出了应用更新。现在，这两款应用在向用户提出请求时都使用了相似的措辞。

例如，在 Tweetbot 中，有几个选项可供选择。顶部选项允许付费用户点击一个按钮，将他们的订阅转移到该公司的新应用象牙。第二个选项写着“我对从 Tweetbot 获得的东西很满意，不需要退款”，并提供了一个蓝色的大按钮“我不需要退款”。第三种选择是让想要按比例退款的顾客什么都不做——退款将通过苹果公司自动进行，这是通常的做法。

与此同时，推特的信息几乎是一样的，但没有提供订阅转移选项。与 Tapbots 不同的是，该公司并没有建立一个 Mastodon 客户，作为订户资金的新目的地，这是有意义的。像推特一样，推特也提供了一个选项，上面写着“我对从推特获得的东西很满意，现在不想退款”，下面有一个蓝色按钮供同意的人点击。想要按比例退款的客户会再次被告知苹果会退款。

此外，推特提供了其他 Iconfactory 应用程序的链接，毫不奇怪，还有它的 Mastodon 账户。

有趣的是，两家公司都不能对他们的更新发表官方评论，这表明苹果可能对其应用商店的常规做了一个特殊的例外。(我们被要求联系苹果公关寻求置评，但没有得到回应。)

没有太多的其他情况下，应用程序会被允许通过订阅支付的方式来要求实际上是捐赠的非功能性应用程序。但鉴于推特事件的高调性质，苹果允许这些应用程序发出这样的请求是有道理的。

当然，这种独特的情况需要用户重新下载应用程序——或者如果仍然安装的话更新它们——然后点击按钮来阻止自动按比例退款的发生。这并不理想。这些公司没有做错，但现在将不得不自掏腰包支付许多退款，因为许多客户一开始就不知道再次打开这些应用程序。
