谷歌希望将其隐私沙盒引入 Android | TechCrunch

谷歌为其 Chrome 浏览器推出的隐私沙盒计划并不完全成功，但它无疑引发了一场关于在线隐私的健康讨论——以及该公司在广告生态系统中的角色。现在，围绕 Chrome 的许多举措仍在不断变化，谷歌还计划将这些工具中的许多扩展到其 Android 操作系统中——这可能会对广告行业产生深远的影响。

但是，如果你在广告生态系统中，不要绝望。谷歌表示，在测试这些新系统期间，当前系统将“至少”再活跃两年。

通常，在 Android 上，广告商使用谷歌的广告 id 来提供个性化广告，并在应用程序中跟踪你的行为，例如，他们可以将你的购买归因于你点击的广告。简化一下，你可以把广告 ID 想象成安卓版的 cookies。你可以关闭这个功能，通过删除你的广告 id，在 Android 广告设置中选择非个性化广告。当你这样做的时候，谷歌会提醒你广告有助于保持许多服务免费——这也是谷歌为今天的变化提出的一个论点。

在今天宣布之前的一次简报中，谷歌负责 Android 安全和隐私的产品管理副总裁特别强调了广告的重要性(当然，广告也是谷歌自身收入的主要来源)。

“强调一些对生态系统至关重要的关键能力是有用的，”他说。“所以像[广告] ID 这样的工具有助于提供更好、更相关的广告体验，打击欺诈，等等。这使得我们今天在移动应用中享受的许多免费内容和服务成为可能。因此，在我们构建下一代移动技术时，确保这些功能得到支持至关重要。”

当然，房间里的大象是苹果，它使用的是谷歌团队认为非常钝的工具，因为它基本上使跟踪变得不可能。这是隐私的胜利，但谷歌认为，在绝望中，广告商只会想出新的方法来采集你的行为和设备的指纹，以获得有利可图的跟踪数据。然而，Meta 表示，由于苹果的变化，它将在 2022 年损失 100 亿美元的广告收入，这一事实似乎会使这一论点无效。如果 Meta 不能找到一个好的方法来解决这个问题，谁能呢？

因此，就像 Chrome 一样，谷歌试图两者兼得:保护你的隐私和广告生态系统。澄清一下，谷歌说它自己的广告系统将遵循与第三方广告商相同的规则。

这里的一些提议是基于谷歌与 Chrome 的合作。它们包括 Topics ，这是 FLoC 和 fragge 的最新替代品，该系统允许广告商根据他们自己对“定制受众”的定义来展示广告，而不必依赖个人标识符。

没有属性报告就没有现代广告生态系统，所以谷歌也在这里向提议一个新系统，它承诺仍然给广告商他们需要的数据，同时仍然改善其用户的隐私。

还将有一个面向开发者的 SDK，它隔离第三方广告代码，这样它将与应用程序自己的代码分开运行。截至目前，这似乎是 Android 13 独有的功能，因为它需要一个不同的整体 SDK 架构，该架构不仅关注这些新的隐私功能，还为任何 SDK 提供额外的安全保证。

谷歌表示，它希望在这个新系统上与广告业合作。到目前为止，它所有的支持报价都来自应用程序开发者，而不是更广泛的广告生态系统。
