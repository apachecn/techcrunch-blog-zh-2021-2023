Suborbital 为其网络组装平台筹集了 160 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/23/suborbital-raises-1-6m-for-its-webassembly-platform/

亚轨道公司(Suborbital)今天宣布，它已经筹集了由 Amplify Partners 牵头的 160 万美元的种子资金。亚轨道公司是开源项目 Atmo 的幕后公司，该项目以 WebAssembly 为中心，用于构建可扩展的服务器应用程序。包括 Jason Warner(GitHub 前 CTO)、Sri Viswanath(Atlassian 的 CTO)、Tyler MC Mullen(Fastly 的 CTO)、Jonathan Beri(Golioth 的创始人)、Vijay Gill(RapidAPI 的工程 SVP)和 Mac Reddin(Commsor 的创始人)在内的多位天使投资人也加入了此轮投资。

此外，该公司今天还宣布公开测试推出亚轨道计算。乍一看，这似乎是一种奇怪的产品。由于 SaaS 服务希望使他们的产品在基本的拖放集成之外具有可扩展性，他们需要允许开发人员在其产品内部编写这些扩展的工具。但是这些用户函数也带来了很多安全问题。通过亚轨道计算，SaaS 开发人员可以让他们的最终用户能够编写自己的功能并扩展他们的产品，WebAssembly 的沙盒属性(Atmos 和亚轨道的其他开源工具的基础)提供了许多防护。

但这只是开始。亚轨道是一个雄心勃勃的项目。首席执行官兼创始人康纳·希克斯告诉我，它的使命是“改变我们作为一个行业思考和部署计算的方式。”Hicks 之前在 1Password platform 团队工作，在那里他开发了 1Password 命令行界面及其企业产品等工具，最终领导了该公司围绕其企业产品的研发工作。但另一方面，他开始涉足构建一个分布式功能即服务系统，首先基于 Docker，这被证明太慢了，然后，最终，围绕 WebAssemly。事实证明，这比他预期的要复杂得多，很大程度上是因为他必须编写所有的粘合代码才能完成这项工作——但大约两年前，事情开始步入正轨。

“我开始更认真地沿着这条路走下去，开始在它上面花更多的时间，它产生了这个 WebAssembly 函数的调度程序，这就是今天我们的 Reactr 项目，”希克斯解释道。虽然 Reactr 是一个 Go 库，但人们开始对纯粹的 WebAssembly 服务感兴趣，这成为了 Atmo 项目，现在是 Suborbital 工作的核心。

希克斯解释说:“Atmo 的大实验是，‘嘿，让我们看看是否可以对 web 服务器应用程序进行声明性描述，并弄清楚如何在用户不需要做任何样板文件的情况下运行它。所以我们可以用这个声明性的描述——和一堆函数——编译 WebAssembly，我们可以弄清楚如何构建这个 web 服务，让它运行，让它安全，让它自动快速运行，用户不必担心任何管道问题。"

通过 Atmo，Suborbital 正在押注服务器端的 WebAssembly，让开发者用 Rust、Swift 或 AssemblyScript 之类的语言编写代码，然后编译成 WebAssembly，由 Atmo 部署和管理，在沙盒环境中运行。Atmo 的核心是一个调度程序，它运行 WebAssembly 模块，并保证以接近本机的性能运行。

Hicks 认为，随着时间的推移，这种方法可能会挑战容器在部署许多应用程序中的作用，尤其是在边缘。“我们认为裸机上的 WebAssembly 将在很大程度上取代这些小型、资源受限的边缘环境中对容器的需求，”希克斯说。

但是为什么要推出这样一款利基产品呢？像“Atmo Pro”这样的东西可能看起来是更合理的选择，但希克斯认为这还为时过早。因为这个想法还是非常新的，市场不会有这样的服务。

希克斯说:“它没有被广泛采用，你需要从托管的 Atmo 服务中赚钱。“在意识到我不能只靠卖 Atmo 的专业版或者主机版赚钱之后，我回去问自己，‘嘿，我们到底能造出什么东西让人们愿意花钱购买，并真正围绕它做生意？’”

Hicks 告诉我，目前由四人组成的团队已经开始围绕合作伙伴关系加大努力，但明年，它计划真正扩大其基础设施和运营能力。
