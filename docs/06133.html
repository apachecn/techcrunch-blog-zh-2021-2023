Stripe 利用其金融技术力量，通过金融连接自动获取银行数据

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/04/stripe-launches-financial-connections-for-customers-to-pull-banking-data-used-to-power-transactions-automatically/

支付巨擘 Stripe ，一年前估值为950 亿美元，现在据报道正慢慢接近 IPO，今天宣布了一款新产品，填补了其在为商户和其他业务提供金融服务层方面的一些重大空白，这些业务的模式是基于促成交易。它正在揭开金融连接的面纱，这将让 Stripe 的客户直接连接到他们客户的银行账户，以访问金融数据来加速或运行某些类型的交易。

这些包括核实支付和支出的账户；在付款前检查余额，以确保有足够的钱；确认账户所有权。类似这样的细节反过来可以用来帮助承担贷款风险；跟踪消费模式并自动支付账单；更多，换句话说，通过 Stripe Connect、ACH payments 或 Stripe Capital-powered loans 等其他 Stripe 服务进行金融交易时有用或必要的金融数据。

Stripe 表示，对于 Stripe 客户的客户来说，当被询问时，他们将输入他们的银行账户详细信息，并获得关于如何使用这些信息的具体信息。(补充说明:看看美国消费者是否乐于在以前没有的情况下分享这些信息，这将是一件有趣的事情。)

这项服务将首先在美国上线，Stripe 表示，它将与超过 90%的银行账户合作。它将按使用量付费——银行账户验证和账户信息将为每个 API 调用收取 1.50 美元；帐户余额检索是每个 API 调用 10 美分；交易拉动尚未推出，因此定价待定，大客户可以通过企业合同购买。

我们询问了 Stripe，但 Stripe 拒绝评论金融连接何时以及是否会扩展到其他市场，鉴于各国的银行体系差异如此之大，这或许并不令人意外。

在数字支付领域，金融连接正处于一个有趣的时刻。电子商务无疑创造了一个让网上支付变得更容易的市场。虽然这为 PayPal 等数字钱包和一些国家的银行直接支付打开了大门，但这种增长的许多战利品已经转移到基于卡支付轨道的服务中。

但是，随着这个领域的发展，我们已经看到技术、服务、法规和消费者需求的激增。数字银行、投资、加密货币交易、获得贷款的新方法、管理费用——所有这些以及更多都是数字服务变得更加复杂的例子，我们对它们的需求也是如此。(电子商务也没有停滞不前:现在购买，以后支付服务和许多其他服务也进入了这个世界，以创造更灵活的服务，在竞争更激烈和利润更薄的情况下净交易更多。)

越来越多的这种情况出现了。就在昨天，我写了一个有趣的初创公司，叫做凯文(好吧，凯文。)这就为账户到账户支付建立了一套全新的支付轨道和 API，直接链接到银行账户，绕过了更难实现的卡轨和传统的账户到账户支付方法。

在这种情况下，Financial Connections 是 Stripe 推出的一个及时的工具。这是新服务浪潮(如 Kevin)的一部分，这些服务正在为数字交易和相关金融服务创造一种更加程序化的方法。虽然公司以前可以从银行获得交易数据，但现在可以以更快、更自动化的方式处理这些数据。

Stripe 的业务主管克拉拉·梁(Clara Liang)在一份声明中表示:“企业一直要求我们提供一种简单、安全的方式来连接和验证他们客户的银行账户。Stripe Financial Connections 提供的正是这一点。"

事实上，它符合当前客户的需求；但对 Stripe 来说重要的是，它围绕他们已经在使用的产品创建了一个更紧密的服务生态系统，这是一个一站式商店，因此他们不需要利用其他方的工具来集成这一功能，例如 Plaid 等其他公司提供的工具。Stripe 已经进行了大量收购，为其平台带来额外的功能，以弥补一些差距——例如，几乎就在一年前，它收购了 TaxJar 以帮助自动计算销售税，并向其客户提供相关工具——但看起来财务连接是内部建立的，但由 MX 和 Finicity 提供支持(正如玛丽·安在此所指出的)。

Stripe 对这些工具的卖点在于，除了与其他产品的无缝集成之外，它还能帮助客户进行更多交易。它声称已经使用该服务的 Connect 客户已经减少了 75%的支付失败；早期资本用户看到的贷款额增加了 55%，因为他们使用了额外的数据来为决策提供信息。

所有这些都意味着 Stripe 平台本身会有更多的交易。任何数字支付的利润都可能很薄——这也是为什么即使是一家看起来正在成长并且做了很多生意的公司仍然可能失败的原因之一:数字需要巨大才能对其有利——但这就是为什么这么多支付公司大规模运作，为什么建立许多额外的增值服务，希望它们被客户(以及客户的客户)接受， 正如 Stripe 在这里所做的那样，这是一项明智的业务，它希望通过这种方式来维持自己的长期(以及可能的上市)运营。
