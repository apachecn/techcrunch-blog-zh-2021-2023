以下是刚刚完成 C 轮融资的 Rippling 如何让公司围绕“员工身份”这一主题

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/11/heres-how-rippling-fresh-off-its-series-c-aims-to-center-companies-around-employee-identity/

上个月末，ripping，这家由连续创业家 Parker Conrad 创立的现在已有五年历史的公司，在一轮 C 轮融资中筹集了 2.5 亿美元，这一轮融资使其估值达到了惊人的65 亿美元。

现在，康拉德正在揭开让投资者对这套设备如此兴奋的面纱，这套设备已经帮助公司管理员工的工资单、福利、设备和应用程序，但从今天开始，它将为其现有阵容增加数量惊人的功能。

事实上，在本周早些时候与康拉德的谈话中，他建立一家非常大的公司的雄心是毋庸置疑的。他甚至将其潜力与 Salesforce 相提并论。他说，Salesforce“正在把你使用的所有业务系统捆绑在一起，围绕着一个共同的客户身份的想法”。涟漪是“同时说，‘嘿，看，你需要用另一种方式来做这件事，那就是围绕一个共同的员工身份将所有这些系统捆绑在一起，并使这个成为你在业务中使用的所有不同业务软件的这种统一模式的核心。”"

考虑到世界上已经存在的人力资源软件和 It 软件的数量，这是一个大胆的想法——其中一些软件是由康拉德开发的。从整体来看，这也很有吸引力。(我们看了投资者演示。)事实上，所有这些新功能都有了自己的品牌；它们现在是 Rippling 的“Unity”平台的一部分，该平台由五个独立的产品单元驱动，每个产品单元都有铃铛和哨子，财大气粗的客户可能会作为一个完整的包订购，因为它们在一起会更强大。(请注意，目前，整个“无限团结”的价格是每人每月 25 美元。)

引擎盖下到底有什么？Unity 的第一个也是看起来最重要的部分是一个“无代码”系统，该系统承诺允许公司高管——无论他们是在人力资源部门、销售部门还是工程部门工作——创建自定义触发器，在 ripping 和连接到 ripping 的第三方应用程序中自动触发操作。(到目前为止，Rippling 已经为其客户的员工建立了第三方应用程序，但没有与他们整合。在 Rippling 的这个新阶段，它非常专注于整合越来越多此类机构的数据和服务——在他们的同意下——比如信用卡公司 Ramp 和对员工进行背景调查的 Checkr。)

Conrad 在这里提供了许多例子，其中包括设置工作流以自动检测潜在薪酬不平等的能力，以及设置跨系统工作流以通过 Slack 通知工程，工程师无法解决 JIRA 的高优先级票据，因为他们不在办公室。

Unity 的第二个桶围绕着一个分析套件，Rippling 说它已经从头开始构建，以支持对非常大的数据集的分析。康拉德用员工薪酬来强调这一点。例如，许多公司在聘用他们推荐的候选人时，会向员工支付推荐奖金。他说，通常情况下，企业必须将这些数据从多个系统导出到一个 Excel 电子表格中，以确定支付给谁、支付多少以及某些员工是否有资格获得推荐奖金；他说，涟漪效应将使人们有可能在一个地方看到这些数据，答案会更快地变得清晰。

涟漪的第三个主要部分集中在访问上，包括轻松配置员工访问公司今天使用的所有各种 SaaS 产品的能力，只需点击几下键盘。类似地，Conrad 称，Rippling 使在公司的公司卡或支出管理系统中添加员工到正确的支出政策，以及正确的兼职系统变得非常简单。(不同的员工根据所处的世界不同，假期也不同。)

康拉德说，当员工和公司分道扬镳时，也有可能一下子取消对所有东西的访问权。

最后，Unity 邀请用户通过一种名为 RQL 的脚本语言进行定制，这种语言可以用来在其分析套件中创建“公式字段”。正如 Conrad 所指出的，它明显借用了 Excel 的核心概念和语法，但他说他也结合了 SQL 的功能，如“select”语句，以及 Python 的一些元素。

与此同时，我们向康拉德询问了一些更大的问题，比如 Rippling 是否倾向于与它进行集成的某些第三方提供商。在这方面，他说，荡漾“希望与每个人合作。”

“我知道我们已经与 Ramp 进行了相当深入的整合，”他表示，“我认为我们也有兴趣与 Brex 讨论如何深化我们与 Brex 的整合。我们还在与 Expensify 讨论深化我们与他们的整合。”

他说，第三方服务越多越好，这表明每一条数据都会让 Rippling 变得更加棘手。当具体谈到企业支出时，他说他预计“在未来三到六个月内，你实际上会在[ripping]中看到许多交易支出管理数据，以便客户可以对他们组织内的支出进行分析和报告。他们可以看到他们所在部门的人员在一段时间内花费了多少钱，钱去了哪里，谁是顶级供应商，属于什么类别，“甚至在支出超过特定类别的特定金额时触发警报，”甚至回到 Brex、Ramp 和 Expensify 等系统内部

他说，在某些情况下，涟漪甚至与第三方系统集成，其产品反映了一些涟漪已经建立的内部。他提到了考勤系统，说 ripping 有自己的系统，但是“有些情况下，对员工使用 ripping 时间跟踪是有意义的，有些情况下，他们使用第三方的考勤系统也是有意义的。我们希望这两种体验都很棒。”
