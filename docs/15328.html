安全研究人员警告一类新的苹果漏洞

原文：https://web.archive.org/web/https://techcrunch.com/2023/02/22/security-researchers-warn-of-new-class-of-apple-bugs/

安全研究人员表示，他们发现了一种“新类别”的漏洞，这种漏洞可能允许攻击者绕过苹果在 iOS 和 macOS 中的安全保护，访问用户的敏感数据。

Trellix 的高级研究中心本周公布了权限提升漏洞的详细信息，这意味着它们允许某人获得更高级别的系统访问权限，影响 iPhones 和 MAC 电脑。Trellix 警告说，这类从中度到高度严重的漏洞，如果不打补丁，可能会让恶意应用程序逃离他们的保护性“沙箱”，访问某人设备上的敏感信息，包括个人信息、位置数据、通话记录和照片。

Trellix 的发现是在谷歌和公民实验室的早期研究之后，后者在 2021 年发现了一种新的零日漏洞，称为 ForcedEntry ，被以色列间谍软件制造商 NSO 集团滥用，在政府客户的要求下远程秘密侵入 iPhones。苹果公司随后通过增加新的代码签名缓解措施来加强其设备安全保护，该措施通过加密来验证设备的软件是可信的并且没有被修改，以阻止该漏洞的利用。

但 Trellix 本周表示，苹果采取的缓解措施不足以防止类似的攻击。

在的一篇博客文章中，Trellix 表示，新的漏洞涉及 NSPredicate，一种允许开发者过滤代码的工具，苹果在 ForcedEntry 漏洞后通过一种名为 NSPredicateVisitor 的协议加强了对它的限制。但是 Trellix 说，NSPredicateVisitor 的几乎每个实现都“可以被绕过”。

虽然 Trellix 没有看到任何证据表明这些漏洞已经被积极利用，但这家网络安全公司告诉 TechCrunch，它的研究表明，iOS 和 macOS“并不是天生比其他操作系统更安全”。

“我们团队本周发现的漏洞从根本上破坏了他们的安全模式，”Trellix 的漏洞研究主管道格·麦基(Doug McKee)说，并补充说，从理论上讲，这些漏洞可能会使受影响的苹果设备暴露在各种攻击媒介下，并使不当访问敏感数据变得更容易。这些错误实质上允许已经获得低特权代码执行(即 macOS 或 iOS 上的基本功能)的攻击者获得高得多的特权

苹果修补了 Trellix 在 1 月份发布的 macOS 13.2 和 iOS 16.3 软件更新中发现的漏洞。苹果的安全支持文档也在周二更新，以反映新补丁的发布。

安全研究员、守护者防火墙应用的创始人威尔·斯特拉发奇(Will Strafach)形容这些漏洞“相当聪明”，但警告说，“除了对安装安全更新保持警惕”，普通用户对这些威胁几乎无能为力

iOS 和 macOS 安全研究员 Wojciech regula 告诉 TechCrunch，虽然这些漏洞可能很严重，但在没有利用的情况下，需要更多的细节来确定这种攻击面有多大。

Jamf 的迈克尔·科文顿说，苹果的代码签名措施“从未打算成为保护设备数据的银弹或单独的解决方案”。Covington 说:“这些漏洞虽然值得注意，但显示了分层防御对于保持良好的安全态势是多么重要。”

当联系到苹果公司时，苹果公司没有提供正式的评论。
