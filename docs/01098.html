AWS 为 CloudWatch 增加了用户监控和 A/B 测试

亚马逊 CloudWatch 早在 2009 年就推出了，旨在帮助 AWS 客户查看他们的云使用和支出数据。今天，在 AWS re:Invent 即将到来之际，该公司的云客户会议于本周在拉斯维加斯举行，云部门宣布了该产品的几项增强功能。

亚马逊一直在建立由 CloudWatch 提供的数据类型，今天它增加了用户监控。通过真实用户监控，AWS 客户可以了解部署何时出现问题，并在客户真正感觉到之前采取纠正措施。

“亚马逊 CloudWatch RUM 将帮助您收集指标，为您提供见解，帮助您识别、理解和改善这种体验。你只需注册你的应用程序，在每个页面的标题上添加一段 JavaScript 代码，然后部署，”亚马逊的杰夫·巴尔在宣布该功能的博客文章中写道。

这并不完全属于惊人创新的范畴。这是 AppDynamics 和 New Relic 等公司多年来一直在做的事情，但正如亚马逊的大多数事情一样，他们正在为 AWS 内部的客户提供从头到尾的体验，这种类型的监控让你知道你的 AWS 应用程序何时可能会出错。

另一个新功能是一个名为 CloudWatch 的新实验工具，显然是，它可以帮助开发人员设置功能标志，并在他们基于 AWS 构建的应用程序中运行 A/B 测试。开发人员可能不只是为每个用户更新应用程序，而是希望在有限的用户子集上进行测试，看看新功能是否有所突破，或者用户是否更喜欢特定的方法或设计。

他们可以通过在代码中设置特性标志并为该特性设置参数来限制看到新特性的人。此外，您可以进行 A/B 测试，这是另一种形式的实验，让您用某个用户子集来测试功能，以了解人们更喜欢哪种功能或设计。

这些都不是新的。像 Split.io 这样的公司做更广泛的功能标志管理已经有一段时间了，像Optimized这样的公司一直在围绕 A/B 测试建立公司。

显然，CloudWatch 已经在亚马逊的九个云区域提供，价格为现收现付，而 CloudWatch RUM 现在也在 10 个区域提供，每 100，000 个收集到的事件花费 1 美元。
