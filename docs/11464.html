被微软收购四年后，GitHub 继续做自己的事情

原文：https://web.archive.org/web/https://techcrunch.com/2022/10/26/four-years-after-being-acquired-by-microsoft-github-keeps-doing-its-thing/

自微软完成对 GitHub的收购至今已经四年了，当时 GitHub 主要是一个代码仓库。今天的 GitHub 看起来有点不同，现在它添加了带有 GitHub 操作和代码空间的 CI/CD 工具作为在线编辑器和计算平台，以及各种安全工具等等。但是据一年前从 Nat Friedman 手中接管 T3 的 GitHub 首席执行官 Thomas Dohmke 说，微软已经非常允许 GitHub 做它最擅长的事情。

“我们保留了 GitHub GitHub，它仍然是微软内部的一个独立实体，类似于 LinkedIn，”他告诉我。“我认为我们在这方面做得非常出色，并保持了 GitHub 的原始形式。你在 GitHub.com 看不到比四年前更多的微软，这帮助我们继续增长，我们对这种发展感到非常兴奋。”

他指出，多年来，GitHub 一直得到包括首席执行官塞特亚·纳德拉在内的微软领导团队的支持。“微软没有忘记我们当初为什么要做这笔交易，以及这笔交易的重要支柱是什么。首要的原则是把开发者放在第一位。这就是我们每天所做的，”多姆克说。

但是，他也承认微软是一家大公司，人们有时会对微软和 GitHub 的关系有自己的想法。不过，到目前为止，双方的领导层似乎都能够将这些想法拒之门外。

多姆克指出，GitHub 显然受益于微软的销售能力，这帮助它获得了许多大客户。这无疑也帮助该公司实现了昨天宣布的 10 亿美元的年度经常性收入。多姆克说，他相信 GitHub 作为一家独立公司也有可能达到这一里程碑。

“我通常是个乐观的人，”他说。“因此，只要专注于自己的使命，任何公司都可以实现这一目标。公司一旦发展到一定规模，面临的最大挑战就是专注。”

今天的 GitHub 显然和四年前的 GitHub 处于不同的位置。举例来说，它的产品组合已经随着 CodeSpaces 和最近的 Copilot 等项目有了很大的扩展。“我认为，如果我们培养出快乐的开发人员，我就完成了我作为首席执行官的使命——快乐的开发人员喜欢做他们的工作，不把安全性、合规性和可访问性视为负担，而是让他们快乐和让他们在生活中表现的一部分，”多姆克说。像这样的项目显然是其中的一部分。

“我认为，我们在这里所做的是，我们用人工智能、副驾驶和代码空间扰乱自己，”他补充道这些都是远离传统 GitHub 的新投资，传统 GitHub 有回购、问题和维基，并不断推动我们认为可能的边界。"

但是，他也强调，这不仅仅是关于大的公告和华而不实的事件，还应该关注那些可能对让开发者满意同样重要的小修补和功能。“我认为这是我们的超能力:我们可以在微小的成功和对我们自己业务的重大干扰之间取得平衡。”
