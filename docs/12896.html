谷歌关闭了网络上的 Duplex，这是它试图将人工智能引入零售网站等

谷歌正在关闭其人工智能驱动的网络双工服务，该服务导航网站，以简化点餐、购买电影票等过程。根据谷歌支持页面上的一条消息,从本月起，将不再支持网络上的双面打印和任何由它启用的自动化功能。

“随着我们继续改善 Duplex 体验，我们正在回应我们从用户和开发者那里听到的关于如何让它变得更好的反馈，”谷歌发言人通过电子邮件告诉 TechCrunch，并补充说，Duplex on the Web 的合作伙伴已经得到通知，以帮助他们为关闭做准备。“到今年年底，我们将拒绝网络上的双工，并完全专注于使人工智能进步到每天帮助人们最多的双工语音技术。”

谷歌在其 2019 年谷歌 I/O 开发者大会上推出了网络双工，这是其呼叫自动化双工技术的产物。一开始，它专注于几个狭窄的用例，包括打开一家连锁电影院的网站，代表用户填写所有必要的信息——暂停以提示座位等选择。但后来，Duplex on the Web将扩展到了密码，帮助用户自动更改数据泄露时暴露的密码，以及协助电子商务零售商结账、航空公司网站的航班登机以及自动折扣查找。

Duplex 在网络上的承诺是，你可以向谷歌助理发出类似“从赫兹为我预订一辆车”的命令，并让 Duplex 调出相关网页，自动填写你的姓名、汽车偏好、旅行日期和支付信息等细节(使用 Gmail 和 Chrome 自动填充的信息)。但是部署开始时很慢，只有有限数量的站点和合作伙伴支持特定的用例。Android 是唯一可以使用 Web 上的 Duplex 的平台，该服务于 2019 年底作为“Chrome 中的助手”进入 Chrome for Android。

网上双工被用来预订汽车。图片来源:谷歌

对于谷歌来说，技术上的提升是不是太大了，以至于无法证明维持网络上的双工是正确的？也许吧。正如 Duplex on the Websupport page所概述的，Duplex 使用了一种特殊的用户代理，它每天最多几个小时爬取网站，定期对它们进行“训练”，微调人工智能模型，以从用户的角度理解网站的布局和功能。这无疑是资源密集型的，如果网站所有者选择阻止爬虫索引他们的内容，很容易出错。

毫无疑问，一些品牌对于谷歌在他们和他们的顾客之间插上一脚的想法也感到不舒服。但也许压垮骆驼的最后一根稻草是谷歌业务助手方面的削减。根据最近的报道,谷歌计划减少投资，为非谷歌制造的设备开发谷歌助手，这是受到企业的其他领域，如硬件，将被证明是更有利可图的想法的刺激。

时间会证明事实是否如此。但是可以肯定的是，网络双工已经加入了被大肆宣传后又被抛弃的谷歌产品的恶名堂。
