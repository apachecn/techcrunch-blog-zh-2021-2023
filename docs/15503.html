OpenAI 为 ChatGPT 推出 API，并为企业客户提供专用能力| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/01/openai-launches-an-api-for-chatgpt-plus-dedicated-capacity-for-enterprise-customers/

旧金山初创公司 OpenAI 开发的免费文本生成人工智能 ChatGPT 称之为“成功”是一种轻描淡写的说法。

截至 12 月，ChatGPT 的估计月活跃用户超过 1 亿。它的吸引了各大媒体的关注，并在社交媒体上催生了无数的模因。它已经被用来在亚马逊的 Kindle 商店中编写数百本电子书。它被认为至少与人合著了一篇科学论文。

但是 OpenAI 作为一家企业——尽管是一家利润有限的企业——必须以某种方式将 ChatGPT 货币化，以免投资者坐立不安。今年 2 月，谷歌推出了一项名为 ChatGPT Plus 的高级服务，向这一目标迈进了一步。今天，它采取了一个更大的举措，引入了一个 API，允许任何企业将 ChatGPT 技术构建到他们的应用程序、网站、产品和服务中。

API 一直是我们的计划。这是 OpenAI 的总裁兼董事长 Greg Brockman(也是联合创始人之一)所说的。昨天下午，在 ChatGPT API 发布之前，他通过视频电话与我进行了交谈。

“让这些原料药达到一定的质量水平需要一段时间，”布罗克曼说。“我认为是这样的，就像是能够满足需求和规模。”

布罗克曼说，ChatGPT API 是由 OpenAI 广受欢迎的 ChatGPT 背后的相同人工智能模型驱动的，称为“gpt-3.5-turbo”。GPT-3.5 是 OpenAI 目前通过其 API 套件提供的最强大的文本生成模型；“涡轮”这个绰号指的是一个优化的、响应更快的版本的 GPT-3.5，OpenAI 一直在悄悄地为 ChatGPT 进行测试。

布罗克曼称，该 API 的价格为每 1000 个令牌 0.002 美元，约合 750 个单词，可以驱动一系列体验，包括“非聊天”应用。Snap、Quizlet、Instacart 和 Shopify 都是早期采用者。

开发 gpt-3.5-turbo 的最初动机可能是为了削减 ChatGPT 庞大的计算成本。OpenAI 首席执行官山姆·奥特曼曾经称 ChatGPT 的费用“令人眼馋”，估计他们每次聊天的计算成本只有几分钱。(有超过 100 万的用户，这可能会很快增加。)

但是布罗克曼说 gpt-3.5-turbo 在其他方面有所改进。

“如果你正在建立一个人工智能导师，你永远不会希望导师只是给学生一个答案。你希望它总是解释它，并帮助他们学习——这是你应该能够(用 API)建立的那种系统的一个例子，”布罗克曼说。“我们认为这将会使 API 变得更容易使用和访问。”

ChatGPT API 支撑着 My AI ，Snap 最近发布的面向 Snapchat+订户的聊天机器人，以及 Quizlet 新的 Q-Chat 虚拟导师功能。Shopify 使用 ChatGPT API 建立了一个个性化的购物推荐助手，而 Instacart 利用它创建了 Ask Instacart，这是一个即将推出的收费工具，允许 Instacart 客户询问食物，并根据该公司零售合作伙伴的产品数据获得“可购物”的答案。

Instacart 首席架构师 JJ·庄(音译)通过电子邮件告诉我:“买菜可能需要很大的精神负荷，有很多因素在起作用，比如预算、健康和营养、个人口味、季节性、烹饪技巧、准备时间和食谱灵感。”。“如果人工智能可以承担这种精神负担，我们可以帮助通常负责杂货店购物、膳食计划和将食物放在桌子上的家庭领导人——并实际上使杂货店购物真正有趣，会怎么样？Instacart 的人工智能系统，当与 OpenAI 的 ChatGPT 集成时，将使我们能够做到这一点，我们很高兴开始尝试 Instacart 应用程序中的可能性。”

然而，那些密切关注 ChatGPT 传奇的人可能会想，发布的时机是否已经成熟——这是理所当然的。

早期，用户能够提示 ChatGPT 以种族主义和性别歧视的方式回答问题，这反映了 ChatGPT 最初接受训练时所依据的有偏见的数据。(ChatGPT 的训练数据包括广泛的互联网内容，即电子书、Reddit 帖子和维基百科文章。)ChatGPT 还会在不披露的情况下虚构事实，这种现象在人工智能中被称为幻觉。

chat GPT——以及类似的系统——也容易受到基于提示的攻击,或者恶意的敌对提示，让它们执行并非其最初目标的任务。Reddit 上的整个社区都在寻找“越狱”ChatGPT 并绕过 OpenAI 设置的任何安全措施的方法。在一个不那么令人不快的例子中，初创公司 Scale AI 的一名员工能够让 ChatGPT 向泄露关于其内部技术工作的信息。

毫无疑问，品牌不想成为靶子。布罗克曼坚称他们不会。为什么这样他说，一个原因是后端的持续改善——在某些情况下是以肯尼亚合同工为代价的。但是布罗克曼强调了一种新的(也是争议更少的)方法，OpenAI 称之为聊天标记语言，或 ChatML。ChatML 将文本作为一系列消息和元数据提供给 ChatGPT API。这与标准的 ChatGPT 相反，后者使用表示为一系列标记的原始文本。(例如，单词“fan”、“tas”和“tic”将被拆分为记号。)

例如，给定提示“我的 30 岁生日有哪些有趣的派对创意？”开发人员可以选择在该提示后添加一个附加提示，如“你是一个有趣的聊天机器人，旨在帮助用户解决他们提出的问题。你要如实回答，用一种好玩的方式！”或者“您是一个机器人”，然后让 ChatGPT API 处理它。根据布罗克曼的说法，这些指导有助于更好地定制和过滤 ChatGPT 模型的反应。

“我们正在转向更高级别的 API。布罗克曼说:“如果你有一种更结构化的方式来表示系统的输入，你可以说，‘这是来自开发者’或‘这是来自用户’……我认为，作为一名开发者，你实际上可以更强大地(使用 ChatML)对抗这种即时攻击。”。

另一个(有希望)防止意外 ChatGPT 行为的变化是更频繁的模型更新。随着 gpt-3.5-turbo 的发布，开发者将默认自动升级到 OpenAI 的最新稳定模型，Brockman 说，从 gpt-3.5-turbo-0301(今天发布)开始。如果开发者愿意的话，他们可以选择继续使用旧的模型，但是这可能会在某种程度上抵消它的好处。

Brockman 指出，无论他们是否选择更新到最新型号，一些客户(主要是预算相对较大的大型企业)将通过引入专用容量计划对系统性能进行更深入的控制。本月早些时候泄露的文档中首次详细描述了今天推出的 OpenAI 专用容量计划，让客户为运行 OpenAI 模型的计算基础设施分配付费——例如，gpt-3.5-turbo。(对了后端是天蓝色的。)

除了对实例负载的“完全控制”(通常，对 OpenAI API 的调用发生在共享计算资源上)之外，专用容量还使客户能够实现更长的上下文限制等功能。上下文限制指的是模型在生成附加文本之前考虑的文本；更长的上下文限制允许模型“记住”更多的文本。虽然更高的背景限制可能无法解决所有的偏见和毒性问题，但它们可能会导致 gpt-3.5-turbo 这样的模型产生更少的幻觉。

Brockman 表示，专用容量客户可以预期 gpt-3.5-turbo 模型具有高达 16k 的上下文窗口，这意味着他们可以接受四倍于标准 ChatGPT 模型的令牌。比方说，这可能会让某人粘贴一页又一页的税务代码，然后从模型中获得合理的答案——这在今天是不可能的。

布罗克曼暗示将来会有一个全面的发布，但不是很快。

布罗克曼说:“上下文窗口开始慢慢变大，我们现在只专注于容量客户的部分原因是因为我们这边有很多性能权衡。”“我们可能最终能够提供同一事物的点播版本。”

鉴于 OpenAI 在获得微软数十亿美元投资后盈利的压力越来越大，这一点也不奇怪。
