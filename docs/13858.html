互操作性和去中心化已经成为今年科技领域的主要主题，这在很大程度上是由不断增加的监管、社会和行业压力以及对加密和网络 3 的大肆宣传推动的。这种上升的趋势正在提升其他船只，例如一种叫做矩阵的基于开放标准的通信协议——它正在为我们数字生活的另一个专有部分带来互操作性:消息传递。

Matrix 的联合创始人之一马修·霍奇森(Matthew Hodgson)表示，Matrix 网络上的人数今年翻了一番，达到了 8030 万用户(这个数字可能更高；并非所有 Matrix 部署都“呼叫总部”统计到 Matrix.org)。

虽然所有这些活动的大部分都是在企业通信领域，但看起来主流消费者平台现在可能也注意到了这一点。

工程师兼应用研究员简·满春·王的一些调查发现了 Reddit 正在测试 Matrix 聊天功能的证据——Reddit 或多或少地向 TechCrunch 证实了这一举动。一位发言人表示，他们正在“寻找一些方法来改善 Reddit 上的对话”，并“测试一些选项”，尽管他们没有特别提到姓名检查矩阵。

考虑到对互操作性的更大支持——这也发生在 数字钱包 和 地图 中——仔细观察 Matrix 可以了解我们是如何做到这一点的。

起初

从上面看手握手机。图片来源:马尔特·穆勒/盖蒂

任何发送过短信或电子邮件的人都不会考虑收件人使用的是什么网络、服务提供商或信息客户端。主要原因是这并不重要——T-Mobile 和威瑞森的用户可以很好地相互发送短信，而 Gmail 和 Outlook 的用户发送电子邮件没有问题。

但情况并非总是如此。在电子邮件的早期，你只能给同一网络上的用户发信息。随着手机在整个 20 世纪 90 年代的普及，如果朋友在不同的移动网络上，人们最初无法给他们发消息。欧洲和亚洲在互操作性方面处于领先地位，到 2000 年伊始，北美的大型电信公司也意识到，如果他们允许消费者在竞争对手的网络上给朋友发信息，他们就能打开一座名副其实的金矿。这对每个人来说都是双赢。

快进到现代智能手机时代，虽然电子邮件并没有像渡渡鸟一样消失，短信仍然在蹒跚前行，但今天卓越的通信工具彼此之间并不友好。那些希望拥抱独立的注重隐私的消息应用如 Signal 的人将会碰壁，当他们意识到字面上他们所有的朋友都在使用 WhatsApp。或者 iMessage。或者电报。或者 Viber …你明白了。

这种趋势也渗透到了企业领域。如果你的工作使用 Slack，那么祝你好运，向你在城市另一端被迫使用微软团队的伙伴发送消息，而那些强行进入 Meta 工作场所的人力资源人员可以再次考虑使用 Salesforce Chatter 在走廊上与他们的销售同事进行 DM。

这当然不是什么新鲜事，但在线消息领域的互操作性问题在 2022 年已经成为焦点。欧洲正在通过数字市场法案( DMA )推进规则以强制在线平台之间的互操作性和可移植性，而美国通过访问法案也有类似的计划。

与此同时，埃隆·马斯克在 Twitter 的到来引发了对乳齿象等替代物的关注，即所谓的“开源 Twitter 替代物”，在 Twitter 的混乱之后，它在拥有了 200 万用户。Mastodon 由开放的 ActivityPub 协议提供支持，并围绕 Fe diversity 的概念构建:一个由互联服务器组成的分散网络，允许不同的 ActivityPub 支持的服务相互通信。Tumblr 最近透露它打算在未来支持 ActivityPub 协议，而 Flickr 首席执行官 Don MacAskill 就照片托管平台和社区是否也应该采用 ActivityPub 对他的 Twitter 粉丝进行了调查。

但是尽管最近几周 Twitter 马戏团对互操作性进行了喧嚣和炒作，这个方向上已经有了一个安静但不断发展的运动；由企业和政府推动的一场运动，旨在避免供应商锁定并获得对其数据堆栈的更大控制。

进入矩阵

Element 创始人和 Matrix 联合创始人 Matthew Hodgson 和 Amandine Le Pape。图像制作者名单:元素

Matrix 于 2014 年在软件和服务公司 Amdocs 内部开发，由 Hodgson 和Amandine Le Pape领导，他们后来离开公司，完全专注于将 Matrix 作为一个独立的开源项目来发展。他们还试图通过一家名为 New Vector 的 公司将 Matrix 商业化，该公司开发了一款 Matrix 托管服务和一款名为 Riot 的Slack alternative应用。2018 年，霍奇森和 Le Pape 成立了 Matrix.org 基金会 ，作为万物矩阵的法人实体和监护人，包括保护其知识产权，管理捐赠和推动协议向前发展。

两年多前，Matrix 的旗舰商业实现被更名为 Element ，今天，Element 在 Automattic、Dawn Capital、idea、Protocol Labs 和其他公司的支持下，被许多组织用来寻找美国科技巨头出售的大牌现任公司的联合替代方案。

Element 本身是开源的，并承诺端到端的加密，而其客户可以访问大多数人期望从团队协作产品中获得的常见跨平台功能，包括群组消息和语音及视频聊天。

行动中的元素。图片来源:元素

Element 也可以托管在公司自己的基础设施上，避免担心他们的数据可能会在第三方服务器上被(错误)使用，确保他们保持对其整个数据堆栈的控制——这是托管敏感数据的实体的交易者或破坏者。

越来越多的法规，尤其是在欧洲，正迫使大型科技公司关注数据主权，谷歌等公司去年与德国电信的 IT 服务和咨询子公司 T-Systems 合作，为德国公司的敏感数据提供“主权云”。

这种监管推动，加上对数据主权日益增长的期望，对矩阵协议来说是一个福音。去年，负责德国医疗保健系统数字化的机构透露它正在向 Matrix 过渡，以确保组成医疗保健行业的 150，000 个个体实体(如医院、诊所和保险公司)可以相互通信，无论他们使用的是什么基于 Matrix 的应用程序。

这基于其他地方的现有矩阵实施，包括法国政府内部的 Tchap 团队协作平台，以及德国武装部队联邦国防军。

霍奇森向 TechCrunch 解释说:“钟摆很明显已经摆向去中心化有一段时间了。”。“我们现在看到，在法国、德国、英国、瑞典、芬兰和美国政府之间或内部，以及在北约和邻近组织中，基于矩阵的分散式通信得到了广泛应用。”

早在五月份，开源企业消息平台迅速发展。聊天透露它将过渡到矩阵协议。虽然这一过程仍在进行中，但考虑到那枚火箭，这代表着矩阵运动的一次重大政变。Chat 声称拥有大约 1200 万用户，遍布奥迪、大陆集团和德国国家铁路公司德国铁路。

“我们认为，任何消息平台的价值都是基于其与其他平台连接的能力而增长的，”火箭说。聊天发言人告诉 TechCrunch。“我们在连接火箭上下了很大功夫。和其他平台聊天。我们不必担心用什么客户端发邮件，当我们互相发信息时也应该如此。”

Rocket.Chat. 图片来源:火箭。闲谈

也许最有趣的是，这与传统的消费者和企业社交网络以及团队协作工具所走的道路背道而驰。

Slack、脸书、微软团队、WhatsApp、Twitter 和所有其他公司都在利用网络效应，即产品的价值与用户数量有着内在联系。最终，人们希望和他们的朋友和同事在一起，这不可避免地意味着坚持使用他们不太喜欢的社交网络，或者同时使用多个不同的应用程序。

开放和可互操作的协议支持一种新的业务，这种业务认识到对不锁定用户的东西的不断增长的需求。

“我们的目标不是强迫人们使用火箭。聊天为了互相交流，“火箭。Chat 的发言人继续说道。“相反，我们的目标是让组织能够安全地协作，并通过他们选择的平台与其他组织和个人联系。”

弥合分歧

Matrix 协议还通过一种名为“桥接的技术支持非本地互操作性，该技术引入了对非 Matrix 应用程序的支持，包括 WhatsApp、Telegram 和 Signal。Element 本身提供桥接功能，作为一个名为 Element One 的以消费者为中心的订阅产品的一部分，用户每月支付 5 美元，就可以将他们所有的朋友聚集在一个界面上——不管他们使用什么应用程序。

Element One 的用户可以将不同的消息应用整合在一起。图片来源:黑客帝国基金会

这是通过科技公司自己开发的公开可用的 API 实现的。然而，使用条款通常会对竞争应用程序如何使用它们进行限制，同时它们也可能会强制执行费率限制或使用成本。

桥接就其现状而言，处于“这是允许的吗？”视角。但随着全球监管目光聚焦于大型科技公司对在线通信的束缚，这些公司可能不会过于严格地执行所有条款和条件。

DMA 上个月在欧洲生效——尽管它要到明年 5 月才能正式适用——它对互操作性和数据可移植性有具体的规定。到那时，我们也许会开始看到世界上的大型技术“守门人”如何计划支持新的法规。事实上，我们所谈论的是开放的 API，它们“正式”允许较小的第三方与他们的大型技术兄弟进行集成和通信。然而，这并不一定意味着这样的 API 会很光滑，易于使用，有清晰的文档，我们可能会遇到一些故意的拖后腿和障碍。

服从

iPhone 上显示的 WhatsApp 和脸书应用程序。贾斯汀·沙利文/盖蒂图片社

WhatsApp 等流行的消息应用程序虽然提供端到端加密，但并不是为企业或政府用例设计的，因为它们不允许组织轻松管理他们的任何消息数据，但这种应用程序在这种情况下被广泛使用。早在 7 月，英国信息专员办公室(ICO) 呼吁政府对个人电子邮件账户和 WhatsApp 等“私人通信渠道”的风险进行审查，指出这种使用缺乏“明确控制”，可能导致关键信息“丢失或不安全处理”

英国信息专员约翰·爱德华兹(John Edwards)当时在一份声明中表示，“我理解 WhatsApp 这样的应用所能带来的即时沟通的价值，尤其是在疫情期间，官员们被迫做出快速决策，努力满足各种需求。”。“然而，使用这些方法的代价，虽然不违反法律，但不得导致缺乏透明度和数据安全性不足。公职人员应该能够展示他们的工作情况，这既是为了保存记录，也是为了保持公众的信心。这就是如何确保对这些决定的信任，并为未来吸取经验教训。”

与此同时，在商业领域，美国证券交易委员会(SEC)最近与 16 家华尔街公司达成 11 亿美元的和解，原因是他们使用 WhatsApp 等私人通讯应用程序的“广泛记录失败”。

“金融最终取决于信任，”SEC 主席加里·詹斯勒(Gary Gensler)当时表示。“自 20 世纪 30 年代以来，这种记录对保持市场完整性至关重要。随着技术的变化，注册人仅在官方渠道内恰当地进行商务沟通变得更加重要，他们必须维护和保存这些沟通。”

保持准确的书面记录，并确保政治家和企业对自己的行为负责，这是游戏的名字——类似矩阵协议的某种控制水平。然而，强制要求每家超过一定规模的公司——正如 DMA 法规所做的那样——必须使其软件能够与其他公司互操作，这引发了一系列关于隐私、安全和更广泛的用户体验的问题。

房间里的加密大象

“房间里的大象”概念图图片来源: Klyaksun/Getty Images

正如凯西·牛顿不止一次在 Platformer 指出的那样，欧洲新的互操作性法规有几个缺陷。其中最主要的可能是它们将为端到端加密制造的障碍——也就是说，确保数据在传输过程中保持加密状态并且无法被解码。

端到端加密是当今大型科技公司的一个巨大卖点，WhatsApp 在屋顶上大声疾呼。但是，在不同公司构建的不同平台之间实现这一点并不容易，许多(如果不是大多数的话)这方面的专家说，不可能实施一个真正安全的、可互操作的、不会以某种方式损害加密的消息传递基础设施。

WhatsApp 可以控制——并因此承诺——自己平台上的端到端加密。但如果数十亿条消息在 WhatsApp 和其他公司运营的无数其他应用程序之间飞来飞去，那么一旦这些消息离开 WhatsApp，WhatsApp 就无法真正知道它们发生了什么。

最终，没有两个服务部署相同的加密，这是霍奇森承认的一个挑战。“端到端的加密平台必须从头到尾讲同一种语言，”他说。

在今年早些时候发表的一篇博文中，Matrix 基金会提出了一些解决加密问题的方法，包括让所有的大型网守转而使用同一个“去中心化的端到端协议”(也就是 Matrix，不出所料)，据基金会自己承认，这将是一个巨大的工程——但它说“我们不应该排除”。

为了说明这一点，霍奇森提到了元素 2020 年对 Gitter 的收购，Gitter 是一个以开发者为中心的社区和聊天平台，从 GitLab 购买，被包括谷歌、微软和亚马逊在内的知名公司使用。在完成交易的两个月内，Element 向 Gitter 引入了原生矩阵连接。

当然，在脸书、谷歌或苹果的规模上协调这样的转变将是一个完全不同的命题；一场可能引发各种连锁混乱的危机。在今年早些时候的一篇博客文章中，加密和安全专家 Alec Muffett 暗示，遵守相同标准协议的消息应用和社交网络将导致不同服务之间“没有实际区别”。

“想象一个 Signal 和 Snapchat 必须互操作的世界，那会是什么样子？”Muffett 在这篇文章的问答中反问 TechCrunch。“具体来说，其中一个产品的哪些功能需要在另一个产品上呈现，围绕这些功能的竞争优势是什么？功能上的冲突如何协调？”

这就是为什么矩阵基金会提出了其他潜在的解决方案，如采用 TLS 证书式的警告，提醒用户他们的跨服务对话没有得到完全保护。这或许可以与苹果的 Messages 应用程序支持加密的 iMessage 文本和(未加密的)SMS 相媲美。但是根据 Muffett 的说法，这会给混合带来不必要的复杂性。

Muffett 说:“除了我能举出的任何其他原因，有大量的用户界面研究解释了安全弹出警告通常不被理解和注意。”“有大量的研究支持这一点——弹出警告是一种' 反模式 。”"

Matrix Foundation 还提出在“桥”中转换加密语言之间的通信流量，尽管这实际上意味着必须破解加密，并在某个地方安全地重新加密流量。

“这些桥可以在客户端运行——例如，Matrix iMessage 桥在 iPhone 或 Mac 上运行客户端——或者通过使用客户端开放 API 在手机本身内部的应用程序之间建立桥梁，”霍奇森说。或者，它们可以在服务器端由用户以分散方式控制的硬件上运行，确保重新加密在尽可能安全的环境中进行，而不是在易受攻击的集中式服务器上进行

无法回避的事实是，破解加密远非理想之举，无论解决方案如何调和这一点。但也许更重要的是，解决由强制互操作性引入的真正加密问题的健壮解决方案还没有真正存在。

尽管如此，霍奇森过去曾表示，欧盟新法规的好处大于坏处。

“总的来说，我们认为强制开放 API 的好处超过了有人将运行一个脆弱的大规模桥梁并破坏每个人的 E2EE 的风险，”他在五月写道。“拥有能够在第一时间获取您的数据的选项，比被禁锢在一个有围墙的花园里要好。”

冰山一角

值得注意的是，Matrix 协议虽然主要以其在消息传递领域的存在而闻名，但也有其他潜在的应用。Matrix 基金会最近宣布了第三个房间，这是一个建立在 Matrix 基础上的分散和可互操作的元宇宙平台。这与由少数把关人控制的潜在未来元宇宙背道而驰，如脸书的母公司 Meta。

就目前而言，Element 仍然是矩阵动力世界的旗舰海报。该公司已经获得了一些知名客户，如 Mozilla，该公司正在使用 Element 作为完全托管的服务，而 Element 表示，它今年与另一家公司(未透露名称)签署了一份价值 1800 万美元的四年期协议。与此同时，它也有战略支持者，其中包括 WordPress.com 母公司 Automattic，它首先在 2020 年向 Element 投资了 460 万美元，然后在去年投资了 3000 万美元的 B 轮融资。

在许多方面，Matrix 的发展从未如此肥沃:它在正确的时间处于正确的位置，因为世界正在寻找一条至少有一点监管支持的从大型科技公司的魔掌中退出的路线。Twitter 也在强调集中控制的缺点方面发挥了不小的作用，迎合了所有鼓吹互操作性的公司的胃口。

霍奇森说:“在建立对集权危险的认识方面，Twitter 的情况绝对令人惊讶，它提供了一个关键时刻，帮助用户发现我们正在进入一个去中心化的黄金时代。”“正如许多用户发现乳齿象是 Twitter 越来越可行的去中心化替代品一样，我们也看到了用户发现 Matrix 作为一种方式来重新获得他们对即时通信(如消息和 VoIP)的独立性的巨大光环效应——特别是我们的长期用户群正在以前所未有的速度增长。”
