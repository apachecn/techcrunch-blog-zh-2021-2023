帕拉丁云推出开源平台

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/11/paladin-cloud-launches-open-source-platform-for-cloud-security-and-governance/amp/

虽然云在现代商业世界中扮演着越来越重要的角色，但迁移到远程基础设施的公司面临着一系列挑战——可以说其中最主要的是安全性问题。

在云中托管应用程序和数据是当今无数公司的现实。但是，由于集成的复杂性，从 API 网关到 Kubernetes，要准确地了解幕后发生的事情并不容易。结果是识别违反安全策略的行为变得有点棘手。

这是一个圣骑士云正在着手解决的问题，一个开源的“安全代码”平台今天向公众发布。

Paladin 的核心存在理由是帮助开发人员和开发人员运营(DevOps)团队在测试和生产环境中保护他们的应用程序和数据——它通过全面了解他们无数云服务和企业系统的安全状况来实现这一点。最终，这一切都是为了自动检测和补救违反安全策略的情况，这可能包括从未经授权的访问或错误配置到不安全的 API 等任何情况。

为了帮助开源项目进入下一个阶段，并进一步走向商业化，位于新泽西州皮斯卡塔韦的初创公司今天宣布，它已经在由 Okapi Venture Capital 和 Bowery Capital 共同牵头的种子资金轮中筹集了 330 万美元，参与投资的有一系列著名的支持者，包括 Samsung Next、T-Mobile Ventures、SaaS 风险投资公司、达阵风险投资公司和 UST。

插件架构

Paladin 采用了一种基于插件的架构，可以帮助开发人员连接和接收来自无数来源的数据，包括代码库、威胁情报系统、API 网关、Kubernetes 等。然后，Paladin 可以发现所有资产，评估和确定违反策略的情况，并执行任何预先配置的自动修复。

虽然 Paladin 旨在保护所有主要的公共云，包括 AWS、Azure 和 Google Cloud，但作为一个开源项目，它具有灵活性和可扩展性。这意味着它可以用作公司混合云战略的一部分，其中他们的一些数据和应用程序托管在公共和私有云基础架构之间——Paladin 可以用于在这些不同的系统之间建立桥梁。

为了更好地了解《圣骑士》进入的世界，这个领域还有许多其他类似的玩家——这些包括 Wiz ，一家60 亿美元的公司； Bridgecrew ，最近被帕洛阿尔托网络以 1.56 亿美元抢购；以及 Accurics，它是 Tenable 以大约相同的金额收购的。更不用说其他羽翼未丰的初创公司，如 Jit，它最近以 3850 万美元的种子资金退出了 stealth。

因此，很明显，在 DevOps 时代，对云原生安全性有着真正的需求。然而，Paladin 将其开源基础作为其核心优势之一。好处是开放源码解决方案带来的常见优势，包括开放源码对于喜欢自己动手修改和测试软件的开发人员来说是一个非常有吸引力的提议，而不必通过巨大的公司障碍。

“开发人员更喜欢部署开源解决方案，而不是封闭的源代码，”Paladin 联合创始人兼首席执行官 Daniel Deeney 告诉 TechCrunch。“迄今为止，闭源玩家将他们的产品作为付费企业解决方案出售给 CISO 或安全团队。许多开发人员不使用这些产品，因为他们经常没有预算来购买昂贵的企业解决方案，并且这些产品不能灵活地集成到其他基于云的系统中。”

圣骑士云创始人史蒂夫和丹尼尔。图片来源:圣骑士云

虽然该领域还有其他开源公司(包括风险投资支持的 Stacklet )，但 Paladin 正在宣传其云安全的“整体方法”，其中包括前面提到的基于连接器的架构，该架构不仅将安全性扩展到主要的公共云，还扩展到 Kubernetes 等广泛的云技术。

故事到此为止

自 2021 年底以来，Paladin 一直以隐形模式作为一个自筹资金的项目存在，在这期间的几个月里，创始人——Deeney 和 CTO Steve Hull——一直在与社区的合作伙伴客户和开发人员合作开发该产品。

今天的公告预示着 GitHub 上的核心圣骑士开源项目正式启动，该项目包括一系列开箱即用的功能，包括 UI 管理仪表板；跨越主要云的最佳实践安全策略；直接连接到基于云的企业系统的策略管理平台；基于角色的访问控制(RBAC)；基于严重性的违反政策的优先顺序；还有更多。

圣骑士仪表板。图片来源:圣骑士云

这也标志着最终完全商业化产品的开始。这将包括具有自助 SaaS 平台的企业级产品，以及服务级别协议(SLA)、基于人工智能的风险评分和增强的合规性报告等高级功能。

帕拉丁表示，它已经为至少一家上市公司提供了商业支持。

“除了许多开发人员使用 OSS(开源软件)产品的测试版，Paladin 还与费城的一家上市公司签署了一份多年的 OSS 支持合同，以推出开源产品，”Deeney 说。"由于保密协议，公司目前不允许透露客户的姓名."

但在大规模货币化之前，Paladin 现在资金充足，可以通过组建团队来为其在开源社区的增长提供资金。

“该公司正计划利用种子轮收益聘请一些关键职位，如开发者关系主管，并建立其开发团队，”Deeney 说。
