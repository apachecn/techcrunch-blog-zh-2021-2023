苹果修复了恶意应用绕过 macOS 安全保护的漏洞

微软表示，它在 macOS 核心安全功能 Gatekeeper 中发现的一个漏洞，可能允许攻击者用恶意软件损害易受攻击的 MAC 电脑。

这个被追踪为 CVE-2022-42821 的漏洞，首先由微软首席安全研究员乔纳森·巴尔或发现，并被称为“阿基里斯”漏洞。Bar Or 表示，这个漏洞可能会让恶意软件绕过 macOS 上的网守保护。

Gatekeeper 于 2012 年首次推出，是一项安全功能，旨在只允许可信软件在 macOS 上运行。该功能会自动验证从互联网上下载的所有应用程序是否来自经过鉴定的开发者，这些开发者的已经过苹果的“公证”，并且他们的应用程序没有恶意内容。

微软的 Bar Or 在一篇博客文章中解释道【macOS 给从网络浏览器下载的应用程序和文件添加了一个“隔离”属性，并指示看门人在文件可以打开之前对其进行检查。但阿基里斯漏洞利用了一种称为访问控制列表(ACL)的文件权限模型，为下载的文件添加了极其严格的权限，从而阻止 web 浏览器正确设置隔离属性。

在利用该漏洞时，用户可能会被诱骗在 macOS 上下载并打开恶意文件，而不会触发 Gatekeeper 的安全保护。

微软在 7 月份报告了阿基里斯漏洞，但苹果直到上周才承认漏洞已经修复。

Bar Or 表示， Lockdown Mode 是苹果今年早些时候推出的一项可选功能，旨在帮助高风险用户阻止一些更复杂的网络攻击，但它无法抵御阿基里斯漏洞，因为 Lockdown Mode 旨在阻止无声和远程触发的“零点击”攻击，无需用户交互。“最终用户应该应用该修复程序，而不管他们的锁定模式状态如何，”Bar Or 说。

阿喀琉斯只是近年来发现的众多守门人旁路之一。2021 年 4 月，苹果修复了 macOS 中的一个零日漏洞，该漏洞使臭名昭著的 Shlayer 恶意软件背后的威胁行为者能够绕过苹果的看门人和公证安全检查。
