亚马逊推出 AWS 私有 5G

原文：https://web.archive.org/web/https://techcrunch.com/2022/08/12/amazon-launches-aws-private-5g-so-companies-can-build-their-own-on-site-mobile-networks/

亚马逊的现金奶牛云部门 AWS 推出了一项新服务，旨在帮助公司部署他们自己的专用 5G 网络——至少最终是这样。

AWS first 在去年年底的早期预览中宣布了 AWS Private 5G，但它现在正式向美国东部(俄亥俄州)、美国东部(弗吉尼亚州)和美国西部(俄勒冈州)地区的 AWS 客户提供，并计划在“不久的将来”在全球范围内推广

但是——这是一个很大的“但是”——尽管它的名字，AWS Private 5G 目前只支持 4G LTE。

“它今天支持 4G LTE，未来将支持 5G，这两种技术都能以超低延迟为你提供一致、可预测的吞吐量水平，”AWS 首席传播者杰夫·巴尔在博客文章中写道。

有了 AWS Private 5G ，公司直接从 AWS 订购硬件(一个无线电单元)和一堆特殊的 SIM 卡，然后 AWS 提供所有必要的软件和 API(应用编程接口)，使企业能够在现场建立自己的私人移动网络。这集成了 AWS 管理控制台，用户可以通过该控制台指定他们希望在哪里构建网络以及他们需要的容量，一旦客户激活了他们的小蜂窝无线电单元，AWS 就会自动进行网络设置和部署。

至关重要的是，AWS 管理的网络基础设施与其他 AWS 服务配合良好，包括其身份和访问管理( IAM )产品，这使 IT 能够控制谁和什么设备可以访问私有网络。AWS Private 5G 还接入亚马逊的cloud watchobservability 服务，该服务提供对网络健康状况以及其他有用数据点的洞察。

在成本方面， AWS 对其安装的每个无线电单元每小时收费10 美元，每个无线电支持 150 Mbps 的速度，跨越多达 100 个 sim(即单个设备)。最重要的是，AWS 将对所有向外传输到互联网的数据收费，按照亚马逊通常的 EC2(弹性计算云)费率收费。

因此，实际上，亚马逊正在承诺一些行业——如智能工厂或其他有高带宽需求的地点(偏远或其他地方)——即时、本地化的 5G，同时将它们推销到其粘性云基础设施中，在那里收取通常的费用。

公共与私人

很明显， 5G 有可能转变许多行业 ，并将成为从机器人和自动驾驶汽车到虚拟现实等一切的基石。但是，目前大多数拥有 5G 设备的消费者所依赖的公共 5G 网络覆盖范围有限，带宽可能由数百万用户共享。最重要的是，公司对网络几乎没有控制权，即使他们的场所在网络范围内。这就是为什么专用 5G 网络是一个有吸引力的提议，特别是对于具有要求全天候低延迟数据传输的任务关键型应用的企业。

AWS Private 5G 使用公民宽带无线电服务( CBRS )，这是一个共享的 3.5 GHz 无线频谱，由联邦通信委员会(FCC) 在 2020 年初授权用于商业环境，因为它之前是为国防部(DoD)保留的。因此，这次更新基本上为 CBRS 打开了无数的用例，包括寻求建立新的 5G 服务或扩展现有 4G/LTE 服务的企业。

与此同时，FCC 宣布授权关键频谱接入系统(SAS)管理员管理 CBRS 频段的无线通信，该流程旨在有效保护“高优先级”用户(如国防部)免受干扰。任何连接到 CBRS 频谱的设备都需要 SAS 管理员的授权，目前包括谷歌、索尼、康普、联邦无线、Key Bridge Wireless 和 Amdocs。

这是新的 AWS Private 5G 服务的一个关键组成部分，它完全集成到 SAS 管理流程中，AWS 代表客户管理一切，包括负责干扰问题以及与频谱接入相关的其他故障排除细节。

因此，亚马逊新的私有 5G 产品可能有点用词不当，因为它目前只支持 4G LTE。但是，OnGo 联盟(当时称为 CBRS 联盟)在两年多前就完成了 CBRS 的 5G 规范，在此期间的几个月里，它一直在为全面商用 5G 服务奠定基础——就在昨天，三星电子美国公司宣布与 Kajeet 合作，在 CBRS 部署一个新的专用 5G 网络。

尽管“AWS Private 5G”是对其未来支持的认可，但当前的品牌可能会在今天寻求本地 5G 部署的感兴趣的各方中引起一些恐慌。
