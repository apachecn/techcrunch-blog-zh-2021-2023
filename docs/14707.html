Twitter 的第三方客户端问题似乎是故意暂停

原文：https://web.archive.org/web/https://techcrunch.com/2023/01/16/twitters-third-party-client-issue-is-seemingly-a-deliberate-suspension/

上周五，包括 Tweetbot、Twitterrific 和 Echofon 在内的大量热门 Twitter 客户端都出现了故障。用户无法登录他们的账户或者查看他们的时间表。起初，这看起来像是 Twitter API 中的一个错误，但 Twitter 的无线电静默和新的细节表明，该公司故意限制对第三方应用的访问。(更新:推特证实在这篇报道之后 API 发生了变化。)

问题

太平洋时间周五晚间，许多用户注意到他们无法访问他们的第三方 Twitter 客户端。应用程序制造商很快承认了这个问题，并表示他们一直在试图联系该公司。

一个日本的开发者在那个时候注意到许多小的 Twitter 客户端运行没有任何故障。社区中的许多人猜测，这可能是 API 的问题，或者是公司正在限制更大客户的访问。

无线电静默

虽然开发者和用户期待 Twitter 以某种方式与他们沟通，但该公司及其新所有者埃隆·马斯克(Elon Musk)对这个问题保持沉默。然而，这位特斯拉首席执行官发布了从最新的猎鹰重型发射到通过发布平台的推文推荐代码在 Twitter 上建立透明度的所有事情。

Twitter 上的内部消息表明，关闭某些第三方客户端是公司的决定，而不是一个错误，周末报道的信息。报道称，一名项目经理告诉产品团队，该公司已经“开始进行沟通”，但没有提供任何官方和经批准的沟通时间表。

开发者受挫

自从故事开始以来，许多开发者已经在推特和 T2 乳齿象上表达了他们的沮丧。twit terrorial-maker Craig hocken berry 发布了一篇名为“狗屎秀”的博客，他在博客中说“就我个人而言，我不干了。带着复仇。”

菲尼克斯开发者 Matteo Villa 在 Twitter 上表示，他正在考虑将客户端从 App Store 中撤出——在撰写本文时该应用程序仍在运行——因为他担心客户端可能会在某个时候停止工作。

Tweetbot 的联合创始人保罗·阿达德甚至试图通过加载旧的 API 密钥来让应用程序工作。这种伎俩奏效了一段时间，一些人能够进入他们的账户。然而，用户开始达到一个 API 限制，客户端后来再次被挂起。

iOS 开发者 Mysk 在其账户上表示，Tweetbot 达到了所有用户每 15 分钟 300 条帖子的限制——这适用于旧的 1.1 版 API。

早些时候，他们已经建立了一个演示客户端，以显示 Twitter 的 API 正在工作，第三方应用程序的暂停不是因为一个错误。

如果 Twitter 禁止第三方客户端，这些开发者中的一部分人担心为那些订阅了专业版或高级版应用的人处理退款。这也意味着他们的年收入会下降，他们将不得不在不赚钱的情况下开发新产品。

前进的道路

一些开发商已经表现出专注于其他项目的意图。哈达德在一封电子邮件中告诉 TechCrunch，Tweetbot 正专注于加速推出其乳齿象客户端 Ivory——目前处于封闭测试阶段。

他说，目前团队专注于改善入职体验，然后修复漏洞，并致力于发布应用商店。

维拉还在苹果的测试平台 TestFlight 上发布了他的乳齿象客户端 Wolly 的测试版。

对于其他一些开发者来说，情况很糟糕——iOS 开发者亚当·德马西指出，一些主要产品是 Twitter 客户端的独立开发者可能会面临困难时期。

自去年马斯克接管 Twitter 以来，该公司已经关闭了几个与开发者相关的项目，包括用于应用发现的 Twitter 工具箱。即使该公司没有宣布正式关闭，其他一些项目也处于停止状态。鉴于该公司尚未明确传达其关于平台支持的计划，开发者对他们的 Twitter 开发计划持谨慎态度。

这些举动破坏了社交网络在过去几年赢回开发者信任的努力。上个月，Twitter 前开发者平台负责人 Amir Shevat 在 TechCrunch 上写了一篇关于新管理层如何辜负开发者信任的文章。这种第三方 Twitter 客户端在没有任何沟通的情况下可疑的暂停不会给社区注入任何信心。

Update，1/17/23，1 PM ET: Twitter 在周二下午发布的一篇帖子中证实了 API 的变化，声称它只是在执行其“长期存在的 API 规则”，并“可能导致一些应用程序无法工作。”正如对帖子的回复和其他社交媒体更新所表明的那样，这一迟来的回应并没有得到开发者或受影响用户的认可。
