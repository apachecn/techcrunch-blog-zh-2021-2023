微软推出 Google Wave | TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/02/microsoft-launches-google-wave/

微软正在重新启用 Google Wave，这个注定要失败的实时信息和协作平台由谷歌于 2009 年推出，并于 2010 年过早地关闭了 T2。

也许我们应该预见到这一点。回到 2019 年，微软宣布流体框架(不要和 Fluent 设计系统混淆)。这里的想法无非是试图重新发明业务文档的本质以及开发人员如何构建实时应用程序。去年，该公司开源了 Fluid ，并开始将其构建到自己的一些办公应用中。今天，在其 Ignite 大会上，它发布了一款基于流体的全新产品:微软 Wave Loop。

Loop 是一个新的应用程序和概念，它采用了 Fluid framework，为开发人员提供了灵活的组件来混合和匹配，以便创建基于实时编辑的应用程序，从而为用户创建新的文档协作体验。在许多方面，这也是 Google Wave 的承诺——实时协作加上开发者框架和协议，让 Wave 无处不在。

图片来源:微软

现在，你可能会说:团队不就是为了这个吗？为什么不把它融入到团队中？是的，所有这些都在工作中，但还会有一个 Loop 应用程序，微软称其“结合了强大而灵活的画布和可移植的组件，这些组件可以自由移动并在应用程序之间保持同步，使团队能够一起思考，规划和创造。”

图片来源:微软

循环有三个元素:循环组件，“生产力的原子单位”(向提出这个短语的人致敬)，像列表、表格、笔记和任务；Loop pages，“灵活的画布，您可以在其中组织组件并引入其他有用的元素，如文件、链接或数据，以帮助团队思考、联系和协作；”循环工作区，共享的空间，在那里你可以以某种方式了解每个人正在做的事情，并跟踪朝着共同目标的进展。

Wave 从来没有的一件事，显然是 Loop 的核心特性，那就是 Loop 实时跟踪你的光标位置。这就是元宇宙目前的状况。毕竟，没有什么比移动你的光标更能体现我在会议中的出席和活跃了。

一些新的 Loop/Fluent 组件即将推出，包括投票表(我想补充一下，这是 Google Wave 的第一天功能)和状态跟踪器。

Google Wave 显然走在了时代的前面。
