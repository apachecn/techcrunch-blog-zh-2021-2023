AWS 的分析服务无需服务器

在其 re:Invent 大会上，AWS 今天宣布，其四项基于云的分析服务亚马逊红移、亚马逊 EMR、亚马逊 MSK 和亚马逊 Kinesis 现已成为无服务器和按需服务。这是 AWS 的客户一直在要求的，AWS 首席执行官亚当·塞利普斯基在今天的主题演讲中说。这些新服务现在可以公开预览。

正如 Selipsky 所说，一些 AWS 竞争对手可能会认为一个数据库可以完成所有工作，但他认为不同的工作负载需要正确的数据库来支持它们，他认为分析服务也是如此。但是客户也不希望担心运行这些服务所带来的基础架构，除了必须管理群集之外，用户还将只需为他们使用的资源付费。例如，对于红移，这意味着用户将只在数据仓库被使用时付费，而不是在它闲置时付费。

AWS 的 Danilo Poccia 在今天的公告中解释说:“Amazon Redshift Serverless 会自动为您提供合适的计算资源。“随着您的需求随着更多并发用户和新工作负载而发展，您的数据仓库会自动无缝扩展以适应这些变化。您可以选择指定基本数据仓库的大小，以进一步控制成本和特定于应用程序的 SLA。”

类似地，AWS 用于处理流媒体数据的服务 Kinesis 现在提供了完全托管的点播模式。借助这种新的容量模式，服务可以根据数据流量自动扩展。

AWS 今天的举动显然是对市场压力的回应。它的许多竞争对手现在都提供类似的无服务器分析产品，许多资金雄厚的初创公司也是如此。
