Metrist 筹集 550 万美元来提供更好的云服务中断数据| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/01/metrist-raises-5-5m-to-provide-better-cloud-service-outage-data/

Metrist 是一家帮助 IT 团队在他们运行自己的应用程序所使用的许多云服务中保持领先的初创公司，今天宣布它已经从 Heavybit 、 Morado Ventures 和 PagerDuty 联合创始人 Alex Solomon 以及 StatusPage 联合创始人 Scott 和 Steve Klein 等人那里筹集了 550 万美元的种子资金。

Metrist 背后的总体思想非常简单，但是很少有公司这样做。虽然像 Twitter 或 StatusPage (现在由 Atlassian 拥有)这样的产品允许公司轻松地向用户传达其服务的问题，但它们并不总是反映每个问题和服务退化——当需要审查 SLA 协议或合同续签时，这一点就会发挥作用，双方对产品的可靠性有着非常不同的看法。虽然像 New Relic 或 Honeycomb 这样的应用性能监控和可观察性工具可以提供一些数据，但这不是它们的核心用例，因为这些服务往往面向内部。

“今天，应用程序是建立在其他应用程序的基础上的，”Metrist 联合创始人兼首席执行官杰夫·马腾斯告诉我。“这意味着，如果其中一个出现故障、降级或出现某种问题，您的应用和业务可能会面临同样的命运。但是当前的可观察性工具并没有为那些外部依赖做任何特别的事情——它们仍然继续关注内部。您可以找到关于外部依赖关系的信息，这并不是说您不能知道，但真正的挑战是验证，以便您可以采取行动，同时也让您的供应商承担责任。”

最重要的是，Metrist 希望成为买家和供应商在讨论停机时可以参考的值得信赖的中立参与者。Martens 说，当 Metrist 作为 SLA 的第三方独立验证者被写入合同时，这项服务将是成功的。

“发布到 StatusPage 的太多事件只是简单地提到上游或第三方提供商，”StatusPage 的联合创始人史蒂夫·克莱恩(Steve Klein)说。“令人兴奋的是，Metrist 正在寻找问题的根源，在以前没有的地方创造可见性。”

Metrist 团队。图片来源: Metrist

在技术方面，Metrist 使用代理或 eBPF 来收集关于公司运行的服务的数据，但它也不断检查 AWS、谷歌云和微软 Azure 的 21 个不同云区域的停机时间和服务降级。开箱即用，Metrist 涵盖了 100 多种服务，但客户也可以托管自己的测试或使用应用内测试。该小组指出，这些测试也远远超出了简单地检查正确的 HTTP 响应代码。

“这不只是像 ping 一个 API，然后说，‘这个 URL 返回 200 还是 202？’假设您正在访问一个端点，它应该会在该平台上创建一个东西，我们实际上会在稍后调用检索 API 来查看创建该东西需要多长时间，”Metrist 联合创始人兼首席技术官 Ryan Duffield 解释道。

客户在何时以及如何收到问题警报方面也有很大的灵活性。例如，对于一些人来说，延迟增加 2%可能是不可接受的，而对于其他人来说，这不是问题。警报可以发送到 Slack、email Datadog 或 PagerDuty(用户也可以使用 webhooks 创建自己的警报系统)。

虽然 Metrist 今天才宣布其融资，但值得注意的是，该团队实际上通过两次不同的融资筹集了这笔资金，包括在产品存在之前的预种子期。马腾斯解释说，这两种情况都是主动发生的，团队实际上并没有出去筹集资金。这发生在经济和融资环境发生变化之前。

“现代应用依赖于由外部供应商管理的越来越多的云产品，但可观察性的整体方法没有改变。Heavybit 的普通合伙人 Joseph Ruscio 解释说:“你不会梦想盲目地运营内部服务，你需要以同样的谨慎管理你的云依赖关系。“Metrist 使团队能够主动了解外部服务何时停止运行，目的是避免或减轻由依赖性引起的事故。Metrist 的第三方可观察性方法可确保团队权威地了解 SLA 何时未得到满足。”

Metrist 提供了一个免费计划，允许您在一天的数据保留期内监控多达三项服务。付费计划的起价为每月 99 美元，包含七项服务和七天数据保留期。
