今天在 AWS re:Invent 上，该公司的客户会议本周在拉斯维加斯举行，AWS 宣布了一个新的开源自动缩放工具，用于 Kubernetes 集群，名为 Karpenter 。

云计算的优势之一是能够自动扩展以满足您所需的资源需求——或者至少理论上是这样。实际上，管理 Kubernetes 集群的管理员不得不仔细监控它们，以确保它们拥有适量的资源或避免服务中断。

Karpenter 旨在将云计算理想变为现实。AWS 的 Channy Yun 在发布新工具的博客文章中描述了它的优势。

“[kar center]通过快速启动适当规模的计算资源来响应不断变化的应用程序负载，帮助提高您的应用程序可用性和群集效率。Karpenter 还提供即时计算资源来满足您的应用需求，并将很快自动优化集群的计算资源占用，以降低成本并提高性能，”Yun 写道。

它通过分析您的 Kubernetes 工作负载来确定您将需要哪些资源，方法是查看由于资源限制而无法启动的 pod。然后，它会向您的云提供商发送信息，根据这些信息添加或删除计算。

这里需要注意的是，作为一个开源工具，这不是专门为 AWS 云资源设计的，可以用来向任何云提供商发送关于底层 Kubernetes 集群的信息。Karpenter 利用 Kubernetes 包管理器 Helm 来确定您的 Kubernetes 工作负载。它还需要以自动化方式调配计算资源的权限。

Karpenter 是在 Apache 2.0 许可下提供的开源工具，现在已经可以使用。
