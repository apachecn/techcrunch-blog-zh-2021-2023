凭借 860 万美元的种子资金，Nx 希望将 monorepos 纳入主流 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/17/with-8-6m-in-seed-funding-nx-wants-to-take-monorepos-mainstream/

Narwhal 是广受欢迎的 monorepo 专注于 JavaScript 代码的开源 Nx 构建系统背后的公司，今天宣布它已经筹集了 860 万美元的种子资金，由 Nexus Venture Partners 和 Andreesen Horowitz 共同牵头。包括 GitHub 联合创始人汤姆·普雷斯顿-沃纳(Tom Preston-Werner)在内的多位天使投资人也参与了此轮投资。

Narwahl 由 Angular 团队的两名前谷歌员工 Jeff Cross(首席执行官)和 Victor Savkin(首席技术官)创建，实际上，Narwahl 最初是一家 Angular 咨询公司，帮助大型银行、航空公司和其他企业——通常使用 Angular 的公司。正如克罗斯告诉我的那样，正是与 Capital One 的合作实际上推动了团队追求 Nx，并将其转化为公司的主要产品。那时，由于他们在谷歌的工作，monorepos 的概念对他们来说已经非常熟悉，谷歌使用世界上最大的 monorepos 之一来管理其代码库。

图片来源: Nx

“他们有自己的登录团队，”克罗斯解释说。“如果你登录 CapitalOne.com，就会发现七个业务部门在构建一个统一的应用程序，而且它分散在如此多的存储库中，他们无法协调部署；他们无法在任何事情上真正协调一致。他们真的需要一个单向回购。因此，我们为他们的用例构建了 Nx，然后让它与我们合作的所有其他客户(大多数是这些大公司)一起工作。”

克罗斯认为，对于大型团队来说，monorepos 天生更容易管理。他说，谷歌的创始人被宠坏了，因为有了 monorepo，任何开发者都可以不费吹灰之力构建谷歌代码库的任何部分。毕竟，一切都使用了相同的工具链和测试基础设施。与此同时，让许多团队在不同的存储库上工作会产生很多摩擦，因为团队必须构建一个通用的 API——并为其创建一个新的存储库，创建集成过程并找出如何发布它。“随着出版业的发展，不可避免的是，每家公司都会在出版业中加入版本控制。因此，我们永远不会“发布每一个提交，并在存储库中立即更新。”这更像是:‘我们发布它，我们用某个地方来说明这是一个突破性的改变，一个小的改变，还是一个补丁？克罗斯说:“大多数公司最终都没有时间真正更新它。

所以 Nx 背后的想法是给每个公司提供工具来管理他们的 JavaScript mono repos——并在必要时将它们迁移到一个。

正如 Cross 解释的那样，开源 Nx 项目和 Nx 云帮助公司在这些庞大的存储库中组织他们的代码，使用 Nx 的项目图的概念。值得注意的是，Nx 受到了谷歌 Bazel 构建和测试系统的极大启发，因此它包括一些熟悉的功能，如跨多台机器分配计算和任务执行的能力。Cross 引用了该公司目前正在合作的一家主要零售巨头，该公司迁移到 Nx 的企业产品，现在由于其分布式缓存系统，每月节省超过 40，000 小时的计算时间。Nx(公平地说，还有 Bazel)的一个很好的特性是，它知道两个开发人员何时试图运行相同的任务，并检查是否已经有一个缓存版本。

Narwhal/Nx 已经领先于大多数处于种子阶段的开源公司，因为它已经有了一个托管服务( Nx Cloud )和一个企业版作为其主要产品。鉴于 Nx 与大型企业客户合作的类型，Nx 为他们提供在私有实例中运行服务并与外部 API 隔离的能力就不足为奇了。

该公司目前只有 30 多名员工，大部分都在偏远地区。其中，25 人是工程师。

最近，Narwhal 还接管了 Lerna.js 的管理权，这是一个流行的开源 JavaScript monorepo 工具，以前一直没有得到维护。Narwhal 现在将为它提供关键的错误修复和安全更新。

“mono repo 的采用在全球范围内呈爆炸式增长，这是由易于协作、共享代码库可见性、依赖性管理和重构等优势推动的，”Nexus Venture Partners 董事总经理 Abhishek Sharma 说。“然而，随着 monorepos 的扩大，强大的工具对于管理它们变得至关重要，构建时间成为一个关键因素。这就是 Nx 的闪光点。我们被 Nx 吸引是因为它的世界级团队、品类领导地位、强大的开发者社区和大规模的全球采用——从初创公司到财富 500 强公司。我们非常感谢杰夫和维克多选择我们作为他们此次旅程的合作伙伴。”
