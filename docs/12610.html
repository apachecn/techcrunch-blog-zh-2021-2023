法律专家警告说，英国在线安全法案中的监控权力对 E2EE 是一种威胁

对一项有争议的英国政府提案的独立法律分析表明，该提案在一个以安全为重点的框架下监管网络言论，也被称为网络安全法案，该法案草案包含了西方民主国家有史以来提出的对公民进行最广泛的大规模监控的权力，它还警告称，这对端到端加密(E2EE)的完整性构成了风险。

这份意见书是由 Matrix Chambers 的律师 Matthew Ryder KC 撰写的，是受倡导言论自由的组织“审查指数”委托撰写的。

赖德被要求考虑该法案的条款是否符合人权法。

他的结论是— 照现在的样子——该法案缺乏对监控权力的必要保障，这意味着，如果不进一步修订，它很可能违反欧洲人权公约( ECHR )。

在线安全法案在议会的进展在夏季被暂停，在执政的保守党的政治动荡之后，10 月在 T21 再次被暂停。在新的数字部长上任和两次更换总理后，政府表示打算对草案进行修订——然而，这些修订主要集中在与所谓的“合法但有害”言论有关的条款上，而不是莱德指出的人权漏洞。

我们向内政部寻求对他的法律意见书提出的问题的回应。

一名政府发言人通过电子邮件回复了一份声明，据称是安全部长汤姆·图根哈特(Tom Tugendhat)的声明，否认了任何担忧:

在线安全法案将隐私作为其提案的核心，确保我们能够保护自己免受包括儿童性剥削在内的在线犯罪的侵害。这并不是禁止任何类型的技术或服务设计。

如果一家公司未能解决其平台上的儿童性虐待问题，Ofcom 作为独立监管机构有权作为最后手段要求这些公司采取行动，这是正确的。

强加密保护我们的隐私和我们的在线经济，但端到端加密可以以符合公共安全的方式实施。该法案确保科技公司不会为最危险的网络掠夺者提供安全的空间。

赖德的分析发现，该法案缺乏关键的法律检查，该法案授予国家广泛的权力，迫使数字提供商“在普遍和广泛的基础上”监控用户的在线通信，但未能包括任何形式的独立事先授权(或独立事后监督)来发布内容扫描通知。

根据莱德的评估，这种缺乏严格监督的行为可能会违反《ECHR》第八条(隐私权)和第十条(言论自由权)。

根据(也极具争议的)【2016 年 (IPA)调查权力法案，现有的授予英国安全部门的非常广泛的监控权力确实包含授权最具侵入性权力的法律制衡——涉及司法部门签署拦截令。

但在线安全法案将发布最具侵入性的内容扫描命令的决定权留给了指定的互联网监管机构——莱德认为，这个公共机构在这一职能方面没有足够的独立性。

“法定计划没有规定 104 通知的独立授权，尽管它可能要求私人机构(在公共当局的要求下)对数百万用户的通信进行大规模国家监控。也没有任何事后独立监督的规定，”他写道。我们认为，在这种情况下，国家监管机构英国通信管理局不能被视为一个独立机构。

他还指出，鉴于 IPA 现有的广泛监控权力，在线安全法案中提议的在线通信“大规模监控”可能无法满足另一项关键的人权测试——即“民主社会的必要条件”

虽然 IPA 下的大规模监控权力必须与国家安全问题相关联，并且不能仅用于预防和侦查英国用户之间的严重犯罪，但他的法律分析认为，在线安全法案授予 Ofcom 类似的“大规模监控”权力， 涵盖的内容比纯粹的国家安全问题更广泛。所以它看起来没有那么局限。

索引审查机构的首席执行官露丝·斯梅思在一份声明中对赖德的法律意见进行了评论，她谴责该法案不自量力，写道:

这份法律意见书明确了围绕在线安全法案的无数问题。这项立法的模糊起草将迫使媒体监管机构 Ofcom 单方面决定如何在英国日常数字生活的几乎每个方面部署大规模监控权力。监管机构的监督可能是一项完全不符合目的的法案中最过分的例子。

对 E2EE 的影响

虽然在线安全法案的大部分争议集中在言论自由的风险上，但还有一系列其他值得关注的问题。该法案去年以草案形式发布，但政府一直在对其进行修改并扩大其范围。包括立法中的内容扫描条款可能如何影响 E2EE，像开放权利组织这样的批评者警告说，该法律将从本质上强制服务提供商打破强加密。

自从今年 7 月 11 日政府修正案提出该法案以来，人们的担忧加剧了——该修正案提出了 Ofcom 的新权力，以迫使信息平台实施内容扫描技术，即使通信在其服务上是高度加密的。该修正案规定，受监管的服务可能需要“尽最大努力”开发或获取技术，以检测和消除私人通信中的 CSEA——私人通信使其与 E2EE 发生冲突。

E2EE 仍然是加密和在线安全的“黄金标准”,可以在 WhatsApp、iMessage 和 Signal 等主流消息平台上找到，为用户的在线通信提供必要的安全和隐私。

因此，任何威胁使用这一标准的法律——或为 E2EE 打开新的漏洞——都可能对全球网络用户的安全产生巨大影响。

在法律意见中，Ryder 将他的大部分注意力集中在在线安全法案的内容扫描条款上，这些条款给 E2EE 带来了生存风险。

他的大部分法律分析都集中在该法案的第 104 条上，该条款授予指定的互联网监管机构(现有的媒体和通信监管机构，Ofcom)一项新的权力，向范围内的服务提供商发出通知，要求他们识别并删除通过其服务“公开”传播的恐怖主义内容或“公开或私下”传播的儿童性剥削和虐待(CSEA)内容而且，再一次，包含“私人”通信是 E2EE 真正棘手的地方。

赖德认为，该法案不会迫使消息平台完全放弃 E2EE，而是会推动它们部署一种有争议的技术，称为客户端扫描(CSS)——作为遵守 Ofcom 发布的 104 通知的一种方式——他预测，“ 可能是授权使用的主要技术。”

"第 104 条款并未提及 CSS(或任何技术)的名称。它只提到了“认可的技术”然而，要求识别、删除和/或屏蔽内容的 104 通知的实际实施几乎不可避免地导致这样的担忧，即 Ofcom 将使用这一权力来授权 CSP(通信服务提供商)使用某种形式的 CSS，”他写道，并补充道: “该法案指出，c.104 中提到的认证技术是一种‘内容审核技术’，意思是‘算法、关键字匹配、图像匹配或图像分类等技术，这些技术[……]分析相关内容’(c . 187(2)(11)。这种描述与 CSS 相对应。”

他还提到了两名 GCHQ 高级官员今年夏天发表的一篇文章——他说这篇文章“认可 CSS 作为 CSEA 内容在加密平台上传输问题的潜在解决方案”——进一步指出他们的评论是“在关于 OLSB(网络安全法案)的持续辩论的背景下”做出的

"任何要求通信服务提供商破坏其端对端加密实施的企图，都会对所有全球在线通信的安全和保障产生深远影响。我们无法想象在什么情况下，对数十亿用户的全球在线通信安全采取这种破坏性措施是合理的，”他接着警告说。

客户端扫描风险

CSS 是指有争议的扫描技术，在该技术中，扫描加密通信的内容，目的是识别令人反感的内容。该过程需要在加密和发送之前将消息转换为加密数字指纹，然后将该指纹与指纹数据库进行比较，以检查是否与已知的不良内容(如 CSEA)相匹配。这些加密指纹的比较可以在用户自己的设备上进行，也可以在远程服务上进行。

无论在哪里进行比较，隐私和安全专家都认为 CSS 破坏了 E2E 信任模型，因为它从根本上挫败了端到端加密的“零知识”目的，并通过打开新的攻击和/或审查媒介产生了新的风险。

例如，他们指出，嵌入式内容扫描基础设施的前景使“审查蠕变”成为可能，因为国家可以要求通信提供商扫描范围越来越广的“令人反感”的内容(从有版权的材料一直到令专制政权不快的政治异议的表达，因为在民主制度下开发的工具不太可能只在世界上一个地方适用)。

苹果公司去年试图在 iOS 用户的设备上部署 CSS,当时它宣布将开始扫描 iCloud 上传的照片，寻找已知的虐待儿童图像，这引起了隐私和安全专家的强烈反对。苹果先是停顿了一下——然后在 12 月悄悄放弃了提及该计划,所以它似乎已经放弃了这个想法。然而，政府可以通过像英国的在线安全法案这样的法律强制部署 CSS 来恢复这些举措，该法案依赖于同样声称的儿童安全理由来在平台上嵌入和执行内容扫描。

值得注意的是，英国内政部一直在积极支持内容扫描技术的开发，这些技术可以应用于 E2EE 服务——去年宣布成立“技术安全挑战基金”，将纳税人的钱花在开发它当时标榜为“在端到端加密的在线消息平台等环境中保护儿童安全的创新技术”上

去年 11 月，作为挑战的一部分，宣布了五个获奖项目。尚不清楚这些原型有多“成熟”和/或准确。但政府正在推进在线安全立法，这位法律专家表示，事实上，这将要求 E2EE 平台进行内容扫描，并推动 CSS 的采用——无论这种技术的发展状况如何。

在讨论政府对第 104 条的拟议修正案时——该修正案设想 Ofcom 能够要求通信服务提供商“尽最大努力”开发或采购自己的内容扫描技术，以实现与该法案还设想监管机构签署的认可技术相同的目的——Ryder 预测:任何此类解决方案似乎都可能是 CSS 或类似的东西。例如，我们认为电信运营商不太可能尝试移除其服务上的所有端到端 加密。这样做不会消除他们分析通信内容以确定相关内容的需要。然而，更重要的是，这将致命地危及用户及其平台的安全，几乎肯定会导致许多用户转向其他服务。”

“如果在所有符合条件的平台上发布 104 通知，这将意味着数百万人几乎所有基于互联网的通信内容——包括他们个人对话的细节——将不断受到服务提供商的监控。当然，这是否会发生，将取决于 Ofcom 如何行使其发布 104 通知的权力，但表面目的与适度使用的需要之间的内在矛盾是不言而喻的，”他补充道。

未能遵守在线安全法案将使服务提供商面临一系列严厉处罚的风险——因此，人们正在组装非常大的棍子，并将其与全面的监控权力放在一起，以强制遵守。

该立法草案允许高达全球年营业额 10%的罚款(或 1800 万英镑，以较高者为准)。该法案还将使 Ofcom 能够向法院申请“业务中断措施”，包括在英国市场上阻止不合规的服务。而不与监管机构合作的供应商高管可能面临刑事起诉。

就英国政府而言，到目前为止，它一直对这项立法对 E2EE 的影响不屑一顾。

在“私人信息平台”一节中，一份政府简报声称内容扫描技术只有在“万不得已的情况下”才会被 Ofcom 授权同一篇文章还暗示这些扫描技术将“高度准确”——但没有提供任何证据来支持这一论断。它写道，“这种权力的使用将受到严格的保护，以保护用户的隐私，”并补充说:“高度精确的自动化工具将确保法律内容不受影响。要使用这种权力，Ofcom 必须确定没有其他措施会同样有效，并且有证据表明某项服务存在普遍问题。”

认为新型人工智能在大规模广泛的内容扫描中“高度准确”的想法显然是有问题的，需要强有力的证据来支持它。

你只需要考虑人工智能在主流平台上被证明是一个多么迟钝的工具，因此成千上万的人类承包商仍然在审查自动报告。因此，内政部已经或将能够促进开发一种比谷歌和脸书等科技巨头在过去几十年里成功设计的更有效的人工智能过滤器，这似乎是非常异想天开的。

至于对使用内容扫描通知的限制，赖德的意见触及了法案第 105 条中包含的保障措施——但他质疑这些措施是否足以解决与这种强大权力相关的所有人权问题。

“OLSB 第 105 条中还有其他保障措施，但这些额外的保障措施是否足够，将取决于它们在实践中如何应用，”他建议道。“目前没有迹象表明 Ofcom 将如何应用这些保障措施并限制 104 通知的范围。

"例如，第 105(h)条提到《ECHR》第十条，要求对干涉言论自由权给予适当考虑。但没有具体规定确保充分保护新闻消息来源，为了防止违反第十条，需要提供这种保护。”

在回应 Ryder 意见的进一步评论中，内政部强调，只有在没有替代性的、侵入性较小的措施能够实现对服务中出现的非法 CSEA(和/或恐怖主义内容)的必要减少时，才会使用第 104 条通知权力，并补充说，将由监管机构评估发布通知是否有必要和相称，同时考虑立法中规定的事项，包括服务中发生的伤害风险以及伤害的普遍性。
