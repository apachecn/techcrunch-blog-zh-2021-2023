New Relic 收购 CodeStream 以在开发人员环境中提供聊天服务，与微软 ide 合作 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/21/newrelic-acquires-codestream-to-provide-chat-in-developer-environments-inks-microsoft-ide-partnership/amp/

图片来源: Getty Images

观察平台 New Relic 已经完成了一项收购，并在其基础上推出了一项新服务，为更大的战略带来了新的维度，使开发者和运营更加紧密。它已经收购了 CodeStream ，这是一个在集成开发环境中工作的应用程序，可以让开发人员在代码本身旁边写相关的注释。

New Relic 已经将其与 New Relic One 集成，New Relic One 是其涵盖指标、事件和日志的全栈数据分析平台，并将在新 Relic CodeStream 中推出一项新服务。它还宣布与微软合作，将新产品集成到自己的 ide 和其他聊天和协作产品中，如 VS Code、Visual Studio、Teams 和 GitHub。

CodeStream 的联合创始人兼首席执行官彼得·佩扎里斯(Peter Pezaris)表示，这笔交易的条款尚未披露，只是说它是“有竞争力的”。CodeStream 是 Y Combinator 年冬季团队的一部分，它已经筹集了大约 300 万美元，并且已经/正在被数以万计的开发者使用，包括微软、赫斯特、Atlassian、Zenefits 和其他一些公司的人员。

(作为一家独立的初创公司，它已经使用 Visual Studio 和 VS Code 以及 JetBrains。它还支持来自 GitHub、BitBucket 和 GitLab 的 pull 请求 ，来自吉拉、Trello、Asana 和其他九家公司的问题管理，来自 New Relic One 和 Pixie 的可观察性，并提供代码讨论将所有这些联系在一起，与 Slack、MS 团队、电子邮件和编辑器内通知集成。随着微软集成的深入，所有这些似乎都将继续得到支持。)

New Relic 拥有 14，100 个客户，而 New Relic 和 CodeStream 在被收购前共有 750 个共同客户。

这笔交易凸显了 DevOps 领域的一些趋势。首先，有一些整合正在进行，New Relic 将自己定位为一个平台，正在寻求将更多的功能引入其工具集。

这也意味着随着时间的推移，CodeStream 的销售可能会更具挑战性，因为客户自然会希望一起购买可观察性和通信工具，并希望看到它们的功能随着时间的推移变得更加接近。

“与新 遗迹合作是显而易见的，因为我们看到了我们两家公司之间的战略联盟，”佩扎里斯告诉 TechCrunch。“在过去的 26 年里，我很幸运地与同一个创始人团队一起工作，在此期间，我创立、建立并出售了四家公司。在任何情况下，我们都能够在更大的公司内更快地扩展并释放更多的价值，因此这是一个特别令人兴奋的时刻。

“我们的团队也感受到了与新 遗迹的价值观的契合，并确定我们可以通过与新 遗迹的联手来加速我们的使命——与新 遗迹内置的资源、团队和客户群相契合。我们在 CodeStream 的团队也对 New Relic One 平台印象深刻，并认为我们的解决方案不仅可以让开发人员更好地扩展代码协作，还可以释放 IDE 中遥测数据和可观察性的力量。"

CodeStream 也适合这种整合，它为 New Relic 提供了一个重要的切入点，可以进入这笔交易触及的第二种趋势。

第二个趋势是，协作是目前企业软件的游戏名称。对于开发人员来说也是如此，他们在不同的地理位置和不同的时间空间进行项目，现在和将来都需要更好的方式来相互交流他们的想法和附加说明。

正如 Romain 在介绍 CodeStream 与 VS Code 的集成时所指出的，许多应用程序现在都有“对话”通道，让那些创建内容的人在空白处更详细地谈论正在发生的事情，Slack(或另一个等效物)不应该成为默认组件，如果你可以更具体地构建一些特别适合用户需求的东西，这一点尤其重要。

在今天新推出的情况下，New Relic 表示，那些有兴趣尝试这项服务的人可以注册 New Relic One 进行免费试用。CodeStream 产品不仅允许用户在代码的空白处互相写下注释，还可以创建一个流，让读者直接从聊天注释跳到有问题的代码行。

第三个趋势是 DevOps 作为企业软件中的一个突出类别的持续扩展。更大的项目、更多的开发人员、新的 IT 架构和技术以及新的安全性和其他挑战都导致开发人员逐渐成为组织如何运行和发展的一个至关重要的组成部分。

这反过来又导致了为他们服务的业务不断增长。New Relic 引用 Redpoint 的估计称，仅可观测性市场就有 350 亿美元的潜在市场，并且每年以两位数的速度增长。

“开发人员工作流是所有现代公司的支柱，作为一种工程实践，可观察性代表了一种未来，在这种未来，这些重要的开发人员工作流由数据驱动——而不仅仅是意见，”新 T2 遗迹公司首席执行官比尔·斯台普斯在一份声明中说。“为了加速这一共同任务，使可观察性成为每个工程师的数据驱动的日常实践，我们将生产遥测和协作工具引入开发人员创建和流动的地方 IDE。通过发布NewRelicCodeStream 并与微软联手，我们很高兴能够为我们共享社区的数百万开发者提供真正以开发者为中心的体验。”

New Relic 的总经理 Buddy Brewer 在给 TechCrunch 的一份报告中补充说:“DevOps 从根本上讲是关于加强开发人员和运营人员之间的协调。”“这就是 CodeStream 让我们脱颖而出的原因。作为致力于解决开发人员代码协作的领先公司，他们已经将让技术团队更有效地理解和交流他们的代码作为自己的使命。使用 CodeStream，开发人员可以在他们的开发环境中讨论他们的代码，并保存这些讨论以构建一个可参考的、不断发展的知识库。通常，开发人员会参与一个沉闷的过程，将特定的代码行复制/粘贴到 Slack、团队或他们正在使用的任何聊天功能中，并解释上下文和他们的问题，这在团队之间造成了大量的摩擦和时间损失。”

这也是微软与该公司合作的原因，尽管对我来说，这确实留下了一个问题，即随着时间的推移，微软是否会试图更直接地参与开发者服务的这一方面。考虑到有多少环境将集成这种特定的解决方案，这是有意义的。

“开发人员对于帮助各行各业的组织加速使用新功能至关重要。微软云+人工智能执行副总裁斯科特·格思里(Scott Guthrie)在一份声明中表示:“我们在微软的目标是提供广泛的服务，以满足客户的现实需求。“有了像 New Relic 这样的合作伙伴，很高兴看到与NewRelicCodeStream 的全面集成支持，跨越多个微软平台和产品:VS Code、Visual Studio、。NET、GitHub、微软团队和 Azure DevOps 等等。开发项目之间更紧密的协作和现有应用程序之间更好的连接只是NewRelicCodeStream 将为开发人员社区提供的一些好处。”
