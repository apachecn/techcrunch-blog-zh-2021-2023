AWS 就是不能休息一下 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/12/22/aws-just-cant-catch-a-break/amp/

图片来源:诺亚·伯杰/Getty Images for Amazon Web Services

今天，AWS 的一个数据中心遭遇了本月第三次停机。今天早上，其 US-EAST-1 地区的停电影响了 Slack、Asana、Epic Games 等服务。

这些问题始于美国东部时间上午 7:30 左右，截至美国东部时间下午 1 点，这些问题的连锁效应继续困扰着该服务，因为 AWS 继续报告该地区许多服务的问题，特别是其 EC2 计算服务和相关的网络功能。最近，该地区的单点登录服务也开始出现错误率增加的情况。

该公司在美国东部时间上午 8 点的更新中解释道:“我们可以确认美国东部 1 区的单个可用区(USE1-AZ4)内的单个数据中心断电。”“这将影响 EC2 实例的可用性和连接性，这些实例是受影响可用性区域内受影响数据中心的一部分。在受影响的可用性区域内启动时，我们还遇到了 RunInstance API 错误率升高的问题。受影响的可用性区域内的其他数据中心或 US-EAST-1 区域内的其他可用性区域的连接和电源不受此问题影响，但我们建议您尽可能远离受影响的可用性区域(USE1-AZ4)。”

如果这是最近几周唯一一次 AWS 停机，那也没什么值得注意的。鉴于现代超级云的复杂性，中断肯定会不时发生。但目前 AWS 每周都会出现停机。12 月 7 日，由于网络问题，同一个 US-EAST-1 区域中断了几个小时。然后，在 12 月 17 日，一次影响其西海岸两个地区之间连接的停电导致网飞、Slack 和亚马逊自己的 Ring 等公司的服务中断。雪上加霜的是，所有这些中断都发生在 AWS 在本月早些时候的 re:Invent 会议上吹捧其云的弹性后不久。

当然，在理想情况下，这些中断永远不会发生，AWS 用户可以通过将系统设计为故障转移到地理上独立的区域来保护自己免受这些中断的影响，但这可能会增加大量成本，因此一些人认为不值得在停机时间和成本之间进行权衡。最终，AWS 将提供一个稳定的平台。虽然很难说该公司是否只是运气不好，或者是否有任何系统性问题导致了这些问题，但如果我现在在美国东部 1 区托管服务，我可能至少会考虑将其转移到其他地方。
