Zed 筹集了 1000 万美元来开发一个代码编辑器

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/15/zed-code-editor-raises-10m/

代码编辑器是复杂且要求高的软件，但从商业角度来看，它们很难赚钱。与此同时，他们没有跟上在家工作等最新趋势。

至少，内森·索博是这么看的。他在接受电子邮件采访时告诉 TechCrunch，“编码本质上是社会性的，然而谈论代码的工具限制了对话的速度和范围，这种摩擦正在阻碍我们行业的生产力。”“我认为唯一的前进道路是将协作作为头等大事集成到创作环境中，就像 Figma 的设计或 Google Docs 的散文已经发生的转变一样。”

Sobo 可能不是一个家喻户晓的名字。但他是 GitHub 的 Atom 编辑器团队的成员，该团队致力于开发(现已废弃的)Atom 代码编辑器。现在，他推出了 Zed ，这是一款专注于“多人”体验、性能和精简极简设计的代码编辑器。

Sobo 与 Atom 的贡献者安东尼奥·斯坎杜拉和马克斯·布伦斯菲尔德合作，共同推出了 Zed。Sobo 将 Zed 描述为改进软件开发方式的“十年之久”追求的顶点。

“2019 年，安东尼奥和我决定，为了实现我们建立终极代码编辑器和协作平台的愿景，我们需要从一个新的技术基础开始，”Sobo 说。“我们在晚上和周末一直致力于 Zed，直到 2021 年春天，当时我们筹集了一轮种子资金，创办了 Zed Industries，从那以后我们一直专注于建设 Zed。”

Sobo 声称 Zed 的关键区别在于性能、设计和多人编辑。Sobo 说，它旨在最大限度地减少分心并“淡出背景”，以便更容易专注于代码。此外，它还提供了一些工具，旨在简化邀请队友进入工作空间的过程，以便他们可以一起浏览。

有了 Zed，当用户加入队友的项目时，他们可以编辑代码，就像在他们的本地机器上一样。用户可以从内置的联系人面板中呼叫另一个 Zed 用户，或者跳到队友的位置，围绕代码跟随他们。他们还可以使用 Zed 内置的屏幕共享工具来跟踪平台外的人，以查看文档或试验正在开发的应用程序。

Zed 的代码编辑界面。形象代言人: Zed

“最终，Zed 是一个伪装成世界级代码编辑器的软件协作平台。我们的目标是让 Zed 成为开源软件开发的标准平台和软件团队的首选工具，”Sobo 补充道。

即使 Zed 像 Sobo 声称的那样提供了很好的体验，我想知道它是否能够打入拥挤的市场。根据 Stack Overflow 的 2022 年开发者调查，Visual Studio 代码仍然是最受欢迎的 IDE，74.48%的受访者表示这是他们的首选平台。第二受欢迎的选择——IntelliJ——远远落后于第二名，获得了 27.97%的选票。

Zed 也不是唯一的 IDE 新贵。例如，的 Stackblitz 最近筹集了 790 万美元来进一步开发其基于浏览器的 IDE。

但投资者似乎正在接受这一愿景。Zed 今天完成了由 Redpoint Ventures 牵头的 1000 万美元的首轮融资，参与方包括 Root Ventures、Matchstick Ventures 和 V1。VC，以及包括 Figma 的 Dylan Field 和 GitHub 的 Tom Preston Werner 在内的天使。Zed 的估值为 4000 万美元，这笔新资金使这家初创公司的总投资达到约 1250 万美元。

红点公司的帕特里克·蔡斯尤其称赞。在一封电子邮件中，他说:“Zed 的使命是让工程师尽可能快地将他们的想法转化为代码，或者他们会说‘以思维的速度编码’。”这意味着一个闪电般快速的编辑器，无缝的团队协作，以及未来更多的功能。没有一个团队比 Zed 背后的团队更有能力解决这些复杂的问题——Zed 的当前版本是近两年专注开发的高潮，但近 16 年来一直痴迷于如何构建一个更好的编辑器。"

拥有七名员工的 Zed 计划主要通过基于服务的模式来赚钱。它目前是预收入和推出测试版开始。但是 Sobo 说，在私人 alpha 测试期间，等候名单增长到了 12，000 人，每周大约有 800 名活跃用户使用 Zed 编码(通过 alpha 测试)。

“疫情帮助我们加速了行业向远程开发的转变。Sobo 说:“由于面对面交流的机会越来越少，软件开发人员越来越强烈地感受到当前协作技术的局限性。“Zed 是为一种新的工作方式而构建的，在这种工作方式中，围绕代码的对话更加丰富，交互更加人性化、更加紧密。这种新的工作方式加速了软件开发本身的进程。”
