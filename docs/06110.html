GitHub 要求双重认证

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/04/github-will-require-all-users-who-contribute-code-to-enable-two-factor-authentication-by-the-end-of-2023/

到 2023 年底，GitHub 将要求所有贡献代码的用户启用双因素认证

下面是一些消息，它们既简单明了，但仍然很重要:到 2023 年底，GitHub 将要求所有在平台上贡献代码的用户启用一种或多种形式的双因素认证 (2FA)。

新闻差不多就这些了。今天，微软旗下的公司说，只有 16.5%的 GitHub 活跃用户和 6.44%的 npm 用户使用 2FA。这不是很多，坦率地说，比我预期的要少。

“受损的帐户可用于窃取私有代码或对该代码进行恶意更改。这不仅会使与受损帐户相关的个人和组织面临风险，还会使受影响代码的任何用户面临风险。GitHub 首席安全官迈克·汉利(Mike Hanley)在今天的公告中写道:“其结果是，对更广泛的软件生态系统和供应链的潜在下游影响是巨大的。

他还指出，该公司正试图确保额外的安全层不会以牺牲用户体验为代价。因此，从今天的声明到何时实施这一政策之间有很长一段时间。“我们的 2023 年底目标让我们有机会为此进行优化，”Hanley 解释道。切换到 2FA 涉及到命令行和 GitHub web 界面上的用户体验的一些变化。

值得注意的是，今年早些时候，GitHub 还在强制 2FA 中招收了前 100 名 npm 包的维护者，以防止软件供应链攻击。它计划本月扩展到前 500 个包的维护者，然后扩展到所有依赖者超过 500 或每周下载超过 100 万的包。
