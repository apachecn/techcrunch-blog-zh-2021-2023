过去的两年对个人和企业来说都是充满挑战的。作为一个全球社区，由于各种限制，我们无法进行长途旅行，但这并不意味着我们不再伸手摘星。

在同一时期，太空探索取得了巨大的进步和成就——从将四名平民宇航员送入太空的 Inspiration 4 SpaceX 的成功发射，到美国国家航空航天局的Lucy mission的发射，该任务旨在研究木星的特洛伊小行星，以寻求关于行星起源和太阳系形成的更深入的知识。

随着火箭实验室和美国维珍轨道公司的成功发射，我们也看到了太空探索被许多私营公司霸占，而太空透视已经开始为一个基于气球的企业筹集资金，该企业将把付费客户带到平流层。

这些进步让年轻人和老年人都感到惊讶，并赋予太空竞赛新的意义。因在《星际迷航》中扮演柯克船长而闻名的演员威廉·夏特纳，最近创造了历史，成为飞向太空的最年长的人。这位 90 岁的老人是蓝色起源第二次载人航天飞行的四名乘客之一。

随着“最后的边疆”现在向私营公司开放，我们如何改变我们不断更新运行在太空中的软件的方式？“数字转型”一词在地球上几乎无处不在，但越来越明显的是，它的概念也将在下一波太空探索中发挥关键的支持作用。

为什么要在太空开发？

新太空产业的一个重大发现是，它正在被软件生吞活剥。软件管理通信卫星和做 Starlink 正在做的事情——为消费者开发低延迟互联网系统——的能力至关重要。

当你观察“新空间”中发生的一切时——观察地球、遨游深空、月球、火星等等。—没有软件，所有这些成就都是不可能的。软件越来越智能，越来越好，越来越容易更新；然而，在太空中执行软件命令所需的计算能力正呈指数级增长。

与此同时，向太空发射有效载荷的成本正在迅速下降，尤其是与五年前相比。目前，轨道上有 2000 多颗运行中的卫星，但计划中的星座将在未来几年增加 40000 多颗卫星。我们将看到越来越多的公司更快地创建更先进的基础设施，以便用更高效、更强大的软件不断升级他们的卫星和星座。

就像我们在边缘计算至关重要的其他环境中看到的一样，例如汽车、能源/公用事业、仓库和最后一英里零售交付等，可靠、安全且持续更新其卫星软件的公司将在竞争中获得巨大优势。

快速释放，否则会有卫星坠毁的风险

太空旅行最大的技术痛点之一是功耗。在地球上，我们开始看到更高效的 CPU 和内存，但在太空中，扔掉 CPU 的热量是非常困难的，这使得功耗成为关键因素。从硬件到软件，再到处理方式，一切都需要考虑功耗。

另一方面，在太空中，有一样东西你拥有很多(很明显):空间。这意味着物理硬件的大小不再是一个问题。重量和功耗是更大的问题，因为这些因素也会影响微芯片和微处理器的设计方式。

一个很好的例子可以在拉蒙空间的设计中找到。该公司使用人工智能和机器学习处理器来构建具有类似地球计算能力的空间弹性超级计算系统，硬件组件最终由它们上面的软件控制。这个想法是优化软件和硬件的使用方式，这样就可以实时开发和调整应用程序，就像在地球上一样。

在这种背景下，DevOps 的编码、测试、验证、分析和分发实践与他们在地球上大致相同，但硬件、仿真、反馈循环和软件的可靠测试的类型非常不同。

我个人的观点是，我们需要创造一种新的方式，在太空中进行连续交付和连续更新。在地球上，许多组织使用 orchestrator 来处理持续更新流程，即系统、应用和服务的自动配置、管理和协调，以帮助 IT 团队高效管理复杂的任务和工作流。到目前为止，在太空中的卫星上还没有类似的东西，现有的也非常有限。

例如，一个指挥者需要从地面发送和控制卫星更新，当涉及到数据安全等问题时，这就产生了很高的风险。

今天的空间创新者需要考虑一种方法，使卫星能够接收正确运行更新所需的所有信息和数据，并快速正确地从坏的更新中恢复，以确保卫星上有一组有效的二进制文件。也就是说，我乐观地认为，我们正处于一场革命的开端，这场革命将很快使高效地向下一代卫星传送二进制文件的能力成为现实。

大胆地去以前没有人(或计算系统)去过的地方

当反思当前的太空竞赛时，很难不去想“星际迷航”以及我们作为一个行业和全球社区如何大胆地去从未有人去过的地方。

我们正在继续适应和改变我们在地球上面临的环境和挑战，这一点现在已经延伸到外层空间。沙特纳的太空之旅让这位传奇演员激动不已；回来后，他告诉蓝色起源和亚马逊创始人杰夫·贝索斯，“我对刚刚发生的事情充满了激动。我只是，太不寻常了，太不寻常了。我希望我永远不会从这件事中恢复过来。”

这种原始的人类情感是使旅行成为可能的创新技术的产物，也是推动边界的好奇心和顽皮愿望的产物。

在这种新的范式下，空间已经从“最后的边疆”转变为具有几乎无限可能性的下一个机会。从这个意义上来说，这是一个独特的活着的时代，我鼓励我在各行各业的 DevOps 同事们继续努力去实现梦想。
