Codefresh 发布其托管 GitOps 解决方案| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/27/codefresh-launches-its-hosted-gitops-solution/

Codefresh 今天宣布推出基于以 Kubernetes 为中心的 Argo 开源项目的托管 GitOps 服务。新的托管服务将免费提供给社区项目、个人和小型团队，旨在让 DevOps 团队能够访问编码了一些行业最佳实践的 GitOps 平台，同时仍让这些团队能够灵活地根据自己的需求进行调整。

值得注意的是，Argo 提供的不仅仅是 Argo CD，今年早些时候，Codefresh 推出了对 Argo 工作流、展示和事件的支持，作为其整体平台的一部分。但那是针对自托管安装的。现在，随着托管 Argo CD 的推出，Codefresh 还提供了一个端到端的托管 GitOps 平台。

“这基本上是在为 Argo 和 GitOps 做 GitHub 为 git 做的事情，”CodeFresh 联合创始人兼首席开源官 Dan Garfied 告诉我。“我们提供完全托管的交钥匙体验。您只需出现，连接您的集群，部署应用程序，就可以开始比赛了。你不需要管理它，不需要更新它，不需要考虑它——它只是为你而被照顾。”

他指出，当该公司在 2020 年推出其首个 Argo CD GitOps 解决方案时，它很快成为最受欢迎的产品。虽然 Codefresh 也提供自己的持续集成(CI)服务，但 Garfield 强调，用户也可以将该服务连接到 Jenkins 和 GitHub 等网站。“我们看到很多人来到 GitOps，他们不想重新考虑等式中的 CI 部分，”他说。“现在，您只需连接它，它就会自动显示在您的 GitOps 仪表盘中，并为您提供所有额外的信息，这些信息对补救和获取这些见解非常有帮助。”

图片来源: Codefresh

托管 GitOps 平台为希望改变部署方式的用户消除了这一障碍，同时让他们保持集成管道不变。加菲尔德强调，这非常符合该公司对整合其他第三方服务的关注，以帮助其用户向客户交付他们的软件。使用 Codefresh 作为集成点，这些用户也可以获得整合的仪表板，无论他们是使用 GitLab、Bitbucket 还是两者都使用。

“Codefresh 向各种规模的企业展示了 Argo CD 固有的价值，这种价值建立在 Codefresh 和社区为未来培养 Argo 和 GitOps 的持续承诺之上，”Codefresh 联合创始人兼首席执行官拉结尔·塔比说。“让每个人都拥有这种能力是我们致力于开源创新的最充分体现。”

目前 Codefresh 的客户包括 GoodRX、Skillshare 和 Monday.com 等公司。展望未来，这些用户也可能会看到 Codefresh 通过其系统和第三方可观察性工具获得更多关于用户开发和部署数据的整体见解。虽然该公司不想进入可观察空间本身。Codefresh 希望成为一个中心枢纽，让用户深入了解他们的部署情况，特别是随着越来越多的开发团队转向渐进交付金丝雀部署和出现问题时不可避免的回滚。
