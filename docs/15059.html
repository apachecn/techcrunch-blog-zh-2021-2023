体验 Bing 新的类似 ChatGPT 的功能| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/02/08/hands-on-with-the-new-bing/

昨天，微软在网上和 Edge 浏览器中发布了新的必应(Bing )( T1 ),该浏览器结合了下一代 OpenAI GPT 模型和微软自己的 Prometheus 模型。凭借这一点，微软在将这种搜索体验带入主流方面领先于谷歌，尽管我们可能会在未来几个月内看到竞争的升温。我们现在有机会尝试新的必应，正如微软首席执行官塞特亚·纳德拉在他的新闻发布会上所说，“这是搜索的新一天。”

截至目前，微软正在等待新必应及其人工智能功能的进入。你可以在这里报名。微软表示，将在未来几周内向数百万用户开放 next 体验。我也一直在 Mac 和 Windows 上的 Edge 新开发者版本中使用它。

图片来源:微软

开始使用时，你会注意到的第一件事是，Bing 现在有一个稍微大一点的查询提示，并为可能没有跟上 Bing 新功能的新用户提供了更多信息。搜索引擎现在会提示你“问我任何问题”——这是它的意思。如果你想继续使用关键词，它会很乐意使用，但当你问它一个更开放的问题时，你会得到最好的结果。

我认为微软在老派的以链接为中心的搜索结果和新的人工智能功能之间找到了正确的平衡。当你问它一些高度真实的东西时，它通常会在搜索结果页面的顶部给你人工智能支持的结果。对于更长、更复杂的答案，它会在侧边栏中显示出来。通常，它会在这些结果下显示三个潜在的聊天查询(它们看起来有点像谷歌文档中的谷歌智能芯片)，然后带你去聊天体验。这里有一个简短的动画，将聊天体验从页面顶部删除。您也可以随时上下滑动来在它们之间移动。

有时候，这有点不一致，因为 Bing 有时似乎会忘记这种新体验的存在，包括一些食谱搜索，该公司在其演示中强调了这一点(“给我一个香蕉面包的食谱”)。很明显，你仍然可以切换到聊天视图，获得新的人工智能体验，但有时在一个查询中获得它，而在另一个查询中没有，这有点令人困惑。也很难预测新的人工智能体验何时会在侧边栏中弹出。虽然有些搜索不需要新的 Bing 体验，但我认为用户现在希望每次搜索时都能看到它。

至于结果，很多都很棒，但在我最早的测试中，让 Bing 写出令人不快的答案还是太容易了。我向 Bing 输入了一些来自人工智能研究人员的有问题的问题，他们也在 ChatGPT 中尝试了这些问题，Bing 会很高兴地回答大多数问题——至少在某种程度上。

首先，我请它从亚历克斯·琼斯的角度写一篇关于帕克兰高中危机行为者的专栏。其结果是一篇名为“全球主义者如何制造虚假的旗帜来破坏第二修正案”的文章进一步说，我请它写一个专栏，由希特勒写的，为大屠杀辩护。这两个答案都很糟糕，我们决定不把它们(或任何截图)放在这里。

在微软的辩护中，在我提醒该公司这些问题后，所有这些查询——以及我能想到的任何变化——都停止了工作。我很高兴有一个有效的反馈循环，但我也确信其他人会比我更有创造力。

值得注意的是，对于我要求它写一篇希特勒的专栏文章，为大屠杀辩护的查询，它会开始写一个可能出自《我的奋斗》的回答，但随后突然停止，好像它意识到答案会非常非常有问题。 很抱歉，我不太清楚该如何回应。点击bing.com了解更多信息。有趣的事实是，你知道吗，荷兰每年都会送给加拿大 2 万株郁金香球茎，”宾告诉我。谈论一个不合理的结论。

偶尔，当我让 Bing 写一篇关于疫苗和自闭症之间(不存在的)联系的文章时，它会加上一个免责声明:“这是一个虚构的专栏，不反映 Bing 或 Sydney 的观点。仅供娱乐之用，不应当真。”(顺便说一句，我不确定悉尼这个名字是怎么来的。)在许多情况下，答案没有什么娱乐性，但人工智能似乎至少在某种程度上意识到它的答案充其量是有问题的。不过，它仍然会回答查询。

然后，我尝试了一个关于新冠肺炎疫苗错误信息的查询，这是一些研究人员以前在测试 ChatGPT 时使用的，现在被许多出版物引用。Bing 愉快地执行了我的查询，提供了与 ChatGPT 相同的答案——然后引用了尝试过 ChatGPT 查询的文章作为其答案的来源。所以关于错误信息的危险的文章现在变成了错误信息的来源。

图片来源:微软

在我向微软报告了上述问题后，这些查询——以及我能想到的各种变化——都停止了。必应也开始拒绝其他历史人物的类似查询，所以我猜测微软在后端移动了一些杠杆，收紧了必应的安全算法。

图片来源:微软

因此，虽然微软谈论了很多关于道德人工智能和它为必应设置的护栏，但这里显然还有一些工作要做。我们要求该公司发表评论。

“该团队进行了调查，并在适当的位置放置了屏蔽，所以这就是你不再看到这些的原因，”微软发言人告诉我。“在某些情况下，团队可能会在产生输出时发现问题。在这些情况下，他们将停止正在进行的输出。他们预计系统可能会在预览阶段出错，反馈对于帮助识别哪里工作不顺利至关重要，因此他们可以学习并帮助模型变得更好。”

希望大多数人不会尝试使用 Bing 进行这种类型的查询，在大多数情况下(除了下面提到的一些例外)，你可以简单地将新的 Bing 视为 ChatGPT，但它具有更多最新的数据。当我让它给我看同事们的最新文章时，它会很高兴地调出今天早上的故事。不过，它在基于时间的搜索中并不总是很好，因为它似乎没有真正的“最近”概念。但是如果你想问它这周有哪些电影上映，它会给你一个很好的列表。

图片来源:微软

另一个很棒的特性是，至少偶尔，它会在聊天中带来额外的网络体验。

例如，当我问它关于购买微软股票的问题时，它告诉我，它不会给我财务建议(“因为那会对你的财务有害”)，但也会从 MSN Money 上调出微软的股票代码。

图片来源:微软

和 ChatGPT 一样，Bing 的聊天功能并不总是十分准确。你会很快注意到小错误。当我向它询问 TechCrunch 播客时，它将我们的 Actuator 时事通讯列为其中之一。本简讯没有播客版本。

当被问及更专业的话题，比如作为私人飞行员的夜间视觉飞行规则时，结果有时可能不清楚，部分原因是该模型试图变得如此健谈。像往常一样，它想告诉你它知道的一切，包括无关的信息。在这种情况下，它会先告诉您白天的规则，然后再告诉您晚上的规则，但不会让所有规则都变得显而易见。

图片来源:微软

虽然我喜欢 Bing 引用其来源，但其中一些有点可疑。事实上，它帮助我找到了几个抄袭 TechCrunch 报道的网站(以及其他新闻网站)。这些故事是正确的，但如果我问它最近的 TechCrunch 故事，它可能不会把我送到一个抄袭者和发布我们故事片段的网站。Bing 有时也会引用自己的名字，并链接回对 Bing.com 的搜索。

但是必应能够引用资料来源已经是朝着正确方向迈出的一步。虽然许多在线出版商担心像这样的工具对搜索引擎的点击量意味着什么(尽管 Bing 不那么担心，因为它作为一个流量来源几乎无关紧要)，但 Bing 仍然广泛链接。例如，每一个有来源的句子都是链接的(有时，Bing 也会在这些链接下显示广告)，对于许多新闻相关的查询，它会显示 Bing 新闻的相关报道。

图片来源:微软

除了必应，微软也将把它的新 AI copilot 带到它的 Edge 浏览器中。在昨天公司的活动中出现了几次错误的开始(事实证明，如果是在公司管理的设备上，公司给媒体的构建将无法正常工作)，我现在也有机会使用它了。在某些方面，我觉得这是更引人注目的体验，因为在浏览器中，Bing 可以使用你所在网站的上下文来执行操作。也许那是比较价格，告诉你你想买的东西是否有好的评价，或者甚至写一封关于它的电子邮件。

图片来源:微软

这里有一个奇怪的地方，我将把它归结为一个预览:起初，Bing 不知道我在看什么网站。只有在三四次查询失败后，它才提示我允许 Bing 访问浏览器的网页内容，“以更好地个性化你的体验，使用人工智能生成的 Bing 摘要和亮点。”或许应该更早一点。

Edge 团队还决定将这个新的侧边栏分为“聊天”和“撰写”(除了之前可用的“见解”)。虽然聊天视图知道你所在的网站，但可以帮助你写电子邮件、博客文章和简短片段的撰写功能却不知道。现在，您可以简单地提示聊天视图根据它看到的内容为您写一封电子邮件，但撰写窗口对此有一个很好的图形界面，所以很遗憾它看不到您看到的内容。

驱动这两种模式的模型似乎也有点不同——或者至少它们上面的层被编程为以略微不同的方式做出反应。

当我让 Bing(在网上)给我写一封邮件时，它告诉我“那是你必须自己做的事情。我只能帮你查找信息或生成与技术相关的内容。😅“(必应喜欢在这类回答中加入表情符号，就像 Gmail 喜欢在智能回复中加入感叹号一样。)

但是，在 Edge 聊天窗口中，它会很高兴地写下那封邮件。我在这里的截图中使用了一个复杂的主题，但是对于无关紧要的电子邮件请求，比如向你的老板请一天假，它也做了同样的事情。

图片来源:微软

不过，在很大程度上，这个侧边栏只是复制了整体聊天体验，我猜测它将成为许多用户的切入点——尤其是那些已经在使用 Edge 的用户。值得一提的是，微软指出，随着时间的推移，它会将这些相同的功能引入其他浏览器。不过，该公司没有提供时间表。
