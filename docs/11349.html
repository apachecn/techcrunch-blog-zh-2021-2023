Zapier 通过第一方数据库和 UI 工具扩展其自动化服务| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/10/20/zapier-extends-its-automation-service-with-first-party-database-and-ui-tools/

在很长一段时间里，2011 年推出的 Zapier 满足于帮助其用户自动化简单的工作流程，并在各种关键业务工具之间建立集成。这对该公司来说是一项伟大的业务，但今天的用户期望更多，在过去几年的过程中，该公司决定是时候扩大其产品组合了。这些新产品中的第一个是去年 10 月推出的 Transfer ，这是一款在应用程序之间转移数据的工具。今天，在其 ZapConnect 会议上，它在这个旅程中迈出了下一步，推出了 Zapier 表和界面，这是一种数据库服务和 UI 生成器，允许最终用户与现有的 Zapier 工作流进行交互。

如今，该公司的用户经常使用 Google Sheets 这样的服务作为他们的数据库，Zapier 从本质上创建业务逻辑，然后可能 Salesforce 或 Trello 作为这些工作流的前端。在今天宣布之前的一次采访中，Zapier 的联合创始人兼总裁 Mike Knoop 指出，目前这项服务的一半使用是由这些软件类型的用例组成的。但这也是一个非常脆弱的系统，电子表格中的任何一个机会都会导致整个系统停止工作。

图片来源:扎皮尔

“我们[从客户那里]听到的最大的痛点之一是，虽然 Zapier 在逻辑方面(如果你想这样想的话，是代码方面)有很好的覆盖面，但他们只是告诉我们所有这些常见的痛点，如必须为 UI 和数据存储层集成第三方工具，”Knoop 说。

由于 Google Sheets 甚至像 Airtable 这样的新工具都不是为自动化系统设计的记录系统，所以像 Zapier 这样的工具可以在其上构建的自动化类型是有限制的。

“构建自动化第一版本的表使我们能够获得高速变化记录，并说，‘好吧，我们会保护这个系统，如果你做出这种变化，我们会自动将其沉入带下划线的系统，或提醒你哪些应用程序依赖于它。’我们基本上只是列出了所有常见故障，”Knoop 解释道。

虽然 Tables 位于等式的一边，但 Zapier Interfaces 代表了另一边，关注的是最终用户。这里的想法是允许用户创建可定制的动态网页，与 Zapier 和数据库一起工作，不管它是不是表。Knoop 指出，今天的用户经常自己构建这些系统，但是它们也很脆弱，在初始设置后很难维护。有了这个新工具，用户可以构建表单、编辑数据、共享数据以及启动触发程序，所有这些都只需一个简单的拖放界面。

图片来源:扎皮尔

所有这些新功能都是 Zapier 新的早期访问计划的一部分，该计划目前包括传输、表格和接口。Knoop 没有明确说明该公司下一步将致力于什么，但显然有许多其他的痛点该公司可以直接解决。

对于扎皮尔来说，这绝对是一个有趣的举动。Knoop 承认，该公司有点自满，最近不得不迎头赶上，以满足客户的需求。围绕创新的公司文化发生了一点变化，但这项工作似乎开始有了回报。

在这种情况下，值得注意的是，除了这两个新的 marquee 产品，该公司还推出了八个用户最需要的功能，包括起草 zap 的能力，版本控制，用于构建更复杂 zap 的新工具，在传输中安排传输的能力，为一些高价层、子文件夹上的用户提供自定义错误通知，以及增加一个超级管理员级别。
