Amazon 安全湖是一个基于标准的安全数据湖

AWS 今天宣布 Amazon Security Lake，这是一个新的专门为安全相关数据构建的数据湖。它可以聚合来自云和内部基础架构、防火墙和终端安全解决方案的数据。它帮助企业使用基于标准的格式将所有安全数据集中在一个数据湖中，并管理这些数据的生命周期。

显然，Security Lake 将从 AWS 自己的服务(无论是 CloudTrail 还是 Lambda)以及自己的安全工具(如 AWS Security Hub、GuardDuty 或 AWS Firewall Manager)中聚合数据。但这里重要的是，该公司还支持新的开放网络安全模式框架 (OCSF)，最近宣布支持该框架。该框架为安全遥测数据提供了一个开放的规范。有了这个，它也能从思科、CrowdStrike 和 Palo Alto networks 等公司获取数据。

图片来源: AWS

“安全数据通常分散在应用程序、防火墙和身份提供商的环境中，”AWS 首席执行官 Adam Selipsky 今天表示。“要发现像协同恶意活动这样的洞察，您必须收集和汇总所有这些数据，使您用来支持威胁检测、调查和事件响应的所有分析工具都可以访问这些数据，然后随着事件的发展不断更新数据管道。这意味着，您真正想要的是一种工具，能够轻松存储、分析、了解趋势，并从安全数据中获得洞察力。”

这个工具，当然，就是安全湖。该服务将自动对输入数据进行分区，并将其转换为 OCSF 格式，然后通过 IBM、Splunk 和 Sumo Logic 等合作伙伴提供。

新服务现已在多个 AWS 区域提供预览版:美国东部(俄亥俄州)、美国东部(北弗吉尼亚州)、美国西部(俄勒冈州)、亚太地区(悉尼)、亚太地区(东京)、欧洲(法兰克福)和欧洲(爱尔兰)。
