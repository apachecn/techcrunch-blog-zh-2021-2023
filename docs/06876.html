迭代推出 MLEM，一个简化 ML 模型部署的工具

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/01/iterative-launches-mlem-an-open-source-tool-to-simplify-ml-model-deployment/amp/

图片来源: Getty Images

大约一年前，宣布2000 万美元 A 轮融资的 MLOps 平台迭代，今天推出了 MLEM，一款基于 Git 的开源机器学习模型管理和部署工具。

该公司表示，这里的想法是通过使用开发人员已经熟悉的基于 git 的方法，在 ML 工程师和 DevOps 团队之间架起一座桥梁。使用 MLEM，开发者可以在整个生命周期中存储和跟踪他们的 ML 模型。因此，它补充了 Iterative 的开源 GTO 工件注册表和 DVC ，该公司的数据和模型版本控制系统。

“拥有一个机器学习模型注册表正在成为机器学习技术堆栈的一个重要部分。Iterative 的联合创始人兼首席执行官 Dmitry Petrov 表示:“当前的 SaaS 解决方案可能会导致 ML 模型和软件应用程序的生命周期出现分歧。“我们的 ML 模型注册方法是提供模块化构建模块，组织可以轻松地将其集成到现有的 MLOps 技术体系中。MLEM 用于提取 ML 模型的元信息并简化部署。DVC 在云中或本地存储中管理大型 ML 模型文件。GTO 提供了 GitOps 功能，用于在 Git 中对模型进行版本控制，并向 CI/CD 系统发送信号以进行模型生产。这些独立的工具为 ML 模型管理和模型操作带来了模块化的 Unix 理念。”

图像信用:迭代

正如团队所指出的，像这样的系统允许业务单元和团队之间更容易地共享模型，同时也使 ML 团队更容易与他们的 DevOps 团队协作。对于高度监管的行业，像这样的系统也为弄清给定模型的谱系提供了单一的事实来源。

Petrov 说:“模型注册通过存储和版本化训练的模型，简化了在 ML 生命周期中跟踪模型的过程，但建立这些注册的组织最终会为机器学习模型和软件开发提供两种不同的技术堆栈。”“ MLEM 作为模型注册中心的构建模块，使用 Git 和传统的 CI/CD 工具，联合 ML 和软件团队，以便他们可以更快地将模型投入生产。”

当然，Iterative 本身提供了一个托管平台，通过它的iterative Studio服务来完成所有这些事情，该服务用于在 ML 模型上进行协作，跟踪实验和可视化，以及它的托管模型注册。
