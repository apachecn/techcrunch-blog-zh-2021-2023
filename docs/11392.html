微软的 Windows Dev Kit 2023 允许开发人员在笔记本电脑上使用人工智能处理器 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/10/24/microsofts-windows-dev-kit-2023-lets-developers-tap-ai-processors-on-laptops/

在 5 月份的 Build 大会上，微软推出了 Project Volterra，这是一款由高通骁龙平台支持的设备，旨在让开发人员通过高通的神经处理 SDK for Windows toolkit 探索“人工智能场景”。今天，Volterra——现名为 Windows Dev Kit 2023——正式上市销售，售价 599 美元，在澳大利亚、加拿大、中国、法国、德国、日本、英国和美国的微软商店均可买到

微软是这样描述的:

借助 Windows Dev Kit 2023，开发人员将能够在一个紧凑的设备上实现他们的整个应用开发流程，为他们提供在 Arm 上构建 Windows 应用所需的一切。

正如之前宣布的那样，Windows Dev Kit 2023 包含一个专用的人工智能处理器，称为 Hexagon 处理器，由基于 Arm 的芯片——骁龙 8cx Gen 3——补充，两者都由高通提供。它使开发人员能够与 Visual Studio(17.4 版本原生运行在 Arm 上)等工具一起构建 Arm 原生和人工智能应用。NET 7(其中有针对 Arm 的性能改进)、VSCode、微软 Office 以及包括 PyTorch 和 TensorFlow 在内的团队和机器学习框架。

微软的 Windows Dev Kit 2023，它封装了一个 Arm 处理器和一个 AI 加速器芯片。图片来源:微软

以下是完整的规格列表:

  • 32GB LPDDR4x 内存
  • 512GB 快速 NVMe 存储
  • 骁龙 8cx 第三代计算平台
  • 以太网的 RJ45
  • 3 个 USB-A 端口
  • 2 个 USB-C 端口
  • 迷你显示端口(最多支持三个外部显示器，包括两个 4K 60Hz 的显示器)
  • 蓝牙 5.1 和 Wi-Fi 6

Windows Dev Kit 2023 在 Windows 中提供了对神经处理单元(NPU)的支持，或为人工智能和机器学习特定工作负载量身定制的专用芯片。专用人工智能芯片可以加快人工智能处理速度，同时减少对电池的影响，在智能手机等移动设备中已经很常见。但是随着人工智能驱动的图像升级器和图像生成器等应用程序得到更广泛的使用，制造商们一直在他们的笔记本电脑上添加这种芯片(见微软自己的 Surface Pro X 和 5G Surface Pro 9 )。

Windows Dev Kit 2023 利用了最近发布的高通神经处理 SDK for Windows，该软件提供了在骁龙 Windows 设备上转换和执行人工智能模型的工具，以及针对具有不同功率和性能配置文件的不同处理器内核的 API。使用它和神经处理 SDK，开发人员可以在采用骁龙硬件的 Windows 设备上执行、调试和分析深度神经网络的性能，并将网络集成到应用和其他代码中。

这种工具有利于基于骁龙 8cx Gen 3 片上系统的笔记本电脑，如宏碁 Spin 7 和联想 ThinkPad X13s。骁龙 8cx Gen 3 的人工智能加速器旨在与苹果基于 Arm 的芯片竞争，可用于将人工智能处理应用于照片和视频。微软和高通打赌，随着 Windows Dev Kit 2023 的推出，用例将会扩展；微软方面已经开始在 Windows 11 中利用人工智能加速器来增强背景噪音消除等功能。

图片来源:微软

在今天发布之前与 TechCrunch 分享的一篇博客文章中，微软指出，开发人员将“需要根据他们在 Windows Dev Kit 2023 上的工作负载的需要安装工具链”，一些工具和服务“可能需要额外的许可证、费用或两者兼而有之。”

“更多的应用程序、工具、框架和软件包正在被移植到 Arm 上的原生目标 Windows，并将在未来几个月内到来，”该帖子继续说道。“同时，由于 Windows 11 强大的仿真技术，开发人员将能够在他们的 Windows 开发工具包上运行许多未经修改的 x64 和 x86 应用程序和工具。"

Windows Dev Kit 是否能扭转 Windows 在 Arm 设备上的命运仍有待观察，后者在很大程度上未能成功。从历史上看，它们不如基于英特尔的设备强大，同时受到兼容性问题和高昂价格的困扰(Surface Pro X 发布时的价格超过 1500 美元)。最初几款基于 Arm 的 Windows 设备上的仿真应用性能往往很差，某些游戏除非使用特定的图形库，否则无法启动，而硬件驱动程序只有在专门为 Arm 上的 Windows 设计时才能工作。

Arm 上的 Windows 情况最近有所改善，这要归功于更强大的硬件(如骁龙 8cx Gen3)和微软的应用保证计划，以确保商业和企业应用程序在 Arm 上工作。但这个生态系统还有很长的路要走，今天最受欢迎的游戏引擎之一 Unity 只是在今天早上宣布了一项承诺，允许开发者在 Arm 设备上安装 Windows，以获得原生性能。
