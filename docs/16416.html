Vue Storefront 抢购 2000 万美元以发展其基于开源的“前端即服务”电子商务工具包和平台| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/23/vue-snaps-up-20m-to-grow-its-open-source-based-frontend-as-a-service-e-commerce-toolkit-and-platform/

当前电子商务增长放缓，一些领域支出直线下降，这似乎不是创建电子商务工具初创公司的最佳时机，更不用说投资了。

但是，波兰的一家初创公司凭借对市场的浓厚兴趣，刚刚筹集了 2000 万美元，该公司为前端开发人员提供了一个构建可组合电子商务网站的平台。 Vue Storefront 正在构建它所谓的“前端即服务”，已经完成了一轮 2000 万美元的融资，它计划将这笔资金用于研发，推出更多针对 B2B 电子商务业务的工具，并在过去 18 个月收入增长 300%后用于业务发展。它拥有大约 2，000 名客户、零售商和商家，目前在美国有相当大的增长

这笔资金由 Felix Capital 牵头，这是一家电子商务投资领域的大型公司，尤其是在欧洲，Mirakl 的创始人兼首席执行官 Philippe Corrot 凪良·勒泰法，米拉克尔的工程主管；之前的支持者 Creandum，Earlybird 和 SquareOne 也参与其中。前脸书高管、现任 Felix 合伙人朱利安·科多尔尼乌(Julien Codorniou)也将加入董事会。

该公司将其描述为“A 轮+”，2022 年的 A 轮将带来 1740 万美元的收入。这是一个“略高”的估值，所以从技术上来说不是这轮融资的一部分，尽管 Vue 的首席执行官和联合创始人帕特里克·星期五及其投资者没有谈论这方面的数字。

Friday 补充说，之所以不称之为 B 轮融资，也是因为该公司希望等到市场稳定下来，以更高的估值进行融资，它认为这将是一轮更大的融资。相当令人不安的是，他认为至少在未来两年内不会出现这种情况。

该公司的根源在于开源，它仍然运行着该软件的一个流行的开源版本(据说该社区有 19000 名成员)，这帮助它在市场上获得了最初的人气和牵引力。

商业版本的立场是，使用一个可组合的框架——本质上是通过使用 API 引入不同的工具来改进功能和可用性，让它们一起工作——来处理前端符合后端开发的发展方向。在所有情况下，这导致开发人员节省时间，并且在理论上，产生更好地工作并且可以更灵活地维护的前端界面，以优化购买转化和参与。

星期五说，这是建筑中“拆除巨石”趋势的一部分。“在过去的 20 年里，人们只会从一个供应商那里购买一切，仅此而已。但是现在你可能从一个供应商那里得到后端，从另一个供应商那里得到搜索，从另一个供应商那里得到 CMS，从另一个供应商那里得到付款，等等。前端是集成的最佳场所。我们将所有这些整合在一起，将所有东西放在一起。”

事实上，Shopify 等公司和 Commercetools 等无头电子商务平台已经改变了小型和大型商家的游戏规则。这里的理论是，Vue Storefront 代表了这一轨迹的下一步，尽管同一领域的其他人可能也在寻求更长期的解决方案。除此之外，其他初创公司也专注于前端，比如美国的幕府将军。

该公司有大约 30 个集成作为其商业产品的一部分，具有托管选项和一些最受欢迎的组件的预构建版本，以及 API 编排工具来管理不同的组件。

除了这一轮，Vue Storefront 还宣布了一位来自 Sendcloud 的新 CTO Tim drij vers(因此，也许我们应该期待看到更多直接的客户工具来将运输和物流添加到前端组合中？).Vue Storefront 的另一位联合创始人、前首席技术官菲利普·拉科夫斯基(Filip Rakowski)将担任“首席开发者体验官”的新角色。

Felix 一直在更深入地关注欧洲欠发达市场，作为其挖掘其他人可能不会关注的较少曝光和更多新兴创业公司的战略的一部分，这笔投资强调了这一点，以及支持那些帮助电子商务企业在当今市场条件下做生意的人的想法。

“电子商务的增长已经放缓，但每个人都在寻找改善客户体验的解决方案，更重要的是，转化率，”Codorniou 说。Vue Storefront 的开源根源也是一大优势，因为潜在客户可以在购买前进行尝试，并依赖于一个非常活跃和充满活力的开发人员和合作伙伴社区，他们每天都在努力使产品变得更好。"
