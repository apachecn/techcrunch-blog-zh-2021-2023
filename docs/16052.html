谷歌警告用户采取行动防范流行安卓手机中可远程利用的漏洞

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/16/google-warning-samsung-chips-flaws-android/

谷歌的安全研究部门就其在数十种安卓机型、可穿戴设备和汽车中的某些三星芯片中发现的一系列漏洞发出了警告，担心这些漏洞可能很快被发现并被利用。

谷歌零计划负责人蒂姆·威利斯(Tim Willis)表示，内部安全研究人员在过去几个月里发现并报告了三星生产的 Exynos 调制解调器中的 18 个零日漏洞，其中包括四个最严重的漏洞，这些漏洞可能会通过蜂窝网络“悄悄地和远程地”危及受影响的设备。

威利斯说:“Project Zero 进行的测试证实，这四个漏洞允许攻击者在没有用户交互的情况下，在基带级别远程破坏手机，只需要攻击者知道受害者的电话号码。”

通过获得在设备基带级别远程运行代码的能力(实际上是将手机信号转换为数字数据的 Exynos 调制解调器)，攻击者将能够几乎不受限制地访问进出受影响设备的数据，包括手机通话、短信和手机数据，而不会提醒受害者。

随着披露的进行，很少看到谷歌或任何安全研究公司在高严重性漏洞被修补之前就发出警报。谷歌指出了公众面临的风险，称熟练的攻击者“能够通过有限的研究和努力，迅速创造出一个可操作的漏洞”。

Project Zero 的研究员 Maddie Stone 在 Twitter 上写道，三星有 90 天的时间来修补这些漏洞，但现在还没有。

三星在 2023 年 3 月的一份安全清单中证实，几个 Exynos 调制解调器容易受到攻击，影响了几家 Android 设备制造商，但没有提供其他细节。

据 Project Zero 称，受影响的设备包括近 12 款三星机型、Vivo 设备以及谷歌自己的 Pixel 6 和 Pixel 7 手机。受影响的设备还包括依赖 Exynos 芯片连接到蜂窝网络的可穿戴设备和车辆。

受影响设备列表包括(但不限于):

  • 三星移动设备，包括 S22、M33、M13、M12、A71、A53、A33、A21s、A13、A12 和 A04 系列；
  • Vivo 移动设备，包括 S16、S15、S6、X70、X60 和 X30 系列；
  • 谷歌 Pixel 6 和 Pixel 7 系列；
  • 使用 Exynos Auto T5123 芯片组的联网车辆，

谷歌表示，补丁将因制造商而异，但指出其 Pixel 设备已经用其三月安全更新打了补丁。

在受影响的制造商向其客户推送软件更新之前，谷歌表示，希望保护自己的用户可以在他们的设备设置中关闭 Wi-Fi 通话和 LTE 语音(VoLTE)，这将“消除这些漏洞的利用风险”。

谷歌表示，其余 14 个漏洞不太严重，因为它们要么需要访问设备，要么需要拥有手机运营商系统的内部权限或特权。
