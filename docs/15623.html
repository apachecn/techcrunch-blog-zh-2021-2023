a16z 支持的 Uno 推出以设计为中心的密码管理器| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/07/a16z-backed-uno-launches-a-design-centric-password-manager/

有很多很好的理由为什么你应该使用密码管理器，从帮助你生成和存储复杂和独特的密码到不需要记住任何密码。但是对于一些人来说，第一次使用密码管理器可能会很麻烦。

为了解决这个问题，a16z 支持的公司 Uno 推出了一款以设计为中心的新密码管理器。这家初创公司的密码管理器是一款适用于 iOS 和 Mac 的应用程序，也是 Chrome 的一个扩展，旨在让人们更容易处理密码和登录。

Uno 包含大量旨在使登录更容易的功能:一键登录、通过可信联系人恢复社交密码、定制和简单的密码共享以及存储私钥、信用卡详细信息和地址的安全保险库。

Uno 的首个密码管理器。图片来源: Uno

当你在桌面上登录网站时，Chrome 扩展为你做了大部分工作。如果你用 Uno 保存了你的登录信息，该公司会一键处理所有的登录过程——包括发送到电子邮件的 2FA 代码。你必须登录 Gmail，并允许该应用程序阅读你的最新电子邮件，但该公司表示，所有这些过程都是在你的设备上处理的，不会向他们的服务器发送电子邮件数据。

该公司表示，该扩展可以识别何时用数据填写地址字段，以及何时填写登录信息。

iOS 和 Mac 应用程序都处于测试阶段，具有基本的安全存储和密码自动填充功能。这家初创公司表示，它已经在开发 Android 版本，但没有给出具体的发布时间表。

图片来源: Uno

如果您丢失了设备，该应用程序会要求您保存一个用于恢复数据的私钥短语。还有另一个稍微复杂的过程来恢复您的数据。您可以将信任的联系人添加到您的 Uno 帐户，对于恢复，他们可以通过验证您的投票来帮助您。但问题是他们都必须是 Uno 用户。因此，除非你找到也使用该应用程序的人，否则你可能更好地坚持传统方法，如从另一台设备恢复或输入你的私人密钥短语。

这个公司

Uno 由 Parteek Saran 创立，他有设计背景，曾与 Lady Gaga、脸书和 Postmates 合作过项目。Saran 还共同创建了一个名为 Form 的交互设计和原型工具，该工具于 2014 年被谷歌收购。收购后，他在这家搜索巨头工作了五年，从事从硬件设计到软件设计的各种产品——最著名的是参与了谷歌的材料设计方法。

该公司已经筹集了 300 万美元的种子资金，目前由 Andreesson Horowitz 领导，Lookout 创始人 Kevin Mahaffey 和 Duo security 的 Dug Song 参与其中。

Uno 的 iOS 版应用。图片来源: Uno

萨兰说，Uno 的灵感来自于黑客控制了他的电子邮件、金融服务、社交账户甚至 Spotify 播放列表。

“在遭到黑客攻击后，我正在升级我的帐户的安全性，我意识到这个过程是技术性的，而且很麻烦。有很多步骤和术语对于非技术人员来说可能难以理解，”Saran 告诉 TechCrunch。“让人们定期使用密码管理器是一个行为问题。影响它的方法是通过观察人类如何与这种软件交互来设计解决方案。”

创始人说，通过 Uno，他希望面向更广泛的人群——包括不太关心密码安全的用户。

安全

虽然密码管理员通过存储大量凭据来提高便利性，但他们也有责任保护数据和用户隐私。

优诺表示，它从用户那里收集的数据很少，所有存储在服务器上的数据都是用存储在用户本地设备上的私钥加密的，该公司无法访问这些数据。它指出，只有电子邮件，电话号码和帐户的公钥被收集。

Saran 表示，该应用程序不会使用分析工具跟踪任何个人数据。该公司的隐私政策指出，“在任何情况下，您的安全保险库中的私人内容都不会以 Uno 可以破译的形式传输给 Uno。”

“我们非常关心人们的隐私和安全。我认为人们有点厌倦了放弃他们的数据，喜欢做所有这些事情。所以我们的立场是——我们不希望这样。我们的应用程序只需要最低限度的权限就能运行，”Saran 说。

考虑到黑客们——尽管是非常熟练的黑客——能够接触到 LastPass 的数据(包括客户的密码库),这也是一个安全问题。Uno 的一个出发点是限制其员工可以访问的客户数据。这家初创公司表示，它希望通过将敏感数据存储在用户的设备上而不是云中，来保持本地优先和客户优先的方法，从而避免此类事件。此外，Uno 指出，由于它对包括密码在内的所有客户数据进行加密，黑客即使拿到了一个人的设备也无法破解。

至于说服客户信任其产品，Uno 表示，它已经联系了更大的供应商，对其应用程序进行正式的安全审计。

“Uno 已经让独立的安全工程师审计代码并进行渗透测试，并且已经启动了联系大型供应商进行正式审计的流程。他们目前正处于公开测试阶段，这就是为什么它没有早点开始。”宇野说。Uno 没有透露早期代码审计和渗透测试的结果，但表示计划公布未来的审计结果。

该公司的目标受众——非技术人员——可能不会问这些问题。但是 Uno 有责任对其高级用户提供足够的保证和数据，对密码管理器的安全实践保持公开和透明。
