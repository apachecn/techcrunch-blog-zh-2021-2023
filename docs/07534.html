Strapi 因其“无头”CMS 平台获得 3100 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/22/strapi-lands-31m-for-its-headless-cms-platform/

开源内容管理系统(CMS)Strapi 今天宣布，它在由 CRV 牵头的 B 轮融资中筹集了 3100 万美元，参与方包括 Flex Capital、Index Ventures 和天使投资人，其中包括前 GitHub 首席执行官 Nat Friedman。随着资本的到来，首席执行官 Pierre Burgy 告诉 TechCrunch，该计划将推出新产品 Strapi Cloud，同时引入更多的技术和解决方案合作伙伴，并进一步开发 Strapi 项目。

Strapi 源于 Burgy、Aurélien Georget 和 Jim Laurie 在大学做自由开发人员时共同开发的软件。由于对传统框架和内容管理系统的局限性感到恼火，他们决定创建自己的解决方案 Strapi，用于客户的项目。

Burgy、Georget 和 Laurie 于 2015 年在 GitHub 上发表了 Strapi。“对我们来说，让这个项目开源是显而易见的，”Burgy 在一次电子邮件采访中说。“我们使用了大量开源软件，我们相信协作会带来更好的软件。我们的大多数客户都想在他们自己的服务器上托管他们的 CMS 定制功能只能通过访问源代码来实现。”

一开始，Strapi 是构建 api 的框架和 CMS 的混合体。“Strapi”这个名字实际上是“Bootstrap your API”这个短语的创造性缩写。如今，Strapi 允许开发人员连接和管理不同的数据库、前端框架和静态站点生成器。用户可以在服务器上运行 Strapi 并连接到 Strapi 的管理界面，依靠前端使用 api 从 Strapi 获取内容。

Strapi 的内容管理仪表板。图片来源: Strapi

“CMS、知识管理和企业维基软件之间有相当多的混淆和重叠，”Burgy 说。“这些平台由于其卓越的技术架构，一直被用户视为独立的解决方案。人们对他们的看法是看他们是什么样的人，而不是看他们取得了什么成就。这正在改变。”

从 Burgy 的角度来看，与 Contentful、Storyblok 和 Sanity 竞争的 Strapi 解决了遗留 CMS 软件的一些主要问题，如缺乏对定制用例的适应性。为了解决其他已知的限制，Strapi 团队在最近几个月推出了一些功能，如 Strapi Market，它提供了一个与第三方平台集成的库。

Burgy 承认 Strapi 并不完美，但他表示近期目标是将资源投入到内容编辑体验中，包括预建模板、出版工作流程、富文本编辑器和自定义字段，以及个性化、可视化和电子商务插件。Strapi Cloud 的工作也在进行中，这是一个完全托管的 Strapi 服务，它将允许客户在本地启动一个项目，根据他们的需求进行定制，并将其部署在 web 上。

Strapi Cloud 计划于明年全面上市。它目前处于测试阶段。

“有了疫情，许多面临不确定性的企业被迫转向网络来适应。另一方面，个人在线消费的内容比以往任何时候都多，他们也需要适应远程工作的现实，”Burgy 说。“因此，公司越来越关注所有利益相关者的整体体验。Strapi 通过提供一个更加通用和可组合的平台来帮助公司实现这一过程，帮助他们的应用程序经得起未来考验，并为所有用户提供跨渠道的一致体验。”

除了成千上万的开源用户，Burgy 说 Strapi 还有 450 个付费用户，包括美国电话电报公司、易贝、IBM 和丰田以及美国和欧洲的政府机构。该公司的员工总数为 70 人；Burgy 预计，到今年年底，这一数字将增长到 110。

迄今为止，Strapi 已经筹集了 4500 万美元。
