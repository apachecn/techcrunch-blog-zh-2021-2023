GitHub 迎来了新的首席执行官

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/03/github-gets-a-new-ceo/amp/

图片来源:大卫·保罗·莫里斯/彭博/盖蒂图片社

GitHub 首席执行官纳特·弗里德曼将于 11 月 15 日卸任，成为微软旗下服务的名誉主席。最近刚刚成为 GitHub 首席产品官的托马斯·多姆克将接任首席执行官一职。

当微软在 2018 年收购 GitHub 时，开发者社区中有相当多的担忧这将是一个霸道的存在，并将代码共享和协作服务变成微软第一的平台。由于弗里德曼的开发者和开源背景，他在接受这份工作时带来了很多社区善意，GitHub 在他的三年任期内保持了独立性和平台中立性。

正如多姆克告诉我的，这在他的领导下不会改变。正如他在今天宣布之前的一次采访中指出的，弗里德曼曾要求他在收购后加入董事会。出生于德国的多姆克最出名的可能是 HockeyApp 的联合创始人兼首席执行官，微软于 2015 年收购了该公司。

“2018 年，Nat 从微软方面挑选我领导 GitHub 交易，”自收购 HockeyApp 以来在微软开发者部门排名上升的多姆克说。“对我来说，这真是一段激动人心的时光。这有点像回到了我的首席执行官时代，因为我领导着所有不同的交易职能。GitHub 在产品和工程世界之外还有很多方面需要解决。然后 Nat 邀请我和他一起加入 GitHub，我欣然接受了。我在 GitHub 负责战略项目和特殊项目。”

新任 GitHub 首席执行官托马斯·多姆克

他的第一个项目是在 2019 年初让所有开发商免费获得私人回购。虽然大多数开发人员可能都意识到了这一点，但多姆克自己也承认，他没有弗里德曼担任首席执行官时的公众形象，但他强调，他作为开发人员和开源倡导者的背景符合过去 GitHub 首席执行官的总体路线。

“我希望人们在回顾过去几年 Nat 作为 GitHub 领导者的时候，会对所有让 GitHub 成为软件开发更好地方的东西感到兴奋和感激，”Dohmke 说。“我希望他们在回顾这个时代时，把它视为 GitHub 的伟大时代之一。在与微软交易期间，人们担心这对 GitHub 的独立性意味着什么。我们离微软越来越近了吗？我认为在过去的几年里，我们已经证明了我们将保持独立，我们将保持云中立，我们为开发者做正确的事情，我们将开发者放在第一位。希望他们将这种转变视为 GitHub 首席执行官作为开发者的传统的延续。以前的 CEO 们，我们都是开发者，Nat 是开发者，我也是开发者。我希望每个人都对我们公司的发展方向以及我们将如何创新和提高开发人员的工作效率感到兴奋。”

不过，他确实指出，他的整体风格有点不同，他更注重执行力，而不一定是一个有远见的人。

“在过去的几年里，我们一直密切合作，我们的许多风格都保持一致，我们几乎每天都有非常密切的对话。我认为 Nat 更有远见，能看到未来——我更有执行力，专注于现在必须发布的内容。但我认为我们都非常重视顾客，”多姆克解释道。

然而，展望未来，仍然喜欢在业余时间从事一些以移动为中心的私人编码项目的多姆克并不打算马上做出任何重大改变。相反，他描述的很多内容是 GitHub 已经走上的轨迹的延续——继续让 1 亿开发者进入其平台。在他看来，这有四个支柱。

首先，该公司将继续打造像副驾驶这样的人工智能项目。正如 Dohmke 指出的，在过去的几年里，GitHub 在软件开发的 CI/CD 方面做了很多工作——在 pull 请求之后发生的所有事情。他说，“有了人工智能和 Copilot，我们将把它带入内循环，带入开发者在笔记本电脑上做的事情，以及他们在编辑器中做的事情。”

他描述的第二个支柱是由 Visual Studio 代码支持的 GitHub 基于云的开发环境 Codespaces 。他说，这里的想法是确保 GitHub 可以满足开发人员的需求——让他们不必为每个新项目建立复杂的开发环境。

毫不奇怪，第三个支柱是 GitHub 社区。“如果你想想今天的创造者社区，我们会想到抖音，YouTube 和 Substack，”他说。“但实际上，开源是可以追溯到 90 年代的原创社区之一。如果你想想开放源码的早期，Linux 的早期——这都是关于创造东西和与世界分享东西，以及彼此合作创造这个软件世界。因此，我们认为 GitHub 正变得越来越像一个创意社区，或者，如果你愿意的话，它在 2008 年开始的时候是最初的创作者社区之一。"

在多姆克看来，安全是 GitHub 从头开始构建的第四大支柱——这显然是该公司一段时间以来的重点。多姆克自己帮助领导了该领域的一些收购，包括 2019 年的dependent bot和semle收购。

多姆克在 8 月份才成为 GitHub 的首席执行官，看起来他没想到会这么快就担任这个新角色。

“8 月份，我不知道自己已经成为首席执行官，但从作为朋友与纳特交谈，从作为同龄人与纳特交谈，作为 GitHub 的合伙人，我知道在某个时候，他的心在别处，他想回到他的创业根源，更多地探索创业领域，”多姆克告诉我。“因此，这种情况已经持续了一段时间，我们正在为有序的过渡做准备。”作为 GitHub 领导团队的一员，他在过去三年里参加了所有的产品审查。

弗里德曼证实，他想回到创业现场。他在今天的公告中写道:“考虑到我们已经取得的成就，以及在微软超过五年的辉煌时光，我决定是时候回到我创业的起点了。”。“推动我前进的是让建设者创造未来。我喜欢和开发人员一起工作，向他们学习，开发新的工具和新的项目，解决棘手的问题，用代码创造奇迹。这就是为什么我要继续我的下一次冒险:支持、建议和投资创始人和开发者，他们用技术创造未来，抓住当今最大的机遇。”

很难责怪他。他在 2016 年将 Xamarin 卖给了微软，并且比大多数人预期的时间都长。在一定程度上，这肯定是因为他被邀请担任 GitHub 的首席执行官。我们会留意他接下来会做什么。毕竟，很有可能会有一只猴子作为吉祥物。
