OpenAI 发布 Point-E，这是一个生成 3D 模型的 AI TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/20/openai-releases-point-e-an-ai-that-generates-3d-models/amp/

图片来源: OpenAI

下一个席卷人工智能世界的突破可能是 3D 模型生成器。本周，OpenAI 开源了 Point-E，这是一个机器学习系统，可以在给出文本提示的情况下创建 3D 对象。根据与代码库一起发布的论文，Point-E 可以在单个 Nvidia V100 GPU 上在一到两分钟内生成 3D 模型。

Point-E 并不创造传统意义上的 3D 物体。相反，它生成点云，或空间中代表 3D 形状的离散数据点集——因此有了这个大胆的缩写。(Point-E 中的“E”是“efficiency”的缩写，因为它显然比以前的 3D 对象生成方法更快。)从计算的角度来看，点云更容易合成，但它们不能捕捉物体的精细形状或纹理——这是目前 Point-E 的一个关键限制。

为了绕过这一限制，Point-E 团队训练了一个额外的人工智能系统，将 Point-E 的点云转换为网格。(网格是定义对象的顶点、边和面的集合，通常用于 3D 建模和设计。)但他们在论文中指出，该模型有时会错过物体的某些部分，导致块状或扭曲的形状。

图片来源: OpenAI

在独立的网格生成模型之外，Point-E 由两个模型组成:文本到图像模型和图像到 3D 模型。文本到图像模型，类似于 OpenAI 自己的 DALL-E 2 和稳定扩散等生成艺术系统，在带标签的图像上进行训练，以理解单词和视觉概念之间的关联。另一方面，图像到 3D 模型被输入一组与 3D 对象配对的图像，以便它学会在两者之间有效地转换。

当给出文本提示时，例如，“一个 3D 可打印齿轮，一个直径 3 英寸、厚 0.5 英寸的齿轮”，Point-E 的文本到图像模型会生成一个合成的渲染对象，该对象会被馈送到图像到 3D 模型，然后生成一个点云。

OpenAI 的研究人员表示，在对“数百万”3D 对象和相关元数据的数据集进行训练后，Point-E 可以产生经常与文本提示匹配的彩色点云。它并不完美——Point-E 的图像到 3D 模型有时无法从文本到图像模型中理解图像，导致形状与文本提示不匹配。尽管如此，它还是比之前的最先进技术快了几个数量级——至少根据 OpenAI 团队的说法。

将点云转换成网格。图片来源: OpenAI

他们在论文中写道:“虽然我们的方法在这种评估上的表现不如最先进的技术，但它可以在很短的时间内产生样本。”“这可能会使它在某些应用中更加实用，或者允许发现更高质量的 3D 对象。”

具体应用是什么？OpenAI 的研究人员指出，Point-E 的点云可以用来制造现实世界的物体，例如通过 3D 打印。通过额外的网格转换模型，该系统可以——一旦它变得更加完美——也可以找到进入游戏和动画开发工作流的方法。

OpenAI 可能是最新一家加入 3D 对象生成器竞争的公司，但正如前面提到的，它肯定不是第一家。今年早些时候，谷歌发布了 DreamFusion，这是该公司在 2021 年推出的一个生成式 3D 系统 Dream Fields 的扩展版本。与梦场不同，DreamFusion 不需要事先训练，这意味着它可以在没有 3D 数据的情况下生成对象的 3D 表示。

虽然目前所有的目光都集中在 2D 艺术生成器上，但模型合成人工智能可能是下一个大的行业颠覆者。三维模型广泛应用于影视、室内设计、建筑和各种科学领域。例如，建筑公司用它们来展示提议的建筑和景观，而工程师则利用模型来设计新的设备、车辆和结构。

E 点故障案例。图片来源: OpenAI

不过，3D 模型的制作通常需要一段时间，从几个小时到几天不等。像 Point-E 这样的 AI 可以改变这一点，如果有一天这些缺陷被解决了，并使 OpenAI 获得可观的利润。

问题是将来会出现什么样的知识产权纠纷。3D 模型有很大的市场，包括 CGStudio 和 CreativeMarket 在内的几个在线市场允许艺术家出售他们创作的内容。如果 Point-E 流行起来，其模型进入市场，模型艺术家可能会提出抗议，指出证据表明现代生成式人工智能大量借鉴其训练数据——在 Point-E 的情况下，是现有的 3D 模型。像 DALL-E 2 一样，Point-E 没有提到任何可能影响其几代人的艺术家。

但是 OpenAI 把这个问题留到以后再说。无论是 Point-E 文件还是 GitHub 页面都没有提到版权。

值得称赞的是，研究人员确实提到他们预计 Point-E 会遭遇的其他问题，比如从训练数据中继承的偏见，以及缺乏对可能被用来创建“危险物体”的模型的保护这也许是为什么他们小心翼翼地将 Point-E 描述为一个“起点”,他们希望这将激发文本到 3D 合成领域的“进一步工作”。
