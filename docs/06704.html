Stripe 推出应用市场、脚本和工具，整合了与 Stripe | TechCrunch 协同工作的第三方 SaaS 应用

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/24/stripe-launches-app-marketplace-scripts-and-tools-incorporating-third-party-saas-apps-that-work-alongside-stripe/

支付新贵 Stripe ，目前估值为950 亿美元，已经推出了许多新产品，以扩展其名声在外的 API，建立其作为金融服务和相关商业基础设施平台的地位。最新的一项服务今天首次亮相:这项新服务名为 App Marketplace，是一项新产品，Stripe 将提供对第三方应用的访问，以及由应用发行商、用户和 Stripe 自己创建的脚本，这些脚本将这些应用与 Stripe 结合在一起。这可能代表着它在支付之外的最大飞跃。

该市场推出了 50 个应用程序，Stripe 的客户已经在使用这些应用程序作为其营销、支付和业务开发堆栈的一部分，如 DocuSign、Dropbox、Intercom、Mailchimp、Ramp 和 Xero。Stripe 告诉我，随着时间的推移，还会增加更多。

最初，将这些应用与 Stripe 结合在一起的脚本将免费使用——目的是获得一些关键的使用量，因此门槛很低——尽管鉴于这是一个市场，并且它将试图鼓励更多的第三方为该生态系统构建应用，Stripe 可能会在未来对其中一些脚本进行收费。与此同时，作为这些脚本一部分的付费服务的第三方应用程序，如果用户还不是订户，将直接向他们收费。但随着脚本本身的发展，Stripe 也可能会在自己的平台上引入新的支付方式。

也许最有趣的是，虽然最初的想法是为一些最常见的用法创建脚本，但理论上这些脚本可以覆盖任何类型的自然存在于 Stripe 中的业务流，即使它不涉及 Stripe 服务本身。

“在应用审查过程中，我们要求它必须具备真正的商业功能，”Stripe 应用产品负责人潘博文(Bowen Pan)在接受采访时说。“所以它不能仅仅是一个问答游戏应用程序。它必须以商业为中心。但是它不必具有现有的条带功能。它可以是有助于处理的生产力工作流程。”

潘告诉我，应用市场的想法来自于该公司对 Stripe 产品使用情况的观察:简言之，支付不是凭空而来的，而是更广泛的商业服务流的一部分，可能涉及销售或营销活动、客户支持团队以及合同或其他协议的发布。

到目前为止，客户通常会一起使用这些工具，要么构建手动集成，要么根本不集成它们。(如果您曾经与客户支持部门通话或互动，并发现自己要等好几分钟才能得到回复…这通常是原因之一:他们正在查看和更新多个数据库。)

这里的想法是提供帮助加速所有这一切的方法——当然，使人们更有可能在 Stripe 生态系统中开展所有业务活动。

该市场的推出紧随 Stripe 最近几个月推出的其他几款产品之后，这些产品使该公司超越了帮助其成名的服务，即主要用于使商户和其他在线支付卡的人更容易这样做的 API(甚至该公司的名称 Stripe 也是指支付卡背面的磁条)。这包括向加密货币迈出期待已久的步伐；Connections 是一种类似格子图案的服务，帮助客户从银行账户中提取财务数据；最近的一个 ETL 产品在雪花和 AWS 红移仓库中更有效地收集和分析条带数据。

公平地说，该公司多年来一直在采取一些小举措，将支付业务扩展到邻近领域，如税务计算、预付款甚至企业注册。然而，这一新的工具浪潮代表了非常大的金融基础设施游戏，表明该公司如何寻求完全填补其超大的估值，鉴于公共和私人融资市场正在发生的事情，这些天或许是一项更重要的任务。
