Aembit 筹集 1660 万美元，将身份管理引入工作负载| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/20/aembit-raises-16-6m-to-bring-identity-management-to-workloads/?tpcc=tcplustwitter

总部位于马里兰州的安全初创公司 Aembit ，专注于帮助开发人员和安全团队管理联合工作负载如何相互对话，今天正式推出其服务，并宣布从 Ballistic Ventures 和 Ten Eleven Ventures 获得 1660 万美元的种子资金。

实质上，Aembit 的工作负载身份和访问管理服务应用了行业知识，从管理用户和设备访问到云工作负载，如 API、数据库和其他云资源，所有这些都不需要开发人员对其代码进行更改。

图片来源: Aembit

联合创始人大卫·戈尔德施拉格和凯文·萨普在一起工作了 17 年。在其他创业公司中，他们共同创立了零信任平台新边缘实验室，后者被 Netskope 收购，以及移动设备管理平台 Trust Digital，后者被 McAfee 收购。

“一路上，人们总是问我们:工作负载到工作负载的应用程序级访问怎么样？一直都有这个问题，这很重要，但我们没有解决它，”戈尔德施拉格解释道。当创始人在 2021 年夏天离开 Netskope 时，他们决定最终应对这一挑战。“这很重要，因为生态系统中发生了所有这些事情，对吗？所有这些 API 都成为了人们应用程序的一部分，”他指出。“如果你想想几年前的开源，人们通过包含开源来构建应用程序。今天，人们通过包含数据库和 API 来构建应用程序，现在您需要支持这些数据库和 API 之间的安全访问。”

Aembit 联合创始人大卫·戈尔德施拉格(左)和凯文·萨普(右)。图片来源: Aembit

他指出，Aembit 的使命不同于 API 网关和安全服务。这些服务位于 API 的前面，帮助开发人员构建这些服务，并将其安全地公开给内部和第三方开发人员。但是 Aembit 的重点是访问 API 的客户端，并确保这个客户端被授权访问它。他将其比作当今的身份管理系统如何帮助企业授权他们的用户。例如，当用户使用 Okta 登录到 Microsoft 365 时，该用户与 Okta 交互，然后获得访问该服务的凭证。

要做到这一切，Aembit 还必须不仅成为所有这些工作负载身份的记录系统，而且成为工作负载本身的记录系统(如今，这些工作负载往往是短暂的，这使得问题更加棘手)。

图片来源: Aembit

“你应该从基础层面开始，也就是你有身份，你有政策。启用访问并记录下来。但你可能希望从所有这些分散的地方发现越来越多的工作负载，然后你可能希望发现访问模式，”Goldschlag 解释说。“我们的系统已经可以做到这一点。我们可以以非强制模式(发现模式)部署系统，告诉我们正在发生什么访问。”

然后，以此为路线图，就可以更容易地了解这些工作负载通常是如何交互的，并在发生变化时采取行动。

“企业已经花费了大量资源来保护人与他们使用的软件之间的连接。然而，随着企业转向云，一个新的快速增长的攻击面已经出现，”Ballistic Ventures 的联合创始人兼普通合伙人杰克·塞德说。“需要识别、保护和管理软件与其他软件对话时产生的工作负载到工作负载的网状连接。Aembit 正在定义这种新的工作负载 IAM 类别，以保护企业最重要的数字资产。从第一天开始，我就很荣幸能与 Aembit 的创始人一起工作，并继续支持他们的旅程。"

Aembit 目前有 11 名全职员工，几乎都是工程师。有了这笔新资金，该公司计划壮大其营销团队，打造自己的产品。具体来说，Aembit 在向大型企业销售产品方面表现出色，计划很快推出一款自助产品，这将使其能够扩展到更多的中小型企业。
