Union.ai 筹集 1000 万美元用于简化 ML 工作流程编排

原文：https://web.archive.org/web/https://techcrunch.com/2022/04/12/union-ai-raises-10m-to-simplify-ai-and-ml-workflow-orchestration/

Union.ai 是一家从 stealth 崛起的初创公司，拥有开源人工智能编排平台 Flyte 的商业版本，今天宣布在 NEA 和“精选”天使投资人的一轮投资中筹集了 1000 万美元。首席执行官 Ketan Umare 表示，收益将用于支持 Flyte 社区，方法是“提高 Flyte 的可访问性、性能和可靠性”，并扩大 Flyte 集成的系统阵列。

虽然公司发现人工智能的预测能力很有吸引力，特别是在组织的数据分析方面，但用人工智能实现有意义的结果往往是一个挑战。诚然，人工智能可以帮助预测收入，例如，通过识别买卖趋势。但是，实现和维护必要的数据管道，以防止人工智能系统漂移到不准确的程度，可能需要大量的技术资源。

这就是 Flyte 的用武之地——一个编程和处理并发人工智能和数据分析工作流的平台。Union 的团队，包括 Umare，在 Lyft 期间帮助建立了 Flyte，在那里它被用来帮助创建一个系统来计算司机从 A 点到 b 点的预计到达时间(ETA)。

Umare 通过电子邮件告诉 TechCrunch:“(Union 的)创始人第一次见面是在 Lyft，在那里我们加入了负责计算 Lyft 司机从 A 点到 B 点的预计到达时间的团队。”“寻找正确的解决方案使团队深入到机器学习技术中，这需要使用大量数据，并一致地将稳健的模型交付给生产……所用的技术已经平台化，该解决方案在 Lyft 得到了广泛应用。”

Lyft 在 2020 年将 Flyte 贡献给开源，一年后将商标授予 Linux 基金会。这时，Union 的团队发现了一个机会，可以将付费服务置于云项目之上。

“Flyte 的一个托管版本，称为 Union Cloud，将允许较小的团队和组织使用 Flyte 的功能，而无需为基础架构团队配备人员，”Umare 继续说道。“我们[创立了 Union]，因为我们相信机器学习和数据工作流与软件部署有着本质的不同。这是因为软件更精确，生命周期更长，而机器学习和数据工作流刚开始是实验性的，可能需要快速生产。”

服用 Flyte

Umare 和 Union 的其他联合创始人 Haytham Abuelfutuh 和 George Snelling 都有深厚的科技行业背景。在加入 Lyft 之前，Umare 是亚马逊的高级软件工程师和甲骨文的首席工程师，在甲骨文，他领导了基础设施即服务和裸机产品的块存储产品的开发。Abuelfutuh 在微软担任了七年工程师，在谷歌担任了三年开发人员，在谷歌，他帮助发布了包括谷歌照片在内的第三方应用程序的内部软件库。Snelling 也是一名微软老将，他共同创立了几家初创公司(Westside、LabKey 和 Patchr)，并在 Salesforce 担任高级工程总监。

Umare 表示，Union Cloud 的推出恰逢 Flyte 版本的发布，其目标是减少(最好是消除)数据科学项目和开发中可能出现的笨拙基础设施。乌马尔指出，在最糟糕的情况下，混乱的抽象可能需要重建基础设施，以将人工智能部署到生产中——这对潜在的投资回报产生了负面影响。

根据 2021 年 Wakefield Research 的报告，企业数据工程师花费将近一半的时间来构建和维护数据管道。69%的调查受访者(主要是数据工程师)表示，如果他们的团队能够为业务决策做出更多贡献，并减少在手动管道管理上花费的时间，业务成果将会改善。

“生产机器学习目前仍处于初级阶段，尤其是在大科技以外的公司。因此，大多数公司从 DIY 开始——这是我们的主要竞争对手，”Umare 说。“我们采用了完全不同的第一原则方法来定义工作流对机器学习和数据科学家的意义。我们一开始的目标是最大限度地减少人为错误，并试图帮助提前预测问题(并与 Spotify、Gojek 和 Freenome 等极其复杂和多样化的合作伙伴密切合作，以帮助完善解决方案。”

Union Cloud 继承了 Flyte 的所有特性和功能，包括计算后端之间的连接器，记录对 AI 管道的所有更改。Union Cloud 还存储所有管道执行的历史，并提供仪表板、命令行界面和 API 来与计算进行交互。

Union Cloud 和 Flyte 将工作流定义为多个任务。工作流和任务可以用任何编程语言编写，并驻留在本地，数据在这些组件之间移动也是如此。

云优势

那么，联合云的附加值是什么？Umare 表示，它通过集中基础设施管理和维护“高”隐私和合规标准，为 Flyte 增加了“灵活性、可再现性和安全性”。“我们的产品是本着零信任原则构建的，因此我们的用户可以使用它来构建一个仍然保持高安全标准的自助服务平台，”他继续说道。“数据科学的学术性很强，直接影响机器学习。学术界有很多很棒的研究和文献，但是很难生产出来。我们需要以结构化和可重复的方式将这两个世界联系起来。”

Umare 还将 Union Cloud 视为一种降低新产品和系统开发成本的方式，这是开源 Flyte 项目无法实现的。虽然他承认其他供应商也有类似的努力，如 AWS Sagemaker ，但他认为他们未能与数据科学生态系统的其他部分很好地集成。

“我们已经研究这个问题五年多了，根据现实世界的反馈和要求完善我们的解决方案和迭代，”Umare 说。“机器学习领域已经很大，在传统公司内部也在增长。然而，我们认为增长潜力不受当前需求规模的限制，而是受我们能够提供的体验的限制，这就是为什么我们完全专注于客户成功和开源采用。这将在不久的将来带来收入增长。”

关于增长的话题，Union 计划在年底前将其 20 人的员工人数增加一倍，因为它专注于产品建设。Umare 没有关于 Union Cloud 兴趣或接受度的统计数据，但重申 Lyft、Spotify、丰田子公司 Woven Planet 以及生物技术和金融品牌等公司的“数千名”用户已经采用了 Flyte。
