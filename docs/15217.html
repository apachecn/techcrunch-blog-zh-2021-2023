在加密领域，有一个流行的格言叫做区块链三元悖论，指的是在区块链网络中同时实现三个理想属性的难度:安全性、可扩展性和去中心化。

三元悖论认为不可能将这三个属性都发挥到极致。随着区块链网络变得越来越安全，其可扩展性也变得越来越差；当它变得更加可扩展时，它变得不那么安全；随着它变得越来越分散，它变得越来越不安全，越来越不可扩展。

设计区块链系统时，必须在三个参数之间进行权衡，以达到特定用例可接受的平衡。

在方便食品领域，按需配送也存在类似的三重困境:

“按需交付三难困境”图片来源:阿里·艾哈迈德

它指的是平衡按需提供方便食品的三个最重要因素的挑战:速度、利润和负担能力。

对于交付预订商品的模型，不可能同时最大化所有三个因素。

这种“按需交付的三难困境”将继续困扰行业现有企业，除非他们能够超越现有的交付预订商品的模式。

例如，为了实现低交付成本以传递给消费者，或者换句话说，实现可负担性，交付系统不得不牺牲利润或速度。为了盈利，公司需要收取高额费用，使其无法负担，或者通过缓慢来改善其成本基础。要想快速发展，公司要么需要盈利，这意味着收取高额费用，要么放弃这些费用，牺牲盈利能力。

下面，我将提供每个权衡的例子，以及为什么在使用交付预订商品的现有模型时不可能绕过它们。

使用这种现有交付模式的在职人员包括:

  1. 送货机器人。
  2. 基于路线的冰淇淋和食品车。
  3. 交付应用程序。
  4. 快速商业。

送货机器人

图片来源:阿里·艾哈迈德

像 Starship，Nuro，Serve，Kiwi 和 Coco 这样的送货机器人公司通过自动化司机来降低成本。这种成本降低有助于他们实现盈利性和可承受性，但他们为此牺牲了速度。

机器人运送东西需要的时间比人长，当我们谈论移动缓慢的人行道机器人时，这一点尤为明显。

基于路线的卡车

图片来源:阿里·艾哈迈德

基于路线的冰淇淋和食品卡车已经存在了几十年，但像尖叫卡车和 Zing 这样的新公司正在试图使它们现代化。

然而，他们经营的是同样的预购商品的基本模式，因此只能实现盈利和负担能力；不是速度。

凭借基于路线的优势，它们把车停在中心位置，迫使消费者来到它们面前，排队购买商品，要么消费，要么拖回家。相当缓慢的经历。
