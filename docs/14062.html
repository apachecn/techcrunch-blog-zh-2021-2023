CircleCI 警告用户在 hack | TechCrunch 之后要转移“任何和所有的秘密”

原文：https://web.archive.org/web/https://techcrunch.com/2023/01/05/circleci-breach/

CircleCI 警告客户在黑客攻击后要轮换“任何和所有的秘密”

CircleCI 公司的开发产品受到软件工程师的欢迎，该公司敦促用户在公司系统遭到破坏后轮换他们的秘密。

总部位于旧金山的 DevOps 公司在周三晚些时候发布的一份咨询报告中表示，它目前正在调查这起安全事件，这是近年来最新的一起事件。

“我们想让你知道，我们目前正在调查一个安全事件，我们的调查正在进行中，”CircleCI 首席技术官 Rob Zuber 说。“在这一点上，我们确信在我们的系统中没有未授权的参与者；然而，出于谨慎，我们希望确保所有客户都采取一定的预防措施来保护您的数据。”

CircleCI 声称其技术被 100 多万软件工程师使用，建议用户轮换存储在 CircleCI 中的“任何和所有秘密”，包括存储在项目环境变量或上下文中的秘密。机密是用于连接和验证服务器的密码或私钥。

对于使用 API 令牌的项目，CircleCI 表示已经使这些令牌失效，用户将被要求替换它们。

2021 年，circle ci宣布了价值 1 亿美元的 F 系列，估值 17 亿美元，没有分享更多关于事件性质的信息，也没有回应 TechCrunch 的问题。

然而，该公司还建议用户审计他们的内部日志，查看 2022 年 12 月 21 日至 2023 年 1 月 4 日之间发生的未经授权的访问，这表明该公司的违规行为大约在两周前就开始了。12 月 21 日，该公司还宣布，它已经发布了该服务的可靠性更新，以解决潜在的“系统性问题”。

2019 年，CircleCI 在第三方供应商被入侵后遭遇数据泄露。这表明黑客破坏了用户数据，包括用户名和电子邮件地址，与 GitHub 和 Bitbucket 相关的用户名和电子邮件地址，以及用户 IP 地址。

11 月，CircleCI 表示也目睹了越来越多的网络钓鱼企图，即未经授权的行为者冒充 CircleCI 来获取用户在 GitHub 上的代码库。
