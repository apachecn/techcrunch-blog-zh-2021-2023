认识 Crowd.dev，这是一个开源的用户导向的发展平台，用于培养开发者社区| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/01/meet-crowd-dev-an-open-source-user-led-growth-platform-for-fostering-developer-communities/

社区主导的增长(CLG)已经成为一种受欢迎的推动业务的机制，因为公司努力培养一个由狂热用户组成的生态系统，有机地吸引新客户，作为数百万人的支持网络，并完全出于自己的意愿敲打公司的鼓。

Stripe、Slack、Canva、idea 和 Figma 等企业在各自社区的支持下大幅增长，这反过来又导致了一系列新技术的出现，这些技术致力于帮助这些企业利用他们的粉丝群，挖掘他们最大的拥护者，并保持 CLG 飞轮的旋转。投资者也注意到了这一点:仅在过去的一年里，我们就看到像 Commsor 这样的公司筹集了5000 万美元的 B 轮融资；公共休息室获得 5200 万美元；Threado 获得了一轮310 万美元的种子资金；最近，Talkbase 筹集了 200 万美元用于推动任何公司的用户导向型增长。

现在，另一家新公司进入了社区主导的增长领域，其方法与现有公司略有不同，专注于开发者社区，以开源为核心。

Crowd.dev 于 2021 年在柏林成立，汇集了来自无数开发者社区的数据，包括 GitHub、Discord、Slack、Twitter、dev 和 Hacker News，并在这些聚合数据的基础上提供分析和工作流自动化。

例如，一家开发工具公司可能希望更好地了解其用户，并与他们和他们的雇主建立关系，以改进他们的产品并找到更好的产品市场匹配。这可能包括在一个界面中收集和查看所有直接和间接的反馈，或者使用 Crowd.dev 的高级工具之一，如 Eagle Eye ，它依靠自然语言处理(NLP)来识别适合参与的社区讨论。

为了帮助事情更上一层楼，Crowd.dev 刚刚在 Seedcamp 和 Lightbird 牵头的种子前一轮融资中为€筹集了 220 万英镑(合 220 万美元)，潜在风险投资公司、天使投资公司和一些天使投资人也参与了进来。最重要的是，这家德国初创公司已经开源了其核心平台，这一举措有助于在日益拥挤的空间中脱颖而出。

但首先，值得考虑的是为什么专注于开发者的公司可能需要一个专用平台来引导他们社区主导的增长努力，因为现有公司已经可以用于任何用户社区——包括开发者。

垂直线

Crowd.dev 首席执行官兼联合创始人乔纳森·雷默(Jonathan Reimer)认为，“社区”一词有广泛的内涵，可以指从社交媒体影响者到在线学习小组的任何东西。最终，“一刀切”的方法是行不通的——一家专注于吸引开发者的公司可能需要不同的工具，而不是一家寻求吸引创造者或加密爱好者的公司。

“有关于社区的宣传，但也有关于使社区建设更容易的新工具的失望，”Reimer 向 TechCrunch 解释说。“我在以前的工作中尝试过[现有的]工具，但从未满意过，因为它们与我的用例不匹配。与 CRMs (客户关系管理软件)类似，我们相信社区软件领域将会出现垂直化。我们是开发人员领域的先锋。”

这种“垂直化”对于建立一个人们真正想要使用的平台是很重要的。以 Crowd.dev 为例，它的目标是创建一种产品，根据开发者社区数据建议用户可以采取的行动，这种方式的专业化使它能够更好地定制产品，并“建立更可靠的模型”，如 Reimer 所说，例如在检测反馈或评估情绪方面。

“同时为所有类型的社区实现这一目标将非常困难，”赖默说。“开发者社区有着惊人的相似性，尤其是对于开源社区，我们可以访问大量的历史培训数据。”

开源因素

开源社区长期以来在推动软件采用方面发挥了重要作用，这也是为什么越来越多的公司选择在开源许可下提供他们的产品。如果开发人员能够以最小的摩擦自己修补软件，贡献一些代码，甚至添加新功能，他们更倾向于在他们的工作场所使用该软件——因此，他们更倾向于说服他们的雇主，在开源产品的基础上为高级功能付费是值得的。这是 Crowd.dev 关注开源开发社区的主要驱动力，也是它开源自己平台的原因。

“我们相信，对于以开发者为中心的开源公司来说，一个必不可少的工具——就像社区管理一样——应该是开源本身，”Reimer 说。

过渡到开源平台可能还有其他好处。例如，寻求更大的透明度和对数据的控制的企业可以在自己的基础设施上托管 Crowd.dev，然后向 Crowd.dev 付费以解锁对无限用户和集成的访问。或者，公司可以选择为托管的 Crowd.dev 付费，除了更高级的企业计划之外，它还包括一个基本的免费层。

在其短暂的生命周期中，Crowd.dev 声称拥有一个相当令人印象深刻的客户名单，如 Linux 基金会和微软，在过去的八年里，这家公司已经越来越多地拥抱开源，而在之前的几年里，它对社区驱动软件的态度有些冷淡。

Reimer 说，微软使用 Crowd.dev 来运行 Flatcar Linux，这是一个用于容器工作负载的 Linux 发行版，它现在在 2021 年收购开发商 Kinvolk 后运行。

“他们使用 Crowd.dev 主要是为了分析社区成员的参与度，在 GitHub 上发现相关的观星者，并创建报告，”Reimer 说。

事实上，微软及其大型科技公司不会是典型的用户，因为 Crowd.dev 的大部分目标客户都是寻求增长的小公司。但这仍然表明了 Crowd.dev 迄今为止成功获得的思想份额，自今年 3 月以来，“数百家组织”加入了该公司的测试产品。

“我们 80%的用户是介于种子和 B 轮之间的公司，他们将社区视为他们的重要增长渠道之一，”Reimer 说。

该公司表示，银行中有 220 万€的新资金，计划在向公众提供之前，添加更多的应用程序和数据集成。
