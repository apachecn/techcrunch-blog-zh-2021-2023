编辑推文仍然是一个雷区，但 Twitter 的解决方案可能会奏效

原文：https://web.archive.org/web/https://techcrunch.com/2022/10/03/edited-tweets-is-a-minefield/

编辑推文仍然是一个雷区，但 Twitter 的解决方案可能会奏效

我一直非常反对经过编辑的推文，主要是因为技术原因。核心问题是 Twitter 不像其他社交媒体网站；与 Instagram 和脸书的帖子不同，嵌入式推文在许多方面推动了新闻。重新发布和转发推文是允许用户编辑的主要风险。让我来分析一些原因，为什么 Twitter 的推出可能具有挑战性，但仍然有效。

自推出以来，尤其是在如果没有 Twitter，特朗普的总统任期基本上不可能发生的时代，该平台已经成为任何人向观众提供任何信息的最快方式。这有积极的一面，但也意味着信息的传播范围更广。在一个假新闻横行的世界里，这是…一个挑战。

编辑推文的能力在技术上一直是微不足道的——你更新一个数据库，工作就完成了。数据库在更新信息的能力方面非常出色，尤其是如果你不关心编辑历史或(上帝禁止)不变性的话。

Twitter 是一个即时的平台:只要你点击“Tweet”，你所有的关注者都会在几秒钟内看到你刚刚写的内容。这使得 Twitter 有别于其他社交媒体。这也意味着从政策的角度来看，编辑是复杂的。

即时能够重新发布内容的概念是 Twitter 的一个强大功能，尽管人们对转发是否构成对观点的认可或仅仅是一种放大的看法不一，但一个想法可以在分享后几秒钟内被转发。这使得编辑推文的概念变得复杂，因为很难知道编辑实际上是什么意思。

如果你开始稍微思考一下经过编辑的推文的含义，你会提出一系列问题，所有这些问题都有两个或更多非常合理的解决方案——但每个解决方案都有自己的挑战。如果有人发了一条推文，你喜欢这条推文，一旦这条推文被编辑，你的“喜欢”会发生什么？如果编辑意味着你不再同意怎么办？或者，如果你的“喜欢”实际上触发了一个 Zapier 脚本，对这条推文做了一些事情——这条推文会发生什么？

想象一下，我写了一些关于我的本地咖啡店是多么不可思议的东西，他们转发了它，我把推文编辑成“我应该去星巴克而不是本地咖啡店”？

看起来 Twitter 正试图暂时限制对这个能力的访问，只有 Twitter 蓝色用户。

该平台还向其 API 用户推出了推文编辑，因此他们可以访问过去的推文，并使用 API 为启用该功能的帐户编辑推文。

看到 Twitter 平台上的用户和滥用者如何使用这些功能将会非常有趣。这有太多古怪的边缘案例；作为一名产品人员，以下几条让我出了一身冷汗:

  • 如果你报告了一条辱骂性的推文，但后来被编辑了，普通用户看不到了，该怎么办？
  • 如果你在一个网站上显示推文，但是你缓存了内容怎么办？你如何确保展示最新版本的推文？
  • 如果你嵌入了一条推文，作为发布者，你要承担一定程度的编辑责任。你如何触发对可能非常旧的推文的重新审核？
  • 如果你打开了一条推文，在你阅读它和点击转发之间，它被编辑了，那该怎么办？
  • 如果你使用了一个调度工具(比如 Buffer)来转发一些带有评论的东西，但原始推文被编辑了，会怎么样？如果你的评论不再有意义，应该如何处理？
  • 编辑过的推文是否应该重新插入时间线，让“最新的先”看到的用户看？
  • 如果我能回到过去，在 2015 年 1 月，建议唐纳德·川普竞选总统只是个玩笑，会怎么样？好吧，我确实这么做了，但是现在有了剪辑，造假就容易多了。不过，还是不好笑。
  • 大调和小调编辑应该有区别吗？比如，一个打字错误还是完全改变推文的内容？而且，在一个逗号可以造成世界差异的世界里，平台如何知道什么是主要的或次要的编辑？

我仍然很好奇启用编辑是否会导致更多的问题，但在新西兰、加拿大和澳大利亚的有限推广将变得非常有趣。滥用的可能性很大，但也许我们最终可以摆脱推文中那些意想不到的愚蠢的小错别字。
