谷歌消灭了 FLoC，代之以 Topics

原文：https://web.archive.org/web/https://techcrunch.com/2022/01/25/google-kills-off-floc-replaces-it-with-topics/amp/

图片来源:Christina Kilgour/Getty Images 提供的图片

FLoC (群组联合学习)，谷歌的一个有争议的项目，通过将用户分组为具有相似兴趣的用户群来取代基于兴趣的广告的 cookies，已经死亡。取而代之的是，谷歌今天宣布了一项新提议:主题。

这里的想法是，当你在网上移动时，你的浏览器会了解你的兴趣。它将保存你过去三周的浏览历史数据，截至目前，谷歌将主题数量限制在 300 个，并计划随着时间的推移延长这一限制。谷歌指出，这些话题将不包括任何敏感类别，如性别或种族。

为了找出你的兴趣，谷歌根据这 300 个话题中的一个对你访问的网站进行分类。对于以前没有分类的网站，浏览器中的轻量级机器学习算法将接管并根据域名提供估计的主题。

图片来源:谷歌

当你偶然发现一个支持用于广告目的的主题 API 的网站时，浏览器将分享你感兴趣的三个主题——过去三周的每一周——从你每周的前五个主题中随机选择。然后，该网站可以与广告合作伙伴分享这些信息，以决定向您展示哪些广告。理想情况下，这将为决定向你显示哪个广告提供一种更私人的方法——谷歌指出，它还为用户提供了比目前标准更大的控制和透明度。用户将能够查看和删除列表中的主题——也可以关闭整个主题 API。

“主题的设计是根据我们从早期 FLoC 试验中获得的经验设计的，”谷歌隐私沙盒负责人 Ben Galbraith 在今天宣布之前的新闻发布会上说。“这导致了社区的大量反馈，我相信您也知道。因此，Topics 取代了我们的 FLoC 提案，我想强调的是，分享提案、进行试验、收集反馈，然后迭代设计的整个过程——这是我们希望用于沙盒的整个开放开发过程，并真正展示了该过程的预期效果。”

图片来源:谷歌

Galbraith 指出，谷歌已经与许多方面进行了交谈，以收集对这一新提议的反馈，但今天标志着该公司与生态系统合作进程的开始。其他浏览器厂商是否有兴趣加入主题 API，还有待观察。由于他们都很快对 FLoC 冷淡了，我有点怀疑他们是否想采用 Topics API，但是观察生态系统的反应会很有趣。

同样值得注意的是，对于广告商来说，主题只是决定向特定用户展示哪个广告的一个潜在信号。在某些方面，它只是成为他们的另一个信号，可以用关于用户当前正在阅读的文章的数据或关于用户的其他上下文数据来增强。

计划是在本季度末开始测试 Topics API，但为了让这个球滚动起来，谷歌今天还发布了一个技术解释器，它对该提议的细节进行了更深入的研究。
