Kubernetes 开发平台 Okteto 在 A 轮 TechCrunch 融资 1500 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/24/kubernetes-development-platform-okteto-raises-15m-series-a/

一家让开发人员更容易快速启动基于 Kubernetes 的开发环境以加快开发进程的初创公司 Okteto ，今天宣布筹集了 1500 万美元的首轮融资。

这轮融资由两家适马风险投资公司牵头，现有投资者 Haystack、Root Ventures 和不相关的风险投资公司也参与其中，还有一些个人投资者，包括 LaunchDarkly 创始人兼首席技术官约翰·科杜马尔(John Kodumal)、复制创始人格兰特·米勒和马克·坎贝尔(Marc Campbell)、FingerprintJS 创始人兼首席执行官丹·平托(Dan Pinto)、Bitnami 创始人埃丽卡·布雷西亚(Erica Brescia)和 Mesosphere 创始人弗洛里安·莱贝特(Florian Leibert)。

该公司参加了 Y Combinator 的 2019 年冬季课程，已经有像 Monday.com 这样的公司，在客户中秘密启动和复制。

图片来源: Okteto

正如 Okteto 联合创始人兼首席执行官 Ramiro Berrelleza 告诉我的那样，这项服务的想法来自于他在微软和 Atlassian 等公司的经历。在那里，他注意到 Kubernetes 和微服务让运营团队的生活变得更加轻松，但对开发人员来说却不一定。

“你有所有这些非常酷的工具，用于生产、部署、容器、Kubernetes、微服务——但我从我作为开发人员的角度看到的是，‘嘿，所有这些都很棒，我明白为什么它对生产非常有用——这一切都很有意义——但这让我作为开发人员的生活变得更加艰难，’”他说，将今天的开发实践与 Ruby on Rails 或他在 Atlassian 构建的各种服务器应用程序进行了比较。当 Berrelleza 会见他的联合创始人 Pablo Chico de Guzman(首席技术官)和 Ramon Lamana(首席技术官)时，他们意识到他们都面临着完全相同的问题，前者曾在 Docker 工作，后者曾在 Atlassian 工作。就像经常发生的情况一样，这给了团队一种动力去尝试一劳永逸地解决这个问题。

图片来源: Okteto

Okteto 背后的核心思想是开发环境应该和生产环境一模一样。“第一个关键是，我们需要让开发人员获得尽可能接近生产的东西，而不是等待部署和经历所有构建过程的复杂性，”他说。

通过 Okteto，开发人员可以在 Kubernetes 集群上运行远程开发环境，并拥有所有将在他们的生产环境中运行的微服务、数据库和其他服务，一旦他们完成工作，他们就可以轻松地关闭环境。

Berrelleza 指出，该公司的部分理念是确保其服务与开发人员的现有工作流程相集成。因此，如果已经有一个 Docker Compose 文件来编译生产环境，Okteto 也可以用它来建立开发环境(或者开发人员也可以编写他们自己的 okteto.yml 清单)。

该团队正在寻求扩展它的特性集，不仅仅是设置开发环境，还包括编写测试的工具，而且还要将其短暂的环境扩展到更多的用例。例如，去年，它发布了公司所谓的“预览环境”，客户可以用它来向自己的客户、设计师或其他利益相关者展示项目的当前状态，这些人通常不会参与软件开发过程。“软件开发过程不仅仅是写代码。你越能在周期的早期引入这些人，对每个人都越好，”Berrelleza 说。

Okteto 不是唯一一家解决这个问题的公司。就在昨天， Signadot 发布了其服务的公开测试版，并且它也承诺为开发者提供更快的反馈循环，这要归功于它能够快速启动类似生产的测试环境。从技术角度来看，两家公司对此采取的方法略有不同，但它们的目标非常相似。
