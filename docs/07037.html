macOS 将很快默认屏蔽未知的 USB-C 设备

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/07/macos-usb-accessory-security/

macOS 将很快默认屏蔽未知的 USB-C 配件

苹果即将推出的 macOS 13 Ventura 中的一项新安全功能将自动阻止新的 USB-C 设备与操作系统进行通信，直到该配件获得用户的批准。

苹果在其发布说明中放弃了新安全功能的细节，这似乎是为了保护运行其定制的 M1 或 M2 芯片的新款苹果笔记本电脑免受潜在恶意附件的攻击。

根据苹果的描述，该功能将默认启用，并要求用户在 USB-C 配件可以与操作系统对话之前批准它——本质上是一个屏幕弹出窗口，要求用户许可。苹果公司表示，这不适用于电源适配器、独立显示器和经批准的集线器的连接——即使你不批准该配件，设备仍然可以充电。苹果公司表示，当更新到新的 macOS 软件时，已经连接的配件将自动工作。

默认限制 USB-C 设备的举动是对一系列不断演变的威胁的默认，这些威胁对苹果用户构成了风险。研究人员已经证明，在看起来很正常的充电电缆中隐藏恶意的植入物是可能的，可以劫持电脑到笔记本电脑的 USB 设备，这些设备被设计成永久烧毁电脑的内部。

新的 macOS 功能也与苹果在 iOS 12 中引入的 USB 限制模式几乎相同，该模式可以防止未经授权的配件和电缆在未经所有者允许的情况下访问 iphone 和 iPads 上的数据，而此时执法部门正在使用电话破解设备来规避苹果的设备安全。

以下是发行说明中的完整描述(最早由 The Verge 发现):

在采用 Apple silicon 的便携式 Mac 电脑上，新的 USB 和 Thunderbolt 附件需要用户批准，然后该附件才能与 macOS 通信，以便直接连接到 USB-C 端口。这不适用于电源适配器、独立显示器或与认可集线器的连接。如果您选取“不允许”，设备仍然可以充电。

您可以在系统设置>安全性和隐私>安全性中更改安全性配置。初始配置要求新的附件。配置辅助功能开关控件会将策略设置为始终允许使用附件。经批准的设备可以连接到锁定的 Mac 长达三天。

自动允许在 macOS 早期版本的软件更新过程中附加附件。重新启动 Mac 之前连接的新附件可能会枚举并运行，但在连接到解锁的 Mac 并得到明确批准之前，不会被记住。
