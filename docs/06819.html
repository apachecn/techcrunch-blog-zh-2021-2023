Mintlify 使用人工智能从代码中生成文档

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/30/mintlify-taps-ai-to-automatically-generate-documentation-from-code/amp/

图片来源:Dmitry Bairachnyi/Getty Images

开发软件文档任务自动化软件的初创公司 Mintlify 今天宣布，它在贝恩资本风险投资公司(Bain Capital Ventures)领投的种子轮融资中筹集了 280 万美元，TwentyTwo Ventures 和 Sourcegraph 的联合创始人 Quinn Slack 也参与了融资。首席执行官汪涵表示，收益将用于产品开发，并在年底前将 Mintlify 的核心三人团队增加一倍。

总部位于纽约州伊萨卡的 Mintlify 由汪涵和哈恩比·李于 2021 年共同创立，两人都是软件工程师。王之前与人合作推出了 Foodful，一家开发基于云的奶牛监测系统的初创公司，以及 Pe ple，一个在线客户社区平台，该平台于 2021 年初被 Tribe 收购。在短暂加入 Duolingo 担任工程师之前，Lee 是 Pe ple 的联合创始人。

王说，Mintlify 的想法来自于他和李在软件开发方面的经验，其中包括处理不总是完整或最高质量的文档。他们的观察结果与 2017 年 GitHub 的一项调查一致，该调查发现，93%的开发人员认为不完整或过时的文档是一个普遍存在的问题。

王在电子邮件采访中告诉 TechCrunch:“我们作为软件工程师在从初创公司到大型科技公司的各个阶段工作过，发现他们都遭受着糟糕的文档记录的困扰，如果这些文档记录真的存在的话。”。“文档是初级工程师和那些进入新代码库的人的生命线。这有助于高级开发人员在将来向他人解释他们的代码时节省时间。对于面向公众的开源产品，文档对用户的采用有直接影响。”

图片来源: Mintlify

Mintlify 的目标是通过自动化解决文档方面的挑战，特别是自动生成文档。该公司的平台读取代码并创建文档来解释代码，利用包括自然语言处理和网络抓取在内的技术。

王拒绝透露更多关于 Mintlify 的技术基础，但用今天的人工智能技术从代码中生成文档是完全可能的。事实证明，Mintlify 有几个竞争对手采取了类似的方法，包括 Documatic，其人工智能系统除了文档外，还会自动从代码中生成更改日志和解释。

王断言，Mintlify 提供了比其竞争对手“显著”更高质量的结果，并且不像某些人那样，不强迫开发者在云服务上托管文档。“Mintlify 的使命是通过将持续文档开发成软件团队的标准实践来解决文档腐败问题，”王补充道。“当工程经理积极寻求更好的文档实践的解决方案时…这就是我们介入的时候。”

除了生成文档，Mintlify 还定期扫描“陈旧”的文档，并检测用户如何处理文档以提高其可读性。(王强调，该平台不存储代码，并对所有静态和传输中的用户数据进行加密。)Mintlify，对个人开发者免费，也与包括 Slack、Dropbox 和 GitHub 在内的现有系统集成，用于自动化任务管理和开发工作流。

据王介绍，Mintlify 的免费计划自 1 月份推出以来，用户数量每周增长 20%。随着用户群现在超过 6000 个活跃账户，该公司计划将重点转移到面向企业客户的优质产品上。

“疫情标准化了分散的异步工作环境。这使得高质量的文档成为实现高效沟通、入职和产品开发的关键，”王说。“我们向工作流程自动化领域的扩张，正通过将我们现有的粉丝群作为冠军来瞄准工程经理，从而应对这一挑战。”
