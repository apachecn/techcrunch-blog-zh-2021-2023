DuckDuckGo 删除了 Microsoft 跟踪脚本的例外

原文：https://web.archive.org/web/https://techcrunch.com/2022/08/05/duckduckgo-microsoft-tracking-scripts/

几个月前，以隐私为中心的搜索老将 DuckDuckGo 遭遇了跟踪争议，该公司宣布，它已经能够与搜索联盟合作伙伴微软修改条款，此前这意味着其移动浏览器和浏览器扩展无法阻止微软脚本在第三方网站上发出的广告请求。

在博客文章中承诺“为 DuckDuckGo 网络跟踪保护提供更多隐私和透明度”，创始人兼首席执行官 Gabe Weinberg 写道:“下周，我们将扩展我们阻止在网站上加载的第三方跟踪脚本，以包括我们的浏览应用程序(iOS 和 Android)和浏览器扩展(Chrome、Firefox、Safari、Edge 和 Opera)中的微软脚本，测试版应用程序将在下个月推出。”

“这扩展了我们的第三方追踪器加载保护，它阻止来自脸书、谷歌和其他公司的已识别追踪脚本加载到第三方网站上，现在包括第三方微软追踪脚本。默认情况下，大多数其他流行的浏览器都不提供这种网络跟踪保护，它位于许多其他 DuckDuckGo 保护之上，”他补充说。

DDG 声称，默认情况下，大多数其他流行的浏览器都不提供这种第三方跟踪器加载保护。

“大多数浏览器的默认跟踪保护侧重于 cookie 和指纹保护，仅在第三方跟踪脚本加载到您的浏览器后对其进行限制。不幸的是，这种级别的保护使得像您的 IP 地址和随加载请求发送的其他标识符这样的信息容易被分析。我们的第三方追踪器加载保护有助于解决这一漏洞，通过首先阻止大多数第三方追踪器加载，提供了明显更多的保护，”Weinberg 在博客中写道。

“以前，由于使用 Bing 作为私人搜索结果来源的相关政策要求，我们在如何对 Microsoft 跟踪脚本应用第三方跟踪器加载保护方面受到限制。我们很高兴这种情况不再发生了。我们过去没有，现在也没有与任何其他公司有任何类似的限制。”

“微软脚本从未嵌入我们的搜索引擎或应用程序中，它们不会跟踪你，”他补充道。“网站出于自己的目的插入这些脚本，因此它们从未向 DuckDuckGo 发送任何信息。由于我们已经通过其他网络跟踪保护措施限制了微软跟踪，如在我们的浏览器中阻止微软的第三方 cookies，这次更新意味着我们现在比大多数其他浏览器更多地阻止跟踪器。

当被问及 DDG 是否会公布其与微软的新合同，或者它是否仍然受 NDA 的约束时，Weinberg 说:“其他什么都没有改变，我们没有其他信息可以分享。”

为 DDG 的搜索供应商开拓的市场是在5 月 通过隐私研究员扎克·爱德华兹进行的独立审计获得的。

当时，DDG 承认存在异常行为，但表示它基本上别无选择，只能接受微软的条款，尽管它也表示对这一限制感到不满，并希望未来能够取消这一限制。

当被问及这场争议引发的公众关注是否有助于说服这家科技巨头放松对其在非微软网站上屏蔽微软广告脚本的限制时，DDG 让我们回到微软。

当我们向这家科技巨头提出同样的问题时，一位女发言人告诉我们:

微软有适当的政策，以确保我们平衡我们的出版商的需求与我们的广告客户的需求，以准确跟踪我们网络上的转换。我们一直在与 DuckDuckGo 合作，以了解该政策的含义，我们很高兴达成了解决这些问题的解决方案。

在今天宣布的以透明度为重点的步骤中，DDG 表示将公布其追踪器保护列表——可在 GitHub 上获得此处——尽管该公司告诉我们该信息之前就可获得，但暗示现在更容易找到。

尽管 DDG 阻止微软追踪请求的能力有所增强，但仍有一些情况下，微软广告脚本在默认情况下没有被 DDG 的工具阻止——这与广告商用来追踪转化(即确定广告点击是否真的导致了购买)的过程有关。

“为了评估 DuckDuckGo 上的广告是否有效，广告主想知道他们的广告点击是否转化为购买(转化)。为了在微软广告中看到这一点，他们使用来自 bat.bing.com 领域的微软脚本，”Weinberg 在博客中解释道。“目前，如果广告商想要检测在 DuckDuckGo 上显示的他们自己的广告的转换，第三方跟踪器加载保护将不会阻止 bat.bing.com 请求在 DuckDuckGo 广告点击之后加载到广告商的网站上，但是这些请求在所有其他上下文中被阻止。对于任何想避免这种情况的人来说，可以在 DuckDuckGo 搜索设置中禁用广告。

DDG 表示，它希望在广告转换跟踪方面进一步保护用户隐私——但承认这不会很快发生。Weinberg 在博客中写道,“最终”它希望能够通过迁移到一个新的私下评估广告效果的架构来取代当前的广告转换检查流程。

“为了最终取代对 bat.bing.com 评估广告效果的依赖，我们已经开始研究一种私人广告转换的架构，这种架构可以在外部被验证为非简档，”他说。

DDG 绝不是唯一的例子。在整个行业，各种各样的举措正在进行，以发展/重新思考广告技术基础设施，以应对隐私反弹——以及与个人跟踪相关的上升 监管风险——如谷歌多年来推动用替代广告技术栈(又名其“隐私沙箱”提案)取代 Chrome 中对跟踪 cookies 的支持；这仍然是[延迟的正在进行的工作)。

“DuckDuckGo 并不是唯一试图解决这个问题的人；Safari 正在研究隐私点击测量(PCM )，Firefox 正在研究可互操作隐私归属(IPA) 。我们希望这些努力能够帮助推动整个数字广告行业将隐私作为默认设置，”Weinberg 补充道。“我们认为这项工作很重要，因为这意味着我们可以改善无数公司赖以提供免费服务的基于广告的商业模式，让它变得更加私密，而不是完全抛弃它。”

当被问及开发这样一个基础设施的时间表时，他说:“我们现在没有时间表可以分享，但这不是一个迫在眉睫的公告。”

尽管 DDG 声称通过其浏览器观看广告是“匿名的”，但其广告披露页面证实，它将一些个人数据(IP 地址和用户字符串)传递给其广告合作伙伴微软，用于“会计目的”(也就是“向广告商收费，并为适当的点击向我们支付费用，这包括检测不适当的点击”，正如温伯格所说)。

“根据我们的广告页面，微软已经承诺】“当你点击出现在 DuckDuckGo 上的微软提供的广告时，微软广告不会将你的广告点击行为与用户资料相关联。当被问及他从微软那里得到了什么保证，保证用于广告转换的用户数据最终不会被重新用于更广泛的个人跟踪和分析时，他说:“除了出于会计目的，它也不会存储或共享这些信息。”。

在与 TechCrunch 的交锋中，DDG 也反复强调其政策声明微软不会将这些数据与行为特征联系起来(或者，事实上，分享用户的实际 IP 地址等)。

然而，Weinberg 承认，DDG 对 数据通过后发生的事情的控制力是有限的——例如，考虑到 adtech 生态系统对共享(和同步)假名标识符(例如标识符的哈希)的偏好，以便数字活动仍然可以链接到个人资料，比如通过第三方数据处理器/富集器链的几次跳跃，从而消除早期的隐私屏蔽… 所以，TL；dr，在一个为无处不在的监控而设计的广告生态系统中运行时，试图保护你的用户的隐私不受窥探的第三方的侵犯(并允许到处蔓延)仍然是一场大规模的交火。

“通过广告技术生态系统保持匿名是一个不同的故事，因为一旦有人点击一个网站(无论他们是否通过 DuckDuckGo 搜索进入)，他们就会受到网站所有者的隐私政策和相关做法的影响，”温伯格承认。“在我们的浏览器中，我们试图通过网络隐私保护来限制这种情况，但我们无法控制网站所有者(“第一方”)的行为，这可能会与广告技术生态系统中的第三方共享数据。”

“广告披露页面明确表示查看广告是匿名的，并进一步涵盖了广告点击，微软承诺不会在广告点击时对用户进行分析，包括他们或其他人的任何行为分析。这一承诺包括不将这些数据传递给任何人，”DDG 还声称。

它补充说:“我们的隐私政策规定，查看所有搜索结果(包括广告)都是匿名的，微软广告(或任何其他人)当时没有获得任何可以解除用户搜索匿名的东西(包括完整的 IP 地址)，无法将单个搜索与单个搜索联系起来或一起纳入搜索历史。”

在该公司今天强调的进一步发展中，DDG 表示，它已经更新了在其应用程序和扩展中显示的隐私仪表盘，以显示关于第三方请求的“更多信息”，根据其博客帖子。

“使用更新的隐私仪表板，用户可以看到哪些第三方请求被阻止加载，哪些第三方请求已加载，以及两者的原因，”温伯格写道。

它还重新推出了其帮助页面——承诺修改后的内容提供“我们跨平台提供的所有网络跟踪保护的全面解释”。

“如果用户想了解我们在他们使用的平台上提供的不同类型的网络隐私保护，他们现在有一个地方可以看。该页面还解释了如何根据每个平台上的技术可能性提供不同的网络跟踪保护，以及我们产品路线图的这一部分正在开发什么，”它的博客帖子建议道。
