通过竞争对手 RideCo 筹集 1600 万美元来扩展按需运输软件——TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/17/via-competitor-rideco-raises-16m-to-expand-on-demand-transit-software/amp/

图片来源: RideCo

提供城市按需交通技术的加拿大公司 RideCo 完成了 1580 万美元(2000 万加元)的首轮融资。RideCo 的联合创始人兼首席执行官 Prem Gururajan 表示，新资金将用于进一步开发其产品，这将涉及大幅增长其工程团队，以及客户服务，销售和营销团队。

该公司的最大竞争对手是 Via ，它运行的是软件即服务的商业模式。与 Via 类似，RideCo 通过收购 Remix 也提供了交通规划技术，RideCo 向交通机构和车队运营商提供底层软件和应用程序，以便他们反过来向 RideCo 的最大客户城市提供按需交通选择。RideCo 还为寻找动态方式接送员工的公司提供这项服务。

该软件为公交车或其他类型的车辆创建动态路线，根据需求指引它们前往下一个上下车区。圣安东尼奥、洛杉矶以及最近的拉斯维加斯等城市已经使用这些功能来解决第一英里和最后一英里交通的接入问题，这是大多数主要由固定路线交通系统运营的城市的常见问题。

“以圣安东尼奥为例，”Gururajan 告诉 TechCrunch。“他们在城市的某些区域拥有良好的公交服务，因为他们的密度很高，而城市的其他许多地方密度很低，他们有第一英里和最后一英里的接入问题。固定路线的交通在那里根本行不通。”

有活力的班车和公共汽车实际上可以到达人们预定乘车的地方和他们想下车的地方，这可以解决这些问题，并在城市中创造更多的流动性。

“所以你现在有一个城市，在我们的服务下，他们可以在中低密度区域使用动态交通，进入高频走廊，那里有轻轨交通或快速公交专用道，他们有需求给公交车加油。”

乘客可以在应用程序上查看他们的车辆，并可以提前、实时甚至定期预订乘车服务，例如工作日的每天上午 9 点。

RideCo 的首轮融资由 Eclipse Ventures 牵头，这是该公司成立七年来首次筹集机构投资。Gururajan 将此归因于几个因素，第一个因素是 RideCo 在最初几年致力于构建其核心技术，并与一些 beta 客户一起测试，以解决某些用例。

当 RideCo 准备将其产品推向市场时，它意识到产品还不太适合市场。

“当我们进入市场时，我们发现有一些客户对此感兴趣，但作为一个整体，市场还没有准备好，”他说。“他们仍然担心任何由应用程序驱动的交通工具，但这一切在 2018 年底左右开始改变。我们有几个大城市的签约，比如圣安东尼奥，在为机构节省资金和提高居民流动性方面，他们都出类拔萃。”

从那时起，RideCo 已经进入了美国和加拿大的城市，从卡尔加里和休斯顿这样的大城市到安大略省科伯格这样的小镇。Gururajan 说，疫情也加速了 RideCo 的增长，因为运输机构开始考虑新的方法来适应乘客不断变化的需求模式以及对现代化和响应网络的渴望。

在过去的一年中，RideCo 的客户群增长了两倍，尽管该公司没有说明增长的基础是什么。

RideCo 正在增加其按需交通服务，而其主要竞争对手 Via 正在使其服务多样化，并更多地将自己定位为整体交通技术提供商，而不是按需交通提供商。这两家公司成立于同一年，但威盛的市场份额却远远超过了它的加拿大同行。去年，这两家公司因技术相似而相互提起专利侵权诉讼。

RideCo 或许仍能在这个新兴产业中占有一些市场份额。在美国和加拿大，近 16%的交通机构已经提供按需交通，随着其他形式的共享移动，这一数字预计将继续上升。该公司必须保持目前的势头。
