AWS 使其图形数据库服务 Neptune 成为无服务器的

近五年前，亚马逊网络服务公司(AWS)推出了 Neptune，这是一种运行需要图形数据库来存储和查询连接数据集的应用程序的服务。现在，为了跟上无服务器的趋势，AWS 正在通过 Amazon Neptune Serverless 扩展产品，这是 Neptune 的一个无服务器选项，可以自动扩展以支持可变的图形数据库工作负载。

与传统数据库不同，图数据库存储节点和关系，而不是表、列和文档。开发人员构建应用程序来跟踪连接的数据点之间的关系，使用图形数据库来了解整个数据集中的关系；图形数据库用例包括接触追踪、欺诈检测、药物发现甚至网络安全。

当然，图形数据库是强大的。但是它们在处理开销方面也是不可预测的。通常，图形数据库需要开发团队持续监控和重新配置计算能力，以保持良好的性能。

Amazon Neptune Serverless 表面上通过自主供应、缩放和管理图形数据库实例的集群来解决这个问题。据 AWS 数据库、分析和机器学习副总裁 Swami Sivasubramanian 称，Neptune Serverless 支持与亚马逊 Neptune 相同的图形查询语言，客户只为他们使用的应用程序付费。

“客户要求我们处理与管理容量和优化成本和性能相关的繁重工作，”Sivasubramanian 在一份新闻稿中说。“现在，有了 Amazon Neptune 无服务器，客户就有了一个图形数据库，它可以自动调配和无缝扩展集群，以提供恰到好处的容量来满足需求。”

从今天起，美国东部(俄亥俄州)、美国东部(弗吉尼亚州北部)、美国西部(加利福尼亚州北部)、美国西部(俄勒冈州)、亚太地区(东京)、欧洲(爱尔兰)和欧洲(伦敦)服务器地区运行 Neptune 的 AWS 客户可以使用 Neptune Serverless。亚马逊表示，它将在未来进入更多地区。

无服务器计算抽象了管理服务器容量的复杂性，是软件开发的一个增长趋势。根据 2020 年的一项调查，50%的 AWS 用户表示他们正在使用某种程度的无服务器功能。CB Insights 估计2021 年，无服务器市场价值 77 亿美元，高于 2016 年的 19 亿美元。

AWS 最后一次大规模扩展其无服务器产品组合是在 4 月，当时它推出了亚马逊 Aurora 无服务器 V2，其无服务器数据库服务，以及 SageMaker 无服务器推理，这是一种运行人工智能系统的解决方案，不需要配置底层基础设施。7 月份发布了几款新的无服务器分析产品，包括 Amazon EMR、Amazon Managed Streaming for Apache Kafka 和 Amazon Redshift。
