DX 退出隐形模式

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/16/dx-exits-stealth-mode-looking-to-help-developer-teams-be-more-productive/

尽管开发人员深入参与了科技公司的产品发布和软件升级，但大多数公司高管对这些团队实际上是如何工作的，或者如何提高他们的生产力知之甚少。 DX 认为已经破解了代码。

早在 2020 年初，DX 联合创始人阿比·诺达(Abi Noda)在 GitHub 担任产品经理，他的任务是找出是什么拖慢了工程团队的速度。他得到了一组从交付时间到部署的指标，但没有一个指标提供了对实际影响或推动这些数字的洞察。他决定耍流氓，开始面试开发商。模式很快出现了。

野田告诉 TechCrunch:“我们对帮助产品更快上市所需的东西的假设，与团队和开发人员所说的阻碍他们的东西大相径庭。”"即使是团队也并不总是意识到他们自己的问题和领导力."

野田决定在 2020 年末离开 GitHub，试图建立一个更好的系统来识别和修复开发团队内部的摩擦点。他让 Greyson Junggren 帮助推出 DX，他在 GitPrime 等开发商物流公司有多年的上市经验。

该平台帮助公司识别开发团队问题，提出基于研究的解决方案，并跟踪进展。DX 现在正悄然崛起，拥有来自 Chime 和 Upwork 等公司的 5000 名开发团队。

当野田开始构建 DX 时，他希望确保它能从其他开发人员生产力工具中脱颖而出，并更多地关注开发人员自己的感性数据，如发布压力或实际期限，而不仅仅是输出数字。

离开 GitHub 后，他联系了该领域的两位顶级研究人员:Margaret-Anne Storey，space 的合著者，一篇关于 DX 正在寻求解决的同一主题的知名论文，以及米凯拉·格雷勒，他曾在包括微软和 Wix 在内的公司从事这方面的工作。经过一年的挖掘，他们确定了这些团队最常见的 40 个摩擦点，从糟糕的跨团队协作到不当的文档记录。

“我们通过显影剂的镜头来测量它们，”野田说。“其他解决方案大多着眼于正在发生的事情的更热门的数据。我们衡量开发人员的看法，这就找到了真正影响团队并使他们慢下来的根本原因。”

Junggren 说，该平台还为用户建议潜在的解决方案，并帮助团队将他们的生产力与其他市场竞争对手进行比较。

野田说，这家 10 人创业公司是盈利的，今年有望增长 5 倍。他拒绝透露收入细节，值得注意的是，该公司从 2021 年第四季度开始销售其产品。DX 已经从支持者那里筹集了大约 200 万美元的资金，这些支持者包括序言风险投资公司和行业高管 Nat Friedman，GitHub 的前首席执行官和 Jason Warner，GitHub 的前首席技术官。

“DX 正在建立一种真正新颖的方法，远远超出了分析或拉动，”序言的普通合伙人法鲁克·阿巴西说。"他们正在以咨询的方式将客户的流程自动化."

既然这家初创公司已经从隐身状态中崛起，野田表示，今年剩下的时间将专注于其走向市场战略和招聘——它希望抢走一些最近被解雇的技术人才。

“DX 表面上的见解对领导人至关重要，”野田说。“开发商为什么要走？为什么团队效率更低？DX 在组织中灌输了一种流程，在这种流程中，当地团队不断进行对话并采取行动。”
