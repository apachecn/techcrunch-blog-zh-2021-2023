开放式银行初创公司 Finverse 希望建立亚太地区的 Plaid TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/26/open-banking-startup-finverse-wants-to-build-the-asia-pacific-regions-plaid/

总部设在香港的 Finverse 公司雄心勃勃的目标是在整个亚太地区实现开放式银行业务。这家初创公司最近摆脱了秘密模式，获得了 180 万美元的种子资金，目前在四个市场(香港、菲律宾、新加坡和越南)开展业务，与 30 家银行建立了联系。创始人兼首席执行官夏羽·乐斯福尔告诉 TechCrunch，Finverse 计划每个季度在一个新市场推出，目标是覆盖每个地方约 75%的消费者和中小企业银行。

Finverse 种子轮的参与者包括 Febe Ventures、金门风险投资公司、SixThirty、Venturra 和天使投资人。

Finverse 是一批开发 API 的金融科技公司之一，这些 API 允许更容易地共享金融数据。最突出的例子包括美国的格子和欧洲的 Tink 和Truelayer(Finverse 的种子资金包括 true layer 员工的天使投资)。

在 2020 年开始 Finverse 之前，Lesaffre 是 NerdWallet 金融数据集成的高级产品经理，与 Plaid 和 legacy player Yodlee 等账户聚合 API 合作。

Lesaffre 说，格纹赢得了美国市场，因为它可靠，对开发者友好。它没有提供像 Yodlee 那样多的数据覆盖面，但“它所做的是一个非常集中的数据集，非常好，非常容易建立。我从 NerdWallet 最终学到的是，坏数据真的比没有数据更糟糕。”

Finverse 希望通过构建可靠的 API 和数据集成为亚太地区做同样的事情。“在核心上，我们基本上是一个基于同意的数据管道，消费者允许 Finverse 连接到他们的账户，并与另一家金融科技或金融机构分享，”Lesaffre 说。

这可能包括有关账户、余额、交易历史和银行对账单的信息。访问这些数据可以让金融机构了解消费者的资产和负债情况，并可以用来进行收入估计、信用检查和评估偿还能力等工作。

Lesaffre 表示，Finverse 的早期采用者大多是金融科技初创公司，包括中小企业贷款提供商和先买后付服务的组合。

Finverse 的 API 可以用于广泛的用例，但其当前的大多数潜在客户都专注于消费者或中小企业贷款。他们中的许多人希望从要求申请人上传文件的大量手动流程过渡到只需一分钟的数字化信贷决策。

Finverse 目前专注于银行消费者，或拥有传统银行账户和信用记录的人，但随着时间的推移，它也计划加入数字钱包、新银行和其他不太传统的机构。随着越来越多的亚洲人开始使用电子钱包、投资应用和在线银行账户，未来的用例包括金融跟踪。

“如果你是一家较小的数字银行，你知道你的许多客户将在一家较大的银行有另一个主要账户，所以许多较小的银行非常希望能够全面了解他们的消费者，”Lesaffre 说。"一种方法是让消费者在一个地方追踪他们所有的账户."

Finverse 的 API 的另一个用例是跨境支付验证、合规和 KYC。

其他专注于东南亚的开放式银行初创公司包括 Brankas 和 Finantier。Lesaffre 表示，Finverse 的方法不同，因为它针对的是整个亚太地区，而不是专注于特定市场。其新的资金将用于发展其工程和业务开发团队。
