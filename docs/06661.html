GitHub 的人工智能编码工具将对学生免费开放

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/24/copilot-githubs-ai-powered-coding-tool-will-become-generally-available-this-summer/

去年 6 月，微软旗下的 GitHub 和 OpenAI 推出了 Copilot ，这是一项为微软 Visual Studio 等开发环境中的整行代码提供建议的服务。Copilot 是一个可下载的扩展，由一个名为 Codex 的人工智能模型提供支持，该模型在数十亿行公共代码上进行训练，以根据现有代码的上下文建议额外的代码行和功能。Copilot 还可以利用其知识库和当前上下文，针对开发人员想要完成的内容的描述提出一种方法或解决方案(例如，“Say hello world”)。

虽然 Copilot 以前是在技术预览中提供的，但微软在 Build 2022 上宣布，它将从今年夏天开始全面上市。Copilot 也将免费提供给学生以及“经过验证的”开源贡献者。关于后一点，GitHub 表示以后会分享更多内容。

Copilot 的体验不会随着上市而有太大变化。像以前一样，开发人员将能够循环浏览 Python、JavaScript、TypeScript、Ruby、Go 和许多其他编程语言的建议，并接受、拒绝或手动编辑它们。Copilot 将适应开发人员进行的编辑，将特定的编码风格与自动填充样板文件或重复的代码模式相匹配，并推荐与实现代码相匹配的单元测试。

除了 Visual Studio 代码，Copilot 扩展还将用于 Noevim 和 JetBrains，或者在 GitHub 代码空间的云中。

Copilot 的一个新功能是 Copilot Explain，它可以将代码翻译成自然语言描述。这是一个研究项目，目标是帮助开发新手或那些不熟悉代码库的人。

GitHub 产品副总裁 Ryan J. Salva 在接受电子邮件采访时告诉 TechCrunch，“今年早些时候，我们推出了 Copilot Labs，这是一个独立的 Copilot 扩展，作为机器学习实验应用的试验场，可以改善开发人员的体验。“作为 Copilot 实验室的一部分，我们推出了‘解释代码’和‘翻译代码’。这项工作符合我们正在测试的实验能力的类别，它让你一窥可能性，并让我们探索用例。也许通过“解释这段代码”，开发人员正在考虑一个不熟悉的代码库，并希望快速了解发生了什么。这项功能可以让您突出显示一段代码，并让 Copilot 用通俗易懂的语言解释它。再说一遍，Copilot Labs 本质上是实验性的，所以可能会出问题。实验室实验可能会也可能不会发展成为 Copilot 的永久功能。"

Copilot 的新功能 Copilot Explain 可以将代码翻译成自然语言解释。图片来源:副驾驶

由于人工智能模型的复杂性，副驾驶仍然是一个不完美的系统。GitHub 警告说，它可能会产生不安全的编码模式、错误和对过时 API 的引用，或者反映其训练数据中不完美代码的习惯用法。Copilot 建议的代码可能并不总是编译、运行或者甚至有意义，因为它并没有实际测试这些建议。此外，在极少数情况下，副驾驶的建议可能包括个人数据，如来自其训练集的姓名和电子邮件，更糟糕的是，“有偏见的，歧视性的，辱骂性的或攻击性的”文本。

GitHub 表示，它已经实现了过滤器，以阻止以标准格式显示的电子邮件和攻击性词语，并且它正在建立一个过滤器，以帮助检测和抑制从公共存储库中重复出现的代码。Copilot 网站上的免责声明写道:“虽然我们正在努力让 Copilot 变得更好，但 Copilot 建议的代码应该像任何其他代码一样，经过仔细的测试、审查和审核。”

虽然 Copilot 自去年在技术预览版推出以来可能有所改进，但不清楚改进了多少。支撑模型 Codex——open ai 的 GPT-3 的后代——的能力已经被 DeepMind 的 AlphaCode 和开源的 PolyCoder 等系统所匹敌(甚至超越)。

“我们看到 Copilot 在生成更好的代码方面取得了进展…我们正在利用我们在[其他]工具方面的经验来提高 Copilot 建议的质量，例如，通过对 CodeQL 扫描的训练数据给予额外的权重，或者在运行时分析建议，”Salva 断言，“CodeQL”指的是 GitHub 的代码分析引擎，用于自动进行安全检查。“我们致力于帮助开发人员提高工作效率，同时提高代码质量和安全性。从长远来看，我们相信 Copilot 会写出比普通程序员更安全的代码。”

缺乏透明度似乎并没有挫伤对 Copilot 的热情，微软今天表示，在技术预览版中，Java 和 Python 等语言的代码中约有 35%是由开发人员生成的。该公司声称，成千上万的人在整个预览过程中经常使用这个工具。
