安全公司集团启动开源项目以简化数据共享

众所周知，安全并不是公司，甚至是大公司能够独自解决的问题。当涉及到网络安全时，需要一个社区共同努力来解决公司今天面临的各种问题。

今天早上在拉斯维加斯举行的黑帽美国会议上；AWS、Splunk 和赛门铁克(现在是博通的部分)宣布了一个新的开源网络安全项目，名为开放网络安全模式框架 (OCSF)项目。

据参与该项目的主要公司之一 AWS CISO 办公室主任马克·瑞兰德(Mark Ryland)称，该项目的想法是为不同的安全工具提供一种共享数据的通用方式。

Ryland 告诉 TechCrunch:“这个项目的原因是，我们收到了越来越多的客户意见和信号，表明他们正在使用大量的安全工具——永远不会有一个安全工具来统治所有这些工具——但与此同时，很难处理数据并使工具相互对话以创建公共的数据池，从而在逻辑上相当于跨工具进行连接操作。”

他说，整个一年，这个小组都聚在一起，因为所有人都从客户那里听到了类似的故事。“所以我认为，从客户那里得到了强烈的信号，你可以从参与此事的供应商名单中看到，他们得到了同样的信号。客户希望有一种方法使数据更具互操作性，并使工具之间更容易交流。因此，我认为这是工作组[走到一起]的核心原因。

除了这三家主要贡献者，还有另外 15 家公司的名人录，包括 Cloudflare、CrowdStrike、DTEX、IBM Security、IronNet、JupiterOne、Okta、Palo Alto Networks、Rapid7、Salesforce、Securonix、Sumo Logic、Tanium、Trend Micro 和 Zscaler。

根据 Ryland 的说法，赛门铁克已经开始构建一些东西，这是构建这个项目的讨论基础。他说，通过开源项目，该集团相信随着时间的推移，它可以更好地满足公司和客户的需求。

“通过将标准和一些支持工具放在 GitHub 上，现在我们有了一个开放的环境来评论、请求和建议。所以我们可以得到整个行业的意见。我们当然也有其他公司表示有兴趣加入核心工作小组。

该框架从今天开始可以在 GitHub 上下载。
