开发者称，新的 Twitter API 层仍未达标

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/30/new-twitter-api-tiers-still-miss-the-mark-developers-say/

在最近宣布了一些改变后，一些 Twitter 开发者表达了他们对 Twitter 新 API 结构的失望，一些人说这些改变仍然不能满足他们的需求。在最初宣布关闭免费 API 的计划几周后，周三晚上(T1)，Twitter 透露(T2)其新的 API 定价计划(T3)。现在，开发者将有三个新的层次:每月 1500 条推文的“免费”层，每月 100 美元的扩展访问的“基本”层和据报道每月 42000 美元的“企业”层。

对于很多开发者来说，前两级可能不够用，企业级又太贵。

声明发布后，许多开发者担心他们要么关闭项目，要么转向提供专注于另一个社交网络的服务。

一个受影响的领域可能是 Twitter 的机器人。新的免费 API 层将允许自动帐户每月发布 1500 次。这大约是一个小时一条推文，可能只对一些机器人账户有用。

基本层每月花费 100 美元，允许开发者访问 10，000 个读取请求和 50，000 个发布请求。但是，每个帐户每月最多只能发布 3，000 个帖子。对于交互式机器人来说，比如总结主题或者用图片回复人们的机器人，这个限制是不够的。这些开发人员的唯一选择仍然是企业级访问，这对于小型企业来说是可望而不可及的。

相比之下，旧的访问级别为 Twitter 的“基本”级别提供了每月 50 万条推文，为“高级”级别提供了每月 200 万条推文。在 15 分钟窗口内，速率限制(存档页面)也更高。例如，旧的 API 允许应用程序在 15 分钟内为每个用户发布 200 条推文。新的 API 只允许每个用户在 24 小时内发 100 条推文。这实质上扼杀了许多用例。

由于这些变化，开发者卢卡·哈默在推特上说他将关闭他的统计应用程序@账户分析。他还提到，他正在试图寻找新的方法来保持fed ifinder——一种让你在 federal 上搜索你的 Twitter 朋友的服务——通过众筹和缓存账户级请求的方式保持活跃。

允许人们在 Twitter 上使用 ChatGPT 的 ChatGPTbot 的开发者 Travis Fischer 说他也将不得不关闭他的机器人。他告诉 TechCrunch，他准备为访问付费，但每个账户每月 3000 条推文的限制将使他的工具毫无用处。fischer——与过去的其他开发者一起——也表达了对一些开发者将转向非官方 API 或浏览器自动化来抓取 Twitter 数据的担忧。

视频翻译机器人的开发者亚历克斯·沃尔科夫告诉 TechCrunch，他将不得不关闭他认为有用的服务。他还运行一个名为 Targum video 的工具，帮助人们翻译来自任何社交网络的视频。他提到该工具的大量使用来自 Twitter，现在他需要转向其他社交网络来使用和营销这项服务。

其他有用的项目，如 #buildinpublic ，允许开发者展示他们的工作，或 Mailclipperhq ，每周给用户发送关于他们书签的电子邮件，也将关闭，项目 在 Twitter 上宣布。

分析工具tweehp的开发者 Joe Clark 在 Twitter 上表示，他已经尝试订阅新的基础级别，并发现 Twitter 立即通知他该应用已经超过了月限额。

除了开发商，研究人员和学者也担心他们项目的未来。Twitter 表示，它正在“寻找新的方式”来迎合社区，但在此之前，他们必须依赖这些现有的层级。

斯旺西大学(Swansea University)网络威胁教授莫拉·康威(Maura Conway)表示，推特的举动“从研究人员的角度来看，绝对是一种嘲弄”，网络仇恨、极端主义和错误信息等不同的研究领域都将受到“严重影响”。数字智库 Demos 社交媒体分析中心的联合创始人卡尔米勒表示，议会应该质疑 Twitter 的举措将如何影响信息战和选举诚信等领域。

对于研究来说，免费的 API 实际上是无用的，学生可能没有钱花在项目上。此外，研究项目通常需要大量的数据访问，这在当前的计划下是不可用的。

正如研究人员 之前指出的，新的 API 访问级别将会成为在自然灾害等紧急情况下提供重要信息的项目的障碍。

埃隆·马斯克的最新举措是为了增加公司的收入。信息很明确:如果你想为大量受众开发任何围绕 Twitter 的工具，就为企业访问付费。这位特斯拉首席执行官已经削减了公司员工倍倍倍以节省成本。重新推出的 Twitter 蓝色订阅表现不佳。分析显示，到目前为止，它在移动领域的收入只有 1100 万美元。

Twitter 已经通过关闭多个项目和屏蔽流行的第三方客户端疏远了开发者社区。通过这一新举措，该公司推动开发工具的人更好地远离 Twitter。

该公司有机会在基础和企业之间引入一个中间层，让独立应用程序制造商注册并从中赚取收入。通过向开发者提供极端的选择，Musk & co .将让他们中的大多数放弃与 Twitter 相关的项目。许多用户对替代 Twitter 客户端的消失感到不满。随着更多的应用和创意机器人被关闭，Twitter 的服务质量将进一步下降。
