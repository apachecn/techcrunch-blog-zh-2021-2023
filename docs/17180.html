开源重塑发展、协作和社会的 5 种方式| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/sponsor/intuit/5-ways-open-source-is-reshaping-development-collaboration-and-society/

 

当竞争激烈时，公司倾向于内向，围绕他们的想法建造更高的围墙。公共领域变得无情和吝啬，发展变得沉寂。但在一个信息至关重要的行业中，被称为开源开发的日益增长的运动正在展示知识产权如何能够，或许应该，自由共享。

直到 10 到 20 年前，软件开发领域还是神秘和孤立的。开发人员在黑暗中辛苦工作，直到每一行代码都完成并商业化，然后再推广他们的项目。另一方面，开源模式促进了透明和基于社区的软件开发，在这种模式下，开发人员可以免费与他人共享他们的代码。反过来，社区提供了有价值的反馈、潜在的用例，有时还有实际的代码行。

说开放源码模式通过改变软件的生产和消费方式颠覆了软件产业是不现实的。我们已经与 Intuit 合作，并聘请了几位专家来阐明开源对科技行业的积极影响，甚至可以作为其他领域的一个模式。请继续阅读，了解更多关于这场运动是如何挑战传统发展观的。

 

1.开源释放创造力

共享一个新程序的内部工作的内在矛盾部分是革命性的，部分是合乎逻辑的。一方面，越来越多的开发者对创新优先于商业更感兴趣。他们希望看到优雅的代码解决紧迫的问题，并通过社区合作对社会产生影响，在这种环境中，在不同行业工作的软件程序员可以为共同的事业做出贡献。

编写有状态测试的简单性的一个例子——空手道上 REST 服务的替身。

合乎逻辑的部分来自投资回报:企业已经发现开源方法有共同的好处。开放采购项目可以加快开发和解决问题的速度，并突出单个公司可能错过的潜在盲点。因为开源项目并不仅仅是朝着底线前进，从事这些项目的人这样做是出于真正的求知欲。

“有了开源，工程师们选择去应对他们内心深处的挑战，”彼得·托马斯说，他是 Intuit 的杰出工程师，也是空手道(一种流行的用于 web 服务 API 测试的开源项目)的创始人。“通常，解决方案也能满足业务需求，这才是真正神奇的地方。例如，使用空手道，我们不仅看到我们自己的工程师接受了它，而且它提高了我们开发人员的速度，改善了我们为最终用户提供的 web 服务的质量。"

 

2.开源证明了多样性和包容性的力量

当一个团队没有不同的观点时，创新的上限会低得多。没有不同的意见和观点，新的想法很难被催化，个人也很难安于现状。

在软件开发的情况下，将一个工具或平台用于单一用途可能意味着错过了产生真正影响的机会。来自世界各地和各行各业的开发人员都在研究开源项目，这种观点的多样性推动了开发的进一步发展。

Intuit 的产品管理总监 Saradhi Sreegiriraju 说:“开源用户可能有其他我们从未想到过的他们希望解决的案例，”Intuit 是由 Argo Workflows 组成的 Argo 项目的成员，Argo Workflows 是一个容器本地工作流引擎； Argo CD ，声明式持续部署解决方案； Argo Events ，基于事件的依赖管理器；以及部署策略控制器 Argo Rollouts 。

这些工具共同帮助组织利用 Kubernetes。“这些组织中的许多都为项目做出了贡献，有时还会要求我们改进项目，这提供了一种新的思考方式。你总是从其他人正在做的事情中学习，通过这种方式，这些项目继续变得更好，并服务于更多的用例。”

 

3.开源揭示了项目的真正价值

个人或公司很难孤立地确定某样东西的真正价值。但是有了开源，社区的反应就更加有机了。如果人们检查了你的项目，耸耸肩，继续前进，那么也许它并不像你想象的那样具有开创性，你的时间用在其他地方会更好。但是，如果它获得了动力、采用和反馈，你可能会有所收获。

对于试图决定项目是否可行的商业领袖来说，社区的投入是无价的。例如，托马斯在 Intuit 的经理就因为这个原因鼓励他开放源代码空手道。通过向开源社区开放这个项目，这个项目获得了响亮的确认票，公司知道时间投资是值得的。

“开源项目健康的真正标志之一是，来自世界各地的人们不仅在使用它，而且还在实际贡献代码，”托马斯说。

 

4.开源允许公司更好地服务客户

除了开源有助于增强创造力、使协作多样化和提供真实世界的验证之外，它还允许项目更快地开发。当多个工具协同开发时，效率会提高。例如，想想平台基础设施，以及运行在其上的工具。当每一个都改进并推动另一个时，开发周期就加强了自身。

所有这些改进最终都会带来更好的软件，帮助消费者和组织更快地创新。

Sreegiriraju 以支票兑现为例，这是一个棘手的金融问题，该领域的许多不同参与者都在试图解决这个问题。对于成千上万的小企业来说，一天与五天之间的支票清算差异可能意味着可持续现金流的差异。

当多家公司开源他们的方法时，这个问题将会得到更快的解决。Sreegiriraju 指出，几家主要的金融公司将 Argo 用于他们自己的解决方案，而这些解决方案又是开源的，例如 Argo Events 。这种合作水平可以对计划的目标用户产生有意义的影响。

 

5.开源挑战我们重新思考工作

面对开源，公司在小范围内工作，然后公开竞争的传统模式暴露无遗。随着公共领域的协作和工作越来越多，随着代码整体质量的提高，整个技术行业都会受益。随着技术在不同应用中变得更加可靠、高效和可扩展，相关问题得到更快的解决，用户的生活也变得更加轻松。

目睹软件开发世界中发生的事情可能会被证明是无数其他行业和工作流的一个有价值的例子。随着强大的新技术允许前所未有的沟通和协作，大规模发生没有技术限制。

Sreegiriraju 说:“(开源)创造了最佳创意胜出的共生关系。”。“公司应该习惯于公开工作，互相帮助完成任务，互相学习，更快更轻松地推进重要的事业。”
