Retool 以 32 亿美元的估值筹集了 4500 万美元，使构建定制软件像购买现成软件一样容易

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/28/retool-raises-45m-at-a-3-2b-valuation-to-make-building-custom-software-as-easy-as-buying-off-the-shelf/amp/

图片来源: enot-poloskun / Getty Images

云计算、宽带网络、智能设备的爆炸式增长，以及打造 SaaS 初创公司的时尚，为企业和消费者创造了一个软件世界:正如苹果曾经说过的一句名言，无论你想要或需要做什么，总有一款应用可以满足你。

但这不是城里唯一的游戏。一家名为 Retool 的初创公司认为，在定制软件领域——为特定用途和特定用户开发的应用程序——仍有大量工作要做。自 2017 年成立以来，它已经在其平台上开发了超过 50 万个应用程序，数十亿次查询表明该软件的使用率很高。今天，它宣布以 32 亿美元的估值进行 4500 万美元的大规模融资，进一步强调了它在市场上的吸引力。

用首席执行官兼联合创始人许豪恩的话说，Retool 的想法是，它提供了“一种构建软件的新方式”

“我们的核心论点是，当你看看过去 20 年或 30 年软件是如何构建的，它真的没有改变，”他说。

你坐在电脑前，不停地打字，你所做的是基于“真正专业的训练，专业的知识，”他继续说:“重组背后的想法是，也许有一种更快的方式来构建软件。”在这里，你为主要组件使用拖放界面，在此基础上编写代码，“为最后的 20%或 30%”的工作，产生一些灵活的和定制的用户需要的东西。

如果重组得以进行，软件可能真的会吃掉世界，但是听起来它会以许多不同的烹饪形式出现。

Retool 向我描述了 C2 系列的资金，这些资金来自红杉资本、Stripe 联合创始人约翰和帕特里克·科利森、GitHub 前首席执行官纳特·弗里德曼、埃拉德·吉尔、丹尼尔·格罗斯和脸书前通信副总裁 Caryn Marooney，他现在是 Coatue 的合伙人。所有人都是该公司以前的投资者。在此轮融资之前，该公司于 2021 年 12 月筹集了更为温和的 2000 万美元 C 轮融资(估值为 18.5 亿美元)。

迄今为止，Retool 一直主要关注一类通常被称为“内部应用”的软件——不是面向客户或面向消费者的软件，而是帮助组织中的人完成工作的工具。

它的客户包括许多你可能认为已经有能力制造这种东西的公司——大型科技公司，如亚马逊、Pinterest 和 Coursera，以及 NFL、NBC 环球和其他公司。这次最新的注入将用于加倍它一直在做的事情，并雇用更多的人才，创造更先进和更深入的功能，并从旧金山的总部扩大地理范围。

今天，Retool 的核心平台是建立在大约 90 个可以组合在一起的“组件”之上的——这并不是一种“低代码”的方法，而是软件开发人员和工程师为了摆脱表单、图表和表格等一些基本构件而构建的。除此之外，它还提供了验证、可访问性和其他必要的工具，以验证一切正常运行。然后，开发人员可以连接任何数据库或 API——任何带有 REST 或 GraphQL API 的东西，以及 PostgreSQL、MongoDB 和其他数据存储——来完成软件的其余部分。

Hsu 的赌注是这种方法使构建和维护定制软件变得更加容易。他说，典型的两周开发时间可以缩短到一天。

对内部应用的关注很有趣。在某种程度上，这意味着 Retool 的知名度仍然相对较低。Hsu 表示，这是该公司做出的战略选择，因为内部应用程序占全球所有应用程序的 50%以上，它们正是组织可能需要更多定制的应用程序的用例，这些应用程序可能运行在私有云或内部，或者只是与他们在传统和更现代的系统中使用的任何应用程序一起工作。

然而，值得注意的是，这不是该公司认为自己的长期未来计划，包括建立功能，让开发人员致力于面向客户的应用程序和潜在的面向消费者的产品。在这方面，Stripe 联合创始人的支持非常有趣:Stripe 的工具正是为了帮助构建后两类应用而设计的。

它肯定有一些现有的客户，他们可能很想利用这种扩张，如果它启动的话。“Retool 平台的灵活性和易用性将我们内部工具路线图的范围从几年缩短到了几个月。Coursera 的高级工程经理 Shon Saoji 在给 TechCrunch 的一份声明中说。

Retool 的发展和定位非常引人注目，尤其是因为它在如此多的事情都在曲折中前进。看到红杉并不令人惊讶，它在另一个听起来不合时宜的想法——一个消息应用程序(WhatsApp)背后投入了大量资金，而当时这个领域似乎也已经被垄断了。

红杉资本(Sequoia)合伙人布莱恩·施雷尔(Bryan Schreier)表示，“重组处于两大趋势的交汇点:开发者的崛起，以及在疫情将十年的数字化转型压缩为两年后，卓越运营变得越来越重要。”。“Retool 使工程师能够以令人难以置信的速度构建内部软件，从而加速公司的运营。Retool 很好地定义了企业内部工具的未来，更广泛地说，定义了开发人员如何实际开发。面对宏观经济的不确定性，Retool 的价值主张更加鲜明。”
