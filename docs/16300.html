Dope Security 获得由 GV 牵头的 1600 万美元，用于构建安全的网络网关，旨在终端上工作，而不是在云中工作

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/22/dope-security-nabs-16m-led-by-gv-to-build-out-secure-web-gateways-designed-to-work-on-endpoints-not-in-the-cloud/

安全网络网关是一种网络安全服务，旨在通过阻止(或允许)某些流量来帮助组织实施有关互联网使用的 IT 策略，是组织在其网络安全配置文件中最常见的投资之一。但这是一个已经存在了几十年的产品，这意味着还有很大的创新和改进空间。

今天，一家名为 dope.security 的 SWG 初创公司表示，它筹集了 1600 万美元来推动这项工作。

与通过云和数据中心或大型昂贵的内部网络设备(目前此类服务的一些流行方法)进行路由相反，Dope 设计了一种安全的 web 网关产品，它运行在组织的端点上:连接到公司网络的计算机和其他设备。它认为这种方法更容易使用，更安全，更不容易失败，速度明显更快，对用户来说更隐私友好。

Dope 计划将这笔资金用于招聘、业务发展，并继续增强其产品的功能。Dope 的创始人兼首席执行官 Kunal Agarwal 在加利福尼亚州山景城总部接受采访时表示，新功能将在今年晚些时候包括 Dope API，以及为需要额外安全级别的组织提供的“零信任”层。

A 轮由 GV (Google Ventures)领投，Boldstart Ventures 和 preamble 也参与了，后者去年支持了这家初创公司，当时它还在隐身。该公司现已筹集了 2000 万美元。该公司没有披露其估值。

Agarwal 说他第一次想到 Dope 的核心前提是在赛门铁克工作的时候，他从 SWG 的一名工程师开始，一路晋升到产品经理，最终成为总经理。赛门铁克是世界上最著名的安全网络网关和其他安全产品的供应商之一，所以 Agarwal 亲眼目睹了赛门铁克花了多长时间对产品进行任何更改，以及产品目前的工作方式存在的问题。

“我每天都在游泳，”他说。“我听说客户抱怨说，当它不能正常工作时，你根本不能上网。”

在他看来，随着计算已经大规模转移到云，将安全性转移到云来管理这种变化已经失败了。典型的安全 web 网关产品是为在云中工作而构建的，在允许或阻止流量之前，它们会将所有流量发送到服务器。这实质上导致了延迟，甚至当某些东西不能正常工作时会导致完全失败。随着越来越多的流量转移到云端，以及越来越多的人在不同的地点工作，这个问题变得更加复杂。

Dope 将其方法描述为“直飞”:终点是流量在被允许通过(或不被允许，视情况而定)之前被评估的地方。减少的跳跃不仅意味着更少的运输时间，也降低了整个系统崩溃的几率。

Dope 使用云，但只用于服务的某些方面。管理员管理云上的白名单和黑名单，并使用 Dope 本身的指导将数据分发到端点，Dope 表示，它正在处理大约 700 的内容，以了解互联网上的威胁。随后，诊断和分析被发送到云，以便网络管理员可以更好地了解其网络上的使用和行为。

这指出了这个和所有安全网络网关产品的另一个重要方面:这些服务的基线是，它们控制人们如何上网，监控他们做什么，并查看数据如何(和什么)进出网络。

但是在一个更大的市场计划中，这个市场在 2022 年底价值 90 亿美元，预测到 2028 年将增长到近 240 亿美元，很明显，在试图保护组织免受不断增长的网络安全威胁的斗争中，这只会成为一种方法。Dope 认为，它提供了一条途径，以一种潜在的侵入性更小的方式来管理这种遏制，这种方式比其他选择更加用户友好。

“dope.security 以安全行业独有的方式深切关注设计和用户体验。GV 的普通合伙人 Sangeen Zeb 在一份声明中说:“它在创造产品时的第一个目标是简单——架构、采购、部署和管理的简单。“dope.security 完全有能力重塑 SWG 的传统方法，我们很高兴与该团队合作，将这种现代方法带入生活。”

Zeb 将在这一轮加入这家初创公司的董事会。

价格从每台设备每年 60 美元开始。
