针对批评，OpenAI 将不再默认使用客户数据来训练其模型

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/01/addressing-criticism-openai-will-no-longer-use-customer-data-to-train-its-models-by-default/

为了回应批评，OpenAI 默认情况下将不再使用客户数据来训练其模型

随着今天早上 ChatGPT 和 Whisper APIs 的发布，OpenAI 正在改变其 API 开发者政策的条款，旨在解决开发者和用户的批评。

从今天开始，OpenAI 表示，它不会将通过其 API 提交的任何数据用于“服务改进”，包括人工智能模型训练，除非客户或组织选择加入。此外，该公司正在为 API 用户实施 30 天的数据保留政策，并“根据用户需求”选择更严格的保留，并简化其条款和数据所有权，以明确用户拥有模型的输入和输出。

OpenAI 的总裁兼董事长 Greg Brockman 声称，这些变化中的一些并不一定是 T2 的变化——open ai API 用户拥有输入和输出数据，无论是文本、图像还是其他。但他说，围绕生殖人工智能和客户反馈出现的法律挑战促使服务条款被改写。

“我们最大的焦点之一是弄清楚，我们如何变得对开发者超级友好？”布罗克曼在视频采访中告诉 TechCrunch。“我们的使命是真正建立一个平台，让其他人能够在这个平台上开展业务。”

开发人员长期以来一直反对 OpenAI 的数据处理政策(现已被否决),他们声称该政策带来了隐私风险，并允许该公司从他们的数据中获利。在自己的一篇帮助台文章中，OpenAI 建议不要在与 ChatGPT 的对话中分享敏感信息，因为它“无法从[用户历史记录]中删除特定提示”。"

显然，OpenAI 允许客户拒绝提交用于培训目的的数据，并提供更多的数据保留选项，试图扩大其平台的吸引力。它也在寻求大规模发展。

关于最后一点，在另一项政策变化中，OpenAI 表示，它将取消当前针对开发者的发布前审查流程，转而支持一个高度自动化的系统。一位发言人通过电子邮件表示，该公司对转移到该系统感到放心，因为“绝大多数应用程序都是在审查过程中获得批准的”，而且自去年这个时候以来，该公司的监控能力已经“显著提高”。

这位发言人说:“改变的是，我们已经从基于表格的前期审查系统(开发者排队等待他们的应用理念在概念上获得批准)转向事后检测系统(我们通过监控其流量并根据授权进行调查来识别和调查有问题的应用)。”

自动化系统减轻了 OpenAI 审查人员的负担。但它也——至少在理论上——允许该公司更多地批准其 API 的开发者和应用程序。在获得微软数十亿美元的投资后，OpenAI 面临越来越大的盈利压力。据报道，该公司预计将在 2023 年赚到 2 亿美元，相比到目前为止已经投入了超过 10 亿美元，这是微不足道的一笔。
