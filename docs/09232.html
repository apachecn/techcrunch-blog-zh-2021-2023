同步计算获得 1550 万美元，用于自动优化云资源

原文：https://web.archive.org/web/https://techcrunch.com/2022/08/16/sync-computing-rakes-in-15-5m-to-automatically-optimize-cloud-resources/

在疫情推动的企业云采用热潮之后，成本终于被放到了显微镜下。据 observability 软件供应商 Pepperdata 最近在进行的民意调查显示，超过三分之一的 T2 企业报告云预算超支高达 40%。来自 Flexera 的另一项调查发现，优化云服务的现有使用是 59%的公司的首要计划，成本是主要动机。

围绕优化云计算，一个由创业公司组成的家庭手工业已经兴起。但是竞争中的一个竞争对手 Sync Computing 声称将成本和运行时间减少等业务目标与底层基础设施配置直接联系起来。作为麻省理工学院林肯实验室的副产品，Sync today 在由 Costanoa Ventures 牵头的一轮风险投资中获得了 1200 万美元(外加 350 万美元的债务)，Engine，Moore Strategic Ventures 和 National Grid Partners 也参与了该轮投资。

Sync 的联合创始人 Jeff Chou 和 Suraj Bramhavar 在创业之前都是麻省理工学院林肯实验室的技术人员。Bramhavar 通过在英特尔担任光子学研究职位的方式来到麻省理工学院，而周则共同创立了另一家初创公司——Anoka Microsystems——设计低成本光开关。

Sync 诞生于林肯实验室开发的创新，包括加速物流应用中常见的数学优化问题的方法。虽然许多云成本解决方案要么提供高级优化建议，要么支持调整工作负载的工作流，但 Sync 更深入， Chou 和 Bramhavar 说，它基于旨在“订购”适当资源的算法，提供特定于应用程序的细节和建议。

周在一次电子邮件采访中告诉 TechCrunch，“我们意识到，我们的方法可以极大地提高所有大规模计算系统的资源利用率。”。“随着摩尔定律变慢，这将成为一个关键的技术瓶颈。”

Chou 声称，Sync 不需要太多的历史数据就可以开始优化数据管道和提供底层云资源。例如，他说，仅通过之前一次运行的数据，一些客户就将其 Apache Spark 作业加速了 80% — Apache Spark 是数据处理的流行分析源引擎。

Sync 最近为 AWS EMR 上的 Spark、亚马逊的云大数据平台和 AWS 上的 Databricks 发布了一个 API 和“autotuner”。Azure 上对数据块的自助支持正在进行中。

“我们的公共 API 的推出将允许用户以编程方式将 Sync autotuner 应用于大量作业，并通过定制集成实现对[云环境]的持续监控，”Chou 说。“C-suite 关心管理云计算成本，我们的 Sync autotuner 在实现这一目标的同时还加快了数据科学和数据分析团队的产出……该产品还允许数据工程师快速更改基础架构设置以实现业务目标。例如，某一天，团队可能需要最小化成本并降低运行时的优先级，但是第二天，他们可能有一个严格的截止日期，因此需要加速运行时。有了 Sync，只需轻轻一点就能完成。”

Sync 首先将其技术应用于麻省理工学院的超级计算中心，然后与更大的政府高性能计算中心合作，包括国防部——它与国防部有 100 万美元的合同。现在，Sync 表示，它的自助服务应用程序有大约 300 名注册用户，还有“几十个”设计合作伙伴进行测试并提供反馈，包括 Duolingo 和迪士尼流媒体服务集团的工程师。

“疫情和最近的经济环境对 Sync 非常有利，因为通过提高效率来控制云成本现在是许多云软件即服务本地公司的首要任务。许多公司正在冻结招聘，需要一个“简单的按钮”来降低云成本，而不增加已经超负荷的团队的负担或开销，”Chou 说。“随着最近的经济衰退，对 Sync 独特方法的需求急剧增加，已经被主要企业客户采用。我们的主要挑战是让开发人员和首席技术官看到我们所构建的东西有什么不同，并认识到使用它可以让双方都受益匪浅。”

Chou 表示，最新一轮融资将把波士顿 Sync 的总资本提高到 2160 万美元，将用于客户获取、营销和销售、产品开发和 R&D，包括增加与现有工程工作流程的集成。Sync 目前有 14 名员工，Chou 预计这个数字到今年年底将增加到 25 名。
