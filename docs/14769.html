推特很棒，推特机器人和其他客户在推特将他们拒之门外后开始卸载他们的应用程序

原文：https://web.archive.org/web/https://techcrunch.com/2023/01/19/twitterrific-tweetbot-app-store-removal-twitter-api/

更新(2023 年 1 月 21 日，IST 下午 2:30):故事发布后，推特和菲尼克斯也被从应用商店移除。为了反映这一点，标题做了修改。

推特机器人制造商 Tapbots 在一条推特上表示，该公司“在过去的 12 年多里一直很荣幸为您服务，但由于我们无法控制的情况，我们不得不关闭推特机器人。”

 

推特最具代表性的第三方推特客户之一今天表示，它已经从应用商店移除了 iOS 和 Mac 应用。制造推特的公司 Iconfactory 在的一篇博文中说，在埃隆·马斯克的管理下，社交媒体网络已经成为“一个我们不再认为值得信赖、也不想再与之合作的推特”。

该应用与推特有着丰富的联系。它是该平台的首批移动和桌面客户端之一，并帮助形成了“推特”这个词事实上，推特的历史可以追溯到 2007 年——甚至在推特开发自己的 iOS 应用程序之前。

推特的消亡是在推特上周五在没有任何解释的情况下故意阻止第三方客户后发生的。本周早些时候，推特开发账户发布消息称，该公司一直在暂停这些应用，违反了“其长期的应用编程接口规则”。但它没有说明违反了哪些规定。

周四晚些时候，推特更新了其开发者术语以列出“使用或访问许可材料来创建或试图创建推特应用的替代或类似服务或产品”。在限制使用其 API 的情况下。一点也不长久。这一举动实质上扼杀了第三方应用程序。

但是，推特网并不孤单。大量其他应用已经开始或准备从不同的应用商店中移除它们的应用。推特机器人的联合创始人保罗·阿达德在一封电子邮件中告诉 TechCrunch，该公司已经将苹果客户端从应用商店中撤出，iOS 应用将很快跟进。

尽管推特发布了公告，一些推特客户仍然在工作，但公司暂停他们的工作可能只是时间问题。

虽然 Twitter 没有对这一举动做出解释，但这可能是为了对用户施加控制，迫使他们使用自己的客户端。

作为一个平台，第三方客户为 Twitter 增添了太多东西。Twitter 在 2010 年收购的应用 Tweetie 是更新大家都熟悉的时间线功能的幕后推手。Twitterrific 已经为类似的鸟标志、人物计数和对话(回复)做出了贡献。看到马斯克公司不重视那些让用户选择以不同方式体验平台的开发者，令人难过。

Twitter 与开发者的嫌隙

Twitter 长期以来一直忽视开发者对生态系统的贡献。该公司于 2012 年开始限制第三方 Twitter 客户端。两年后，它终止了与合作伙伴的协议，从而限制了对其数据的访问。

Twitter 忽视非本地客户的一个经典例子是 Tweetdeck，它于 2011 年收购了这家公司。该公司去年关闭了 Mac 版的 Tweetdeck，并一直在测试一个新的网络版本，选择了一些用户。但鉴于马斯克对该公司的处理方式，完全发布的希望不大。

在过去的几年里，这家社交媒体公司开始与开发者重建信任。2020 年，它发布了新的 API ,提供多种访问级别，以满足许多开发者的使用需求。2022 年，它推出了 Twitter Toolbox，一种展示和推广第三方应用的方式。在接受 TechCrunch 采访时，当时负责 Twitter 开发者平台的阿米尔·谢瓦特(Amir Shevat)表示，该公司正在探索建立某种应用商店。

但在马斯克接管公司后，这一切都崩溃了。Twitter 工具箱和许多其他开发者项目不再继续进行。

上个月，在 TechCrunch 的一篇专栏文章中，Shevat(现已离开该公司)写道新的 Twitter 管理层辜负了开发者的信任。他还批评了 T4 公司现在与开发商沟通的方式。

前进的道路

开发者对这一举动感到心碎，因为他们的应用程序的专业和高级订阅为他们的收入做出了贡献，现在它突然消失了。有些已经开始了其他项目。Tweetbot 制造商 Tapbots 正在构建一个名为 Ivory 的 Mastadon 客户端，并计划很快发布。菲尼克斯开发商 Matteo Villa 也发布了他的 Mastadon 应用程序 Wooly 的测试版本。

Twitter 的举动可能会让其他为该平台开发工具的开发者感到不快。去年 12 月，Typefully 和 Chir App 等作曲家应用程序告诉 TechCrunch ，尽管他们很谨慎，但他们希望继续为 Twitter 开发。内容审核工具保镖也指出，它希望缩减与 Twitter 相关的开发。随着第三方客户端陷入困境的最新举措，其他开发者可能会撤回他们围绕 Twitter 的项目。
