skiff bags 1050 万美元用于构建私人/协作工作空间

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/30/skiff-series-a-encrypted-workspaces/

在推出其端到端加密文档编辑器的六个月后，Skiff 又获得了 1050 万美元的新资金，用于为其迅速增长的客户群建立私人和协作工作空间。

我们去年在 Skiff 发布前写了一篇关于 Skiff 的文章:Skiff 是一款网络应用，它拥有与 Google Docs 大致相同的文档编写和共享功能，但它是建立在端到端加密的基础上的，所以 Skiff 不能像 Google 那样访问用户的文档。这家初创公司已经有超过 20，000 人使用其平台，超过了去年 5 月我们第一次与该公司交谈时的 8，000 名等待用户。

但是 Skiff 所依赖的端到端加密平台掌握着公司未来的关键。现在，Skiff 的联合创始人 Andrew Milich 和 Jason Ginsberg 在银行获得了 1050 万美元的 A 轮融资，他们告诉 TechCrunch，该公司正在努力成为“去中心化网络的应用层”。

该公司努力的一个核心部分是去中心化，这是一个允许用户拥有自己数据的过程。在过去的一年里，Skiff 与 Protocol Labs 合作，提供分散存储，称为 IPFS，或星际文件系统，允许 Skiff 加密他们的文档，并将它们分散在存储主机网络中，以及集成 MetaMask，允许用户使用便携式加密钱包而不是电子邮件地址登录 Skiff。

Skiff 的首席技术官金斯伯格在一次电话会议上说:“我们认为 Web 2.0 实际上是关于移动信息，而 web3 是关于移动价值。”。“数据是互联网上最有价值的东西，我们的目标是，你真的应该拥有自己的数据。”

金斯伯格表示，该公司正专注于增加其产品供应，如通信，并允许用户在其平台上分享更多类型的数据。

“我们看到数亿人选择的隐私产品并没有真正满足远程合作的需求，所以这就是我们看到 Skiff 出现的原因。有很多不同的产品，我们可以沿着这些路线做。金斯伯格说:“我们现在最感兴趣的是开发不仅能处理文档方面的产品，还能处理通信方面的产品。

这家初创公司的首席执行官米利奇表示，这轮由红杉作为回归投资者牵头的融资将帮助该公司开发出那些也依赖于端到端加密的新产品，如通信。Milich 说，Skiff 目前有一个 15 人的团队，分布在全球各地。首轮融资使 Skiff 的总融资额达到约 1400 万美元。

红杉资本的合伙人康斯坦丁·布勒(Konstantine Buhler)表示:“Skiff 正在建立一个令人惊叹的团队，并推出富有远见的产品来引领这一时刻。“我们对双倍下注再兴奋不过了。”
