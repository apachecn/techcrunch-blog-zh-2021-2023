thatDot 推出 Quine，这是一个流图引擎 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/23/thatdot-launches-quine-a-streaming-graph-engine/

总部位于俄勒冈州波特兰的初创公司 thatDot ，专注于流式事件处理，今天宣布推出奎因，这是一个新的麻省理工学院许可的数据工程师开源项目，将事件流与图形数据结合起来，创建公司所谓的“流式图形”

这听起来很复杂，部分原因是因为它是，也因为它是一个相对较新的概念。基于 DARPA 支持的多年研究，Quine 背后的想法是获取大量数据流，并将它们构建成状态图。正如你肯定已经猜到的那样，奎因以逻辑学家威拉德·范·奥曼·奎因的名字命名，然后可以使用该团队称之为“常设查询的方法来查询这个图表这些基本上是对传入数据的实时计算，然后 Quine 可以将这些数据传输到其他应用程序。

“我们已经开发了流图，以真正针对目前行业中的问题——我们都处于其中的岩石和坚硬的地方，”奎因的创造者和 thatDot 首席执行官兼联合创始人瑞安·赖特告诉我。“一方面，数据量巨大。在过去的 10 年里，大数据刚刚成为必要条件，它现在是一件普通的事情，只会变得越来越大。但另一方面，你如何解读所有这些数据？”

如今，这些数据经常处于移动状态，对于许多工作负载来说，延迟很重要。Wright 认为，现有的解决方案，如开源的 Apache Kafka 事件流平台，结合 Apache Fink 来分析流数据，迫使企业投入数十名工程师来构建他们的数据平台和管道，以分析所有这些传入的数据。至于其他现代方法，有像 Neo4j 和 TigerGraph 这样的工具，它们在开发人员中普及了图形数据库，但 Wright 认为它们都是从数据库的角度来解决这个问题的。

“在这种思维模式下，你会受困于所有传统的老问题，这些问题涉及技术细节，以及如何快速、轻松、可扩展地实现这些技术的困难。所以业内通常发生的情况是，如果你有大量的数据进来，你不能真正考虑图表解决方案，因为它们太慢了，”Wright 说。他认为，大多数当前的解决方案可能每秒能够处理几千个事件，但 Dot 与之交谈的那类客户需要每秒能够处理 250，000 个事件的解决方案，Wright 相信 Quine 能够处理这一点，甚至更多。

Tripwire 的首席工程师 Matt Splett 表示:“使用 Quine，我将复杂的自定义逻辑和 SQL 查询页面替换为简单的流计算汇总值查询，该值会在每次底层事件发生变化时更新。

莱特和他的联合创始人以及首席运营官·罗布·马尔纳提指出，这些用户涵盖了从安全公司到观察公司、日志处理公司、金融科技公司、广告公司和欺诈检测公司的各个领域。像其他开源公司一样，thatDot 的使命是支持高容量的企业 Quine 用户，但该公司也将 Quine 作为一个平台，在之上构建新的解决方案。

该公司在 2020 年筹集了超过 200 万美元的种子资金，由俄勒冈州风险基金牵头，并预计在今年晚些时候筹集 A 轮融资。
