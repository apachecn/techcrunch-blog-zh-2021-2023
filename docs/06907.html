数据狗发现无服务器计算正在成为主流

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/02/datadog-finds-serverless-computing-is-going-mainstream/

来自 Datadog 的一份新报告发现，随着超过一半的组织在亚马逊、微软和谷歌三大云之一上使用无服务器计算，无服务器计算可能会成为主流。

该公司在 2020 年的一份报告中发现，尽管一些客户正在使用亚马逊的服务功能(FaaS)T4(Samantha Lambda)T5，但其他云却落后了。今年的报告显示，Datadog 用户在所有三种云上都使用了无服务器技术，其中亚马逊领先 70%以上，微软和谷歌各超过 50%，这表明它在所有主要云平台上都大受欢迎。

报告中的另一个关键点是，公司经常将无服务器技术与容器结合使用，这两种技术看起来很适合对方。容器通常有一个有限的货架，可以根据需要部署，而无服务器提供了自动化资源部署的美妙之处。

事实上，报告发现 20%的 Lambda 用户通过 Docker 容器部署 Lambda 功能。这是无服务器和容器的结合，可能是我们没有想到的，但它正在形成。当报告查看使用这种方法的增长时，发现使用这种部署方法的 Lambda 用户从 2021 年 1 月的 0%增长到 2022 年 1 月的 20%,这是一个显著的上升趋势。

图片来源:数据狗

虽然该报告浮出了其他有趣的花絮，但它也发现，通过其系统的绝大多数 Lambda 函数被用于调用单个 API 网关，仅此而已，这与 DigitalOcean 的 Gabe Monroy 在上个月发布其公司的 FaaS 产品时告诉我们的情况一致。

“开发人员可以在我们的平台上运行容器中运行的 Django 应用程序或 Ruby on Rails 应用程序，然后在相同的应用程序旁边运行一些面向功能的 API，连接到他们需要的相同数据存储，”Monroy 当时告诉我们。

Datadog 公司产品和社区部的 SVP 伊兰·拉宾诺维奇说，总的来说，无服务器的好处是开发人员根本不用担心供应问题，他们可以直接编码。

“每一个云提供商都开始提供将容器作为无服务器机制运行的方式，这样你就不必担心基础设施，甚至在功能方面，他们也让你有可能将容器作为部署机制。因此，你上传一个 Docker 容器，而不是上传一个 zip 文件来运行 Lambda 函数，他们也会为你运行它，”Rabinovitch 解释道。

Datadog 的无服务器产品经理 Alex Cuoci 表示，跨平台工具的增加使无服务器变得更容易使用，这就是他们看到更多使用的原因。“我们从客户那里听到的，我们试图在报告中强调的是，这些新技术减少了团队首次采用无服务器的时间和资源，这为组织和团队打开了更多的大门，”Cuoci 说。

无服务器代表了云计算的理想状态，在这种情况下，您只使用您需要的资源，而不是更多。这是因为云提供商仅在特定事件发生时提供这些资源，并在事件结束时关闭这些资源。不是缺少服务器，而是不需要部署服务器，因为提供商会自动为您处理。

当人们在 2008 年左右开始谈论云计算时，优势之一是弹性计算，或者只使用您需要的东西，根据需要进行扩展或缩减。实际上，开发人员不知道他们需要什么，所以他们通常会过度配置以确保应用程序保持正常运行。

该公司基于通过其监控服务运行的数据创建了该报告。虽然它只代表来自客户的活动，但 Rabinovitch 将其视为优质数据，因为它拥有使用其服务的广泛客户。

“我们确实认为我们代表了整个行业，我们相信我们代表了真正的生产工作负载，”他说。这意味着人们很可能不监控工作负载，他们只涉足无服务器，这为数据增加了更多价值。
