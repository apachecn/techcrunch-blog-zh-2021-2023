安全代码初创公司 Jit 获得了 3850 万美元的种子资金

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/23/security-as-code-startup-jit-comes-out-of-stealth-with-38-5m-in-seed-funding/

Jit 是一家帮助开发者自动化产品安全的初创公司，通过将他们的安全计划和工作流编码为代码，然后在 GitHub 这样的代码库中进行管理，今天宣布它已经筹集了 3850 万美元的种子资金，由 boldstart ventures 领投，Insight Partners、Tiger Global、TeachAviv 和一些战略天使投资者也参与其中。这家公司是由波士顿-以色列创业工作室 FXP 孵化的

随着这一宣布，Jit 也走出了秘密，并宣布前 Puppet CTO 和 Cloud Foundry Foundation 执行董事 Abby Kearns 加入其顾问委员会。

Jit 首席技术官 David Melamed 表示:“网络安全领导者正在添加更多工具，其速度超过了他们的团队实施、调整和配置工具的速度，从而增加了风险支出。“对于高速开发和产品团队来说，创建一个安全计划或程序太耗时了。Jit 通过合规性复选框简化了工程团队的技术安全性，同时降低了开支。我们提供了实现 DevSecOps 的最简单的方法，从一开始就将产品安全性内置到软件中，同时还提供了一种用开发人员理解的语言(代码)持续维护产品安全性的方法。”

形象学分: Jit

Jit 背后的想法是提供公司所说的“最低可行安全”(MVS)。该服务为开发人员提供了现成的 MVS 计划，这些计划已经编写了一套最基本的工具和工作流，他们将需要这些工具和工作流来保护他们的应用程序和运行的基础设施。

该公司解释说:“与其研究、配置、实现并完成将开源安全工具集成到您的堆栈和 CI/CD 管道中的工作，不如由 Jit 的安全研究团队花时间来策划和选择将为您的应用程序提供第一道防线的工具，而无需您自己找出答案。”。

该公司认为，其方法还意味着开发人员只有在出现他们必须立即做出反应的重要漏洞时才会收到警报，然后可以从他们现有的工作流程中修复这些漏洞。该工具将在“拉”请求中创建自动安全检查，或者发现 AWS 错误配置或 npm-audit 等第三方服务的安全控制问题。

通过这种方式，该服务还可以为企业提供一个展示其当前状态的仪表板，从而使企业更容易开始对 SOC2 或 ISO 27001 等合规计划进行差距分析。

boldstart ventures 的创始人兼管理合伙人 Ed Sim 表示:“随着开发和管理的应用程序数量的快速增长，产品安全性需要简单、易于用作代码，以及在当前的 CI/CD 管道中工作。“Jit 确保现代工程团队可以通过设计构建安全的基于云的应用程序，同时简化持续的安全性。Jit 是独一无二的，因为它统一了各种开源安全工具，同时将整个安全性作为代码体验本机集成到当前的开发人员工作流中。”
