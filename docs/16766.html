DataDome 利用人工智能来防范基于机器人的攻击，融资 4200 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/30/datadome-raises-42m-series-c-bot-detection/amp/

图片来源: ArtemisDiana / Getty Images

在线业务面临着不良机器人活动的风险，现在肯定比过去更大。根据 Imperva 的一项调查，2021 年 42.3%的互联网流量不是人类的，而是恶意运行自动程序的机器人。

鉴于机器人可能造成的损害——例如，窃取内容和库存，降低网站和应用程序的性能，接管账户和进行支付欺诈——出售对抗机器人技术的供应商吸引大量投资并不奇怪。一个典型的例子是，为移动应用程序、网站和 API 提供 bot 保护服务的 DataDome 在 InfraVia Growth 领投的 C 轮融资中筹集了 4200 万美元，大象和 ISAI 也参与了融资。

考虑到当前的宏观经济环境，这是一个相当高的数字。据 Crunchbase 数据显示，2022 年，网络安全初创公司的资金减少了三分之一，从 2021 年的 228 亿美元降至去年的 153 亿美元。在网络安全行业之外，硅谷银行倒闭的余波继续影响初创公司获得资金的能力。

DataDome 联合创始人兼首席执行官 Benjamin Fabre 表示，C 轮融资的收益将用于支持公司的商业推广和 R&D 项目。

“这一轮使我们的总融资额达到 8200 万美元，”法布尔在电子邮件采访中告诉 TechCrunch。“我们很幸运能够以非常负责的现金效率实现显著增长。说实话，我们这次并不打算筹集资金。但 InfraVia 与我们一样，将 bot 缓解作为在线欺诈防范的基础，这是一个令人信服的案例。”

法布尔在 2015 年与长期商业伙伴杨奇煜·格雷尼尔一起创立了 DataDome，此前两人发现大多数公司都无法检测和阻止机器人。他们试图建立一个可以防止基于 bot 的威胁的平台，但以一种独特的方式——一种避开机器学习的静态规则的平台，该规则评估对网站、移动应用程序或 API 的每一个请求。

当然，DataDome 不是唯一一家这样做的公司。有一家反机器人初创公司 Kasada ，它使用其专有的反机器人平台 Polyform 来帮助打击在线机器人。 ThreatX 为 API 和 web 应用开发 bot 防御技术，而perimerx使用机器学习来帮助理解正常行为，并以自动化的方式关闭异常的、由 bot 驱动的行为。

那么是什么让 DataDome 与众不同呢？该公司声称是少数几个使用漏洞扫描规则、设备指纹和行为分析等技术重新实时分析每个流量请求的公司之一。该平台与现有的应用程序一起工作，提供了像验证码这样的功能来挫败机器人。它还可以扩展，每天处理超过 3 万亿个数据信号。

图片来源: DataDome

“想想最近的头条新闻，比如泰勒·斯威夫特和 Ticketmaster 的惨败，”法布尔说。“我们的客户非常清楚他们因此而经历的痛点……我们的方法是横向的，跨越了上述这些孤岛。在这方面，我们像攻击者一样思考，并评估每个请求的意图。”

在人工智能和机器学习方面，DataDome 利用几个人工智能模型来尝试识别恶意机器人。一旦检测到异常情况，DataDome 会部署一系列技术(包括数据挖掘)来推断是什么导致了变化。

检测后，检测信号被存储在数据库中，在数据库中，它们可以用于提高检测质量，了解攻击期间发生了什么，或者训练机器学习模型。

据 Fabre 称，在过去的几个月里，DataDome 的重点一直是推出其 CAPTCHA 产品，以及新的人工智能模型，以阻止来自 CAPTCHA 的机器人威胁-解决机器人和 CAPTCHA 农场，以及改进的分析和报告工具。客户似乎对这一轨迹感到满意；收入比 2021 年 10 月翻了一番。

“这仍然是一个高速增长的市场，”法布尔说。“根据我们的经验，大约 40%的新客户从 web 应用防火墙或内容交付网络(CDN)的基本保护升级，30%从 bot 专家转向，30%从内部解决方案发展而来。在第一种情况下，许多客户选择我们来增强现成的 bot 管理功能 cdn，如 Cloudflare 和 Akamai 产品。”

DataDome 声称在电子商务、媒体、旅游、票务、医疗保健和公共部门等行业拥有 300 多家客户。根据 Fabre 的说法，对于这些品牌来说，该平台正在保护全球 4800 多名活跃用户的 40 多万个域名，从北美到西欧，印度和澳大利亚。

Fabre 补充说:“疫情创造了向所有在线事物的巨大转变，这反过来扩大了自动化威胁的前景。”。“在 COVID 高峰期之前、期间和之后，我们的解决方案对数字业务至关重要。”
