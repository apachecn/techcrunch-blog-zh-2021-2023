Arrikto 通过 Kubeflow 即服务扩展其 MLOps 平台| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/10/arrikto-expands-it-mlops-platform-with-kubeflow-as-a-service/

Arrikto 的使命是让数据科学家能够更快地构建和部署他们的机器学习模型。该公司在 2020 年底筹集了1000 万美元的首轮融资，正在基于 Kubeflow 构建平台，这是一个用于构建机器学习操作的云原生开源项目，最初由谷歌开发，但现在主要由社区管理。到目前为止，Arrikto 的主要产品是一个自我管理的 Kubeflow 企业发行版，适用于希望在其数据中心或虚拟私有云内运行 Kubeflow 的企业(恰当地命名为“企业 kube flow”)。如今，该公司还推出了完全管理版本的 Kubeflow。

Arrikto 首席执行官兼联合创始人 Constantinos Venetsanopoulos 告诉我:“将 ML 模型从实验一直推向生产是非常复杂的。“我们认为这有几个共同的原因。第一，数据科学家本质上不是运营专家，运营人员也不是数据科学家，他们也不想成为数据科学家。第二，在过去的几年里，我们已经看到了 ML 工具的爆炸式增长。它们极度分散，需要大量整合。我们看到的是人们努力将所有东西缝合在一起。这两个因素造成了巨大的进入壁垒。”

图片来源:到达

通过其完全管理的 Kubeflow，Arrikto 旨在为企业提供一个平台，帮助他们加速 ML 管道，使数据科学家不必担心基础设施，同时还允许他们继续使用他们已经熟悉的工具(想想笔记本电脑、TensorFlow、PyTorch、拥抱脸等)。).“我们希望打破阻止大多数公司部署真正的机器学习能力的技术障碍，”Venetsanopoulos 说。

该公司认为，通过 Kubeflow 即服务，数据科学家将能够即时访问端到端的 MLops 平台。它本质上是 Arrikto 的企业 Kubeflow，上面有许多定制的自动化工具，用来抽象出它所基于的 Kubernetes 平台的所有细节。

目前，Arrikto 将只在单一云上运行，但从长远来看，该计划将支持三大云提供商，以确保低延迟(并减少在云之间移动大量数据的需求)。

有趣的是，Venetsanopoulos 认为，该公司目前最大的竞争对手不是 AWS 的 SageMaker 等其他托管服务，而是试图通过拼接开源工具建立自己的平台的企业。

Venetsanopoulos 表示:“Kubeflow 即服务为数据科学家和 DevOps 工程师提供了在 Kubernetes 上使用 MLOps 平台的最简单方法，而无需向他们的 IT 部门请求任何基础架构。“当一个组织在生产中部署 Kubeflow 时，无论是在本地还是在云中，Arrikto 的 Kubeflow 即服务都会加速这一过程。”

该公司目前拥有约 60 名员工，除了这一全新的完全托管服务之外，还将继续提供 Kubeflow Enterprise。
