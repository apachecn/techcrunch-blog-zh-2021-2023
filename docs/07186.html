苹果用 Stage Manager 调整 iPad 的工作流程

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/13/apple-resizes-the-ipads-workflow-with-stage-manager/

虽然 iPad 基于其友好的用户界面和单一应用程序的焦点从一开始就获得了巨大的成功，但对于那些渴望更多深度的人来说，它 已经开始感到有点过时了 。长期以来，它一直是世界上最轻、功能最强大的中型计算机之一，但它所能胜任的工作和它实际欢迎的工作之间的差距越来越大。

在过去的几年里，苹果的 重大 iPad 更新 一直专注于将该设备从漫长的妊娠期拖到一个与 Mac 有更多语言相似性的世界。上周我们得到的 iPadOS 16 预览版也不例外。像 Stage Manager、桌面类应用和对连续性的增强等少数功能都是针对 延续 iPadOS 15 在这方面的工作 。

上周，我有机会与软件工程的苹果 SVP 公司的克雷格·费德里吉简短地谈论了旨在增强多任务和多应用工作的新 iPadOS 功能。我们聊了聊这些公告的时机、执行和反应。

Stage Manager 是今年 iPad 多任务处理增强功能的核心。该功能以系统管理的磁贴形式呈现每组最多四个应用的集合。群组排列在左侧，允许您在这些工作区之间快速点击。它本质上是 Spaces 的一个更加可见和持久的版本 Mac 功能，允许多个桌面在 macOS 上悬停在你的屏幕两侧。如果你甚至不知道空间的存在，你会被原谅，因为它是相当模糊的，对任何从未访问过任务控制屏幕的人来说，没有很多(如果有的话)视觉线索。

“iPad 有一个独特的主张，一套独特的关于交互的期望，我们希望从那里开始，而不仅仅是从几十年前或另一套基于不同基本原则的系统中拖拉东西。因此，我认为舞台监督是进化过程中的重要一步，”费德里吉说。

Federighi 说，让单一窗口聚焦屏幕的想法在苹果根深蒂固。首先是 Mac OS X 测试版早期的单一应用模式。几年后，一种感觉像舞台经理的体验的内部原型。

“在苹果电脑上，有很多不同的工作方式。有些人使用空间，有些人进出任务控制。有些人是命令标签的人，有些人喜欢制造混乱，有些人清理他们的混乱，有些人使用最小化。我的意思是，这里没有错误的答案，在 Mac 上工作有很多有效的方法。”

这种工作空间管理方式看起来确实非常明显是以 iPad 为中心的。但 Federighi 表示，苹果的两个独立团队，一个在 iPad 端工作，另一个在 macOS 端工作，试图让多个工作空间更加明显和友好，达成了类似的概念，并在中间相遇。他说，这意味着这种方法体现了两种观点。

“我们当中有很多人每天都在使用苹果电脑，他们真的想要这种专注的体验，让我们获得平衡。所以我们站在 Mac 的立场上，提出了这个想法，并说我们认为这是可以实现的，我们希望实现这个想法。另外，在 iPad 方面，我们也在考虑这个问题。信不信由你，两个独立的团队集思广益，设计出几乎相同的想法。”

他承认，可能会有一群拥有 40 年 Mac 历史的人超出他们的预期。会有人期待这个特性以某种方式运行，而且它是完全不同的。而且，从本质上来说，这不会是许多 Mac 忠实用户立即使用的工具。

“人们需要一些时间来适应他们的肌肉记忆和他们的期望，然后为一套新工具优化他们的流程。我是说，这是给 iPad 用户的。而在 Mac 上，它是为那些发现自己想以这种方式工作的用户提供的。”

“如果 Mac 上 20%的用户最终说这是他们囊中的又一个好工具，”费德里吉说，“那就太棒了。”

为什么是现在，为什么是 M1？

每一次苹果产品的推出都有一个不变的因素，那就是希望它早点出现。“最终”这个词已经成为苹果观察人士的一个笑话。大多数产品发布背后的真相是，这些事情往往比看起来要复杂得多。尤其是在质量和可用性方面，这将满足数十亿用户的需求。苹果并不总是在第一次尝试时就击中目标，随着软件服务于越来越多的用途，边缘案例以及软件的粗糙端也在增加。但其软件团队的整体一致性仍然被认为是任何构建平台级软件的人的基准。

但是时机问题仍然值得一问，所以我问了。Federighi 说，为了让 Stage Manager 和其他最近对 iPad 多任务处理的增强功能面向用户，苹果必须做很多事情。因此，尽管很明显人们已经准备好接受更多的功能——尤其是在 iPad Pro 上——但一些基础工作还需要做。

分割视图、幻灯片和现在的 Stage Manager 都要求开发者在苹果自己的框架中支持完整的应用大小调整。苹果需要建立基础设施支持，以协调同时在屏幕上运行的多个应用程序。还有硬件需求。

“建造 M1 也很重要，”费德里基说。“从一开始，iPad 就一直保持着这种极高的响应性和交互性标准。互动的直接性在于，每个应用程序都可以即时响应每一次触摸，就好像你正在触摸屏幕下的真实事物一样。我认为人们有时很难理解实现这一目标的技术限制。

“当你在游戏中添加多个应用程序以及大量的屏幕空间时，你必须确保这些应用程序中的任何一个都可以即时响应触摸，这是你在桌面应用程序中所没有的。间接操纵给了你一些空间，所以这是一组不同的约束。”

Stage Manager 利用了 M1 芯片带来的更强大的 GPU、更快的虚拟内存 I/O、更快的存储和更多的 RAM。Federighi 说，所有这些都必须一起保持体验的流畅，今年他们做到了。

这加剧了人们围绕 Stage Manager 需要苹果 M1 处理器才能运行的一些讨论。鉴于围绕哪些组件需要新硬件以及哪些是选择问题存在各种各样的理论，我在这里要求澄清一些问题。

Federighi 说，苹果对交互响应的极高要求是问题的核心。他指出，在 iPad 上，交互响应和体验的门槛非常高，你能接触到的每个应用程序都需要能够即时响应。这意味着，从历史上看，用户可以立即访问的任何应用程序都需要完全驻留在 RAM 中。macOS 上的应用程序就不是这样了，它们大量使用虚拟内存。

为了达到那样的响应水平，几个因素需要相互碰撞。首先，需要大量 RAM 和“极快 IO 虚拟内存”的组合来托管活动存储桶中的多个应用程序。

Federighi 说:“只有 M1 的 iPads 将高 DRAM 容量与超高容量、高性能的 NAND 结合起来，使得我们的虚拟内存交换速度超快。”“现在我们让你在一个面板上最多有四个应用程序，再加上另外四个——最多八个应用程序，以实现即时响应和充足的内存，但我们在其他系统上没有这种能力。”

不过，导致苹果将 Stage Manager 限制在 M1 ipad 上的不仅仅是内存的可用性。

“我们也将 Stage Manager 视为涉及外部显示器连接的整体体验。M1 上的 IO 支持我们以前的 iPads 不支持的连接，它可以驱动 4K，5K，6K 显示器，它可以以缩放的分辨率驱动它们。我们无法在其他 iPads 上做到这一点。”

显卡性能也是一个限制因素。

“我们真的设计了 Stage Manager 来充分利用(M1 的)优势。如果你看看应用程序倾斜和阴影的方式，以及它们如何进出动画。要在超大显示器和多显示器上以超高的帧速率实现这一点，需要其他任何人都无法提供的最高图形性能。

“当你把所有这些放在一起时，我们无法在任何较小的系统上提供完整的舞台经理体验，”Federighi 说。“我的意思是，我们希望在任何地方都可以使用它。但这是它所要求的。这是我们将带到未来的经验。我们不想将我们的设计局限于次要的东西，我们正在为未来设定基准。”

为什么这样？

Stage Manager 是苹果方式(my phrase 等)的一个有趣化身。)的设计。主要的原则，特别是在 iPad 上强调的，集中在为用户提供一种通过存在而不是记忆导航的方式。信息设备方法的本质是，iPad 应该体现手头的任务，尽量减少干扰。当这个规则被打破时，哲学的支柱——直接交互，显示不告诉和清晰的导航面包屑——介入，以防止用户迷路。

虽然表面上是一个窗口管理器，但这里的方法绝对不是免费的。苹果正在为用户做出一些非常具体的选择，以便让用户感觉事情可以直接操作。窗口捕捉到预定义的大小，你永远不能通过直接放置另一个应用来隐藏一个应用。

作为一个用户，你会感激你没有不断地积累杂物，你没有清理东西，你没有管理东西在哪里，你只是做你想做的事情。它就在那里。这一切都是为你管理的，”Federighi 谈到他们对 Stage Manager 的设计方法时说。“既干净又专注。传统的窗口环境则相反。默认情况下，它们会造成混乱，你打开的所有东西都会造成混乱。所有的事情都需要你去管理事物的位置，以及事物如何互相掩盖等等。然后你就要一直负责打扫自己的卫生。”

“但即使是那些东西的排列方式，我们也要确保窗户总是可以进入的，不会互相遮挡。所以不用管。然而，当你进入下一个应用程序时，你会回到一个干净的单一窗口体验。我们会让你继续接触一切。所以你有你最近的应用程序，它不是眼不见心不烦，它们是为了快速多任务处理而存在的。但是他们不会碍事，不会把一切都弄乱。

“我们觉得这个设计真正保留了我们喜爱 iPad 的精髓，同时给你多任务处理体验的灵活性和流畅性，这是一些想要进一步扩展 iPad 的人真正想要的。”

在我早期的印象中，这种方法处理得非常好。虽然会有一大群用户想要无限可调的窗口，允许一个窗口遮住另一个窗口的能力，以及尽可能凌乱的自由——但在我看来，这根本不是 iPad 的基础，也不应该在这里迎合。

我认为，如果 Stage Manager 继续得到调整和完善，它可以轻松取代 iPad 上相对模糊的 Spaces、Mission Control 和其他多应用视图。例如，如何让一个应用程序快速全屏显示，然后再返回，这方面的改进将会非常好。我觉得，它需要是一个简单的点击或挤压手势——目前它在一个模糊的图标后面两次点击。但是，快速切换到“全屏”来让 iPad 成为应用程序，然后再切换到“全屏”来促进应用程序之间的交流，这应该是主流的一部分，而不是一个分支。

尽管如此，该功能有一个足够灵活的框架来承担更强大的任务，并在内存中保存更多的应用程序，以及一个令人感觉轻快的功能和强大的即时性。作为一个多年来一直主要在 iPad 上开车的人，并且已经习惯了在多应用工作流中对抗单应用方法，这真是一个不错的惊喜。

从测试版构建

Federighi 表示，苹果将在整个夏天继续开发 Stage Manager。他说，尽管苹果软件的测试版最近受到了更多的审查，但他们的主要工作仍然是让开发者采用他们的新 SDK 并准备好他们的应用程序。

之后，他们希望体验的基础到位，这样他们就能确保他们得到的反馈确实有用。但他们仍在研究这个功能，Federighi 说他们有一些“正在进行中”的东西，他们知道这些东西是他们可以在后续测试版中加入的改进或调整。

“我们已经计划了一些，因为它们与 Mac 和 iPad 上的舞台管理器有关，”费德里基说。“我们收到的一些反馈是这样的，‘是的，我的意思是，这将是第二或第三阶段！’我们已经确定了这些事情，或者是错误，或者只是不完整的元素，或者是对行为的调整。

“我们还没有看到任何让我们思考的东西，哇，这真是意想不到的消息。他们中的许多人要么是我们预期的那些还没有适应这个系统的人的反应，要么是我们在飞行中有所改进的领域。所以，是的，我们肯定会继续这样做。”
