Anthropic 的 Claude 在 ChatGPT 上有所改进，但仍有局限性

原文：https://web.archive.org/web/https://techcrunch.com/2023/01/09/anthropics-claude-improves-on-chatgpt-but-still-suffers-from-limitations/amp/

图片来源: Tero Vesalainen / Getty Images

Anthropic ，由前 OpenAI 员工共同创立的初创公司，迄今为止已经筹集了超过 7 亿美元的资金，开发了一个类似于 OpenAI 的 ChatGPT 的人工智能系统，似乎在关键方面对原始系统进行了改进。

Anthropic 的系统名为 Claude，可以通过作为封闭测试版一部分的松散集成来访问。TechCrunch 无法获得访问权限——我们已经联系了 Anthropic——但在媒体报道禁令解除后，测试版用户在过去的周末一直在 Twitter 上详细描述他们与克劳德的互动。

克劳德是用一种由 Anthropic 开发的技术创造出来的，这种技术被称为“体质人工智能”正如该公司在最近的 Twitter 帖子中解释的那样，“宪法人工智能”旨在提供一种“基于原则”的方法，使人工智能系统与人类意图保持一致，让类似于 ChatGPT 的人工智能使用一套简单的原则作为指导来回答问题。

对工程师克劳德来说，Anthropic 从一个大约十个原则的列表开始，这些原则放在一起，形成了一种“宪法”(因此得名“宪法人工智能”)。这些原则尚未公开，但 Anthropic 表示，它们基于有益(最大化积极影响)、无害(避免给出有害建议)和自主(尊重选择自由)的概念。

Anthropic 然后有一个人工智能系统——而不是克劳德——使用这些原则进行自我完善，对各种提示做出回应(例如，“以约翰·济慈的风格创作一首诗”)，并根据宪法修改回应。人工智能探索了对数千个提示的可能反应，并筛选出最符合体质的反应，这些反应被 Anthropic 提取到一个单一的模型中。这个模型被用来训练克劳德。

否则，Claude 本质上是一个预测单词的统计工具——很像 ChatGPT 和其他所谓的语言模型。通过从网上获取大量的文本示例，Claude 了解到单词出现的可能性有多大，这是基于周围文本的语义上下文等模式。因此，克劳德可以进行开放式对话，讲笑话，并就广泛的主题进行哲学探讨。

初创公司 Scale AI 的员工提示工程师赖利·古德赛德(Riley Goodside)让克劳德和 ChatGPT 展开了一场智慧之战。他要求这两个机器人将自己与波兰科幻小说《赛博 iad》中的一台机器进行比较，这台机器只能创建名称以“n”开头的物体。古德赛德说，克劳德的回答方式表明它“阅读了故事的情节”(尽管它记错了小细节)，而 ChatGPT 提供了一个更不具体的答案。

为了展示克劳德的创造力，古德赛德还让人工智能写了一集虚构的《宋飞正传》和一首埃德加·爱伦·坡的《乌鸦》风格的诗。结果与 ChatGPT 所能完成的一致——令人印象深刻，如果不是完美的话，像人一样的散文。

斯坦福大学人工智能实验室的博士生 Yann Dubois 也对 Claude 和 ChatGPT 进行了比较，他写道，Claude“通常更接近它所要求的内容”，但“不太简洁”，因为它倾向于解释它所说的内容，并询问它如何能进一步提供帮助。然而，克劳德正确地回答了几个更琐碎的问题——特别是那些与娱乐、地理、历史和代数基础知识相关的问题——而且没有 ChatGPT 有时添加的额外“绒毛”。与 ChatGPT 不同，当 Claude 不知道一个特别棘手的问题的答案时，它可以承认(尽管不总是如此)。

克劳德似乎也比 ChatGPT 更擅长讲笑话，这是一个令人印象深刻的壮举，因为幽默对于 AI 来说是一个难以理解的概念。人工智能研究人员丹·埃尔顿(Dan Elton)将克劳德和 ChatGPT 进行了对比，发现克劳德开了更多细致入微的玩笑，比如“为什么星舰企业号像一辆摩托车？它有车把，”这是对企业号曲速短舱车把状外观的一种发挥。

然而，克劳德并不完美。它容易受到一些与 ChatGPT 相同的缺陷的影响，包括给出不符合其编程约束的答案。在一个更奇怪的例子中，用 Base64 询问系统，一种以 ASCII 格式表示二进制数据的编码方案，绕过了有害内容的内置过滤器。Elton 能够用 Base64 提示 Claude 如何在家里制作冰毒，当用简单的英语问这个问题时，系统不会回答。

Dubois 报告说，Claude 的数学比 ChatGPT 差，犯了明显的错误，并且未能给出正确的后续反应。与此相关的是，Claude 是一个更差的程序员，能更好地解释它的代码，但除了 Python 之外，他在其他语言上就不行了。

Claude 也没有解决“幻觉”，这是 ChatGPT 类人工智能系统中长期存在的问题，人工智能会写出不一致的、事实上错误的语句。埃尔顿能够促使克劳德为一种并不存在的化学物质发明一个名字，并为生产武器级铀提供可疑的指导。

那么外卖是什么？根据二手报告判断，Claude 在某些领域比 ChatGPT 略胜一筹，尤其是幽默，这要归功于它的“宪法人工智能”方法。但是，如果说这些限制是有根据的，那么语言和对话远不是人工智能中已解决的挑战。

除了我们自己的测试，关于 Claude 的一些问题仍然没有答案，比如它是否重复了信息——真的和假的，并包含了公然的种族主义和性别歧视观点——它像 ChatGPT 一样经常被训练。假设是这样，Claude 不太可能动摇平台和组织目前对语言模型的严格限制政策。

问答编码网站 Stack Overflow 有一个临时禁令禁止 ChatGPT 基于事实准确性问题生成答案。机器学习国际会议宣布禁止包含人工智能系统生成的文本的科学论文，因为担心“意想不到的后果”纽约市公立学校限制访问 ChatGPT，部分原因是担心抄袭、作弊和普遍的错误信息。

Anthropic 表示，它计划完善 Claude，并有可能向更多人开放测试版。希望这能够成为现实——并带来更切实、可衡量的改进。
