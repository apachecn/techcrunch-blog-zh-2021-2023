苹果将在 iOS 16 中引入新技术，让你跳过验证码

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/21/apple-is-introducing-new-tech-in-ios-16-to-let-you-skip-captchas/amp/

图片来源:伊曼纽尔·杜南/法新社/盖蒂图片社

通过验证码验证非常烦人，你必须识别奇怪的弯曲字母或点击船的图像，直到什么都没有了——只是为了证明你不是机器人。苹果可能会在即将到来的 iOS 16 和 macOS Ventura 更新中推出一项新功能来解决这个问题。

该公司正在引入一种新的自动验证设置——它将生活在设置>苹果 ID >密码&安全>自动验证之下——它将在网站或应用程序上验证你的身份，而无需填写验证码。

“通过允许 iCloud 自动和私下验证你的设备和帐户，绕过应用程序和网络上的验证码，”苹果在 iOS 上对该功能的描述如下。

在举行的全球开发者大会(WWDC)上的一场演讲中，苹果辩称，用户填写验证码很麻烦，通常不会通过跟踪用户的 IP 地址来遵循隐私最佳做法，并且他们可能会阻止那些难以完成验证码挑战的残疾用户。

相反，该公司表示，网站可以使用私人访问令牌(pat)来验证是否有人在访问它们。服务器可以使用 HTTP PrivateToken 身份验证方法请求令牌。因此，服务器只能获得有关验证的信息，而不知道任何面向用户的数据，如 IP 地址。

正如 AppleInsider 解释的那样，苹果使用一个基于 iCloud 的证明者来签署这些令牌，而你设备的 secure enclave 提供一个证书。它还会检查机器人很难模仿的动作，比如用 Face ID 解锁你的 iPhone 或通过该设备上的 Safari 访问网站。

图片来源:苹果

苹果已经与云服务提供商如 Cloudflare 和 Fastly 合作支持 PATs，因此用户可以过上无验证码的生活。Cloudflare 指出，在过去几年中，通过使用其托管挑战平台来识别用户，它已经管理了 91%的验证码使用，现在该平台支持 PATs。

Cloudflare 挑战赛。图片来源:云辉

好消息是 PATs 可以跨平台，因为 Google、Apple、Cloudflare 和 Fastly 都为开发这个协议做出了贡献。然而，Android 端没有实现这一点，所以只有苹果设备用户能够跳过验证码。

在 WWDC 的活动中，苹果宣布了安全和隐私方面的功能，如独立于系统更新的实时安全更新、无需任何密码即可登录服务和隐藏和删除照片的锁定文件夹。
