微软修补影响所有 Windows 版本的新零时差

微软发布了针对一个影响所有受支持 Windows 版本的零日漏洞的安全修复程序，该漏洞已在现实世界的攻击中被利用。

零日漏洞被追踪为 CVE-2022-37969 ，被描述为 Windows 公共日志文件系统驱动程序中的特权提升缺陷，该驱动程序是用于数据和事件日志记录的子系统。该漏洞允许攻击者获得易受攻击设备的最高级别的访问权限，称为系统权限。

微软表示，运行 Windows 11 和更早版本以及 Windows Server 2008 和 Windows Server 2012 的用户会受到影响。Windows 7 也将获得安全补丁，尽管将于 2020 年退出支持。

微软表示，该缺陷要求攻击者已经能够访问受损设备，或者能够在目标系统上运行代码。

“这种性质的漏洞经常被包装成某种形式的社会工程攻击，例如说服某人打开一个文件或点击一个链接，”零日倡议(ZDI)的威胁情报负责人达斯汀·查尔兹说。"一旦他们这样做了，额外的代码以提升的权限执行以接管系统."

微软感谢来自 CrowdStrike、DBAPPSecurity、Mandiant 和 Zscaler 的四组不同的研究人员报告了该漏洞，这可能表明该漏洞在野外被广泛利用。

Mandiant 的高级首席漏洞工程师 Dhanesh Kizhakkinan 告诉 TechCrunch，该公司“在一次主动进攻任务组漏洞搜索任务中”发现了该漏洞，并补充说该漏洞似乎是独立的，不是攻击链的一部分。

微软没有透露利用这一漏洞进行攻击的细节，也没有回应我们的置评请求。

这些修复程序是微软定期每月发布的安全修复程序的一部分，名为 Patch Tuesday，其中包括各种微软产品中的 63 个漏洞，包括 Microsoft Edge、Office 和 Windows Defender。

微软还发布了第二个零日漏洞的补丁，跟踪为 CVE-2022-23960 ，它描述为一个被称为“幽灵 BHB”的缓存推测漏洞，影响基于 ARM 的系统的 Windows 11。Spectre-BHB 是 Spectre v2 漏洞的变种，它可以让攻击者窃取内存中的数据。

本周早些时候，苹果在 iOS 和 macOS 的主动攻击下开始修补零时差。
