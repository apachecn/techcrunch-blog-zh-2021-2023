Mozilla 给 Firefox 带来了免费的离线翻译

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/02/mozilla-brings-free-offline-translation-to-firefox/

Mozilla 为 Firefox 添加了一个官方翻译工具，它不依赖云处理来完成工作，而是在你自己的计算机上执行基于机器学习的过程。对于一项与谷歌和微软这样的巨头紧密相连的流行服务来说，这是一个巨大的进步。

名为 Firefox Translations 的翻译工具可以添加到您的浏览器中。它在第一次翻译一种语言时需要下载一些资源，如果需要，它可能会下载改进的模型，但实际的翻译工作是由你的计算机完成的，而不是在几百英里外的数据中心。

这很重要，不是因为许多人需要在离线时在他们的浏览器中进行翻译——就像潜艇的纱门一样，这不是一个真正有意义的用例——而是因为目标是减少最终对别有用心的云提供商的依赖，以完成不再需要他们资源的任务。

这是欧盟资助的佛手柑项目的成果，Mozilla 与几所大学合作开发了一套机器学习工具，使离线翻译成为可能。通常这种工作是由数据中心的 GPU 集群完成的，在那里大型语言模型(数十亿字节大小和数十亿参数)将被部署来翻译用户的查询。

但是，尽管谷歌和微软(更不用说 DeepL 和其他新贵竞争对手)基于云的工具准确且(由于拥有近乎无限的计算能力)快速，但将数据发送给第三方进行分析并发回存在基本的隐私和安全风险。对于一些人来说，这种风险是可以接受的，而另一些人如果没有必要的话，宁愿不要涉及互联网广告巨头。

如果我谷歌翻译餐前小吃店的菜单，我会成为香肠促销的目标吗？更重要的是，如果有人在翻译已知设备 ID 和位置的移民或医疗论文，ice 会敲门吗？无论在什么情况下，对于那些担心使用云提供商进行翻译会影响隐私的人来说，离线翻译都是有意义的。

相当不错！当然，它在标签标题中将 El País 翻译为“The Paris”，还有许多其他有问题的措辞(尽管它确实将 every |翻译为“哦，这是件好事”——相当滑稽)。但是很少有人能理解要点。

最终，这也是大多数机器翻译的目的:报告基本意思。对于任何一种细微差别或微妙之处，即使一个大型语言模型也不可能复制习语，所以一个真正的双语人是你最好的选择。

主要的限制可能是缺乏语言。谷歌翻译支持一百多种语言——火狐翻译支持十几种语言:西班牙语、保加利亚语、捷克语、爱沙尼亚语、德语、冰岛语、意大利语、挪威语博克马尔语和尼诺斯克语、波斯语、葡萄牙语和俄语。这遗漏了相当多的内容，但请记住，这只是一个非营利组织和一群学者首次发布一个项目，而不是一个数十亿美元的全球互联网帝国的标志性产品。

事实上，创作者正在积极寻求帮助，通过曝光一条训练管道让“爱好者”训练新模型。他们也在征求反馈以改进现有的模型。这是一个可用的产品，但远不是一个成品！
