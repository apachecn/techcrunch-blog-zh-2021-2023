现在预测最近以太坊区块链合并的所有影响还为时过早，但它肯定解决了关于过度能源消耗的最频繁(也是最有效)的批评。批评者可能仍然会找到反对 ETH 的新理由，但我希望这次合并会带来其他东西:我们也有机会将 Web 2.0 的最佳之处与 web3 最令人兴奋的地方合并。

硅谷的裂痕似乎越来越大，传统的 Web 2.0 行业和蓬勃发展的 web3 生态系统被描绘成相互对立的。夹在中间的是新兴的创业公司。

我在这三个群体中都很活跃，而且我相信这种争议大部分是基于风投和其他非开发人员的传道者的疯狂声明和炒作。例如，NFT 滴滴不断的名人促销活动，让人们觉得整个 web3 是一个庞氏骗局。事实上，NFT 只是 web3 生态系统的一小部分，在我看来，甚至不是最有趣或最具变革潜力的。

虽然 Web 2.0 和 web3 可能看起来不兼容，但我认为最好将区块链和 ETH 这样的技术视为所有公司都面临的可扩展性挑战的潜在后端解决方案。同样，web3 的支持者应该认识到 Web 2.0 的成熟使得它对于许多核心用例来说是不可或缺的。

尽管 web3 有巨大的潜力，但开发 Web 2.0 应用程序仍然容易得多，因为生态系统已经成熟，并且拥有一个庞大而繁荣的开发者社区。

为了捕捉现在 web3 开发中发生的事情，我们必须回到 Web 2.0 时代之前。

在网络繁荣时期，有很多关于开源、Linux 和 Red Hat 等热门公司的讨论。虽然很少有消费者会继续安装 Linux 作为他们的操作系统，但这种热潮有助于实现同样重要的事情。在后台，很少有人注意到，Linux 迅速成为运行 100 万个顶级域名中 96.5%的后端服务器的首选操作系统——更不用说巨大的 Android 市场了。
