Stripe 通过数据管道扩展其基础设施业务，与亚马逊和雪花同步财务数据| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/18/stripe-expands-its-infrastructure-play-with-data-pipeline-to-sync-financial-data-with-amazon-and-snowflake/

stripe——价值 950 亿美元的支付巨头——正在进行产品冲刺，以扩展其服务和功能，超越目前构成其业务核心的基本支付。今天，该公司推出了数据管道(wrapse Data Pipeline)，这是一种基础设施产品，可以让用户在他们的 Stripe 交易数据和他们在亚马逊红移或雪花的数据云中保存的数据存储之间建立联系。

此举突显出 Stripe 如何将自己定位为不仅仅是一家支付提供商。它雄心勃勃，要成为一个更大的金融服务和数据中心，用它自己的话说，就是一个“企业金融基础设施平台”。

就在几周前，该公司宣布了金融连接，让 Stripe 客户与他们的客户的银行服务连接，以获取这些用户更完整的金融数据。

迄今为止一直处于封闭测试阶段的 Data Pipeline 已经获得了一些早期客户: ChowNow、Housecall Pro、HubSpot、Lime、Shipt 和 Zoom，Stripe 表示使用它来“自动化下游报告并确定增长机会”。换句话说，支付仍在进行，但现在 Stripe 正在将支付数据结果转化为自己的利润中心。

该产品将使用户能够更全面、更方便地将条带化财务数据与其他业务信息相结合，这反过来将使这些用户能够在更广泛的业务智能工作、财务报告以及监控欺诈、安全问题等业务活动的工作中利用这些数据。

值得注意的是 Stripe 推出了一个基础设施产品，特别是在 ETL(提取、转换、加载)领域，它从内部开始构建，旨在为其用户取代第三方产品。然而，这并不是该公司针对更广泛的企业分析领域的第一款产品:2017 年，该公司推出了跟踪支付数据的工具适马。

“适马允许你在 Stripe 仪表盘中访问/查询你的 Stripe 数据，而数据管道允许你直接在你的雪花或亚马逊红移数据仓库中访问你的 Stripe 数据，” Stripe 的产品负责人 Vladi Shunturov 在一次电子邮件采访中说。"通过这种方式，数据管道可以更容易地结合您的其他业务数据来查询您的条带数据。"

Snowflake 和 Amazon 与其他第三方 ETL 提供商合作，Stripe 拒绝评论与这些特定合作伙伴存在什么财务安排(如果有的话)。该公司还拒绝就是否会将其他数据仓库提供商加入该名单发表评论。

“我们一直在考虑如何扩展我们的服务，更好地为我们的用户服务，但目前没有任何具体的计划可以分享，”Shunturov 说。

他说，通过亚马逊和雪花的整合，Stripe 与这两家公司合作，使用各自的数据共享技术来构建其产品。具体来说，条带启动数据共享，使我们能够将用户的条带数据存储在条带群集中，然后我们向用户提供对此数据的读取权限。“这样，用户可以访问他们的数据，而无需对他们的群集进行写访问，”他说。“我们致力于不断提高数据新鲜度，扩大业务就绪型报告和指标的范围。要实现这一点，我们需要在条带上以本机方式构建这一功能。”

Shunturov 补充说，该产品的动力，也许是该公司如何构建这些新服务浪潮的战略路线图，都源于用户的要求。

他说:“Stripe用户，尤其是较大的用户，已经要求更简单的方法，不仅导出他们的Stripe数据，而且持续地将他们的Stripe数据同步到他们的数据仓库，以便他们可以将他们的Stripe数据与其他业务数据集中在一起，而不必自己构建或维护 API 集成。”。“通过提供产品级报告和指标，我们还显著降低了用户将原始数据转化为业务洞察所需的数据工程投资。由于用户需求高，雪花和亚马逊红移被选为我们的初始发布合作伙伴。事实上，这两个都是在用户群体中使用最广泛的数据仓库。

Data Pipeline 目前只在美国，面向同时使用亚马逊红移或雪花数据云的 Stripe 用户。
