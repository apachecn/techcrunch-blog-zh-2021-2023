digital 希望将银行账户转变为微服务 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/04/numeral-wants-to-turn-bank-accounts-into-microservices/

来认识一下数字，一家想要升级公司银行账户的法国初创公司。当客户使用现代应用编程接口(API)与 digital 进行交互时，这家初创公司直接连接到银行服务器，上传支付文件，并与过时的信息系统进行交互。通过抽象这一层复杂性，您可以将您的银行账户视为您架构中的另一个微服务。

上个月，digital宣布获得了由 Balderton Capital 领投的 1480 万美元(€1300 万)的融资。亚历山大·普罗特、汤姆·布洛姆菲尔德、纪尧姆·普林森和基马风险投资公司也参加了会议。digital 团队最初是在 Logic Founders的项目中开始工作的，这是一个由 eFounders 创建的创业工作室。

描述数字的最好方式是描述它不是什么。digital 不是一个面向消费者应用的开放式银行聚合器。它不与叮叮、 TrueLayer 或 Yapily 竞争。

digital 也不是银行即服务提供商。该公司不提供银行账户，不生成 IBANs，也不发行卡。

“我们是科技公司的支付自动化平台，”联合创始人兼首席执行官爱德华·曼东(douard Mandon)告诉我。“我们让科技公司连接到他们的银行账户，以自动化支付操作。”

虽然零售银行刚刚开始提供 API，但企业银行在许多年前就开放了他们的银行平台。但是不要期望 REST API 有文档页面。许多银行希望你上传一个文本文件到 SFTP 服务器。该文件也应该以一种非常特殊的方式进行格式化。

digital 将其产品出售给严重依赖银行转账的金融科技、保险科技或房地产公司。例如，该公司的第一批客户是斯彭代斯克和 T2 斯维尔。digital 已经为其首批客户创建了集成，因此 Spendesk 和 Swile 可以使用 API 与他们的银行账户进行交互。

到 2022 年底，digital 计划为十几家不同的银行提供服务。“现在，我们一半的客户通过一家法国银行发现了我们的服务，该银行将 Right 描述为他们不提供的 API，”Mandon 说。

集成完成后，数字用户可以将支付功能和特性集成到他们的应用程序中。这家初创公司还为非技术人员提供了一款网络应用。通过这种方式，他们可以核对支付和账户，而不必使用企业银行提供的传统 web 应用程序。

然后，digital 可以在其 API 之上添加一些额外的特性。例如，您可以设想建立一个审批工作流、一个通知系统等。

这家初创公司还在考虑编排功能。如果一个客户有多个银行账户，他们可以根据几个规则将付款转到正确的账户。digital 还可以用于主动管理多个账户的现金余额。

这对在多个国家拥有账户的全球客户尤其有用。曼登在进入 Numeral 之前曾在 iBanFirst 工作，因此他对在多个国家拥有多家合作银行略知一二。

随着融资的进行，digital 计划发展到一个 30 到 40 人的团队。除了与法国银行进行新的整合，该公司还计划将其覆盖范围和客户群扩大到其他欧洲国家，如德国、英国、西班牙和意大利。
