Onymos 筹集 1200 万美元为应用程序提供插件功能

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/14/onymos-raises-12m-to-provide-plug-in-features-for-apps/amp/

图片来源: 乔格·格雷埃尔 /盖蒂图片社

Onymos ，一个“功能即服务”的应用程序开发平台，今天宣布完成了一轮 1200 万美元的 A 轮融资，由 Great Point Ventures 领投，Benhamou Global Ventures、Engineering Capital 和 Industry Ventures 参与。首席执行官 Shiva Nathan 表示，这些资金将用于产品开发和加强 Onymos 的上市活动，以及产品和服务开发。

内森通过电子邮件向 Onymos 推销:“软件即服务的蔓延给企业及其应用带来了复杂性和缺乏可见性，”他告诉 TechCrunch。“对于一个企业工程团队来说，平衡已经从不得不长期从少数几家供应商中选择转变为被无数的选择所淹没。即使在选择了服务之后，每个应用都受到价值数万亿美元的公司的支配，这些公司迟早会弃用这些服务，改变它们的实现方式，或者引入与您当前构建环境不兼容的更新。我们希望改变这种模式。”

在内森看来，应用程序开发既耗时又费钱。根据 2018 年 Stripe 调查显示，普通开发人员每周花费超过 17 个小时处理调试和重构等维护问题。此外，Stripe 估计，他们每周大约花四个小时在“坏代码”上，这相当于全球每年损失近 850 亿美元的机会成本。

图片来源:奥尼莫斯

Onymos 旨在通过提供现成的功能来帮助开发人员减轻一些工作，这些功能可以嵌入新的和现有的应用程序中。该平台提供登录、生物识别、聊天、数据存储、位置服务和通知模块，包括必要的组件，包括在云中处理数据所需的用户界面、底层逻辑和服务器端功能(例如，像亚马逊网络服务、微软 Azure 和谷歌云平台这样的公共云)。

Onymos 承诺的免维护、完全安全的应用程序开发未来可能好得难以置信——每个平台，无论多么全面，终究会有缺陷。但像 Onymos 这样的服务可以提供的是花在其他更重要的产品 R&D 任务上的时间，至少在理论上是这样。

“公司可以让他们的软件开发人员花时间在 Meta 的 Graph API for 脸书登录中调试角落案例，或者利用他们的创造力和洞察力来构建真正的增值功能，帮助他们的雇主取得成功。我们认为选择是明确的，”内森说。“on ymos”…平台作为软件供应商的集成层，允许客户在单个端点上进行构建，从而消除了应用开发的复杂性。当企业使用 Onymos 功能时，他们不必担心操作系统和 API 更新、设备兼容性或维护。"

“功能即服务”是一个古老的概念，至少可以追溯到几年前。有用于社交的 Localytics 和用于定位的地图框。说书人让任何人都可以在自己的应用或网站上添加 Instagram 风格的故事。同时， WorkOS 为开发者提供了单点登录和应用目录同步等企业功能。

内森认为扩张——特别是在机器学习领域——是 Onymos 脱颖而出的途径。

“我们计划将我们的产品组合扩展到机器学习领域，引入新的端到端功能，并支持更多的开发框架，”内森说。“我们在与医疗保健领域的客户合作中发现，他们非常需要光学字符识别和数据提取。对于企业来说，这些流程仍然太耗时，现在有机会引入新的效率。”

Onymos 目前有 30 名员工，预计到年底将达到 40 至 60 人。迄今为止，该公司已经筹集了 1500 万美元。
