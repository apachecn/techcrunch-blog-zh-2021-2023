总部位于奥斯陆的 nabs 斥资 1400 万美元建立开源功能管理工具 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/24/oslo-based-unleash-nabs-14m-series-a-to-build-open-source-feature-management-tool/

当开发人员在软件中构建新功能时，他们必须确保新功能不会破坏已有的功能。因此，他们通常会慢慢推出它，并使用在代码中放置功能标志的技术来看看会发生什么。这是一种经过验证的控制首次展示的方法，但它需要另一个层次的管理。这就是总部位于奥斯陆的 Unleash 的用武之地。

“Unleash 是一个功能管理解决方案。它可以让您定义细分用户的规则，以便您可以完全控制如何启用新功能，”公司联合创始人兼首席技术官 Ivar sthus 解释道。他说，这种方法可以进行更多的实验，同时降低将某些东西放入产品代码库的风险，这将为用户带来重大问题。

2014 年，sthus 在他工作的公司 Finn 遇到了这个功能管理的问题。所以在业余时间，他开始研究解决方案。第二年，他在 GitHub 上开源发布了《释放 T2 》,并在接下来的几年里继续从事他的工程师工作，培育这个项目。

他发现，随着时间的推移，公司开始要求该工具的企业功能，他看到了商业投资的机会。到 2019 年，Ivar 和他的兄弟 Egil 决定一起创办一家公司，在 Ivar 开发的开源产品之上建立一个商业层。

“到 2018 年底，我开始看到开源采用的势头很大，人们开始询问企业功能。我甚至收到了一封来自一家公司的电子邮件，问我是否可以为他们托管它，这样他们就不必这么做了。所以对我来说，这显然是一个早期信号，这也是我引进埃吉尔(创办公司)的时候，”他说。

该公司于 2019 年春天推出了第一款商业产品——主机版的“释放”,该业务开始初具规模。这两个人辞去了日常工作，全心全意为公司服务。

成为首席执行官的 Egil 拥有管理和商业背景，两兄弟开始与公司交谈，看看这个想法作为商业业务的可行性如何。“我认为我们花了差不多 90%的时间与客户交谈，以更好地了解我们的竞争对手，也了解我们用户的需求，”Egil 说。

如今，该公司已逐渐从纯开源模式转向开放核心模式，为需要额外安全保障的客户提供单点登录等企业功能。

Egil 说，多元化对他和他的兄弟来说是一个重要的问题，因为他们建立了公司，也是他们的关键 KPI 之一。他认为这是对领导力的挑战，为了让他们负起责任，他们每月都要向董事会报告，从候选人的角度来看，他们做得有多好。

“因此，我们决定做的真正重要的事情是每月向董事会报告我们候选人渠道的多样性。因此，这是我们将向他们报告的指标之一，这意味着每当我们填补候选人管道时，我们需要报告这些数字，以及我们是在这些多样性数字之内还是之外，”他说。

该公司有 120 名付费用户，其中约 40%在美国和加拿大，另外 40%在欧盟，其余分布在世界各地。他们有 16 名员工，计划在今年年底前增加一倍。

今天宣布了 1400 万美元的 A 轮融资，随着业务开始起飞，这家初创公司应该有资本进行投资。今天的投资由 Spark Capital 领投，Frontline Ventures、firstminute capital、Alliance Venture 和 Arkwright X 参与了投资。
