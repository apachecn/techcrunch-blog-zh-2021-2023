Sigstore 为开源项目推出免费软件签名和验证服务

软件供应链在过去几年中迅速成为热门话题，尤其是随着高调攻击的数量增加以及白宫卷入其中。 Sigstore ，一个由 Google、GitHub、Chainguard 和 RedHat 等支持的开源项目，已经成为签署、验证和保护软件项目的标准——以及它们使用的依赖——以确保你在机器上安装和运行的软件没有被操纵。毕竟，如今没有多少软件项目不依赖于至少一个——通常是多个——开源库，而开源库本身可能也依赖于其他库。由于这些项目中有许多是由志愿者维护的，他们很容易成为黑客的目标。

今天，在底特律 CNCF kube con/CloudNativeCon 大会的同地活动 SigstoreCon 上，Sigstore 社区宣布其开源项目的免费软件签名服务全面可用。Sigstore 已经是有史以来最快被采用的开源项目之一，目前已经有超过 400 万个签名。Kubernetes 和 Python 社区都使用它来签署他们的发布。流行的 JavaScript 包管理器 npm 目前正在集成 Sigstore，以确保其包的出处。

图片来源: Sigstore

Sigstore技术指导委员会成员、Chainguard 软件工程师 Priya Wadhwa 表示:“Sigstore 已经迅速成为签署、验证和保护软件的标准，因此，在软件供应链安全比以往任何时候都更加重要的时候，宣布 Sigstore 的全面上市为更广泛的采用消除了最后一个障碍，这真是太好了。“我们希望, Sigstore 的下一阶段将增强开源软件生态系统的其余部分的能力，以增强采用该技术的信心，并从其可靠和稳定的体验中受益。”

Sigstore 社区承诺 99.5%的正常运行时间和寻呼机支持——超过了大多数免费项目所能提供的。值得注意的是，Sigstore 是一个由开源安全基金会资助的非营利性项目。Sigstore 本身由许多用于签名容器的项目组成，将信息保存在的不可变分类帐中，当然，首先创建那些证书。
