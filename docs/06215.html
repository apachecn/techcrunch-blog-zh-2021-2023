大家好，金融科技领域经历了过山车般的一周，我分别发表了两篇关于初创公司裁员的文章，并在几个小时内完成了一轮九位数的融资。这也是 FinTwit 上充满大量活动的一周，fin twit 或“金融推特”，因为它更正式地被称为——感谢不仅仅是关于条纹和格子的一些来回。所以，拿起你的爆米花，坐一会儿，我会试着为你分解它。想在每周日早上收到这封邮件吗？报名这里！

本周早些时候，我开玩笑说，当我开始报道金融科技时，我预计这将是一个相当枯燥和低调的节奏。现在我嘲笑我的天真。

或许本周金融科技领域最大的新闻是 Stripe 推出了其新的金融连接产品，TC 的 Ingrid Lunden 在这里报道了。是的，产品发布本身就有新闻价值。但是提升它在新闻价值世界中的地位的是它引发了一些争议，因为它几乎正是 Stripe 的前合作伙伴 Plaid 所做的。那就是，它为 Stripe 的客户提供了一种直接连接到其客户银行账户的方式，以访问财务数据来加速或运行某些类型的交易。这也是格子花呢的作用。

在一条后来被删除的推文中，Plaid 首席执行官兼联合创始人扎克·佩雷特(Zach Perret)回复了 Stripe 总理杰伊·沙阿(Jay Shah)的一条推文，从本质上质疑了 Stripe 可能收集产品信息的“方法”。沙阿用自己的一条微博回应了那条微博，为他和他的公司的行为辩护。

几个小时后，Perret 在 Twitter 上承认他已经删除了他的推文，并指出:“删除推文。也许是误解或不同的风格。推定积极的意图。”

与此同时，在 Stripe 内部，高管们用一份内部备忘录解决了这场纠纷。具体来说，帕特里克·科利森说他对 Stripe 新产品的“热情”被 Perret 的“指控”所“冲淡”显然，他很受伤，因为 Plaid 可能会对 Stripe 披露这款竞争产品感到有点不安，即使在两家公司之前曾合作进行集成之后。

嗯。

他在内部报告的结尾承认，Stripe 应该“肯定有可能”把事情处理得更好。他承认这一点很好，但也很难相信这些高管们不知道这一举动会导致紧张局势。帕特里克甚至继续说，也许条纹应该给格纹一个提醒，“这样他们就可以私下表达他们的任何担忧。”他补充说，虽然 Stripe 不一定有义务这样做，但如果它早点告诉 Plaid，它可能会避免随之而来的公开辩论。

与此同时，帕特里克的兄弟和联合创始人约翰发推文称，佩雷特删除了他最初的推文，真是“仁慈”。他补充说:“我们理解他对整个事情的观点可能仍然不同。不管怎样，我们仍然用格子做很多东西。他们是一家伟大的公司，我们期待找到更多的合作方式。”

我联系了两家公司，询问他们各自的看法，但两家公司都拒绝接受采访。的确，我们可能永远也不会知道在这个特殊的事件中到底发生了什么。但我所知道的是，这场争议引发了一场完全不同的对话，包括有人声称这不是 Stripe 第一次被指控行为不谨慎。这些包括【未经证实的】指控该公司之前假装有兴趣收购其他公司或雇佣人员试图榨取他们的信息。据报道，当 Stripe 向投资者红杉施压，要求其放弃一项有竞争味道的投资时，这件事再次浮出水面。

我在这里不是要做出任何判断，因为这个故事可能仍在上演，我们还不知道什么是真的。也就是说，我的拙见是，无论你是多么大或富有，或多么小或不富有，不道德的行为都是不值得的。我宁愿不那么富有，知道我对与我打交道的人做得对，也不愿意富有，让我的诚信一再受到质疑。但这只是我。
