研究发现，代码生成人工智能会引入安全漏洞

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/28/code-generating-ai-can-introduce-security-vulnerabilities-study-finds/amp/

图片来源: monsitj / Getty Images

最近的一项研究发现，使用代码生成人工智能系统的软件工程师更有可能在他们开发的应用程序中造成安全漏洞。这篇论文由斯坦福大学的一组研究人员共同撰写，强调了代码生成系统的潜在陷阱，因为 GitHub 等供应商开始认真营销它们。

“代码生成系统目前无法取代人类开发人员，”斯坦福大学的博士生、该研究的主要合著者尼尔·佩里(Neil Perry)在接受电子邮件采访时告诉 TechCrunch。“使用它们来完成自己专业领域之外的任务的开发人员应该有所顾虑，那些使用它们来加速他们已经擅长的任务的开发人员应该仔细检查输出以及它们在整个项目中的使用环境。”

斯坦福大学的研究特别关注由旧金山研究实验室 OpenAI 开发的人工智能代码生成系统 Codex。(Codex 为 Copilot 提供动力。)研究人员招募了 47 名开发人员——从本科生到拥有数十年编程经验的行业专业人士——使用 Codex 来完成跨编程语言(包括 Python、JavaScript 和 c)的安全相关问题。

Codex 对数十亿行公共代码进行了训练，以在给定现有代码的上下文的情况下建议额外的代码行和功能。该系统响应于对开发者想要完成的内容的描述(例如“Say hello world”)，利用其知识库和当前上下文来呈现编程方法或解决方案。

根据研究人员的说法，与对照组相比，能够访问 Codex 的研究参与者更有可能写出不正确和“不安全”(从网络安全的角度来说)的编程问题解决方案。更令人担忧的是，与控制组的人相比，他们更有可能说他们不安全的答案是安全的。

斯坦福大学的研究生 Megha Srivastava 是这项研究的第二位合著者，她强调说，这些发现并不是对 Codex 和其他代码生成系统的彻底谴责。首先，这项研究的参与者并不具备能够让他们更好地发现代码漏洞的安全专业知识。除此之外，Srivastava 认为代码生成系统对于风险不高的任务确实有帮助，比如探索性研究代码，并且可以通过微调来改进它们的编码建议。

Srivastava 说:“开发自己的[系统]的公司，也许会在内部源代码方面得到进一步培训，可能会更好，因为这种模型可能会被鼓励产生更符合他们的编码和安全实践的输出。”

那么，像 GitHub 这样的供应商如何防止开发人员使用他们的代码生成人工智能系统引入安全漏洞呢？合著者有一些想法，包括一种机制来“精炼”用户的提示，使其更加安全——类似于主管检查和修改粗略的代码草案。他们还建议密码术库的开发者确保他们的默认设置是安全的，因为代码生成系统倾向于坚持默认值，这并不总是没有漏洞。

“人工智能助理代码生成工具是一项非常令人兴奋的开发，可以理解这么多人渴望使用它们。这些工具带来了需要考虑的问题，尽管…我们的目标是对代码生成模型的使用做出更广泛的声明，”Perry 说。"在探索这些问题和开发解决这些问题的技术方面，还需要做更多的工作."

佩里认为，引入安全漏洞并不是代码生成人工智能系统的唯一缺陷。Codex 培训的代码中至少有一部分处于限制性许可之下；用户已经能够提示 Copilot 从 Quake 中生成代码，从个人代码库中生成代码片段，从《掌握 JavaScript》和《思考 JavaScript》等书中生成示例代码。一些法律专家认为，如果公司和开发人员无意中将 Copilot 中受版权保护的建议整合到他们的生产软件中，Copilot 可能会让他们面临风险。

GitHub 试图纠正这一点，它是一个过滤器，于 6 月首次引入 Copilot 平台，根据公共 GitHub 代码检查代码建议及其周围约 150 个字符的代码，如果匹配或“近似匹配”，则隐藏建议。但这是一个不完善的措施。德克萨斯 A&M 大学的计算机科学教授 Tim Davis 发现，启用过滤器会导致 Copilot 泄露大量他的版权代码，包括所有的属性和许可文本。

“由于这些原因，我们很大程度上对使用这些工具来取代对初级开发人员进行强编码实践的教育表示谨慎，”Srivastava 补充道。
