Confluent 推出可视化流数据管道设计器| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/10/04/confluent-launches-visual-streaming-data-pipeline-designer/

在开源 Apache Kafka 项目基础上建立流媒体服务的公司 Confluent ，一直致力于帮助公司捕捉数据流。先是在 prem 上，后来在云中。通过将流媒体服务迁移到云中，it 部门能够抽象出许多与管理底层基础架构相关的复杂性。

今天，在当前会议上，该公司推出了一款名为 Stream Designer 的新工具，可以更容易地在可视化工作流中构建流数据管道。用户可以轻松地连接一组数据组件来构建定制的数据流，Confluent 在后台为他们处理编码，本质上是将抽象从基础设施提升到设计。

公司联合创始人兼首席执行官 Jay Kreps 表示，虽然它仍然针对开发人员，但目标是让更多人能够接触到 stream design，而不是 Apache Kafka 专家。

“我们仍然为各种类型的开发人员提供服务，但这使得您只需点击一下，就可以建立一个连接事物、转换数据、将事物从一个地方发送到另一个地方的数据管道。它继承并运行在相同的底层流基础设施上。这意味着所有的数据流都是可重复使用的，它们都是实时的，它们都是可水平扩展的，”克雷普斯告诉 TechCrunch。

他说，尽管该程序为开发人员编写底层代码，但需要使用它的开发人员仍然可以访问它。“我们这样做的一个很酷的地方是，即使你不需要写任何代码，所有的代码都在那里。你可以清楚地看到它在引擎盖下做什么。你可以确切地看到我们将对这些数据进行的 SQL 转换，所以如果你需要这样做，它可以与现有的开发人员工具链一起工作，”他说。

汇流流设计器示例。图像来源:汇合

对于 Kreps 来说，这是 Kafka 发展的一部分，Kafka 是 LinkedIn 最初构想的一个移动海量数据的项目。起初，这是一项高度技术性的工作，但随着时间的推移，该公司一直在努力让组织内更多的人能够越来越多地接触到它。

除了 Stream Designer，该公司还宣布了 Stream Governance，这有助于确保数据得到正确使用，只有授权用户才能看到数据。

“对于许多组织来说，几乎有两种相互竞争的压力:一种是释放数据，利用数据提高效率，更好地为客户服务，提高效率。另一个是把它锁起来，确保安全，不要让任何不好的事情发生。除非你有工具在这两个方面都有帮助，否则你会有点停滞不前。流治理有助于用户确保他们以安全和合规的方式使用流经这些数据流的数据。

在此期间，Confluent 宣布了一项新的 Confluent for Startups 计划，旨在让初创公司使用该平台，获得免费积分和专业知识，以帮助他们开始使用数据流技术。

他说，Stream Designer 和 Stream Governance 从今天开始作为融合云服务订阅包的一部分提供，新功能不会额外收费。
