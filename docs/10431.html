Web3 bug-bounty 平台 Immunefi 为其首轮融资筹集了 2400 万美元

大约在去年的这个时候，我们报道了T2】Immunefi——DeFi 的新兴 bug 赏金和安全服务平台之一——已经筹集了 550 万美元的资金。考虑到今年到目前为止，黑客和诈骗已经在加密领域损失了近 20 亿美元，这看起来是一笔相当低的投资。

果然是这样。因为 Immunefi 现在已经筹集了 2400 万美元，作为其 a 轮的一部分。该轮由 Framework Ventures 领投。其他投资者包括三星 Next、Electric Capital 和 Polygon Ventures。这使得现在的筹款总额达到了 2950 万美元。

Immunefi 将需要检查代码和保护代码的 web3 项目与 whitehat 黑客联系起来，这些黑客报告漏洞并要求金钱奖励。有时这些奖励可能高达 1000 万美元——当如此多的加密货币处于危险之中时，这并不奇怪。包括苹果和微软在内的大多数科技公司都使用类似的 bug bounty 方法，但这种做法在 web3 中没有得到很好的应用，部分原因是黑客有时更有动力去偷钱而不是报告 bug，特别是当数百万美元可能被提供时。

Immunefi 于 2020 年 12 月推出，它表示已经向 whitehat 黑客支付了 6000 万美元，并声称已经从被黑客攻击中挽救了超过 250 亿美元的资金。

但是 crypto 中的 bug 补偿必须以不同于 Web 2.0 的方式工作。在 1 亿美元的资金可能岌岌可危的情况下，支付 5000 美元是微不足道的。因此 Immunefi 开发了一个 bug 奖励标准，鼓励项目为大漏洞支付相当于潜在风险基金 10%的奖励。

这意味着一些巨大的漏洞奖金——例如为在虫洞(一种通用的跨链消息协议)中发现的漏洞支付的 1000 万美元，以及为在极光(一种以太坊的桥接和扩展解决方案)中发现的漏洞支付的 600 万美元。这与苹果公司提供的 200 万美元的最大常规 bug 奖金形成了对比。

首席执行官兼创始人米切尔·阿玛多在一份声明中说:“开放代码和直接货币化的利用使 Web3 成为世界上最具对抗性的软件开发空间。通过向 whitehats 转移激励，Immunefi 已经节省了数十亿美元的用户资金。跨越 crypto 的项目很快意识到，使用 Immunefi 比公开乞求黑客返还资金或支付赎金更好。我们利用这次加薪来扩大我们的团队，以应对这一巨大挑战。

然而，Immunefi 确实有竞争对手；HackerOne 从 Web 2.0 转换到 web3，Safeheron 最近筹集了700 万美元来使私钥更加安全。
