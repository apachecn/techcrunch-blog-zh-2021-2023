Docker 发布了其 WebAssembly 工具的第一个预览版

Docker 仍然存在，并且很可能在做得更好——至少在财务方面——超过了它早期的宣传周期，该周期开启了容器革命(后来被 Kubernetes 及其生态系统超越)。今天，该公司宣布了其 WebAssembly (Wasm)支持的首次技术预览。

浏览器厂商首创 Wasm 以原生速度运行 web 应用，代码由 C、C++、Rust 等语言编译，运行在安全的沙箱中。目前，您可以将大约 40 种语言编译到 Wasm。但与 node.js 将 JavaScript 带到服务器类似，Wasm 现在也在向后端迁移。例如，Cloudflare 在其边缘计算服务中支持它。

随着风投们开始意识到这一潜力，我们也开始看到这一领域的一些融资回合，例如，Cosmonic 今天宣布为其新的 Wasm PaaS 提供 850 万美元的融资回合。本月早些时候，Fermyon 宣布了 2000 万美元的 A 轮融资。Docker 显然也想成为这一领域的先行者。

该公司指出，这在很大程度上仍然是一个技术预览，事情可能会打破。在这种情况下，Docker 引擎使用与 Docker 生态系统的其余部分相同的 containerd 容器运行时，但它没有使用 runc 来运行容器进程，而是使用 WasmEdge 运行时。虽然 Docker 在这里没有详细说明，但 WasmRdge 的承诺是，与 Linux 容器相比，它提供了明显更快的启动时间，而且 WasmEdge 应用程序明显更小(并且运行更快)。

图片来源: Docker

“我们认为 Wasm 是对 Linux 容器的补充技术，开发者可以选择使用哪种技术(或者两者都用！)取决于使用案例，”Docker 的 Michael Irwin 在今天的公告中写道。“随着社区探索 Wasm 的可能性，我们希望使用您熟悉和喜爱的经验和工具，帮助 Wasm 应用程序更容易开发、构建和运行。”

除了产品新闻，Docker 今天还宣布它将加入字节码联盟，这是一个支持 WebAssembly 和 WebAssembly 系统接口的非营利组织，使这些新项目成为可能，成为投票成员。
