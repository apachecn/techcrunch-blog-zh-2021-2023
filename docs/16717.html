Hygraph 筹集了 3000 万美元来扩展管理数字内容的新联合方法| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/29/hygraph-raises-30m-to-scale-out-a-new-federated-approach-to-managing-digital-content/

数字内容和我们消费数字内容的方式在不断发展，帮助在幕后管理所有这些内容的技术也在不断发展。

在最新的发展中，柏林的一家初创公司 Hygraph 已经围绕一种新的“联合”内容管理方法建立了一个平台——想想看:许多数据源和许多使用它的端点，并使用可组合的架构，从一个平台协调所有这些——已经筹集了 3000 万美元。这家初创公司现在有大约 400 个客户，其中包括三星、飞利浦和快速消费品企业集团奥特克尔博士(Dr Oetker)等主要品牌，计划利用这笔资金继续开发该平台，以及如何和在哪里使用它，并扩展到新的地理区域。

这轮融资是 B 轮融资，由欧洲投资者 One Peak 牵头，之前的支持者 OpenOcean 和 SquareOne 以及新的个人投资者 Boris Lokschin(Spryker Systems 的联合创始人兼首席执行官)也参与了。这家初创公司此前筹集了 1300 万美元。该公司没有披露其估值。

许多初创公司已经发现了改善组织创建和管理在线内容的机会。其中一些专注于特定的垂直行业，如 Shopify，而其他如 Wix 已经建立了框架，以快速启动和运行网站。新一代的“无头”系统可以更加灵活地控制界面，同时还可以在后端处理潜在的大量数据。甚至还有一些新的参与者，如 Vue Technologies ，他们采用开源方法来改善前端管理。Hygraph 代表了这个故事的另一个篇章。

它的定位有点像这样:当涉及到数字内容时，在过去的 15 年里，情况发生了巨大的变化。我们不仅有过多的屏幕，从传统的电视和电脑到电话、手表等等，而且我们在这些屏幕上看到的格式也在不断地分裂和演变。用于创建和管理数字内容的数据源也是如此。

虽然市场上已经有许多服务来解决这种情况，但没有一个平台可以整理体验，并将后端和前端资源和数据管理挑战集中到一个地方。这就是 Hygraph 的目标；它称其方法为“联合内容”

首席执行官兼联合创始人迈克尔·卢卡斯奇克(Michael Lukaszczyk)在一次采访中说，“我们正在无头转向。“我们不仅是前端不可知论者，也是后端不可知论者。我们可以轻松地将后端和前端集成到一个通用内容 API 中。”

这样做的一个很大的好处是，通过围绕 API 构建所有这些，您可以更有效地更新信息并将其发送到其他系统，基本上是实时的，这使得系统随着时间的推移工作得更好、更便宜。Lukaszczyk 承认，对于基本网站来说，这可能是一种“压倒性”的方法，但对于大型组织来说，这是处理大量流量和多平台、市场、社交媒体网站等多种体验的必要方法。这可能有一个电子商务的重点，但同样适用于管理视频等媒体机构。

该公司最初被称为 GraphCMS:它的框架基于 GraphQL，以及它是如何围绕可组合架构的概念构建的。去年九月，它还发布了一个 API ,该 API 对 Hygraph 在哪里以及如何使用进行了增压。

“尽管可组合企业兴起，但许多组织无法从其内容中释放出全部价值，”One Peak 的联合创始人兼管理合伙人大卫·克莱因(David Klein)和董事塔季扬娜·卡斯特(Tatjana Kast)在一份联合声明中表示。“Hygraph 解决了定制和集成方面的挑战，这些挑战继续减缓新的或复杂的数字服务的大规模创建。我们对 Hygraph 的强劲和资本高效的增长轨迹印象深刻，并相信其联合内容平台提供了一个相当大的市场机会。One Peak 非常高兴能够支持优秀的 Hygraph 团队及其创始人进入下一个爆发式增长阶段。”
