Cloudflare 阻止了一次大规模的 2 Tbps DDoS 攻击| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/15/cloudflare-terabits-ddos-attack/amp/

图片来源:布莱斯·德宾/ TechCrunch

Cloudflare 表示，它已经阻止了一次分布式拒绝服务( DDoS )攻击，峰值不到 2 Tbps，这是有史以来最大的攻击之一。

互联网公司在博客文章中表示，攻击是由大约 15000 个机器人发起的，这些机器人在被利用的物联网(IoT)设备和未打补丁的 GitLab 实例上运行原始 Mirai 未来组合代码的变种。

DDoS 攻击发生在 Rapid7 警告 GitLab 漏洞的两周前，git lab 漏洞在 CVSS 严重性等级上被评为 10.0 级，攻击者可以利用该漏洞在受影响的服务器上远程运行代码，如僵尸网络恶意软件。Rapid7 发现，60，000 个面向互联网的 GitLab 实例中至少有一半仍未打补丁，并警告说，随着漏洞细节的公开，预计“利用将会增加”。

该公司没有错；Cloudflare 表示，仅一周后就阻止了大规模 DDoS 攻击。根据对攻击的分析，Cloudflare 认为这是一种多向量攻击，结合了 DNS 放大攻击和 UDP 洪水。

Cloudflare 表示，持续不到一分钟的攻击是迄今为止它目睹的最大规模的攻击。就在一个月前，微软表示，它缓解了针对其欧洲 Azure 客户之一的“破纪录”2.4 Tbps DDoS 攻击。

虽然 Cloudflare 在几秒钟内缓解了攻击，但它警告说，上个月它目睹了多起太比特级的 DDoS 攻击，并补充说这不太可能是一种很快就会放缓的趋势。

Cloudflare 产品经理 Omer Yoachimik 表示:“我们在第三季度 DDoS 趋势报告中的另一个重要发现是，网络层 DDoS 攻击实际上比上一季度增长了 44%。“虽然第四季度尚未结束，但我们再次看到了针对 Cloudflare 客户的多起 TB 级攻击。”

Rapid7 已经敦促 GitLab 用户尽快更新 GitLab 的最新版本。“此外，理想情况下，GitLab 不应该是面向互联网的服务，”该公司补充道。"如果你需要从互联网访问你的 GitLab，考虑把它放在 VPN 后面."
