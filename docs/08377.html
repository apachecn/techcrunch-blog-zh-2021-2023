亚马逊推出新的 Alexa 开发工具

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/20/amazon-debuts-new-alexa-developers-tools-including-support-for-multiple-voice-assistants/

今天，在一场专注于 Alexa 开发者和合作伙伴新闻的活动中，亚马逊宣布了新功能和集成，包括一系列旨在使 Alexa 更加通用的 API 和软件开发工具包(SDK)。一个月前，该公司在拉斯维加斯举行的年度 re:Mars 会议上展示了即将推出的实验性 Alexa 功能，包括将短音频剪辑合成较长语音的能力。

今天活动的一个亮点是亚马逊通用设备命令(UDC)和代理传输(AT)的预览，这两项技术旨在简化在同一设备上与多个语音助手交互的任务。在支持 Alexa 和另一个语音助手的设备上，如 Sonos 的 Sonos Voice Control，UDC 将允许用户使用兼容的唤醒词(例如，“嘿，Sonos”)说出命令(例如，“调高音量”)，即使目标助手最初不是用来发起请求的。与此同时，ATs 将允许语音助手在其他助手没有能力满足用户请求时，将用户请求(例如，“要求 UberEats 下单”)转移给其他助手。

两年前，亚马逊在一份白皮书中公布了其 UDC 和 AT 计划，该白皮书概述了对语音互操作性倡议(VII)成员的设计建议，该公司的计划旨在确保支持语音的产品配备多个语音助手(不出所料，包括 Alexa)。包括杜比和脸书在内的 VII 成员同意开发与其他人“无缝”合作的语音助手，并开发至少配有一个助手而不是自己助手的设备。

亚马逊表示，它将在明年为 UDC 和 ATs 启用“云端”基础设施。在对未来的展望中，Skullcandy 和技术咨询公司 Native Voice一起合作使 Alexa 和“嘿，Skullcandy”命令可以在 Skullcandy 的 Push Active 和 Grind 系列耳机上同时使用。

UDC 和 at 的成就是有限的，因为谷歌和苹果目前拒绝加入 VII。但即使没有 Siri 或谷歌助手，亚马逊也证明了 UDC 和 at 可以通过结合多种助手的功能来提供卓越的体验。

智能家居工具

在一个不太相关的发展中——至少在与智能设备有关的意义上——亚马逊推出了 Alexa Connect Kit(ACK)SDK for Matter，这是一个软件包，将现有的 ACK 扩展到支持 Matter Wi-Fi 连接标准的设备。ACK 于 2019 年推出，运行固件，支持设备与 ACK 管理的服务(如日志记录、使用指标和无线更新)之间的通信。从本质上讲，ACK(或者说 ACK for Matter)允许公司使用亚马逊和 Alexa 云服务，同时在提供这些服务的硬件上拥有自由。

用于 Matter 的 ACK SDK 包括亚马逊的“无挫折设置”(FFS)技术，该技术让智能家居设备在开机的一分钟内自动连接到本地网络。关于 FFS，亚马逊表示，它计划从 Matter 设备开始，将 FFS 从通过亚马逊购买的设备扩展到通过第三方零售商购买的设备。

亚马逊还表示，它将很快推出一个凭证 API，允许选择加入的客户与 Alexa 和其他设备制造商的应用程序和服务共享他们的网络凭证。当这一功能可用时，客户将能够选择与他们的哪些设备和服务提供商共享网络详细信息，并设置这些设备，而无需输入 SSID 或密码。亚马逊表示，该 API 将支持 Matter 设备的凭证，并随后提供额外的扩展支持。

Alexa Ambient Home Dev Kit 也于今天发布，提供针对智能家居用例的 API 和服务，包括设备和组同步、安全和安全性以及多管理员设置。(多管理员允许多个智能家居系统同时控制 Matter 设备。)正如亚马逊在博客中解释的那样:

例如，如果客户离开家，启动了他们的安全系统，他们希望机器人吸尘器知道他们走了，这样它就可以开始清洁。如果他们打开 smart lights 应用程序，将客房重命名为办公室，他们希望这一变化能够级联到其他系统，包括 Alexa。…环境家居开发套件将使这些体验以及更多。

Ambient Home Dev Kit 中的新 API 将亚马逊的声音识别安全功能 Alexa Guard 扩展到第三方应用程序，允许客户在检测到烟雾和一氧化碳警报等声音时通过定制应用程序收到警报。其他 API 允许客户向开发人员公开(有点令人毛骨悚然)他们家的状态，比如“家”、“假期”、“晚餐时间”。亚马逊表示，通过使用房屋内各种设备的信号，它可以确定最有可能发生的事情，并将这些信息传递给第三方服务、设备和应用程序——例如，在“睡眠”状态下调节恒温器和锁门。

新 SDK

Ambient Dev Home Kit 与 Alexa Voice Service (AVS) SDK 3.0 一起推出，后者将两个现有的 SDK(Alexa Smart Screen SDK 和 AVS Device SDK)合并为一个工具包，用于创建跨设备的 Alexa 语音和多模式技能。其中一个亮点是第三方 Alexa 设备之间的视频通话，这一功能此前仅限于亚马逊 Echo 设备。AVS SDK 3.0 还支持第三方智能家庭摄像机，让用户可以通过麦克风和扬声器看到实时摄像机画面，并与摄像机旁的人聊天。

在多媒体方面，AVS SDK 3.0 附带了一个笨拙的名称“AVS Downchannel 上的 Alexa 视频技能工具包(VSK)”，可用于创建控制智能电视的 Alexa 技能。亚马逊表示，使用 AVS SDK 3.0 的应用和设备可以让 Alexa 用户搜索和播放内容，打开应用，改变音量等。

一个独立的新 SDK，Alexa for Apps SDK，允许开发者建立 Alexa 功能，包括语音搜索和导航，以及移动应用程序中的自定义交互。亚马逊之前通过的 Alexa for Apps ，一个 API，为移动设备提供了基于 Alexa 的功能，但是这项服务的功能有限。SDK 极大地扩展了 Alexa 与移动应用程序集成的方式——这或许标志着亚马逊的雄心。

“Alexa for Apps SDK 是亚马逊第一个支持在新的或现有的应用程序中开发 Alexa 功能的软件，”亚马逊 Alexa 语音服务和 Alexa 技能副总裁 Aaron Rubenson 通过电子邮件告诉 TechCrunch。“例如，一个健身应用程序可以使用 Alexa for Apps SDK 来设计一个由 Alexa 作为私人教练的健身程序。”

促销

今天宣布的几个工具与业务开发密切相关，如 Alexa Skill Deals，它使开发人员能够对技能内购买和付费技能进行折扣，有效期长达一年。至于推广技能，他们让开发者运行付费活动来推广他们的 Alexa 技能。

当被问及推广技能时，鲁本森解释说:“(它们)被设计成类似于卖家和图书作者在亚马逊网站和移动资产上宣传他们产品的方式。今年晚些时候，开发者将能够为 Echo Show 主屏幕设计技能推广或活动。为了开展活动，开发人员将上传他们的创意，选择活动类型，设定开始和结束日期，然后出价。”

鲁本森说，提升技能的价格尚未确定。

与此同时，Alexa Routines Kit 使技能创作者能够在与他们的技能互动时向客户展示预先构建的例程，而 Alexa Shopping Kit 允许开发人员提供产品推荐(并赚取 10%的佣金)，客户可以将其添加到购物车和列表中，然后购买。据亚马逊称，西班牙语图书出版商 Grupo Planeta 正在使用 Alexa 购物工具包，通过该公司的正念技能 Diana 提供图书推荐。

亚马逊上一次吸引开发者加入生态系统的努力大约始于去年 12 月，当时公司降低了对收入低于 100 万美元的开发者的 Alexa skill 开发者收入的提成，从 30%降至 20%。正如我的同事莎拉·佩雷斯(Sarah Perez)当时指出的那样，尽管亚马逊做出了努力，但相对而言，很少有开发者能够利用 Alexa 在美国消费者家中的巨大足迹来创造有利可图的业务。

这并没有阻止该公司的尝试。今天，亚马逊宣布，收入低于 100 万美元的开发者也将获得其技能收入的 10%，作为额外的“价值回报”激励，以现金形式在今年支付。亚马逊在一篇博客文章中澄清说，增加的收入份额将从 7 月份的收益开始生效。
