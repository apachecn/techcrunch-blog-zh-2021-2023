Ethyca 筹集了 750 万美元，开源软件让开发者将隐私工具直接构建到他们的代码库中 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/03/ethyca-fides-privacy-codebase/

Ethyca 是一家隐私设计技术初创公司，它建立了一套独特的 API、检测工具和分析方法，使组织更容易遵守像 GDPR 这样的数据隐私政策。随着它越来越深入数据保护业务，它今天迈出了几大步。

首先，这家总部位于纽约的初创公司开源了一套开发工具，名为 Fides ，这样开发者就可以将隐私工具和监控机制直接构建到他们的代码库中。其次，它获得了额外的 750 万美元资金，以便继续开发和商业化其专有工具，特别是 API，使那些建立或监控隐私合规性的人更容易做到这一点。

这笔资金是其 2020 年 6 月A 轮1350 万美元的延伸，来自现有投资者——特别是李·菲克斯尔、IA Ventures、拉奇·格鲁姆的 LGF 和 Table Management(比尔·阿克曼)，它使 A 轮达到 2100 万美元。到目前为止，Ethyca 已经筹集了 2750 万美元，并积累了几十个客户，包括 Away、IDEO 和 InVision 等。

Ethyca 创始人兼首席执行官 Cillian Kieran。图片来源: Ethyca

创始人兼首席执行官 Cillian Kieran 表示，Fides 实际上是最初启动 Ethyca 的动机。他在 2018 年的论点是，隐私正成为构建和运行数字服务的一个更加关键和重要的方面——不仅因为数据保护法规和网络安全挑战的需求越来越需要隐私，而且这些服务的用户也越来越清楚他们的数据是如何被使用的(有时是被滥用的)。

当然，除此之外，还有开发人员或组织本身在等式中的角色。许多人认为，要获得正确的数据保护，您必须从一开始就将其作为优先事项和系统工作方式的实际技术部分来构建，这一概念通常被称为“设计隐私”

虽然 Ethyca 的 API 旨在从本质上构建这些工具的版本，以使现有工作流中的监控和隐私更容易，但 Fides 为那些正在构建的人提供了所有这些工具的核心，从一开始就对其进行编程。Ethyca 将 Fides 本身描述为一种开源定义和配置语言，用于描述数据和软件系统中的隐私结构。它最初与两个 OSS 工具 Fides Ops 和 Fides Control 一起发布，这两个工具分别涵盖组织数据基础设施中的隐私权利编排，以及 CI/CD 工作流中隐私规则的验证。

“本质上，我们已经将隐私插入到 CI/CD 管道中，”基兰说。他将它的工作方式比作 Snyk 这样的公司，开发人员使用 Snyk，在代码投入生产之前就开始寻找安全漏洞和弱点。" Fides 也可以对隐私做同样的事情."我要指出的是，Snyk 最近的估值飙升，这表明目前在 DevOps 级别解决安全性和数据保护问题受到了高度重视。

在 Fides 的情况下，这个想法是，举例来说，工具包将能够检测到代码何时同时调用用户行为、数据和位置。如果业务不允许这样做，Fides 将标记代码的相关部分，以便在投入生产之前修复它。它还可以用于自动化数据权限请求，例如当一个人请求取消订阅或从系统中删除他们的数据时，这通常需要花费数小时或数周的时间来手动执行(这种情况经常发生)。

Ethyca 的 Pro(付费)工具已经与大约 700 个应用程序集成，以监控数据保护和隐私政策的一致性，除了其书籍中的许多私营公司外，还与许多公开交易的大型企业和技术公司合作，但它拒绝透露这些公司的名称(但请相信我……它们很大，正是需要更好的隐私和数据保护措施的那种公司)。

与此同时，Fides 已经与亚马逊的 DynamoDB 和 Redshift、Snowflake、Databricks、MongoDB、MariaDB、微软的 SQL Server、MySQL 和 PostgreSQL 等数据库平台进行了集成，并签署了 Slack 和 GitHub 作为其开源社区的支持者。

这笔资金将使 Ethyca 继续雇佣员工来建立商业业务，同时继续为其认为未来如何建立隐私的核心内容做出贡献。

IA Ventures 的布拉德·吉莱斯皮(Brad Gillespie)在一份声明中表示:“我们对 Ethyca 的投资和持续的兴奋反映了对隐私和代码合规性的开发者优先方法的需求。”“Fides 的发布是该团队三年工作的高潮，也是朝着定义急需的隐私开放标准迈出的第一步。通过 Fides，Ethyca 表明他们正在积极思考如何最好地解决更高层次的挑战，这些挑战将影响未来十年的数据隐私讨论。我们很高兴能够支持他们雄心勃勃的计划，通过开源开发工具将隐私转移到软件开发生命周期中。”
