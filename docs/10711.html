微软称两个新的 Exchange 零日漏洞受到攻击，但没有立即修复 TechCrunch

原文：https://web.archive.org/web/http://techcrunch.com/2022/09/30/microsoft-exchange-zero-days/

微软表示，两个新的 Exchange 零日漏洞受到积极攻击，但没有立即修复

微软已经证实，两个未打补丁的 Exchange Server 零日漏洞正在被网络犯罪分子用于现实世界的攻击。

越南网络安全公司 GTSC 于 2022 年 8 月首次发现这些漏洞，作为其对客户网络安全事件的回应的一部分，该公司表示，这两个零日已被用于攻击其客户的环境，可追溯到 2022 年 8 月初。

微软安全响应中心(MRSC) 周四晚些时候在一篇博客文章中表示，这两个漏洞被确定为 CVE-2022-41040，一个服务器端请求伪造(SSRF)漏洞，而第二个被确定为 CVE-2022-41082，当 PowerShell 可被攻击者访问时，允许在易受攻击的服务器上远程执行代码。

“目前，微软意识到利用这两个漏洞进入用户系统的有限目标攻击，”这家技术巨头证实。

微软指出，攻击者需要对易受攻击的 Exchange Server 进行身份验证访问，如窃取凭据，才能成功利用这两个漏洞中的任何一个，这将影响内部部署的微软 Exchange Server 2013、2016 和 2019。

微软没有分享任何关于攻击的进一步细节，并拒绝回答我们的问题。安全公司趋势科技给这两个漏洞的严重性评级为 8.8 和 6.3(满分 10 分)。

然而，GTSC 报告称，网络犯罪分子将这两个漏洞串联起来，在受害者的系统上创建后门，并在受损的网络中横向移动。GTSC 说:“成功利用漏洞后，我们记录攻击以收集信息，并在受害者的系统中建立一个立足点。”。

GTSC 表示，它怀疑一个中国威胁组织可能对正在进行的攻击负责，因为 webshell 代码页使用简体中文字符编码。攻击者还在持续远程访问攻击中部署了 China Chopper webshell，这是中国政府支持的黑客组织通常使用的后门。

安全研究员凯文·博蒙特(Kevin Beaumont)是周四在一系列推特(tweets)中最先讨论 GTSC 的发现的人之一，他说他知道该漏洞“正在被广泛利用”，并且他“可以确认大量的 Exchange 服务器被人利用了后门”

微软拒绝透露补丁何时可用，但在其博客帖子中指出，即将到来的修复是“加速的时间表”

在此之前，该公司建议客户遵循 GTSC 共享的临时缓解措施，包括在 IIS 管理器中添加阻止规则。该公司指出，Exchange Online 客户目前不需要采取任何措施，因为零日只会影响本地 Exchange 服务器。
