谷歌为开发者推出开放健康栈

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/14/google-introduces-open-health-stack-for-developers/

谷歌为开发者推出开放健康栈

谷歌宣布了一个名为 Open Health Stack 的新开源项目，供开发者构建健康相关的应用。这些工具于本周在该公司的“体检”特别活动上发布，包括用于 Android 的软件开发工具包(SDK)和健康应用的设计指南。

这家搜索巨头表示，该堆栈围绕着快速 医疗保健 互操作性 资源(FHIR)标准。这使得开发人员更容易获取信息，医护人员也更容易访问这些信息。FHIR 已被许多主要的电子健康记录(EHR)提供商采用。

开放健康堆栈允许开发者访问 Android FHIR SDK 来构建安全的应用程序，这些应用程序也可以离线工作；帮助开发人员轻松获取数据的设计指南；和 FHIR 分析，以获得对框架复杂结构的见解，以及 FHIR 信息网关，以将基于角色的数据访问分配给各种利益相关方。最后两个组件可以在早期访问中获得，谷歌正在开发更多的功能。

谷歌高级产品经理 Fred Hersch 在一篇博客文章中说:“这一套组件使开发者更容易快速构建应用程序，使医疗工作者能够访问他们做出明智决策所需的信息和见解。”。

为了建立这个堆栈，谷歌与世界卫生组织合作，根据该机构的建议在 Android 上开发应用程序。该公司还声称，开发者工具符合数字公共产品联盟的数据隐私最佳实践。

谷歌表示，全球多个合作伙伴，如 Ona、IPRD 解决方案、Argusoft 和 Intellisoft，已经建立了一些解决方案，正在撒哈拉以南非洲、印度和东南亚使用。

相比之下，苹果通过其 HealthKit API 支持 FHIR 端点，但它不是一个开源项目。

去年，谷歌宣布与三星合作推出健康连接计划，让用户可以轻松地在应用程序间分享他们的健康和健身数据。虽然新的开放健康堆栈也插入到 Android 生态系统中，但它更多的是帮助医疗保健工作者在偏远地区捕获和访问健康数据。

今天，除了这些新的开发工具，谷歌还为美国用户宣布了一项新的搜索功能，以帮助他们找到提供免费或低价护理的医疗中心。
