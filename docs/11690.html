Google Play 改进以突出更高质量的应用程序，提供新的推广功能 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/02/google-play-revamp-to-highlight-higher-quality-apps-offer-new-promotional-capabilities/

谷歌今天宣布，它将对谷歌 Play 商店进行几项更改，这些更改将影响安卓应用的可发现性、开发者如何向消费者推销他们的应用以及各种信任和安全问题。最重要的是，谷歌现在建议开发者，Play Store 将开始优先考虑那些提供技术和应用内质量的应用，在 Play Store 的更多地方推广它们，让消费者能够发现它们。

这些变化暗示了谷歌的意图，即从编辑的角度来看待应用程序如何在 Play Store 上展示和分发。这是苹果前几年通常更加关注的一个领域——特别是在 2017 年对自己的应用商店进行改造之后，苹果将游戏和应用分成各自的标签，并在商店的主页上引入编辑内容，包括文章和提示。

然而，Play Store 并没有走那么远。相反，谷歌表示，它现在将开始通过改变它如何决定哪些应用程序将在平台上更加可见，来引导消费者远离低质量的应用程序。

具体来说，它正在实施新的质量阈值，将排除超过特定崩溃率和“应用无响应”(ANR)率的应用，无论是整体还是每款手机。谷歌表示，不符合这些门槛的应用程序将被排除在 Play Store 的一些区域之外，包括推荐，而其他应用程序甚至可能会在其商店列表中包含一个警告，以设定适当的用户预期。

图片来源:谷歌

除了的技术质量， Play Store 的编辑还会考虑一系列因素，比如应用程序或游戏是否有完美的设计，内容是否能吸引用户，入职流程是否清晰，广告是否整合良好，应用程序是否易于访问，导航、控件和菜单是否易于使用，等等。他们还将检查应用程序是否符合安卓的质量准则和最佳实践，详情见安卓开发者网站。

此外，该公司将向开发者推出新的推广内容格式和新型定制商店列表，旨在帮助更多用户使用应用程序。

在前一种情况下，开发者将能够利用live ops——在 Play Store 上推广应用程序的特殊商品单位。如今，它们被用于推广折扣和优惠、重大应用更新、应用内活动、预注册公告等。苹果也有类似的功能，去年推出。各种营销单位给了应用商店一种更实时的感觉，因为他们可以根据下载和发布应用的理由进行营销现在，而不仅仅是作为一种普遍的促销。

图片来源:谷歌

谷歌指出，与没有利用 LiveOps 的类似产品相比，使用 live ops 的开发者收入增加了 3.6%，28 天日活跃用户增加了 5.1%。现在，它将把 LiveOps 重命名为“推广内容”，以反映扩展该功能以支持新内容类型的长期计划——包括那些将看到推广单元更深入地集成在 Play Store 的用户主页中、搜索和发现区域中、标题列表中以及通过深度链接直接在应用中出现的内容。

开发者也将很快能够创建一种新的列表类型，这将允许他们专门针对被搅动的用户(尝试过该应用程序或游戏，然后放弃它的人)。这种“搅动用户定制商店列表”格式将在接近年底时推出，将能够显示旨在重新获得先前用户的特定信息。

另外两个变化集中在应用安全和保护开发者——以及下载他们的应用的消费者——免受协同攻击。

谷歌将更新 Play Integrity API ，这有助于防范风险和欺诈性流量，提供更多功能。开发人员将能够定制 API 响应，在 Play 控制台中设置测试，并使用新的报告来分析他们的 API 响应。他们还可以在任何设备上从 Play Store 应用程序的开发者设置中调试 API 响应。

此外，谷歌表示，它将推出一项新计划，旨在解决对应用评级和评论的协同攻击。该公司没有提供太多关于该计划如何运作的信息，但如果用户或竞争对手对他们的应用程序进行不公平的虚假评论，这可能会给开发者提供一种反击的方式。这是一个令人担忧的领域，最近成为新闻，事实上，当排名第一的新社交应用程序 Gas 突然成为一个骗局的目标，该骗局声称它被用于人口贩运，导致用户删除他们的账户。

此前，Play Store 进行了更新，旨在帮助消费者更好地发现在智能手表、电视或平板电脑上运行的非智能手机应用。今年早些时候，谷歌还警告开发者说，会隐藏并阻止过时应用的下载。谷歌提醒开发者从 2022 年 11 月 1 日起，如果他们希望他们的应用程序能够被运行新版 Android 操作系统的新用户在 play Store 上发现，他们必须将 API 级别 30 (Android 11)或以上作为目标。
