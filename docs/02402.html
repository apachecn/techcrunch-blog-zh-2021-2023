Harness 发布了开放版本的可访问源代码的持续交付产品

来自 AppDynamics 创始人 Jyoti Bansal 的初创公司 Harness 在过去五年中一直在构建一个开发者工具平台。2020 年，该公司首次涉足开源领域，购买了持续集成工具无人机。今天，随着该公司连续交付产品的免费开放版本的发布，他们进一步拥抱了开放性。

Bansal 不愿意使用“开源”这个术语，因为它带有一些很大的内涵，他不想歪曲公司正在做的事情。但是从今天开始，他将持续交付工具社区版的源代码提供给任何开发者，让他们在“源代码可访问 PolyForm Shield 许可下使用。”

“我们正在做一切事情[以同样的方式]作为开源的力量，但我们使用的是源代码可用的许可策略，我们将 100%的源代码放在开放的回购中……一切都类似于完全开源的项目，只是我们的竞争对手不允许将其货币化，”Bansal 解释道。

在过去的几年里，像 Elastic 和 MongoDB 这样的公司看到了一些像 Amazon 这样的云提供商利用开源许可来创建他们自己版本的相同产品。他们提出异议，并创建了一个稍微更严格的许可证来防止竞争对手的大规模挖走。

开源倡议(OSI)的执行董事 Stefano Maffulli 表示，Harness 在设计这款产品时非常谨慎和尊重。“Harness 通过明确哪些地方使用 OSI 批准的许可证，哪些地方不使用，做了正确的事情。这样做是尊重社区，建立信任，”马福利说。

Bansal 表示，他们正在努力平衡拥有一个开放和可用的产品的想法，同时保护他们在构建产品方面的重大投资。他认为，通过将其开放，他可以消除过程中的任何摩擦，允许开发者下载开放版本，而不必经过经理的检查或获得许可，这在涉及购买时是常见的情况。

“我们从一开始就提供了我们产品的免费版本，但我们想让它更上一层楼，我们只是消除了障碍，这样开发者就没有理由征求他们经理的许可。他们可以去下载并试用它。只有当产品开始在他们的环境中工作，并且他们成功了，如果他们需要，我们才会进行商业讨论，”他说。

除了开放产品之外，Harness 还提供产品的付费企业版和托管版。Bansal 表示，该计划是最终将平台上的每个产品(在有意义的地方)都移入这种模式，但他们从持续交付产品开始，因为无人机持续集成产品已经是开源的。

Bansal 补充说，无人机创始人真正向他展示了社区的价值，以及让外部开发人员参与项目如何能够改进产品。

“(无人机创始人)带给我们的是向我们展示，你可以建立一个伟大的社区。我们一直在打造伟大的产品，我们知道如何打造伟大的产品，但如果我们能够让我们的用户社区和公司外部的开发人员社区更紧密地参与进来，我们甚至可以打造出更好的产品。”
