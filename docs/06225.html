Cloudflare 致力于实现 web 应用环境的互操作性

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/09/cloudflare-heads-effort-to-make-javascript-environments-interoperable/amp/

图片来源:迈克尔·肖特/彭博/盖蒂图片社

Cloudflare 今天宣布，它正在与 Deno(Deno runtime 的开发公司)和 Node.js 开源项目的个人贡献者合作，创建标准，使开发人员能够在 Deno、Node.js 和 Cloudflare 的无服务器应用平台 Cloudflare Workers 之间编写代码。Cloudflare 联合创始人兼首席执行官马修·普林斯(Matthew Prince)表示，这项努力将允许应用程序在 Workers、Deno 和 Node.js 之间转移，而无需重写，这是目前无法实现的壮举。

“数百万开发人员使用 JavaScript，它通常是他们学习的第一门语言。迄今为止，JavaScript 标准一直完全集中在前端，比如浏览器，”普林斯在一次电子邮件采访中告诉 TechCrunch。“通过在考虑前端和后端的情况下标准化核心 JavaScript APIs，我们可以让前端开发人员以一种熟悉的方式获得比以往更多的后端访问权限。”

Node.js 建立在谷歌开源的 V8 JavaScript 引擎之上，被认为是围绕单一编程语言 JavaScript 统一了 web 应用程序开发，允许开发人员在页面被发送到用户的 web 浏览器之前，在服务器端运行脚本来生成动态网页内容。但是尽管经过了十多年的发展，Node.js 仍然缺乏互操作性。开发人员不得不依靠隐性知识和构建工具来跨各种 JavaScript 环境共享代码。

“JavaScript 的未来是通用的，能够在许多环境之间移动 JavaScript。Node.js 的技术管理机构 Node.js 技术指导委员会的成员 Myles Borins 在一份声明中说。“只有像 Web 互操作运行时社区组这样的合作项目，才有可能实现‘一次编写，随处运行’的承诺。”

Cloudflare、Deno 和来自 Node.js 技术指导委员会的选定 Node.js 贡献者将开发基于 API 的互操作性标准，作为万维网联盟下的 Web 互操作运行时社区组的一部分。总之，他们的目标是使跨运行时利用 JavaScript 社区的工具和集成更加容易，消除特定于平台的细微差别，并允许应用程序随着时间的推移而发展和变化，而无需重写。

“Deno 广泛的生态系统成功在很大程度上可以归功于我们严格遵守网络平台标准，”Deno 软件工程师 Luca Casonato 在一份声明中说。“我们从一开始就知道，Deno 只有在浏览器互操作性上全力以赴，才能取得成功——开发者和广泛的生态系统趋势已经向我们表明，这个赌注是正确的。除了我们与… W3C 的现有标准工作之外，我们很高兴能与 Cloudflare 和 Node.js 合作者合作，实现更好的运行时互操作性。”

Prince 补充道:“当 JavaScript APIs 在最流行的 JavaScript 环境(Node、Deno 和 Workers)中外观、感觉和行为一致时，开发者就赢了。我们的目标是将开发人员从一个环境中解放出来——迁移到一个新的环境并不意味着重写您的整个应用程序。”

当然，这将有利于 Cloudflare，该公司在 2018 年推出了 Workers 平台的测试版。Prince 表示，Cloudflare 的收入驱动因素——工作人员对成批的服务器端任务收取费用——超过 45 万名开发人员在该平台上开发了超过 300 万个应用程序。毫无疑问，Cloudflare 希望通过说服开发团队从其他 JavaScript 环境迁移来提高这些数字——特别是考虑到该公司令人沮丧的收益预测。

并非巧合的是，Cloudflare 今天也宣布在 Apache V2 许可下开源 Workers 运行时。该公司将此举称为对供应商锁定的回答，但它也提高了员工的形象——或多或少地充当免费广告。

“仅仅写下标准是不够的。通过开源 Cloudflare Workers 运行时，我们正在广泛提供这些 API，并为开发人员提供一种采用新标准的简单方法，”Prince 说。“开发人员在采用任何新的开发平台时，有两件事会犹豫不决。第一:他们担心被锁在里面。无论你多么看好这项技术，如果你把公司的未来押在一个开发平台上，你都不希望有被勒索的可能。第二:作为开发人员，您希望本地开发环境能够快速迭代和测试您的更改。开源 Workers 运行时解决了这两个问题，它为开发人员提供了一个可以在任何地方运行的标准，这意味着既可以在任何托管环境中运行，也可以在他们的本地机器上运行，以便进行快速测试和迭代。”
