微软 365 获得一系列新的人工智能功能

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/16/microsoft-365-gets-a-host-of-new-ai-powered-features/amp/

图片来源:微软

在今天一场聚焦人工智能的新闻发布会上，微软发布了微软 365 Copilot，这是其将人工智能嵌入其生产力和企业应用套件的最新举措。目前，Copilot 正在与选定的(约 20 家)商业客户进行测试，它将包括 OpenAI 最近宣布的 GPT-4 在内的人工智能模型的能力与商业数据和微软 365 应用程序(如 Word、Excel、PowerPoint、Outlook 和 Teams)相结合。

微软首席执行官塞特亚·纳德拉在一份声明中表示:“今天标志着我们与计算互动方式的下一个重要步骤，这将从根本上改变我们的工作方式，并开启新一轮生产力增长。”。“通过我们新的 copilot for work，我们给了人们更多的代理，并通过最通用的界面—自然语言—使技术更易于使用。”

Copilot 根据使用它的应用程序处理不同的任务。例如，在 Word 中，Copilot 编写、编辑、总结和生成文本，而在 PowerPoint 和 Excel 中，Copilot 将自然语言命令转化为设计好的演示文稿和数据可视化。

微软表示，Excel 中的 Copilot 还可以揭示相关性，提出假设情景，并根据用户的问题建议新的公式——根据这些问题生成模型。在 Word 中，Copilot 可以建议语气(包括“专业”、“热情”、“随意”和“感激”)，同时提供试图加强论点或解决矛盾的建议。

图片来源:微软

PowerPoint 功能特别棒。使用 Copilot，用户可以创建一个基于 Word 文档的演示文稿，并配有精美的幻灯片、演讲者注释和来源引用。然后，他们可以通过要求副驾驶“在这张幻灯片上添加动画”或“在演示文稿中应用现代风格”来完善该幻灯片

在 Outlook 中，Copilot 可以帮助合成和管理收件箱，以及通过切换来适应长度或语气的草稿回复。(它会识别一个提示，比如“起草一份感谢他们的回复，并询问他们第二点和第三点的更多细节；缩短此稿，使语气专业”)。与此同时，在团队中，Copilot 提供实时总结和行动项目，如在对话的背景下确定跟进人员和创建会议议程。

Copilot 更吸引人的元素之一是业务聊天，它将来自文档、演示文稿、电子邮件、日历、笔记和联系人的数据汇集在一起，以帮助总结聊天内容、编写电子邮件、找到关键日期，甚至根据其他项目文件编写计划。通过“告诉我的团队我们是如何更新产品战略的”等提示，业务聊天(首先出现在团队中)将根据上午的会议、电子邮件和聊天线索生成状态更新。

图片来源:微软

在一篇博客文章中，微软强调，驾驶 Copilot 的模型没有经过客户内容或个人提示的培训。它说，定价和许可的细节将很快分享。

AI 是出了名的容易出错——即使是像 GPT-4 这样的尖端模型也会犯愚蠢的错误。那么副驾驶呢？微软不否认它会出错。但与此同时，该公司强调了 Copilot 用来提高提示质量的“基础”。

在今天的现场演示中，微软 365 负责人 Jared Spataro 解释说，输入到 Copilot 的提示首先通过微软的统一数据 API Microsoft Graph 进行过滤，以获得额外的上下文。这些修改后的提示随后被发送到 GPT-4，响应通过 Microsoft Graph 进行安全、安保和合规性检查，然后被发送回 Microsoft 365 apps。

斯帕塔罗说:“有时副驾驶会做对，有时会做错，给你一个不完美的想法，但仍然给你一个好的开始。”。“我们通过指出限制、链接到来源，并提示用户根据主题专业知识审查、核实和调整内容，来明确系统是如何做出决策的。”

图片来源:微软

很难相信斯帕塔罗的话，考虑到微软最近解雇了其人工智能组织内的一个主要道德团队。该团队一直致力于识别微软在其软件和服务中采用 OpenAI 语言模型所带来的风险。但在演示中，斯帕塔罗说，Copilot 的推出是为了满足微软客户“未满足的需求”。

“我们必须快速负责地行动，边走边学，”斯帕塔罗补充道。“我们正在与一小组客户[包括 8 家财富 500 强企业]一起测试 Copilot，以获得反馈，并随着规模的扩大改进我们的模型，我们将很快扩展到更多客户。”

无论如何，这些都不是新概念。粗略的谷歌搜索会弹出几十个人工智能工具，提供写作推荐；生成邮件、文本和幻灯片；总结会议以及更多关于 Copilot 所能完成的事情。但是微软认为 Copilot 可以做得更好，也更安全。这还有待观察。

图片来源:微软

在微软 365 推出 Copilot 之前，微软推出了 Dynamics 365 中的 Copilot，这是微软的企业资源规划和客户关系管理工具组合，这有力地证明了该公司没有放缓在人工智能和自动化方面的投资。就在一月份，微软向 OpenAI 投资了数十亿美元，open ai 是一家开发 Copilot 各种化身背后技术的初创公司，这个科技巨头显然渴望看到投资回报。

它还试图领先竞争对手谷歌一步，谷歌本周宣布对其生产力和协作工具集 Workspace 进行全面更新，这将把生成式人工智能带到该套件的几乎每个部分。

微软和谷歌的推出都明显感觉有些仓促。谷歌在声明中表示，与 Copilot 一样，新的人工智能工作区功能将在发布时只提供给“可信的测试人员”，尚未定价。

微软咄咄逼人的做法产生了后果。根据昨天《信息》的一份报告，该公司正面临运行其人工智能所需的服务器硬件的内部短缺——即 GPU。(与之相关的是，OpenAI 昨天透露，它与微软合作，在 Azure 中建造了一台“超级计算机”，以开发 GPT-4。)据报道，微软被迫对一些开发其他人工智能工具的内部团队的硬件进行配额限制，以确保其有足够的能力处理必应的新 GPT-4 驱动的聊天机器人和即将到来的新发布的微软 365 Copilot 工具。
