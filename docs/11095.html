Diagrid 获得 2420 万美元的资金，为 Kubernetes TechCrunch 推出了完全托管的 Dapr 服务

原文：https://web.archive.org/web/https://techcrunch.com/2022/10/12/with-24-2m-in-funding-diagrid-launches-its-fully-managed-dapr-service-for-kubernetes/

早在 2019 年，微软就推出了 Dapr ，这是一个开源项目，旨在让开发者更容易在 Kubernetes 之上构建微服务。Dapr 代表“分布式应用程序运行时”，它处理许多用于构建分布式应用程序的原语(如发布/订阅、状态管理、秘密管理、事件触发器等)。)，允许开发人员专注于他们的分布式应用程序的业务逻辑。如今，Dapr 是 CNCF 的一个孵化项目，也是 CNCF 发展最快的项目之一，拥有 2000 多名贡献者和 19400 名 GitHub 明星。正如许多流行的开源项目一样，一个小型的创业生态系统正在围绕它形成。

Diagrid ，今天从秘密行动中走出来，并宣布获得总计 2420 万美元的资金，有点领先:它是由 Dapr 和 Kubernetes 事件驱动自动缩放(KEDA)项目的创始人马克·富塞尔和亚龙·施耐德共同创建的。两人悄悄地筹集了由 Amplify 领投的 420 万美元种子轮，现在又筹集了由 Norwest 领投的 2000 万美元首轮融资。它的天使投资者是云原生生态系统的名人，包括 Kubernetes 和 Heptio 的联合创始人乔·贝达，特使创始人马特·克莱恩和浮力首席执行官威廉·摩根，以及微软 Azure 首席技术官马克·鲁西诺维奇(联合创始人在微软工作期间曾在他手下工作)，前 Atlassian 首席技术官斯里·维斯瓦纳特和前 Heroku 首席执行官亚当·格罗斯。

图片来源: Diagrid

“我们看到微服务应用在各地的巨大增长，对于开发人员来说，这是构建这些应用的实际方式。在过去的两年里，你可以看到 Kubernetes 的增长和集装箱的增长发生了巨大的变化，”Fussell 说。“我们开发 Dapr 是因为我们想帮助所有这些开发人员提高构建应用程序的效率。所以我们决定离开微软，因为我们想打造一条道路——特别是围绕 Dapr 开源项目构建服务。”

不出所料，这正是该团队所做的。该公司可能会在不久的将来从 10 人增长到 30 人左右，今天正式推出其第一款产品:Diagrid Conductor，这是一个为 Kubernetes 提供的完全管理的 Dapr 平台。福塞尔解释说，指挥将帮助运营团队在生产中操作和管理 Dapr。“Kubernetes 是一个托管和运行分布式系统平台的出色平台，但 Kubernetes 本身——以及任何部署在其上的技术——都有其挑战，”Fussell 说。“因此，Diagrid Conductor 帮助团队在生产中操作 Dapr，它实际上提高了 Kubernetes 上 Dapr 的可靠性和操作能力。”

图片来源: Diagrid

Schneider 还指出，Conductor 提供了对应用健康的操作洞察。通常，现在运行 Dapr 的公司必须自己监控和修补它，而 Conductor 为他们处理所有这些。

Diagrid 按 Kubernetes 集群向用户收费。目前，这仍然是一个有点手动的入职流程，但该公司计划明年推出一个自助服务工具。

Amplify Partners 的普通合伙人兼 Diagrid 董事会成员 Lenny Pruss 表示:“基于云的微服务架构给工程团队带来了实际成本，开发人员不得不成为基础设施辩论者和分布式系统理论家，仅仅是为了部署一个简单的应用程序。“Dapr 开源和 Diagrid 构建的商业解决方案触及了这个问题的核心，提供了一套开发人员友好的工具和 API，抽象出了许多分布式系统的复杂性，从而使工程团队能够专注于重要的事情:他们的业务应用程序。”
