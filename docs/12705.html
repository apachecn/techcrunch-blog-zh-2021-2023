AWS 为分析数据提供数据清理室 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/29/aws-gets-data-clean-rooms-for-analytics-data/amp/

图片来源: AWS

AWS 今天在推出了一项新服务，该服务将帮助广告或营销机构内部的用户与公司内部的其他员工或外部合作伙伴共享数据，而不会有无意中共享个人数据的风险。

这项新服务是亚马逊新广告 AWS&营销计划的一部分，该计划旨在利用现有的 AWS 服务——以及来自其合作伙伴的服务——为他们提供专门构建的服务。洁净室是该倡议的第一个主要新产品。

AWS 首席执行官 Adam Selipsky 在今天的主题演讲中解释说:“数据洁净室是受保护的环境，多方可以在不暴露原始数据的情况下分析组合数据。“干净的房间很难建造。他们的复杂需求需要几个月的时间来开发，一旦你建立了一个干净的房间，你必须不断更新数据，同时满足新的合作者和数据类型的要求。”

图片来源: AWS

例如，一家拥有客户忠诚度数据的公司可以与另一家拥有用户广告点击行为数据的公司合作，以创造对用户行为的新见解，而无需分享用户的原始和可识别数据，Selipsky 认为。

“有了这些见解，我们可以制作更多相关的广告，同时维护每个人的隐私，”他说。“品牌和媒体出版商使用洁净室控制台或 API，他们建立了一个洁净室，并开始与其他公司合作，只需点击几下鼠标。因此，您不需要花费数月的开发时间来定制查询和限制的类型，而只需要允许合作伙伴与您一起运行这些洁净室。”

这里的想法是提供一个单一的服务，公司可以使用它来进行数据协作，同时仍然保护底层数据，使用一组可配置的控件。所有的合作者都可以贡献他们自己的数据，无论是纯文本的、散列的还是预加密的。然后，他们可以使用这些干净的房间来协作处理这些数据，而无需向对方透露原始数据。

总共可以有五个合作者，他们的数据存储在 AWS Glue 数据目录中。当任何人对这些数据进行查询时，Clean Rooms 将读取这些数据，无论这些数据位于何处，然后该服务将自动应用预设的规则来保护每个参与者的原始数据。每个表都可以有自己的规则，这些规则限制了允许的查询类型。运行这些查询时，加密数据将保持加密状态。

图片来源: AWS

AWS 广告和营销技术解决方案总监蒂姆·巴恩斯(Tim Barnes)表示:“广告和营销行业的客户一直在寻求新的方式与他们的合作伙伴进行交互，同时保护消费者数据，减少工程团队的负担。“随着 AWS Clean Rooms 和 AWS for Advertising and Marketing 的推出，AWS 客户现在有了一套广泛的解决方案，使他们能够更轻松地安全协作，以 Pb 级和毫秒级延迟经济高效地运营，并在广告测量和客户体验等领域更快地创新。”
