苹果 iCloud、Twitter 和《我的世界》易受“无处不在”的零日漏洞攻击

据报道，许多流行的服务，包括苹果 iCloud ，Twitter，Cloudflare，《我的世界》和 Steam，都容易受到影响流行的 Java 日志库的零日漏洞的攻击。

这个漏洞被 LunaSec 的研究人员命名为“Log4Shell ”,并被归功于阿里巴巴的陈兆军,它是在 Apache Log4j 中发现的，Apache Log4j 是一个开源日志记录工具，在大量应用程序、网站和服务中使用。Log4Shell 首先在微软旗下的《我的世界》被发现，尽管 LunaSec 警告说，由于 Log4j 在几乎所有主要的基于 Java 的企业应用程序和服务器中“无处不在”,因此“许多许多服务”容易受到这种攻击。在一篇博客文章中，这家网络安全公司警告说，任何使用 Apache Struts 的人都“可能容易受到攻击”

到目前为止，拥有服务器的公司确认容易受到 Log4Shell 攻击，包括苹果、亚马逊、Cloudflare、Twitter、Steam、百度、网易、腾讯和 Elastic，尽管可能有数百家甚至数千家其他组织受到影响。在给 TechCrunch 的一份声明中，Cloudflare 表示，它已经更新了系统以防止攻击，并补充说，它没有看到任何利用的证据。

美国国家安全局(NSA)网络安全主管罗伯特·乔伊斯(Robert Joyce)证实，该机构开发的免费开源逆向工程工具 GHIDRA 也受到了影响:“Log4j 漏洞是利用的一个重大威胁，因为它广泛包含在软件框架中，甚至包括 NSA 的 GHIDRA，”他说。

新西兰的计算机应急响应小组(CERT)德国电信的 CERT 和 Greynoise 网络监控服务都警告说，攻击者正在积极寻找易受 Log4Shell 攻击的服务器。根据后者，大约 100 个不同的主机正在扫描互联网，寻找利用 Log4j 漏洞的方法。T9】

HackerOne 的高级安全技术专家 Kayla Underkoffler 告诉 TechCrunch，这个零日突出了“开源软件作为世界关键供应链攻击面的一个越来越大的部分所带来的威胁。”

“开源软件是几乎所有现代数字基础设施的背后，平均每个应用程序使用 528 个不同的开源组件，”Underkoffler 说。“2020 年发现的大多数高风险开源漏洞也已经在代码中存在了两年多，大多数组织缺乏对供应链中开源软件的直接控制，无法轻松修复这些漏洞。对于任何依赖这种软件的组织来说，保护这种通常资金不足的软件是必不可少的。”

Apache 软件基金会今天发布了一个紧急安全更新，以修补 Log4j 中的零日漏洞，并为那些无法立即更新的人提供了缓解措施。游戏开发商 Mojang Studios 也发布了一个紧急《我的世界》安全更新来解决这个问题。
