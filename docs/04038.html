Harness 将 GitOps 引入其软件交付平台 TechCrunch

图片来源:卡万图片/盖蒂图片

Harness ，由 AppDynamics 联合创始人 Jyoti Bansal 掌舵的 ML 增强软件交付平台，正在让搭上 GitOps 列车。

虽然该服务最初专注于持续交付，但后来它扩展了其功能集，努力成为初创公司和企业的全栈 DevOps 平台。这些包括持续集成、云成本管理、功能标志等等。

班萨尔说，公司的总体使命没有改变。“Harness 在开始时有一个雄心勃勃的目标:几乎世界上所有的企业都想成为软件企业——我们能为他们的软件工程师带来与谷歌、脸书、亚马逊或网飞同等复杂的工具吗？这类公司有一个很大的优势，那就是他们拥有其他公司所没有的真正完善的软件工程实践和工具，”他说。

该公司现在拥有 600 名员工，随着 GitOps 服务的推出，现在又增加了一项功能。随着 Kubernetes 变得越来越流行，甚至在传统企业中，GitOps 作为一种开发和部署应用程序到这些集群中的方法也在兴起。难怪 Harness 的客户中包括一些世界上最大的零售商和银行，它也想支持这种模式。

Harness 正在基于 ArgoCD 构建其 GitOps 平台，ArgoCD 是一款面向 Kubernetes 的声明性 GitOps 工具，最初由 Applatix(2018 年被 Intuit 收购)开发，现在隶属于云计算原生计算基金会(Cloud Native Computing Foundation)。“它很强大。人们喜欢它的轻量级，您可以自动化您的开发人员工作流和大多数部署机制，”Bansal 解释说。“GitOps 面临的挑战是，它非常非常难以在真正复杂的企业环境中运作。”

这就是 Harness 的用武之地，它提供 SaaS 风格的 GitOps 服务，在 ArgoCD 上宣传企业能力。“我们正在让所有企业使用 GitOps 变得更容易、更实用。真的是企业级 GitOps。人们喜欢 GitOps，但如果不使用大量的创可贴和胶带，他们很难使用它——现在，您有了一个真正的企业解决方案，也是第一个面向 GitOps 的 SaaS 产品。”
