Payload 为其开发者首款无头 CMS 筹集了 470 万美元

开发无头开源内容管理系统(CMS)的 Payload 今天宣布，它已经筹集了 470 万美元的种子轮投资，由谷歌专注于人工智能的 Gradient Ventures 领投。其他投资者包括 MongoDB Ventures、Y Combinator、SV Angel、Grand Ventures 和 Exceptional Capital，此外还有一些天使投资人。

与大多数 CMS 工具不同，密歇根的 Payload 把重点放在了开发者身上。自 2021 年启动以来，该平台背后的团队认为，典型的应用框架为开发人员提供了创建后端的工具，而不是管理应用及其内容所需的 CMS 风格的用户界面。

图片来源:有效载荷

“对开发者来说，‘内容管理系统’通常是一个骂人的词。如果一个工程师被指派了一个 CMS 项目，那就不那么令人兴奋了。他们希望避开路障，编写代码，打造他们引以为豪的东西——但现有的 CMS 左右逢源，”首席执行官詹姆斯·米克鲁特说，他与丹·里本斯(首席运营官)和埃利奥特·德诺夫(首席技术官)共同创立了这家公司。“我们不会与 Webflow 或 Squarespace 竞争，相反，我们将为有才华的工程师提供一个他们可以信赖的工具来构建关键的内容基础设施。”

该团队没有构建另一个无代码 CMS，而是反其道而行之，构建了一个更类似于框架的东西，而不是“仅仅”一个纯粹的无头 CMS。首先，开发人员在 TypeScript 中描述他们的有效负载配置，该服务创建一个 Mongo 数据库，设置 REST 和 GraphQL APIs，处理文件存储、认证和访问控制，当然，还创建 admin UI，默认为干净、简约的外观。

该公司目前正处于其第一个发布周的中期，这一概念最近似乎正在初创公司中流传。今年早些时候，该团队还推出了 1.0 版本，现在它已经筹集了第一轮资金，计划是扩大团队并投资于围绕有效载荷的开源社区。和大多数开源创业公司一样，该公司计划推出一项托管服务 Payload Cloud，以支持其货币化战略，并作为部署有效载荷应用的中心。
