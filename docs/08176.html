谷歌云推出首款基于 Arm 的虚拟机

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/13/google-cloud-launches-its-first-arm-based-vms/

这是一个漫长的过程，但谷歌云今天宣布了其第一个基于 Arm 的虚拟机，继 AWS 和 Graviton 实例之后，Azure 也于最近推出了 Arm 虚拟机。但是，虽然 AWS 建立了自己的定制芯片，谷歌云正在通过使用来自 Ampere 的芯片来跟随 Azure 的领导。这些新虚拟机现在处于预览阶段，将加入谷歌云的 Tau 虚拟机系列，命名为“Tau T2A”。这条生产线大约在一年前推出，使用 AMD 米兰处理器，提供更好的性价比。

“我们很高兴能够扩展我们已经与英特尔和 AMD 合作提供的丰富选择，并进入 Arm 生态系统，为我们的客户提供更多选择和灵活性。谷歌云副总裁兼基础设施总经理 Sachin Gupta 在今天宣布之前的新闻发布会上表示:“我们支持操作系统、数据库、编程语言和其他工具的广泛生态系统。

新的芯片将采用预定义的 SKU，最多 48 个 vCPUs，每个 vCPUs 最多 4GB 内存。虚拟机将提供高达 32 Gbps 的网络带宽，并支持谷歌云生态系统中常见的存储选项。谷歌表示，这些 CPU 规格将使这些机器可用于广泛的工作负载，包括作为网络服务器和运行容器化的微服务、数据记录应用程序等。

像 AMD 驱动的 Tau 芯片一样，谷歌将这些视为其性价比优化的解决方案。例如，谷歌云美国中部地区的 32 核 Tau T2A 虚拟机每小时的成本为 1.232 美元。

用户将能够在这些机器上使用 RHEL、CentOS、Ubuntu 和 Rocky Linux 等软件，此外还有谷歌自己的容器优化操作系统来运行容器化应用。在这一点上，Arm 支持已经成为大多数操作系统和软件供应商的赌注，这反过来也大大增强了这些虚拟机(以及谷歌竞争对手的虚拟机)的有用性。

新虚拟机现已在少数地区推出，包括美国中部(爱荷华州-A、B、F 区)、欧洲西部 4(荷兰-A、B、C 区)和亚洲东南部 1(新加坡-B、C 区)，但随着时间的推移，这些虚拟机还会出现在其他数据中心。

Ampere Computing 首席产品官 Jeff Wittich 表示:“Ampere Altra 云原生处理器是为了满足现代云应用的需求而全新设计的。“我们与 Google Cloud 的密切合作促成了新的性价比优化的 Tau T2A 实例的推出，该实例能够快速高效地部署要求苛刻的横向扩展应用。”

除了使用这些虚拟机作为谷歌云计算引擎的一部分，谷歌现在还支持它们作为其 Kubernetes 引擎的一部分，即数据流流和批处理服务以及批处理，这是一个新的完全托管的批处理作业调度程序，谷歌今天也发布了。“这项新功能将有利于面向吞吐量的计算的主要用例，如天气预报和电子设计自动化，”古普塔说。“这项新服务的主要目的是为批处理作业在时间、位置和云容量成本方面提供前所未有的灵活性。”
