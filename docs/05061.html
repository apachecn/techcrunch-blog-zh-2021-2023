欧洲如何扩大其颠覆大型科技公司的努力

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/25/how-europe-has-expanded-its-bid-to-disrupt-big-tech/

昨天晚些时候，欧盟联合立法者就数字竞争规则的重大改革达成政治协议，这将对最强大的互联网巨头引入预先义务和限制(字面意思是“该做和不该做的事情”)，如果他们不能满足要求，将被处以巨额罚款和其他类型的处罚。

数字市场法案(DMA)是欧盟多年来对数字市场系统性不当行为的回应。

欧盟针对亚马逊、谷歌和苹果等科技巨头的一系列重大反垄断案件，以及随之而来的对大型科技公司的主导地位继续巩固的沮丧情绪，为该法规提供了信息，因为案件需要数年才能结案，同时滥用行为基本上没有得到遏制。

欧盟习惯于让科技巨头定义自己的补救措施，即使他们(最终)受到反垄断执法的打击——只有一个一般性的声明，即认定的侵权行为必须停止——这也让平台有足够的回旋空间来继续摊牌。(因此，多年后，委员会不得不再次介入谷歌安卓案，向谷歌施压，要求其放弃竞争对手从一开始就宣称不公平的付费拍卖模式。)

DMA 建议通过预先固定条件并对范围内的巨头适用遵守固定道路规则的预期来扭转这种后知后觉的动态，目标是开创一个更加积极有效的技术监管的新时代。欧盟坚信，事前竞争机制将补充通常的事后反垄断程序，以确保数字市场保持公平和可竞争。

尽管欧盟政策制定者花费了多年时间考虑是否以及如何采取行动，但正式的立法提案直到 2020 年 12 月才提交——因此欧盟机构只用了不到 18 个月的时间就临时文本达成一致。这看起来非常快，突显出欧洲对于控制大型科技公司市场力量的必要性有多么一致。

欧盟也加倍努力——昨日同意扩大 DMA 对大型科技公司的要求，包括针对信息平台的新的互操作性义务。

该法规是否能真正成功地促进由核心平台服务主导的数字市场的竞争，这确实是€750 多亿美元的问题。

欧盟认为，在大型科技的单一市场中建立一套共同的规则将促进创新、增长和竞争力，并支持小型平台、中小企业和初创公司的规模扩大——欧盟认为，这些公司将受益于欧盟层面单一、清晰框架的存在。

但一些专家对这一论点表示怀疑——他们认为，改善数字市场竞争的最佳方式可能是鼓励网守巨头之间更直接的竞争，而这不是欧盟委员会的做法。

DMA 是否会像欧盟所希望的那样，在激发新的竞争和创新的同时，阻止平台巨头不公平地滥用权力，可能需要更长的时间来评估，而欧盟在相对较短的时间内就新制度的细节达成一致。但是有一件事是清楚的:改变它的到来——而且它的到来相当快。

请继续阅读，了解欧洲理事会、欧洲议会和欧盟委员会昨日达成妥协的主要进展。

DMA 将适用于谁？

该规定将适用于具有一定规模和市值的中介平台，这些平台扮演着“看门人”的角色——这意味着这些公司可以通过其平台 T & Cs 并作为其市场力量的结果，为其他企业和消费者设定“游戏规则”。

给出的长期存在的例子包括搜索引擎和社交网络。市场和预订平台似乎也属于这个范畴。

欧盟联合立法者还在列表中添加了虚拟助手和网络浏览器，显然是着眼于进一步完善该法规。

被指定为看门人的标准是什么？

欧盟机构同意在这一点上折中:该法律将适用于市值至少为 750 亿€或年营业额至少为 75 亿€的科技巨头(拒绝了委员会最初提出的略低的门槛和一些欧洲议会议员提出的更高的门槛)。

公司还必须在欧盟拥有至少 4500 万月度最终用户和 10，000+年度商业用户。

可能的嫌疑人包括苹果、亚马逊、谷歌和 Meta(脸书)。欧洲预订平台巨头 Booking.com 也可能加入事前俱乐部。中国电子商务巨头阿里巴巴也是如此。

指定守门人将是欧盟委员会的工作，因此一旦该机制开始运作，欧盟将有一段前期冲刺工作，在进入更广泛、持续的监测、调查和执法工作之前，确定所有的守门人(并消除对指定的任何法律挑战)。

为了遵守 DMA，看门人必须做什么和不做什么？

欧盟希望有一长串的要求来塑造市场巨头的行为，以确保数字市场保持开放和可竞争(或者，嗯，可以在他们可能已经倾斜的情况下被打破)。

自委员会最初的提案以来，其中许多都得到了保留，或者得到了加强和扩展。还增加了几个新的。

第 5 条和第 6 条如果 DMA 就是这些关键列表出现的地方。

通过 trilogue 过程引入的一个主要新需求是消息传递平台的互操作性。

这侧重于“基本”功能，如发送短信、照片、视频和文件的能力，而不是全功能对等。它也将从一对一的信息传递开始；群聊将在两年内逐步推广，视频电话/会议将在四年内逐步推广。

其工作方式是，较小的消息平台将能够向网守请求互操作性(网守有义务提供互操作性)。但这类平台没有义务接受这一权利；这是他们的选择。

他们的用户还需要选择能够跨平台发送消息——这样用户就不会仅仅因为服务选择插入网守的 API 而被迫接受跨平台消息。因此，共同立法者认为这是一个“不对称的”互操作性需求。

我们知道 DMA 中没有文字限制会阻止一个网关守护设备向另一个网关守护设备请求互操作性。但是平台巨头们——比如苹果的 iMessage 服务或者脸书的 Messenger——是否会选择这么做是另外一个问题。

欧盟立法者强调，他们非常关注消息互操作性的安全元素，规定所有跨平台通信必须保持相同的安全级别(例如，如果是 E2EE，则不能降低到较低的加密级别)。

除了消息传递，该集团的共同立法者只同意在未来评估社交媒体的互操作性——所以现在还没有讨论，可能是因为认为额外的技术复杂性。

他们还同意成立一个新的高级别顾问小组，向委员会提供跨领域的技术部门咨询，以支持其在互操作性等领域的工作。

在 DMA 规定的另一个主要新内容中，议会推动包括限制个人数据如何用于跟踪广告的努力在 trilogue 谈判中幸存下来。

此外，还有进一步的协议，以确保这个问题也将在 DMA 的姐妹法规中得到解决，更广泛地应用数字服务法案(仍在通过 trilogue)。

因此，这里的共识跨越了两个独立的(如果有联系的)立法。这是值得注意的，因为广告部分是后来才加入的，而且追踪广告行业已经做了多少反游说来试图规避限制。

至于其中的 DMA 部分，网守必须获得用户的明确 同意，才能将他们的个人数据组合起来用于广告——这一条款最终可能会迫使 Meta(脸书)向欧洲用户提供在使用其服务时不被跟踪和建档的选择。

欧盟联合立法者还同意扩大强制性的选择屏幕，让消费者选择他们自己喜欢的搜索引擎、浏览器和虚拟助手——也就是说，而不是看门人能够通过捆绑预先选择或强制使用他们自己的产品。尽管立法者似乎拒绝进一步扩大其他关键服务(如电子邮件)的范围，因为他们担心用户体验负担过重的风险。

核心服务的公平访问权——最初是在 DMA 中，重点关注第三方开发者和运营移动应用商店的看门人——也已扩展到搜索和社交媒体。

这使得网守有义务对他们应用于商业用户的条款保持透明，并提供一种争端解决机制。(这也有助于委员会更快地发现潜在的不公平条款和/或行为，以便更快地解决问题；但委员会本身不会像脸书·佩奇那样监督弗兰德。)

移动操作系统允许应用程序和应用商店侧装的义务也被保留——但经过一些修改，试图达成妥协，平衡消费者的选择和引入恶意软件的风险等安全问题(苹果等科技巨头在游说反对这一条款时多次提出这一论点)。

这种妥协的具体细节尚不清楚，但我们知道它将涉及以某种方式让用户定义他们自己的风险级别，例如通过他们在设置级别可用的选项。

在其他地方，对网守自己服务的自我偏好的禁令仍然有效，比如在他们策划并呈现给用户的内容排名中；规定网守不能阻止用户卸载预装的应用程序，以及更广泛的支持措施来实现服务切换，等等。

如果不遵从，把关者将面临哪些处罚？

对于违反该制度的看门人，最高可处以全球年营业额 10%的罚款，对于屡次违反该制度的看门人，最高可处以 20%的罚款。

后者指的是系统性不遵守的情况，我们的理解是，这种情况被定义为八年期间至少三次不遵守决定。(还需要进行一项法律测试，表明有问题的看门人保持或加强了他们的地位。)

DMA 的处罚制度还允许在系统违规的情况下进行非经济处罚，保留了委员会可以下令采取结构性补救措施的可能性，例如解散一个看门人的商业帝国。

也就是说，该政权显然有意避免这种单向的结果，因为这种权力在很大程度上是以延期储备的形式持有的(作为一种核选择；更多的是害怕它的存在，而不是使用)，大部分执行资源将用于实现对预先市场规则的遵从。

此外，委员会能够与看门人进行监管对话，以确保他们理解规则和要求，因此它也可以推动平台做出改变，帮助他们从一开始就避免罚款。

然而，对于那些决定无视欧盟事前竞争规则的看门人来说，DMA 处罚的另一个有趣之处是看门人可能会被暂时禁止进行并购。

这一举措看起来是为了防止杀手级收购现象。但这种 M&A 禁令会持续多久还不清楚。

在今天的新闻发布会上，竞争专员和 EVP Margrethe Vestager 围绕这个话题进行了交谈，顺便提到了最近的谷歌-Fitbit 收购案(去年完成了)——以及欧盟接受的允许继续进行该收购的条件(包括有时限地禁止谷歌使用 Fitbit 健康数据进行广告定位)，并继续指出欧盟一直在试图调查更多大型技术收购，以评估“实地”影响，并因此已经加强了其合并执法。

维斯特格认为，DMA 对限制他们做 M&A 的能力的看门人实施制裁在行为框架内是合适的，并指出有多少欧洲初创公司被大型科技公司抢购。她补充说:“这整个立法完全涉及把关者的行为，以及如何确保市场保持竞争，当然，在我们看来，合并在我们具体的合并控制中发挥着作用。”

作为 DMA 一般报告要求的一部分，所有守门人也必须在 M&A 之前通知委员会。

谁来执行 DMA？

欧盟委员会本身将是唯一的执行者——但其最初的提议已经修改，允许国家当局在调查方面有更大的参与。

这至少在一定程度上可以衡量欧盟委员会在 DMA 方面承担了多少(新的)工作。

与拥有相关专业知识的成员国机构开展一些合作，可能有助于减轻负担和加快执行。

也就是说，如果一个国家当局开始调查，委员会随后接手，我们理解国家调查预计将会结束。

该委员会表示，预计需要增加约 80 人来处理 DMA 的工作量。这些员工中的一部分将通过重新部署现有员工来实现，另一部分将是新员工——后者的重点是增强其技术专长。

欧盟委员会预计 DMA 将于何时生效？

在这个阶段，10 月份的某个时间据说是“可能的”——尽管还没有确定。

临时文本仍需要检查，以产生最终的法律文件(在所有各种欧盟语言)，供理事会和议会批准(后者在全体会议上表决)。但欧盟立法的主要障碍是昨日达成一致的政治谈判。

成员国有六个月的时间将泛欧盟制度转化为国家立法。所以可能 2023 年才会看到真正的 DMA 烟火。

到目前为止，对该计划的反应如何？

有趣的是，看到美国的行业观察人士对昨天的 trilogue 协议公告有多么震惊/惊讶甚至恐惧，他们似乎没有注意到欧盟立法者几年前详细提出的旗舰改革。

另一方面，欧洲人，包括消费者和企业，以及多年来一直倡导竞争改革以打破数字市场的无数公民社会团体，听起来——广泛地——表示支持，同时非常希望提醒欧盟委员会，精心制定的监管只取决于与之相伴的执法质量。

该地区所有人都将关注欧盟委员会如何应对这一巨大挑战。

至于更具体的批评，受到最多批评的措施是消息应用程序的互操作性义务，这从技术和/或产品体验的角度受到了一些方面的批评，人们担心平台可能在滥用监控或内容审核等领域对安全或其他安全特定流程产生潜在影响。

担忧包括互操作性可能会引入漏洞，也可能会破坏脆弱的安全系统——有时还会伴随着强烈的偏执情绪，即欧盟试图以竞争改革为借口，呃，破坏强加密。

尽管如此，还是有可能找到相反的技术观点:总部位于欧洲的消息公司 Element——它在分散矩阵协议上开发应用程序——从一开始就热衷于支持互操作性。

尽管它主张欧盟立法者更进一步，采用基于标准的方法，这将支持更全面的互操作性，而不是委员会选择的开放 API 路线和核心功能。

即便如此，Element 的联合创始人和首席运营官 Amandine Le Pape 仍然很高兴消息传递的(一些)互操作性被包含在 DMA 中。“任何互操作性都比目前的围墙花园要好，”她告诉我们。“开放的标准方法可能会随后出现，特别是如果它是由行业自下而上推动的，行业最终可能会明白，通过所有人都说同一种语言，每个人的生活都变得更轻松、更安全。”
