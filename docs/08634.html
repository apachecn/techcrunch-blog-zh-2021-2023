Paragon 希望成为 SaaS 应用程序的格子图案

原文：https://web.archive.org/web/https://techcrunch.com/2022/07/27/paragon-wants-to-be-the-plaid-of-saas-apps/

初创公司 Paragon 建立了一个平台，为企业客户集成和聚合各种软件即服务(SaaS)应用程序，该公司在由灵感资本(Inspired Capital)领投的 A 轮融资中筹集了 1300 万美元，之前的投资者包括 FundersClub 和哥鲁达风险投资公司。首席执行官兼联合创始人 Brandon Foo 表示，这部分资金将用于“扩展”和扩大 Paragon 的工程和上市团队。

Paragon 是 Y Combinator 2020 年冬季系列的一部分，旨在允许软件产品与第三方应用程序集成，而不会中断现有的工作流程。公司可以使用该平台将 SaaS 集成构建到他们的产品中，然后提供给他们的最终用户，其功能包括完全托管的身份验证和预构建的集成界面。

Foo 于 2019 年与前优步工程师 Ishmael Samuel 一起创立了 Paragon。Paragon 是 Foo 继专注于协作的电子邮件应用 Polymail 之后的第二次创业。

“在构建 Polymail 的过程中，我们不得不花几个月的时间，成为我们构建的每个集成的不同供应商特定的身份验证方法、API 和文档的专家。感觉我们每次都在重新发明轮子。然而，客户不断要求更多的集成，这很快就让我们无法跟上，更不用说维护我们已经建立的所有这些集成了，”Foo 在一次电子邮件采访中告诉 TechCrunch。“我后来意识到，这是今天每个 SaaS 公司都面临的问题。当 Ishmael 和我创建 Paragon 时，我们试图解决一个我们作为软件开发人员亲身经历的永无止境的问题。

Foo 表示，Paragon 目前支持大约 45 个与 SaaS 应用程序的预建集成，包括 Salesforce、HubSpot、Slack 和 Shopify。最近，该公司推出了一个集成构建器，允许客户在 Paragon 上使用公共 SaaS API 创建自己的定制集成，而无需编写代码。

图片来源: Paragon

Paragon 提供两种版本的服务:云托管和内部部署。两者都存储和管理终端用户身份验证凭据，因此公司不必单独为每个应用集成手动构建和维护身份验证。它们都存储集成数据，用于记录和观察。

“软件公司必须提供集成才能在市场上保持竞争力——这已经成为 SaaS 买家的期望……然而，从零开始构建集成需要大量的工程资源——更不用说在 SaaS API 不断变化的情况下维护集成所需的工作了，”Foo 说。Paragon 提供了一个简单的产品化解决方案，将每个 SaaS 集成的复杂性抽象到一个软件开发工具包中，该工具包可以原生嵌入到任何产品中，以提供无缝的最终用户体验

Foo 声称 Paragon 目前每月为其客户群中的大约 1 亿个请求提供服务。这是一个良好的开端，但随着竞争产品的出现，挑战将是保持增长。虽然没有解决完全相同的问题， Pipedream 提供了一个集成平台，用于构建工作流和连接云应用程序和服务。着眼于金融应用，Stripe 最近推出了应用市场，这是一个包含第三方 SaaS 应用的脚本和工具集合，与 Stripe 的支付处理一起工作。

不过，Foo 表示，科技行业的放缓是意外的巧合(不包括裁员)。

“它实际上是 Paragon 的催化剂，因为工程效率比以往任何时候都更加重要。软件公司需要用更少的资源做更多的事情，但是不能因为不能满足客户的集成需求而继续失去有价值的交易。Paragon 解决的最大挑战是工程资源。将内部工程资源花在外部集成上会占用他们构建核心产品的宝贵时间。"

迄今为止，Paragon 已经筹集了 1650 万美元的资本。
