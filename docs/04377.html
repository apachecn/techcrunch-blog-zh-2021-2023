OpsLevel 筹集 1500 万美元帮助开发者管理他们的微服务 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/01/opslevel-raises-15m-to-help-developers-manage-their-microservices/

帮助开发团队在集中式开发者门户中组织和跟踪他们的微服务的初创公司 OpsLevel 今天宣布，它已经筹集了 1500 万美元的 A 轮融资。这轮投资由 Threshold Ventures 牵头。之前的投资者 Vertex Ventures，领投了该公司的 500 万美元种子轮，S28 Capital 和 Webb Investment Network 也参与了这轮投资，此外还有一些天使投资人。

该公司由前 PagerDuty 工程师 John Laban 和 Kenneth Rose 共同创建，其核心是为开发团队提供集中的服务目录。随着 DevOps 越来越受欢迎，工程师越来越多地承担部署和操作他们编写的代码的任务。但在现实中，以及在生产中，通常不清楚谁拥有给定的微服务。

OpsLevel 联合创始人约翰·拉班和肯尼斯·罗斯。图片来源: OpsLevel

“当我们[在 page duty]时，我们看到所有这些公司都有微服务或分布式架构，他们都在努力适应服务所有权和拥有服务的概念，”拉班告诉我。“从那以后，随着向基于微服务的分布式架构的转变，我们看到这种趋势变得更加糟糕。像 DevOps、Shift Left 和 DevSecOps 这样的运动也随之兴起，所有这些额外的责任都堆积在开发团队身上。”

拉班认为，这个问题的答案是自动化。因此，为了帮助这些工程团队，OpsLevel 为他们提供了一个开发人员门户，该门户提供了一个单一界面来管理他们所有的服务、系统和工具。为此，该服务集成了各种各样的服务，从 Git 和 GitLab 到 Jenkins、GitHub Actions、PagerDuty、AWS ECR、Datadog、Sentry、Snyk 和 SonarQube。

图片来源: OpsLevel

“我们希望为开发者铺平道路[…]，成为他们使用服务做任何事情的第一站。例如，如果一个团队想要构建一个新功能，通过开发人员门户，他们可以直接创建服务来支持该功能，用正确的工具链装配它，然后持续监控未来的运营成熟度，”Laban 解释道。

该公司的客户包括像 Auth0，Chegg，车队，Hootsuite，安德玛和 Zapier。该团队尚未准备好披露任何财务数据，但拉班告诉我，他们看到了强劲的增长，例如，仅 1 月份的收入就超过了上一季度。

当然，这也是投资者希望看到的增长。Threshold Ventures 的合伙人莫·伊斯拉姆(Mo Islam)表示:“在过去十年中，构建和交付可扩展软件的两个最重要的发展是微服务的兴起和 DevOps 的采用。“OpsLevel 正在构建领先的开发人员门户，以便工程和运营团队能够大规模拥有和运营他们的生产基础设施。”

图片来源: OpsLevel

该公司计划使用新的资金来扩大其工程团队，以加快其产品开发工作。“我们已经看到市场上对(OpsLevel)的大量需求，”拉班说。“总的来说，这是一个释放开发人员生产力的巨大机会，我们认为自动化是其中的一个重要部分，使开发人员可以轻松快速地交付产品，并在交付产品时做正确的事情，也就是说，在构建服务时考虑安全性和可靠性。”
