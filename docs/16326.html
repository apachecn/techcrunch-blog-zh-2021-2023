微软循环，一个类似概念的中心，用于管理在微软 365 应用和服务之间同步的任务和项目，今天在公开预览中发布。

拥有 Microsoft 帐户或 Azure Active Directory 帐户的用户可以使用 Loop。iOS 和 Android 的配套应用即将推出；微软没有提供确切的时间。

在 Ignite 2021 上发布的 Loop 在某种程度上是微软对谷歌工作空间的回应，谷歌工作空间为实时、数字优先的项目合作提供了仪表盘。与大多数基于团队的生产力平台一样，Loop 拥有项目进度跟踪工具和与服务的双向同步，包括 Trello。

那么它有什么独特之处呢？嗯，Loop 由三个主要元素组成——Loop 组件、Loop 页面和 Loop 工作区——它们一起可以用于将实时内容块粘贴到 Outlook、Microsoft Teams 和 Word 等应用程序中。(我的同事 Frederic Lardinois 曾经将这种体验比作命运多舛的 Google Wave 。)微软表示，它旨在弥合在虚拟团队中工作的差距——这种差距在开发 Loop 的疫情期间变得越来越明显。

图片来源:微软

循环组件本质上是 JavaScript 小部件，让用户在工作流程中进行协作，例如在聊天、电子邮件、文档或在线会议中。组件可以像列表、表格和注释一样简单，也可以像 Dynamics 365 中的客户销售机会一样复杂，它们在 Microsoft 365 应用程序中始终保持同步。

由于同步功能，对循环组件(例如表格)的编辑将会反映在嵌入或共享的任何位置。微软表示，未来计划从 Dynamics 365 记录开始，添加方便业务工作流程的循环组件，并允许开发人员构建完全自定义的循环组件。

至于循环页面，它们是灵活的画布，用户可以在其中组织他们的循环组件，并拉入链接、文件或数据等元素。(Loop 为快速设置提供了许多页面模板。)循环工作区的范围更广，代表了共享平台，用户可以在其中查看和分组对他们的项目重要的一切。

事实上，在创建工作空间时，Loop 可以搜索并推荐相关的文档和同事。多达 50 人可以同时编辑一个工作区，用表情符号和评论对编辑做出反应，但微软建议两到 12 人一组，这样界面不会感觉太幽闭。

图片来源:微软

同样值得注意的是，微软正在将其新的微软 365 Copilot 系统构建到 Loop 中。目前在私人测试中，人工智能驱动的 Copilot 将提供建议，以创建头脑风暴或蓝图，并允许一个或多个用户编辑这些建议，然后在 Outlook 和 Teams 等应用程序中分享它们。

这是一系列令人印象深刻的特性，但是 Loop 能和 concept 竞争吗？这还有待观察。作为 The Verge 的循环笔记报道，idea 并没有满足于现有的成就，最近推出了一个人工智能系统，该系统可以分析会议笔记，创建摘要，显示重要信息，甚至重写和生成文本。

对于 Loop 来说，现在还为时过早，但微软——尽管无可否认地受益于微软 365 内置的庞大用户群——仍有艰巨的工作要做。
