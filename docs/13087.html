苹果为 iCloud 数据推出端到端加密技术

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/07/apple-launches-end-to-end-encryption-for-icloud-data/

苹果今天推出了高级数据保护，这是一种新的可选端到端加密方案，可以防止客户 iCloud 中的数据在“不可信”的设备上被解密。具体来说，高级数据保护将确保 iCloud 账户的大部分数据安全，即使在苹果遭到黑客攻击的情况下，而且——正如《华尔街日报》的一篇文章所指出的那样——将会让阻止苹果访问 iCloud 手机备份，以回应执法部门的请求。

高级数据保护目前在美国面向 Apple Beta 软件计划的成员提供，并将于今年年底面向所有美国用户(世界其他地区将于 2023 年推出)，它使 iCloud 用户的受信任设备(如 iPhones 和 MAC)能够单独访问其大部分数据的加密密钥。(对于门外汉来说，加密密钥是专门为加扰和解扰数据而生成的随机位串。)一旦该功能被启用，苹果服务器就不能代表用户修改 iCloud 的某些设置，也不能访问存储在 iCloud 备份、照片、笔记和第三方开发者选择标记为加密的 CloudKit 字段中的数据。

在推出高级数据保护之前，iCloud 用户无法阻止苹果公司倾倒设备备份的内容，包括短信和联系人，如果它选择这样做的话。读者可能还记得这家科技巨头与联邦调查局(FBI)就圣贝纳迪诺枪击案枪手的加密 iPhone 数据展开的斗争，在此期间，该机构试图通过法院迫使苹果解锁一部受保护的 iPhone。当时，苹果辩称，美国联邦调查局可以通过其服务器上未加密的 iCloud 备份来获取它所寻求的数据。

图片来源:苹果

值得注意的是，高级数据保护不能与 iWork 协作工具、照片中的共享相簿、iCloud 邮件、通讯录或日历一起使用；苹果归咎于互操作性要求。为了启用该功能，用户必须为他们的 Apple ID 注册双因素认证，并在他们的设备上设置密码或通行码，以及将这些设备更新到最新的可用软件(iOS 16.2、iPadOS 16.2、macOS 13.1、tvOS 16.2、watchOS 9.2 和最新版本的 iCloud for Windows)。

高级数据保护尚不支持受管理的 Apple IDs 和儿童帐户，Apple 在一份支持文档中澄清了。

给聪明人一个忠告:未能为高级数据保护设置恢复方法的代价是相当大的。苹果指出，如果恢复失败——例如，如果恢复联系人的信息过期——任何加密的 iCloud 数据都等同于丢失。

除了先进的数据保护，苹果今天早上宣布其产品生态系统将加入另外两项安全相关的功能:iMessage 联系人密钥验证和安全密钥。

苹果表示，iMessage 联系键验证允许“面临非凡数字威胁”的用户，如记者和政府成员，选择进一步验证他们只与他们想要的人发送消息。苹果公司表示，如果对手侵入云服务器，窃听加密通信，并允许用户当面、在 FaceTime 上或通过安全呼叫比较特殊的身份验证联系验证码，iMessage 联系键验证将发出警报。

与此同时，安全密钥建立在苹果现有的双因素认证系统上，要求硬件安全密钥作为认证个人 Apple ID 凭据的两个因素之一。硬件钥匙有系列的口味和价位，通常使用蓝牙、NFC 或 USB 进行认证。

苹果公司表示，iMessage 联系人密钥验证和 Apple ID 的安全密钥将于 2023 年开始在全球范围内提供。
