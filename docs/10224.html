打车巨头优步表示，继上周发生“网络安全事件”后，其服务已经开始运行，黑客入侵了该公司的网络并访问了存储大量客户数据的系统。

在周一之前，优步很少提及此事。安全研究人员在与黑客的对话中发布到 Twitter 上的优步内部网络截图显示，可以访问内部仪表盘、公司 Slack 及其 HackerOne 账户。优步在周一的更新中表示，黑客窃取了一些内部信息和垃圾信息，但没有敏感信息——如信用卡数据和旅行记录——被窃取，这就留下了其他个人用户信息是否被泄露的问题。

这名自称 18 岁的黑客告诉安全研究人员，他们通过窃取一名员工的密码，并诱骗该员工批准攻击者的优步多因素认证(MFA)推送通知，闯入了优步的系统。

一旦他们在优步的网络上有了关键的立足点，黑客声称发现了一个包含高特权凭证的网络共享，允许他们几乎不受约束地访问公司系统的其余部分。

优步周一表示，该黑客隶属于 Lapsus$，一个在今年早些时候入侵了 Okta 、微软、英伟达、 Globant 和 Rockstar Games 的组织，侵入了一个优步承包商的用户账户。优步表示，在漏洞发生后，它短暂关闭了一些内部工具，客户支持业务“受到的影响很小，现在已经恢复正常。”

优步事件的最终分析可能还需要一段时间才能知道，但安全专家已经开始剖析黑客是如何进入优步的系统的——通过轻松击败该公司的 MFA 安全系统。

并非所有的 MFA 选项都是平等的，在输入用户名和密码后，您必须完成额外的步骤来验证登录的是您本人而不是攻击者；有些人比其他人更强壮。通过短信发送的代码可能会被拦截或窃取，但在很大程度上已被移动认证应用所掩盖，这些应用不断生成循环随机代码或发送几乎不可能被拦截的推送通知。但是，随着攻击变得越来越聪明，一些最强的 MFA 保护正在被利用人类行为中的漏洞所击败。

如果世界上最大的公司之一都能被这样攻破，你如何防范另一次优步黑客攻击？

黑客是怎么打败 MFA 的？

据研究人员称，员工的凭证可能被安装在员工电脑上的 RedLine 等窃取密码的恶意软件窃取。Lapsus$也被称为利用 Redline 窃取员工密码。优步说，黑客可能从黑市上购买了被盗的密码。

一旦被盗，黑客必须击败优步的多因素认证，这增加了一个额外的障碍，以防止攻击者使用被盗的凭据闯入公司的网络。

在推特上发布的一段对话中，黑客确认他们通过社交工程进入了优步的网络，使用窃取的凭据向该员工发送了一个多小时的重复推送通知，然后“在 WhatsApp 上联系了他，自称来自优步 IT，告诉他如果他想停止，他必须接受，”黑客说。“嗯，他接受了，我添加了我的设备，”黑客写道。

这就是一些人所说的 MFA 疲劳，黑客利用员工在整个工作日必须重复登录和重新验证其访问权限的情况，向员工发送大量推送通知，通常是在工作时间之外，希望员工最终出于愤怒接受登录请求。

社交工程专家兼 SocialProof Security 首席执行官雷切尔·托巴(Rachel Tobac)表示，MFA 疲劳攻击是绕过 MFA 入侵组织的“最简单的方法”之一。

“是的，有时 MFA 疲劳看起来像是在受害者睡觉时重复请求，直到他们接受为止，但通常只是在工作日开始时连续发送 10 次请求，或者只是在会议期间发出令人讨厌的垃圾请求，直到受害者接受为止，”Tobac 告诉 TechCrunch。

在欺骗员工接受推送通知后，黑客可以像员工一样发送 MFA 推送通知，授予他们持续访问优步网络的权限。

解决方法是什么？

安全专家普遍认为任何级别的 MFA 都比没有强，但是 MFA 本身并不是万能的。优步不是唯一一家使用多因素身份认证但网络仍然受到威胁的公司。

2020 年，黑客通过诱骗一名员工将他们的凭据输入他们设置的钓鱼页面，闯入了 Twitter 的网络，黑客用这个页面生成了发送到该员工设备的推送通知。根据纽约州政府的调查，这名员工接受了提示，让攻击者进入。最近，短信巨头 Twilio 被利用类似的网络钓鱼攻击攻破，Mailchimp 也被社会工程攻击所黑，该攻击诱骗一名员工交出他们的凭证。

所有这些攻击都利用了多因素身份认证中的弱点，通常直接针对相关个人，而不是在这些高度审计的系统中寻找安全缺陷。

Cloudflare 是最近一系列网络攻击中唯一一家被锁定的公司，这些攻击阻止了网络入侵，因为它使用了无法被钓鱼的硬件安全密钥。在一篇博客文章中，Cloudflare 承认，虽然一些员工“确实中了钓鱼邮件的圈套”，但它使用硬件安全密钥，要求员工在输入凭据后将 USB 设备插入他们的计算机，从而阻止了攻击者闯入其网络。Cloudflare 表示，攻击的目标是员工和系统，“我们认为大多数组织都有可能被攻破。”

安全钥匙被视为 MFA 安全的黄金标准，但它们也有自己的挑战，尤其是钥匙的成本和维护。“我们花时间争论硬件安全密钥对所有人的必要性，但在该领域，一些组织仍在为内部访问的强制性 SMS 双因素身份验证或 MFA 提示而斗争，”Tobac 说。

虽然随机生成的代码或推送通知的 MFA 绝不是完美的，正如优步的违规所证明的那样，“我们不能让完美成为好的敌人，”托巴说。"小的改进随着时间的推移会产生很大的不同。"

“我现在从组织那里得到的最大问题是如何配置现有的 MFA 工具，以限制我们在优步、Twilio 和 Twitter 黑客中看到的攻击方法，”Tobac 说。“这有助于组织思考可以快速做出的小改进，这样他们就不会陷入数月(甚至数年)的内部更新争论中。”

一个重要的改进是 MFA 号码匹配，它通过在登录者的屏幕上显示一个代码，并必须将该代码输入到该人的验证设备上的应用程序中，使社交工程攻击变得更加困难。这个想法是，攻击者需要目标的凭证和他们验证过的设备，类似于安全密钥。

微软、 Okta 、 Duo 提供 MFA 号匹配。但是正如安全研究员 Kevin Beaumont 所指出的，微软的解决方案仍然处于预览阶段，Okta 的号码匹配产品捆绑在昂贵的许可层中。优步依靠 Duo 进行 MFA，但据报道在被入侵时没有使用号码匹配。

博蒙特在推特上写道:“在其他新闻中，你可以看到一群青少年正在实时改造网络安全行业。”

toba 说，网络卫士还可以设置警报和限制用户可以获得多少推送通知，并在 Twitter 帖子中提到，并开始向一组测试用户推出安全密钥，目的是每个季度增加该组用户。

就其本身而言，优步周一表示，它正在加强其外交部政策，以回应其违约行为。

至于黑客是如何利用一个承包商偷来的密码获得其余关键系统的高权限凭证的，优步可能仍有很多问题要回答。
