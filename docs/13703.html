Okta 证实黑客窃取源代码后又有一个漏洞 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/22/okta-breach-source-code-github/amp/

图片来源: 迈克尔六世 / Getty Images

Okta 已经证实，它正在对另一个重大安全事件做出回应，此前一名黑客在其 GitHub 仓库被攻破后访问了它的源代码。

这家身份和认证巨头在周三的一份声明中表示，本月早些时候，GitHub 向其通报了对其代码库的“可疑访问”。Okta 后来得出结论，黑客利用这种恶意访问复制了与该组织面向企业的安全解决方案“员工身份云”( WIC)相关的代码库。

Okta 在一份声明中说:“当 Okta 得知可能的可疑访问时，我们立即对 Okta GitHub 仓库的访问进行了临时限制，并暂停了所有 GitHub 与第三方应用程序的集成。”

当被 TechCrunch 询问时，Okta 拒绝透露攻击者是如何设法进入其私人资料库的。

Okta 表示，不存在对 Okta 服务或客户数据的未经授权的访问，与 Auth0 相关的产品(该产品于 2021 年收购)不受影响。“Okta 并不依赖其源代码的保密性来保证其服务的安全性。Okta 服务保持全面运作和安全，”Okta 说。

该公司表示，自从被警告违规后，它已经审查了最近对 Okta 软件库的访问，审查了所有最近对 Okta 软件库的提交，并轮换了 GitHub 证书。Okta 表示，它也已通知执法部门。

Okta 没有明确表示，它是否有日志等技术手段来检测自己的系统是否被访问，或者其他哪些数据可能已经被泄露。

本周早些时候，在 Okta 发布声明之前，该公司的最新事件首先由哔哔声计算机报道。

今年早些时候，Okta 成为臭名昭著的 Lapsus$敲诈集团的目标，该集团获得了 Okta 第三方服务提供商 Sykes 的一名客户支持工程师的账户，并发布了 Okta 应用程序和系统的截图。今年八月，Okta 经历了第二次妥协，此前它成为了的目标，另一次黑客攻击攻破了 100 多个组织，包括 Twilio 和 DoorDash 。
