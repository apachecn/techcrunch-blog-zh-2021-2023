尽管开源软件分发的前提是“免费的”，但像 RedHat、MongoDB、GitLab 和 Elastic 这样价值数十亿美元的公司已经开始以开源软件为核心建立盈利业务。

但是，一个更小的开源项目有可能进入这片充满商业机会的土地吗？

COSS 正在加速

总的来说，商业开源(COSS)的趋势令人鼓舞。像 Meilisearch 和 Supabase 这样的新产品获得的关注比 MongoDB 这样的 COSS 传奇要快得多，MongoDB 的创立要早得多。

让我们给上面的图表更多的上下文。从 2010 年到今天，GitHub 用户数量已经从 50 万爆发到 1.03 亿。人们可能会认为，新用户涌入社区是明星数量增加的驱动力。

你的开源项目可以从一个宠物项目开始，但前提是你必须投入时间。

但是，与此同时，项目(存储库)的数量以更高的速度增长:从 60 万增加到 3 . 59 亿。对开源产品的投资几乎增加了两倍，从 2015 年的 58 笔交易增加到 2021 年的 144 笔交易。

重要的是，每个存储库的平均用户数量从 0.8 减少到了 0.3。这意味着 GitHub 明星的竞争比以往任何时候都更激烈，这表明上面的超级明星确实是局外人，很难复制他们的成功。

从这些数字、投资和明星动态来看，COSS 目前正处于一个甜蜜点。

也就是说，我们必须记住，COSS 和开发工具仍然只占据一个利基市场。毕竟全球只有 2500 万到 3000 万的软件开发者。尽管这个行业的生产率远高于许多其他行业，但这个数字只是金融或零售等其他大市场的一小部分。

此外，从某种程度上来说，将为开发者构建的产品货币化仍然是一个悬而未决的问题。

如何将开源货币化

开源赚钱有多种策略。

先说个简单的:众筹和捐款。赠款和捐款属于同一类，唯一的区别是你如何筹集资金。基金会是从大型赞助者或大量赞助者那里收集捐款的一种手段。

可悲的是，这样的收益不太可能覆盖日益增长的成本。以 PostCSS 为例，这是一个由 Andrey Sitnik 构建的广受欢迎的 CSS 框架。通过一个开放的集体中心，GitHub 上有 27，000 颗星，他每年只收取大约 12，000 美元，尽管像 Meta 或 Google 这样的大公司使用 PostCSS，并可能支持该产品。
