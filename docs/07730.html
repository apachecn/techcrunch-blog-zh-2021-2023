Helios 想要驯服微服务开发的混乱

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/29/helios-wants-to-tame-the-microservices-development-chaos/

总部位于特拉维夫的初创公司 Helios 希望让开发人员在开发阶段更容易理解、排除故障和测试他们的分布式系统，该公司今天宣布正式上市，并筹集了由 entre Capital 和 Amiti VC 共同牵头的 500 万美元种子轮投资。包括 Benny Schnaider、Guy Podjarny(Snyk 的联合创始人)、Adi Sharabani 和 Yair Amit(sky cure 的联合创始人)以及 Guy fig hel(New Relic 的总经理)在内的多位天使投资人也参与了此轮投资。

“作为开发人员，我们希望帮助您(甚至在生产之前)加快开发过程、故障排除过程和测试过程，因为今天，我们知道开发人员确实在努力。Helios 首席执行官伊莱·科恩(他与首席技术官兰·诺齐克共同创立了该公司)告诉我:“当你在系统的一个地方做出改变时，你不一定知道它会如何影响系统的其余部分。”。

由于开发人员很难理解他们的代码如何与系统的其余部分交互，开发过程通常会变慢。与此同时，即使一个微服务或 API 中的一个小错误也可以迅速关闭一个大型分布式应用程序。

Helios 背后的想法是帮助开发人员准确理解他们的代码如何与应用程序的其余部分进行交互。使用开放遥测，一个开源可观测性工具的集合，Helios 从应用程序中提取分布式跟踪数据，并将其放入开发人员的上下文中。例如，这意味着他们可以准确地重放他们的代码如何与大型应用程序交互，以便更容易地发现和重现问题。

entre Capital 的普通合伙人埃兰·比尔斯基(Eran Bielski)在一封电子邮件中说，“成长中的科技公司经常努力扩展他们的云原生架构。“公司希望专注于建立一个成功的企业，但开发人员的日常工作往往充满摩擦，从而减缓了进度。Helios 对开发速度和生产率的关注正是市场所需要的，我毫不怀疑世界上的每一家软件开发公司都将很快使用这样的工具。"

公司开放的遥测 SDK 目前支持 Go，Java，。NET、Node.js、Python 和 Ruby，该服务可以与 Postman 等工具集成，构建 Nx.dev 等系统，Airbrake 和 Sentry 等错误监控解决方案，以及 Databricks 等数据仓库和一些流行的可观测性平台和测试框架。这应该涵盖了相当多的用例，但有了新的资金，该公司计划随着时间的推移扩大集成的数量。

这项服务提供基本的免费计划和每月 99 美元起的付费计划，现已全面推出。不过，如果你想在全投入之前看看它是什么样子，该公司还提供了一种简单的方法，在预先填充的沙盒环境中测试它，这是更多公司应该做的事情。
