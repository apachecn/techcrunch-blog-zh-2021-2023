Dylibso 筹集 660 万美元帮助开发者将 WebAssembly 投入生产

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/24/dylibso-raises-6-6m-to-help-developers-take-webassembly-to-production/

WebAssembly(简称 Wasm)是一个开放标准，它使基于浏览器的应用程序能够以接近本机的性能运行。它还扩展到支持非浏览器环境，这是最近围绕它进行大量宣传的原因。但是像任何新兴技术一样，它需要一个更强大的工具生态系统来实现其全部潜力。

朝这个方向努力的公司之一是 Dylibso T1，这是一家初创公司，其使命是帮助开发团队将 Wasm 投入生产。该公司今天在巴塞罗纳 Wasm I/O 大会上宣布了一轮 660 万美元的种子资金，并以 Extism 而闻名，它的开源通用插件系统允许开发者将 WebAssembly 集成到他们现有的非 WebAssembly 代码库中。它还推出了 Modsurfer ，这是一个用于 WebAssembly 的记录和诊断工具系统，今天正式上市。

该公司的这轮融资由 Felicis 牵头，参与方包括 Boldstart Ventures 、 Pebblebed 和 Crew Capital 。

Dylibso 联合创始人兼首席执行官 Steve Manuel 之前在 Cloudflare 工作，在那里他为该公司的工人产品提供了 Wasm 支持，并在量子计算初创公司 Rigetti Computing 工作。

Extism 标志。图片来源: Dylibso

Manuel 说:“作为一个喜欢其他语言胜过 JavaScript 的人，我想:你知道，我必须使用这个 WebAssembly 运行时来为 Cloudflare 工作人员带来另一种语言。”作为一个附带项目，他构建了一个从 Rust 编译到 WebAssembly 并与 Cloudflare 的 API 集成的框架。但在这个过程中，他也意识到，一旦代码在生产中运行，他就无法真正观察到它是如何运行的，这使得调试成为一个挑战。

“有很多很多地方——以及很多很多现有的代码和运行时——开发人员喜欢在其中添加一点他们选择的语言，”他解释道。“从安全性和性能的角度来看，WebAssembly 为您提供了一个很好的解决方案，使您能够在另一个环境中可靠地运行代码，但代码是用不同的语言编译的。”

这就是 Dylibso 在 2022 年 12 月发起的 Extism 背后的核心思想。这是一个开源的插件系统，允许开发人员运行任何代码，这些代码可以在任何以前用不同语言编写的程序中编译成 WebAssembly。“我们的目标是尽可能容易地将 WebAssembly 集成到您的程序中，无论您的程序是否已经使用 Wasm。没关系。这很容易做到，”曼纽尔说。他强调，这不是一个商业产品，但主要是为了加速 WebAssembly 的采用——特别是对于运行在浏览器之外的代码。

Modsurfer 旨在解决 Manuel 的另一个问题:理解 WebAssembly 代码如何在生产中运行——以及它可能失败的原因。它对开发者是免费的，但它也是 Dylibso 的第一个商业产品，因为它也将提供 Modsurfer 的企业版。

图片来源: Dylibso

Manuel 解释说:“Modsurfer 为您提供了一个单一窗口，让您可以查看所有您感兴趣并需要跟踪的 WebAssembly 代码。它是记录系统的一部分，允许开发人员跟踪他们正在运行的所有 WebAssembly 代码，并且是这些代码的一部分分析工具。由于 WebAssembly 是一种二进制格式，代码有点像一个黑盒，Modsurfer 旨在为开发人员打开这些模块。它分析代码(通常可以检测出它最初是用什么语言编写的)，并返回关于它的复杂性、导入和导出、名称空间等数据。Manuel 说:“我们分析并进行所谓的圈复杂度分析，在这里我们实际上确定了在您的环境中运行这些代码的风险状况。

Modsurfer 现已全面上市。

图片来源: Dylibso

“由于技术缺乏工具，开发人员经常难以使用 WebAssembly，”企业战略集团首席分析师 Paul Nashawaty 说。“我们在我们的研究中看到，预计 WebAssembly 的市场及其对开发人员的价值可以与容器的采用相媲美，如果不是更大的话，但需要有正确的工具来帮助开发人员通过整个开发生命周期，并将 WebAssembly 投入生产。Dylibso 使开发人员能够通过一套产品轻松使用 WebAssembly，帮助他们集成 WebAssembly，并获得对其二进制代码的关键洞察力和可见性。”
