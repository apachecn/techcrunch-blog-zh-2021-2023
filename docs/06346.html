谷歌更新其 Firebase 后端即服务，以加快应用程序开发 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/11/google-updates-its-firebase-backend-as-a-service-to-make-app-development-faster/

在其 I/O 开发者大会上，谷歌今天宣布了对 Firebase 的大量更新，Firebase 是谷歌广受欢迎的后端即服务平台。这里的重点主要是与谷歌其他开发者工具和平台的更深入集成，以及整体开发者生态系统，以及一些将帮助开发者更好地保护他们的应用程序的更新。

第一批重大公告之一是，Android Studio 现在将推出一个新的应用质量洞察窗口，让开发人员可以直接访问 Firebase 的 Crashlytics 崩溃数据，允许开发人员查看他们的堆栈跟踪，并确定触发崩溃的特定代码行。“现在，开发人员可以在构建功能时融入流程。他们也可以看到，‘哦，我上一个版本中的这行代码有一堆错误。’他们可以点击进入，查看 Crashlytics 数据，了解崩溃的严重程度，它们可能发生在哪些设备上，等等。，以便他们能够真正快速地解决这些问题并重现它们，”Firebase 产品负责人 Francis Ma 解释道。

Flutter 开发者也将获得更好的 Crashlytics 支持。现在，他们只需点击几下鼠标，就可以为自己的应用程序设置 Crashlytics，并获得改进的崩溃报告，以及记录 Flutter 应用程序中的致命错误和接收 Crashlytics 的崩溃警报的能力。

图片来源:谷歌

对于 web 开发人员来说，Firebase it 使开发人员更容易使用 Angular 和 Next.js 等现代 web 开发框架，但也帮助他们更容易部署这些 web 应用程序。毕竟，现代 web 框架可能非常强大，但在部署应用程序时，它们也引入了许多复杂性。现在，开发人员可以简单地使用“firebase deploy”命令，firebase 将自动计算出应用程序的哪个部分应该部署在哪里，而不必担心依赖性。目前，这适用于 Angular 和 Next.js，但该团队计划在未来添加对更多框架的支持。

跨平台，Firebase 也使开发者更容易使用第三方 API，允许他们定制 Firebase 扩展以使用 Stripe 和 Twilio 等服务。现有的预打包扩展使开发人员更容易利用第三方 API，但正如经常发生的情况一样，开发人员经常遇到边缘情况或想要做一些稍微不同的事情。“我们认识到，开发人员可能会在他们的应用程序中使用 20 到 40 个 APIs 虽然扩展对于开发人员快速部署这些解决方案非常有效，但我们从他们那里听说，他们希望能够进行更多的定制，以便能够采用这种基线部署，并真正使之成为他们自己的，”马说。

图片来源:谷歌

该团队现在还在为 Snap 添加第三方扩展，例如，允许用户使用 Snap 登录，以及新的 Stream 扩展，以帮助开发人员在他们的应用程序中实现聊天，以及新的 RevenueCat 扩展，以管理应用程序内订阅。

在安全方面，Firebase 现在集成了新的 Play Integrity API，允许开发人员相信正在与其后端通信的给定 Android 应用程序没有被操纵(这是游戏经常发生的事情)。

对于苹果开发者来说，Firebase 团队正在改进对苹果 Swift 语言的支持。Swift 支持并不是全新的，但正如 Ma 指出的，该团队现在已经达到了一个里程碑，全面覆盖了 Swift。“只使用 Swift 的苹果开发者在使用 Firebase SDKs Swift 时可以期待那种直观的、更原生的支持，”马说。
