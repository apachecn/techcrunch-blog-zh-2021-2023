嵌入式设备是下一个勒索目标吗？TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/01/22/embedded-devices-ransomware/

2021 年将被铭记为勒索软件团伙将注意力转向关键基础设施的一年，目标是围绕制造业、能源分销和食品生产建立的公司。

仅 Colonial Pipeline 勒索软件就导致了 5500 英里管道的关闭，原因是人们担心对其 IT 网络的勒索软件攻击会蔓延到控制燃油分配管道的运营网络。

运营技术(OT)网络控制着对生产线、发电厂和能源供应的持续运营至关重要的设备，因此通常与公司面向互联网的 IT 网络相分离，以更好地隔离关键硬件免受网络攻击。针对 OT 网络的成功攻击很少，但是在殖民勒索软件攻击之后，CISA 警告说对关键基础设施所有者的威胁越来越大。

现在，安全研究人员正在警告位于这些 OT 网络上的嵌入式设备所带来的风险。嵌入式设备安全提供商 Red Balloon Security 在新的研究中发现，在现实网络中使用的嵌入式系统上部署勒索软件是可能的。

该公司表示，它在施耐德电气 Easergy P5 保护继电器中发现了漏洞，该设备是现代电网运行和稳定的关键，如果发现故障，就会触发断路器。

这个漏洞可能被用来部署勒索软件有效载荷，这是一个“复杂但可重复”的过程，红气球表示它已经实现了。施耐德电气的一位发言人告诉 TechCrunch，“它对网络威胁保持高度警惕，”并且“在得知施耐德电气 Easergy P5 保护继电器的漏洞后，我们立即着手解决这些问题。”

Red Balloon 的创始人兼联合首席执行官崔昂告诉 TechCrunch，尽管勒索软件攻击已经袭击了关键基础设施提供商的 it 网络，但 OT 嵌入式设备的成功入侵可能“更具破坏性”

“公司不习惯或没有经验从嵌入式设备本身的攻击中恢复过来，”他说。如果设备被破坏或变得不可恢复，则需要寻找替换设备，这可能需要几周时间，因为供应有限

安全资深人士 Window Snyder 去年创办了一家初创公司，帮助物联网制造商可靠、安全地向他们的设备提供软件更新，他表示，嵌入式设备可能会成为一个容易的目标，特别是在其他切入点变得更加灵活的情况下。

谈到嵌入式系统:“它们中的许多都没有权限分离，它们中的许多都没有代码和数据的分离，并且它们中的许多都是基于空气隙网络的想法开发的——这是不够的，”Snyder 告诉 TechCrunch。

Red Balloon 表示，其研究表明，这些设备内置的安全性(许多已有几十年历史)需要改进，并呼吁政府和商业部门的最终用户要求制造这些设备的供应商提高标准。

“发布固件修复是一种被动、低效的方法，无法解决我们最关键的行业和服务的整体不安全问题，”崔说。"供应商需要将更多的安全性带到嵌入式设备层面."他还认为，美国政府在监管层面还需要做更多的工作，并认为需要向设备制造商施加更多的压力，因为他们目前没有动力在设备层面建立更多的安全性。

然而，Snyder 认为监管主导的方法不太可能有所帮助:“我认为最有帮助的事情是减少攻击面和增加条块分割，”她说。“我们不会通过监管来摆脱更安全的设备。必须有人走出去，为他们培养适应能力。”
