Acquia 利用开源入门套件跳上了无头 CMS 的潮流

在过去十年左右的时间里，内容管理系统已经从由 IT 部门管理的单一系统发展成为通过 API 向开发人员提供的一组服务。更现代的方法是将前端的表示层与后端的管理层分开。

今天， Acquia ，开源 Drupal 项目的幕后公司，宣布正式进入 headless CMS 市场，毫不奇怪，它是基于 Drupal 和开源的。Acquia 的 Drupal 总经理 SVP·吉姆·肖(Jim Shaw)说，该公司将这种产品称为“入门套件”，这是一种根据需要利用无头功能的方式。

“如果你来到 Acquia 并开始一个数字体验项目，我们通常会为你提供 Acquia CMS 作为开始使用 Drupal 的起点。但现在我们也在其中提供了一个无头套件，这样你就可以使用无头功能通过 API 来移动内容，”肖告诉 TechCrunch。

他指出，虽然它正在发布 headless kit，但该公司将其视为混合战略的一部分，其中包括成熟的 Acquia CMS 与 headless piece 一起工作。“对我们来说，这不仅仅是拥有一个专用的无头产品，而且只做无头产品。它是关于从允许你做混合以及无头的平台获得那些能力，”他说。

事实上，肖并没有看到很多顾客完全没有头脑。“我认为人们不太可能 100%没有头脑，他们只会有没有头脑的用例。我认为我们看到的是人们都在做，因此我们正在做的是在 Acquia 平台内建立这些能力，”他说。

除了 headless starter kit 之外，该公司还推出了一个用于 next.js 的工具包，这是一个开源的 web 开发框架，可以帮助开发人员构建前端。他说:“(我们还提供)next.js 的入门套件，它可以让 next.js 前端与无头后端一起工作。”

他说，一些客户已经自己采取了这种方法，该公司希望建立一些东西，使之更容易做到这一点，而不是建立一个成熟的产品。“这就是为什么我们提供的是入门套件，而不是全新的产品。它们是内部的层，我们拥有这些层，然后允许我们的客户(希望还有新客户)在我们的平台上加速那些无头项目，并且仍然有所有选项可供他们使用，就混合方法而言。”

这两款产品将于今天在 Acquia Engage 上发布，该公司的客户会议将于本周在迈阿密举行。这些工具包今天就可以下载了。
