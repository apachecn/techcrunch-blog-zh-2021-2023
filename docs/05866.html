ARMO 筹集 3000 万美元建立端到端开源 Kubernetes 安全平台

广受欢迎的开源 Kubernetes 安全平台kubscape背后的总部位于特拉维夫的 ARMO 公司今天宣布，它已经筹集了由 Tiger Global 牵头的 3000 万美元的首轮融资。新投资者 Hyperwise Ventures 以及现有投资者 Pitango First 和 Peled Ventures 也参与了此轮投资。

开发人员可以通过命令行界面或基于浏览器的用户界面来访问 Kubernetes，它可以帮助企业扫描他们的 Kubernetes 集群、YAML 文件和掌舵图，以发现错误配置、潜在漏洞和用户配置问题。该平台还支持多种安全和合规框架，如 NSA 和 MITRE，但它也允许企业创建自己的定制框架。可能与发现这些问题同样重要的是，kubscape 还向这些团队展示了为什么某个控件失败了——在某些情况下，允许他们在 Kubescape UI 中进行正确的更改，如果他们想这样做的话——以及如何修复它们。

图片来源: ARMO

ARMO 首席执行官兼联合创始人 Shauli Rozen 告诉我，该公司的计划是为 Kubernetes security 建立一个端到端的解决方案，并保持开源。如今，许多公司要么将多个开源解决方案拼凑在一起，这很快变得难以管理，要么不得不使用难以适应其特定用例的专有系统。

老虎环球合伙人约翰·柯蒂斯也提到了这一点。“Kubernetes 是开源的，Kubernetes 的安全也应该是开源的，遵循同样的透明和协作文化，”他说。“ARMO 是独一无二的，因为他们致力于为 Kubernetes 提供完整的开源安全解决方案，所以每个人都可以从最安全的平台中受益，并为之做出贡献。”

ARMO 的商业计划是提供一个托管版本的 Kubescape，定价从每个工人节点/月 59 美元起，数据保留三个月(以及年度计划的折扣)。对于最多只使用 10 个工作节点的用户，还有一个数据保留一个月的免费计划。当然，较大的企业也可以与公司协商企业计划。

“Kubernetes 只是保持增长；保护 Kubernetes 意味着保护所有现代微服务应用所依赖的基础设施，”Hyperwise Ventures 的管理合伙人 Nathan Shuchami 说。“但这也意味着在许多不同层面存在严重的错误配置和潜在的漏洞。Kubernetes 安全性是一个复杂的问题。ARMO 对整个挑战的开源、端到端解决方案的承诺使其脱颖而出，而 Kubescape 的成功正好表明了它是多么需要。"
