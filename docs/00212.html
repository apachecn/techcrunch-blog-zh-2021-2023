健康应用 Docket 中的一个安全漏洞暴露了新冠肺炎疫苗记录

原文：https://web.archive.org/web/https://techcrunch.com/2021/10/27/docket-vaccine-records-covid-security/

健康应用摘要中的一个安全漏洞暴露了新泽西和犹他州接种新冠肺炎疫苗的居民的私人信息，该应用得到了州官员的认可。

Docket 允许居民通过从他们所在州的卫生当局提取疫苗接种记录来下载和携带他们的免疫接种的数字副本。的数字副本与新冠肺炎的纸质卡有着相同的信息，但是被国家数字签名以防止伪造。Docket 是美国几种所谓的疫苗护照之一，允许居民出示他们的疫苗接种记录——或可扫描的二维码——以参加活动、餐馆或进入需要疫苗的国家。

但有一段时间，该应用程序允许任何人访问其他接种疫苗的用户的二维码——以及其中编码的所有个人和疫苗信息。这些信息包括姓名、出生日期和个人的新冠肺炎疫苗接种情况，例如他们接种了哪种疫苗以及何时接种的。

TechCrunch 在周二发现了这个漏洞，并立即联系了该公司。Docket 首席执行官迈克尔·佩雷塔(Michael Perretta)表示，几个小时后，这个漏洞就在服务器层面得到了修复。

该漏洞是在 Docket 应用程序如何从其服务器请求用户的二维码时发现的。用户的二维码以智能医疗卡的形式在服务器上生成，这是一种在全球范围内广泛接受的验证个人疫苗接种状态的标准。二维码与用户 ID 相关联，用户 ID 在应用中不可见，但可以通过使用 Burp Suite 或 Charles Proxy 等现成软件查看其网络流量来查看。

但是 Docket 的服务器没有进行检查，以确保请求二维码的人被允许请求二维码。这意味着任何应用程序用户都可以更改他们的用户 ID，并请求其他人的二维码。更糟糕的是，Docket 用户 ID 是连续的，因此只需将用户 ID 更改一个数字，就可以枚举新的 QR 码。

不知道是否还有其他人发现了这个 bug。Perretta 表示，该公司“目前正在审查日志，以确定该平台上是否存在任何恶意活动。”Perretta 还表示，该公司正在努力向各州政府通报这一失误，但没有说明该公司是否计划向用户通报这一安全失误。

新泽西州卫生部发言人南希·科尔尼在一份声明中说:

我们的供应商 Docket 向新泽西州卫生部通知了一个代码漏洞，该漏洞与最近发布的与该应用程序相关的二维码有关。Docket 向该部门保证，他们已经识别并修复了代码中的漏洞。该应用的其他功能没有受到影响。Docket 用户的隐私和安全仍然是最重要的。目前，Docket 正在调查任何可能被泄露的潜在记录。新闻部继续与 Docket 合作，确保他们对这一问题保持警惕。

明尼苏达州卫生部的一名发言人也没有做出回应。(Docket 面向明尼苏达州居民，但该州尚未部署二维码。)

犹他州卫生部发言人汤姆·胡达科说:

犹他州卫生部致力于确保犹他州居民的隐私，并希望其承包商和合作伙伴保持同样的承诺。Docket 周二]通知我们，其系统中存在一个漏洞，可能会允许用户接收其他用户的个人信息。Docket 向我们保证，他们已经确定了导致错误的原因，并解决了这个问题。

“我们正在与 Docket 和我们自己的数据安全团队合作，以确定任何可能不恰当地共享其信息的用户，并向这些人提供适当的通知，”Hudachko 说。

但是关于这个漏洞是如何从一开始就溜走的问题仍然存在。目前还不清楚到底有多少接种过疫苗的人的记录处于危险之中。上周，Docket 在一条后来被删除的推文中表示，它的用户已经达到了 100 万。在撰写本文时，新泽西和犹他州共有 850 万居民至少接种了一剂新冠肺炎疫苗。

当被问及在发布前进行了何种安全测试时，Perretta 拒绝透露。

犹他州的 Hudachko 说，Docket 经过了医疗保险和医疗补助服务中心(CMS)和国家卫生信息技术协调员办公室(ONC)的“彻底的安全审查”，这两个办公室位于美国卫生与人类服务部(HHS)。ONC 发言人推迟了对 CMS 和 HHS 的评论，这两家公司都没有回应我们的置评请求。

批准该应用的疾病控制和预防中心(CDC)也没有回应该机构是否进行了安全审查的问题。

Docket 不是唯一面临安全问题的疫苗护照应用程序制造商。Docket 应用程序中发现的漏洞与一个名为 Aura 的应用程序中发现的问题几乎相同，该应用程序暴露了数千个包含员工和学生接种疫苗状态的二维码。今年早些时候，总部位于卡尔加里的疫苗接种证明应用程序 Portpass 在其网站不安全后暴露了数十万人的个人信息，而一名黑客能够使用魁北克的官方疫苗接种证明应用程序创建一个完全伪造的疫苗护照。
