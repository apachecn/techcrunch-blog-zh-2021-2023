Amazon 宣布推出 Eventbridge Pipes，这是一种连接多个服务事件的更简单的方法

由于开发人员使用多个 Amazon 服务来组成一个应用程序，他们通常必须编写粘合代码来在不同的服务之间进行通信，这可能是一项耗时的手工练习。但今天在拉斯维加斯举行的 AWS re:Invent 上，该公司宣布了一种更简单的连接 AWS 服务的方法，称为亚马逊 Eventbridge Pipes。

亚马逊首席技术官沃纳·威格尔在今天的主题演讲中介绍了这一新功能，称其为流程带来了管道的简单性。“我们的许多客户希望在不同的服务之间建立这种连接，他们不得不编写一些粘合代码。和往常一样，我一直在想为什么我们不能用管道原理呢？所以我今天很高兴地宣布，Eventbridge Pipes 可以让你轻松地将 AWS 服务连接在一起，”他说。

它非常灵活，允许开发人员使用 lambda 函数或 API，或者他们使用的任何东西，在数据到达消费者之前操纵管道内部的数据。“所以我们的想法是，你不应该再编写粘合代码，因为你可以很容易地将这些服务缝合在一起。如果你想在事件到达消费者之前对其进行操作，你可以提供一个 lambda 函数或者一个指向步骤的函数或者 API 网关来运行一些代码来操作流经管道的事件，”他解释道

正如该公司在宣布新功能的博客文章中所写的，“使用亚马逊 EventBridge Pipes，您可以将受支持的 AWS 和自管理服务作为事件生产者和事件消费者以简单、可靠、一致和经济高效的方式集成到您的应用程序中。”

据 AWS 称，Eventbridge Pipes 现在在所有地区都有售。
