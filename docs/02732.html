让青少年特斯拉黑客成为一个可教的时刻 TechCrunch

Assaf Harel

领导 Karamba Security 的研究和创新活动，负责监管创新产品和服务的广泛知识产权组合。

关于 19 岁的特斯拉黑客大卫·科伦坡(David Colombo)的传言是当之无愧的。第三方软件中的一个缺陷让他能够远程访问 13 个国家的 25 辆世界领先电动汽车制造商的汽车。黑客分享说，他能够远程解锁车门，打开窗户，播放音乐并启动每辆车。

他利用的漏洞不在特斯拉的软件中，而是在第三方应用程序中，所以科伦坡能够完成的事情有一些限制；他无法控制方向，加速或减速。但他能够打开车门，按喇叭，控制手电筒，并从被黑的车辆中收集私人数据。

电动汽车很好玩。它们连接得非常好，不断更新，提供了很好的用户体验，但它们是汽车，不是手机。阿萨夫哈勒

对于网络安全专业人士来说，这种远程代码执行或窃取应用程序密钥的行为每天都会发生，但我希望我们不要对违规披露变得如此麻木，以至于错过了利用这一机会向联网汽车生态系统的利益相关者进行教育的机会。

这次妥协是一个网络安全卫生 101 问题，坦率地说，是一个不该发生的错误。有问题的第三方软件可能是一个自托管的数据记录器，因为在科伦坡发布 Twitter 帖子并通知他们的第二天，特斯拉突然否决了数千个认证令牌。其他一些 Twitter 用户支持这一想法，指出该应用程序的默认配置为任何人远程访问车辆提供了可能性。这也与科伦坡最初声称该漏洞是“所有者的错，而不是特斯拉的错”的推文一致。

最近的汽车网络安全标准 SAE/ISO-21434 和联合国第 155 号法规要求汽车制造商(即 OEM)对其整个汽车架构进行威胁分析和风险评估(TARA)。这些法规要求原始设备制造商对网络风险和暴露负责。责任到此为止。

有些尴尬的是，像特斯拉这样成熟的原始设备制造商监督向第三方应用程序开放其 API 的风险。低质量的应用程序可能没有得到很好的保护，使黑客能够利用他们的弱点，并将应用程序作为进入汽车的桥梁，就像这里的情况一样。第三方应用程序的完整性取决于汽车制造商:他们有责任筛选这些应用程序，或者至少阻止其 API 与未经认证的第三方应用程序提供商的接口。

是的，消费者有责任确保他们经常从其原始设备制造商认可或检查的应用商店下载和更新应用，但原始设备制造商的部分责任是在其 TARA 流程中识别此类风险，并阻止未经授权的应用访问他们的车辆。

我们 Karamba Security 在 2021 年开展了几十个 TARA 项目，看到原始设备制造商在安全准备方面有很大差异。然而，他们都非常重视识别尽可能多的风险，并在生产前解决这些风险，以维护客户安全，并符合新的标准和法规。

以下是我们建议原始设备制造商采用的最佳实践:

  1. 保护秘密/证书——这确保了依赖于成功模仿某人或某事的一长串攻击失败(替换固件、伪造凭证等)。).
  2. 分段访问和功能(以对用户透明的方式)-即使一个点出现故障，损害也是有限的。
  3. 不断测试你自己(或者建立一个奖励计划让其他人来做)——并且快速修复你发现的任何问题。
  4. 通过强化您的外部连接系统(如信息娱乐、远程信息处理和车载充电器)来抵御远程代码执行攻击。
  5. 关闭您的 API。不要让未经授权的人使用它们。这种做法本可以避免最近的袭击。

我们给消费者的建议是严格避免下载不在 OEM 商店上的应用程序。尽管看起来很诱人，但这类应用可能会让司机和乘客面临极大的网络和隐私风险。

电动汽车很有趣。它们连接得非常好，不断更新，提供了很好的用户体验，但它们是汽车，不是手机。侵入车辆危及司机安全和隐私。
