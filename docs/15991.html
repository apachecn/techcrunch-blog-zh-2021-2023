GitHub 发布萌芽开源项目办公室蓝图

GitHub 已经发布了自己的内部指南和工具，介绍如何着手建立一个开源项目办公室(OSPO)。

GitHub 上新的 GitHub-OSPO 库(还有哪里？)的目标是在建立他们的第一个 OSPO 的第一年的企业，包括从覆盖贡献者许可协议的政策(CLA)到关于存档库的指南的一切。它基本上是关于帮助小规模的开源项目发展成更加充实和有组织的东西。

OSPO 的崛起

开源软件几乎与现代技术栈的每个方面都有交集，从云计算和数据库到服务器和超级计算机。许多公司也选择开源他们自己的内部项目以促进行业认同和社区参与。但是，希望保持所有开源组件、合规性、安全性和许可义务的公司将很快意识到他们面临的巨大挑战。这也是为什么 OSPO 正在成为现代公司的一个重要组成部分，将以前可能是跨越无数部门和角色的松散的员工集体正式化。

虽然大多数巨头如微软和Spotify都有专门的 OSPOs，但我们已经看到各种组织纷纷效仿。就在上个月，荷兰政府透露继去年世界卫生组织(世卫组织) OSPO 启动之后，它正在建立一个 OSPO。与此同时，高盛早在 2021 年就开设了 OSPO 。

此外，美国安全开源软件法案是在关键 Log4Shell 安全缺陷之后出现的拟议立法，它专注于改善政府系统中开源软件安全性的方法——这包括漏洞检测/披露、sbom(软件材料清单)以及在联邦机构内设立 OSPO 的规定。

根据 GitHub 最近的十月报告，30%的财富 100 强公司都有 OSPO。OSPO 兴趣的高涨正是 GitHub 希望通过发布自己的内部政策和工具作为他人效仿的蓝图来支持的。
