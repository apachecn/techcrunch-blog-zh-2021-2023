蜻蜓，一家为 Redis 开发“嵌入式”替代品的初创公司，融资 2100 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/21/dragonfly-a-database-startup-developing-a-drop-in-replacement-for-redis-raises-21m/

蜻蜓，一家为 Redis 开发“简易”替代品的初创公司，融资 2100 万美元

Oded Poncz 和 Roman Gershman 在他们以前的工程职位上经历了管理和扩展 Redis(开源数据库)的痛苦。两人在谷歌和 Ubimo 一起工作，罗曼是亚马逊 ElastiCache 服务的负责人。

Poncz 在一次电子邮件采访中告诉 TechCrunch:“开发者已经厌倦了手握基础设施。“对于数据库行业来说，主要挑战是跟上现代应用程序的性能需求，同时保持操作对大多数开发人员来说足够简单。”

为了解决他们作为开发者遇到的一些挑战，庞茨和格什曼创建了蜻蜓。正如庞茨所描述的，蜻蜓 Redis 的替代者是一个“大规模实时”数据库，旨在简化生产，同时提高应用程序性能。

“Redis 创建于 14 年前，虽然从那时起普通应用程序消耗的数据量急剧增长，但 Redis 的处理能力……处理模式却没有跟上步伐，”Poncz 说。相比之下，蜻蜓[可以]支持每秒数百万次操作和来自每个实例的万亿字节大小的数据量。"

Poncz 声称，总的来说，蜻蜓可以提供更快的用户体验，帮助节省硬件成本，并降低运营复杂性，因为维护它所需的工作不太复杂。“蜻蜓拥有独特的缓存算法和多线程处理模型……使其性能比 Redis 高 25 倍，并且更易于扩展，”他说。

这一点尚无定论——TechCrunch 无法独立证实这些说法。但是值得一提的是，本周发布了 1.0 版本的开源蜻蜓已经有了一些用户和投资。

Poncz 声称“成千上万”的用户已经为他们的应用程序部署了蜻蜓。与庞茨和格什曼共同创立的商业化初创公司蜻蜓迄今已从红点(Redpoint)和安静资本(Quiet Capital)获得了 2100 万美元的种子和首轮融资。

庞茨说，这笔资金将用于扩大蜻蜓的开发团队和产品 R&D，特别是改善蜻蜓的架构，以处理更大和更复杂的工作负载。

Redpoint 的总经理 Satish Dharmaraj 通过电子邮件说:“随着应用程序和用户变得更加全球化，边缘计算继续增长，内存数据存储市场也将增长。蜻蜓的架构与众不同，对于内存数据存储市场来说，这是一个真正令人兴奋的时刻，我们预计随着分布式应用的不断增加，这个市场将继续繁荣。”

蜻蜓计划在年底前将其 7 人的员工队伍扩大一倍。
