最近被关闭的第三方应用如何促进 Twitter 的发展

上周，Twitter 更新了其开发者条款,以有效禁止任何替代 Twitter 客户端应用。该公司的变化发生在许多热门应用程序——包括twit terrious、Tweetbot、Echofone 和菲尼克斯——被该公司的开发者平台团队暂停后，没有任何通知或解释。

这家社交媒体公司一直没有公开其避开第三方客户的决定，只是说这是在执行“长期存在”的规则(然后它不得不赶去记录)，这使得开发者更加不信任这个平台。尽管 Twitter 与应用开发者的关系多年来一直不稳定，但第三方客户为 Twitter 今天的核心体验贡献了许多关键功能。

下面，我们总结了一些例子，说明第三方应用程序完成的工作是如何成为社交网络不可或缺的一部分的:

  • 根据该应用的联合创始人之一 Ged Maheux 的博客，Twitter 最早的客户之一 Twitterrific 开始使用小鸟标志，其他客户也紧随其后。这只鸟在 2010 年才成为推特官方标志的一部分。
  • “推特”这个词:没错。tweet 这个词在早期并不属于该公司的词汇。Maheux 的另一篇博客文章引用了 Twitter 创始人比兹·斯通写给用户的一封电子邮件，其中使用了“Twitter-ers”和“Twitter”这样的词 2007 年 1 月初，Twitterrific 开始使用术语“Twit”帖子指出，一名在 Twitter 工作的工程师建议将名称从“Twit”改为“Tweet”。
  • 拉式刷新:今天，很难想象推特没有拉式刷新按钮。然而，推特当时没有应用程序。相反，这个功能是由开发人员 Loren Brichter 开发的名为 Tweetie 的应用程序首创的。

  • 苹果应用程序:顺便说一下，推特在 2010 年收购了推特，然后作为本地官方客户发布。推特是第一家在苹果手机和苹果电脑上发布本土推特应用的公司。
  • 付诸行动:这也是推特的一个标志性功能，在推特收购该公司时被终止。当用户刷推特时，它会显示回复、转发、喜欢和分享等动作。该功能使其在 2012 年回到了官方推特应用程序。
  • 内嵌照片和视频:在推特的早期，社交网络允许用户在推特上发布照片和视频的链接，而不是直接在时间轴上显示。 Brizzly 是最早支持在线照片和视频预览的推特客户端之一。
  • 图像托管:最初，推特没有任何图像托管解决方案。2011 年，该公司与 Photobucket 合作，使用户能够上传照片。与此同时，其他服务如推特和 Yfrog 填补了推特的空白。2015 年，在照片分享服务关闭前几个小时，推特最终收购了推特。
  • 编辑推文:去年，Twitter 首次向 Twitter 蓝色用户推出了编辑推文功能，第三方客户也尝试了不同的方式来提供这一功能。twit terrorial在 2014 年推出了编辑推文功能，在后台删除和转发推文。2020 年，Brizzly 推出了编辑推文功能，在发布推文之前在本地创建并存储一段时间。
  • 线程阅读: Twitter 线程近年来已经成为一种流行的分享故事的工具。但在原生应用上阅读线程是一件痛苦的事情。为了解决这个问题，该公司收购了 Threader ，并将其作为 Twitter Blue 订阅的一部分。
  • 策展:在 Twitter 去年的裁员中，该公司裁减了其策展团队。如果 Twitter 的 API 允许的话，第三方应用可以填补这个空白。过去，Favster 提供了热门推文跟踪和 Listorious 策划的有趣列表以供关注。

随着第三方应用的时代即将结束，一些公司发布了愤怒的博客或其他颂扬 T21 的文章，Twitter 所有者埃隆·马斯克声称他现在想把 Twitter 打造成一个“一切应用”(即“超级应用”))但鉴于马斯克对待开发者社区的方式，从 Twitter 应用生态系统中获取未来想法的机会现在已经丧失给了该公司，这可能会阻碍未来的发展。
