债务即服务提供商 Sivo 希望推动下一代贷款创业公司

一家名为 的初创公司 Sivo 希望为债务做 Stripe 为支付所做的事情:让获得债务工具像插入 API 一样容易。由于 Stripe 可以轻松地将支付嵌入其应用程序和服务，它帮助推出了一系列新业务，Sivo 希望推动新一代贷款产品。

早期回报是积极的:在推出后的短短三个月内，Sivo 收到了约 40 亿美元的需求，并实际上从希望利用其债务即服务产品的发起人那里签署了 15 亿美元的条款清单。据创始人兼首席执行官凯特·希斯考克斯(Kate Hiscox)称，该公司正在招募 600 名发起人，他们希望利用其程序化债务额度。

获得并验证贷款模式

从历史上看，当一家初创公司推出新的信贷或贷款产品时，它必须借出其资产负债表，这意味着通过股权融资筹集资金，然后用这些资金测试贷款模式并创造一些历史。

然后，当他们有足够的数据，他们会去银行或其他金融机构筹集债务工具。这一过程可能需要几个月时间，因为该银行将要求大量数据，谈判条款，并围绕债务融资条款达成复杂的法律协议。

相比之下，根据 Hiscox 的说法，Sivo 的债券发行没有权证，没有预付款利率、安排费、设施费或大型复杂协议。相反，Sivo 的工作方式有点像安全信用卡。也就是说，当公司注册时，他们通过 Sivo 的平台开设一个商业银行账户，并存入一定数量的抵押品，以确保初始债务限额。

平台上的发起人可以配置他们的债务额度，以满足他们的业务需求，然后 Sivo 通过其银行产品评估他们的贷款计划的执行情况。如果发起人通过达到某些预定义的关键绩效指标来证明他们的模型，那么 Sivo 将扩大他们可以借到的更多杠杆。
