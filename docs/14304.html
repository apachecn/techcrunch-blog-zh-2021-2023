XetHub 为其基于 Git 的数据协作平台| TechCrunch 筹集了 750 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2023/01/09/xethub-raises-7-5m-for-its-git-based-data-collaboration-platform/

总部位于西雅图的 xet hub T1 是一家让企业使用 Git 进行数据管理变得容易的初创公司，今天宣布它已经筹集了由 T2·马德罗纳 T3 牵头的 750 万美元的种子融资。这里的基本思想是允许开发人员像处理代码一样处理数据，包括像 Git 这样的工具支持的所有协作特性。该团队将 XetHub 描述为“用于数据管理的协作存储平台”。

该公司由 Yucheng Low(首席执行官)、Ajit Banerjee 和 Rajat Arya 共同创立，这是一个拥有多年大型数据平台工作经验的团队。事实上，洛曾与人共同创立了 ML 初创公司 Turi，艾莉亚是该公司的第一名员工。苹果在 2016 年收购了该公司，允许 Low 和 Arya 在苹果的 ML 平台堆栈的各个部分工作，例如 Arya 领导苹果的数据平台团队。也是在苹果，两人遇到了班纳吉，他曾在 Inktomi、亚马逊和脸书工作。他之前还创办了两家创业公司。

XetHub repository view 设计用于导航和可视化数据存储库，同时保持 GitHub 的敏感性。XetHub 自动总结通用文件格式(CSV)并支持自定义可视化。图片来源: XetHub

在苹果从事数据平台工作期间，该团队意识到数据管理领域仍有很大的改进空间。

“这真的不应该令人惊讶，但数据远比其他一切都重要。比模型更重要，比其他任何东西都重要，”洛告诉我。“管理您存储这些数据的位置，以及如何协作处理这些数据是非常重要的。然而，我们看到的是，我们今天管理数据的方式真的很像 30 年前的源代码，这意味着版本控制或协作是通过复制粘贴来完成的，有时会有更复杂的版本，但如果我想确保没有人会接触我正在做的事情，最终还是复制粘贴。”

就像开发人员已经转向 Git 这样的工具来协作开发他们的源代码一样，XetHub 希望允许他们使用这些熟悉的原语来处理数据。

“我们认为这是第一次，我们真正让开发人员能够以与代码完全相同的方式处理数据，”Low 说。他指出，该团队的目标是创建一个工具，不仅模仿类似 Git 的体验，而且保留核心 Git 用户体验——包括开发人员熟悉的所有集成。

XetHub 扩展 Git 以支持大文件，通过重复数据删除提供高效的存储和传输，同时保持完全的 Git 兼容性。图片来源: XetHub

目前，该服务可以处理高达 1TB 的数据存储库，并计划很快将其扩展到 100TB。很少有开发人员会想要克隆这样的大型存储库，因此这里的一个漂亮特性是，开发人员也可以挂载这些存储库，并使它们像本地文件系统一样工作，无论是在他们的笔记本电脑上还是大型 GPU 集群上。同样值得注意的是，该工具与文件格式无关。

从营销的角度来看，该团队正在将精力集中在 AI/ML 团队上，但用户显然可以使用 XetHub 来管理任何类型的数据。

XetHub 现已推出免费社区版，您可以使用它来管理多达 20GB 的重复数据消除存储。Low 告诉我，该公司已经在与一些企业客户洽谈，但团队还没有准备好指名道姓。

“十多年来，Yucheng 和杰出的 XetHub 团队一直在机器学习方面进行创新，然后将他们的技能应用于最具标志性的消费技术公司——苹果。XetHub 使开发人员能够与他人协作，处理大型数据集，以构建智能的生成性应用程序。“开发和部署这些应用受到传统基础架构和复杂数据工作流的限制，XetHub 从开发人员的角度解决了这些难题。”
