微软开始在 Windows 11 上测试安卓应用

当微软宣布 Windows 11 时，它没有提到任何关于它希望用一个功能较少的任务栏来烦扰用户的事情，但它确实谈了很多关于它的新操作系统将如何支持 Android 应用程序的内容。然而，很快就清楚了，仅仅几周前发布的第一个版本实际上并不支持 Android 应用。然而，现在，微软与亚马逊及其应用商店合作，将大约 50 个安卓应用程序的子集引入 Windows 11 内部计划。

如果你在测试版(不是开发版，因为那是在不同的开发周期)，你现在可以尝试亚马逊自己的 Kindle 应用程序，华盛顿邮报应用程序，国王的冲突，硬币大师和乐高得宝世界。

所有这些都应该在 AMD 和英特尔设备上运行，应用程序运行在微软所谓的“Android Windows 子系统”上，该子系统本身由英特尔的英特尔桥技术提供支持。值得注意的是，微软正在使用基于开源版本 Android 11 的 Linux 内核和 Android OS。这样，它几乎获得了 Android 的所有功能，但没有谷歌为其 Android 版本带来的一些应用程序和附加功能。因为亚马逊也是这么做的，所以亚马逊应用商店的应用程序在 Windows 子系统上也能很好地工作也就不足为奇了。

当然，你的电脑必须启用虚拟化，并满足微软对运行 Windows 11 的总体要求。你还需要一个亚马逊账户来访问这个商店。
