贝恩资本牵头的金融科技初创公司 Fidel API 融资 6500 万美元

原文：https://web.archive.org/web/https://techcrunch.com/2022/04/06/london-based-fidel-api-lands-65m-in-series-b-funding-proving-fintech-infrastructure-is-hotter-than-ever/?utm_campaign=Fintech&utm_medium=email&utm_source=Revue%20newsletter

金融基础设施初创公司 Fidel API 在由贝恩资本风险投资公司(Bain Capital Ventures)牵头的 B 轮融资中筹集了 6500 万美元。

总部位于伦敦的公司 提供身份、数据和支付产品，该公司表示，这些产品为开发人员提供了一种获取同意许可并将支付卡“安全”连接到服务或应用的方法。

“有了这个基础设施，开发者能够在购买点创建高度情境化和事件驱动的用户体验，”它声称。 Fidel API 不受行业限制，客户数以百计，从初创公司到谷歌、加拿大皇家银行和英国航空公司等巨头都有。开发者使用该公司的工具来增强一系列功能，包括数字收据、全渠道归属、忠诚度和奖励、费用管理和个人财务管理。

根据首席执行官兼联合创始人 Dev Subrata 的说法，Fidel API 表示，自 2018 年推出以来，其增长指标已增长两倍，卡基数同比增长四倍，但他拒绝透露硬收入数据。 目前为全球“数千万”持卡人和“数十万”商户提供服务。

现有的支持者 NYCA 伙伴公司、QED 投资者和其他未透露姓名的投资者也参与了 B 轮融资，这使得该公司自成立以来的总融资额达到 8800 万美元。其他支持者包括花旗风险投资公司、加拿大皇家银行资本和商业风险投资公司。该公司没有披露其估值。

此次融资是金融科技领域基础设施公司继续保持弹性的又一个例子，尽管全球融资整体明显放缓。事实证明，在金融服务领域增强其他公司能力不仅有利可图，对投资者也很有吸引力。就在上周，Cross River Bank 获得了一笔 6.2 亿美元的投资，使其估值超过 30 亿美元。今年 2 月， 区块链基础设施初创公司 Alchemy 宣布，它已于 完成了一轮 2 亿美元的 C1 股权融资 ，对该公司的估值为 102 亿美元。

在 Fidel API 的案例中，Subrata 说公司已经 建立了可以帮助开发者创造“可编程货币”的基础设施

“我们在生态系统中的角色是让其他开发者能够在现有资金流的基础上构建我们所说的可编程体验，这可能非常过时，”Subrata 说。“我们所做的是建立一个现代化的连接层，它位于全球网络核心网的支付系统之上。然后，我们通过一个现代的 API 向其他开发者展示这一点，让他们在此基础上进行创新。”

他补充说，该公司的技术消除了流程中的“大量”复杂性，以及合规性需求。

Subrata 说:“这将构建时间从两年压缩到两天，并将数千行代码压缩到仅仅七行代码。

他补充说，具体来说，该公司的数据产品可以捕捉持卡人在任何地方使用他们的卡进行交易的任何时间。其身份产品提供了一种验证持卡人身份的方法。此外，其支出产品为公司提供了一种将钱返还到员工卡上的方式。

Subrata 说:“一旦获得消费者的同意，开发人员就可以实时捕获支付信息，然后使用该 支付事件以编程方式对每次购买做出实时决策，然后触发事件。”“一旦你做到了这一点，你就可以开始创造体验，真正改变我们做事的方式，修复破损的流程，为消费者创造新的价值，否则这是不可能的。”例如，公司可以实时询问采购是否符合其政策。

Fidel API 在总部伦敦、里斯本、葡萄牙和纽约都设有办公室。目前，它有 130 名员工分布在全球各地。它计划利用新资本将员工人数增加一倍，重点是工程、销售和产品招聘，以及投资于现有产品和跨身份验证、同意管理和支付的新产品功能。

“我们正处于高速增长模式，”Subrata 告诉 TechCrunch。“这笔资金将帮助我们继续沿着这条道路加速前进。愿景和前进的道路非常清晰。今天，这真的只是执行。”

自 2018 年推出产品以来， Fidel API 表示，该公司已进入北美、欧洲、亚太和中东市场。

解决费用管理

在拥挤的费用和支出管理领域，有许多美国公司，包括 Ramp、Airbase、Brex 和 TripActions 等。目前，他们主要集中在北美。

贝恩资本风险投资公司的合伙人梅里特·胡默说，她不认为 Fidel API 是费用管理公司的竞争对手，而是他们的推动者。

她告诉 TechCrunch:“事实上，Fidel API 可以通过允许他们跟踪任何第三方卡上的消费，而不仅仅是费用管理公司发行的品牌卡，来增强费用管理平台的商业模式。”“对于根本不想发卡的费用管理平台，Fidel API 为他们提供了与自己发卡的平台相匹配的能力。最终，员工应该能够使用他们想要的任何卡，而不会降低他们的用户体验或报销速度。Fidel API 让未来成为可能。”

她认为 Fidel API 从其他基础设施参与者中脱颖而出，因为它提供了到主要卡网络的实时连接。

“没有其他平台能够让初创公司和企业利用实时信用卡交易数据。”

她继续说道:实时、精细的信用卡交易数据的应用非常广泛。

“看看现在费用管理的创新浪潮就知道了。Hummer 告诉 TechCrunch:“创业公司正在世界各地涌现，以解决管理员工支出的普遍痛点。“Fidel API 非常适合这种用例，因为它提供实时、高度精确的开支数据。所有这些因素在员工支出方面都很重要，因为准确、及时的信息是管理审批和向员工报销的关键。”

同样，Ramp 今年早些时候宣布将扩展到旅行领域时表示，它现在能够让公司制定旅行政策，其中包括航空旅行、住宿和每日津贴限制，这些限制将“自动执行”例如，当员工违规消费时，经理会收到实时警报。

与此同时，Hummer 补充说，总的来说，金融服务行业正在“朝着更快、更数字化和更便宜的阶段前进”，自动化正在取代整个行业的人工流程。

她告诉 TechCrunch:“像 Fidel 这样的 API 平台在这一转变中发挥了重要作用，因为它们促进了各方之间的无缝信息传递。”。“过去，像卡交易数据这样的信息必须批量传输，或者需要人工参与。”
