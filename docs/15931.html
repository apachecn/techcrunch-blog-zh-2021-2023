会见开发开源 ChatGPT 替代方案的团队| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/14/meet-the-team-developing-an-open-source-chatgpt-alternative/

冒着明显的风险，人工智能聊天机器人现在很热。

这些工具可以写文章、电子邮件等，只需几个基于文本的指令，就能吸引科技爱好者和企业的注意。OpenAI 的 ChatGPT ，可以说是其前身，拥有估计超过 1 亿用户。通过一个 API ，包括 Instacart、Quizlet 和 Snap 在内的品牌已经开始将其构建到各自的平台中，进一步提高了使用数量。

但令开发者社区中的一些人懊恼的是，开发这些聊天机器人的组织仍然是一个资金充足、资源丰富的独家俱乐部的一部分。Anthropic、DeepMind 和 OpenAI 都是少数几家成功开发出自己的现代聊天机器人技术的公司之一。相比之下，开源社区在创建一个的努力中受到了阻碍。

这很大程度上是因为训练支撑聊天机器人的人工智能模型需要巨大的处理能力，更不用说必须精心策划的大型训练数据集了。但是一个新的、松散联合的研究小组自称为 一起 旨在克服这些挑战，成为第一个开源类似 ChatGPT 的系统。

一起已经取得了进展。上周，它发布了任何开发人员都可以用来创建人工智能聊天机器人的训练模型。

“Together 正在为开放基金会模式建立一个可访问的平台，”Together 的联合创始人 Vipul Ved Prakash 在一次电子邮件采访中告诉 TechCrunch。“我们认为我们正在构建的东西是人工智能 Linux 时刻’的一部分。”我们希望通过一个汇集数据、模型和计算的平台，让研究人员、开发人员和公司能够使用和改进开源人工智能模型。"

普拉卡什之前曾联合创立了网络安全初创公司 Cloudmark ，Proofpoint 在 2017 年以 1.1 亿美元收购了该公司。2013 年，在苹果收购了普拉卡什的下一个合资企业——社交媒体搜索和分析平台托普斯之后，他在苹果担任了五年的高级主管，随后离开苹果，共同创业。

上周末，Together 推出了其首个主要项目 OpenChatKit ，这是一个用于创建专门和通用人工智能聊天机器人的框架。GitHub 上提供的工具包包括前面提到的训练模型和一个“可扩展”的检索系统，该系统允许模型从各种来源和网站获取信息(例如最新的体育比分)。

基本模型来自 EleutherAI，一个研究文本生成系统的非盈利组织。但他们使用 Together 的计算基础设施 Together Decentralized Cloud 进行了微调，Together Decentralized Cloud 汇集了包括来自互联网志愿者的 GPU 在内的硬件资源。

Prakash 说:“我们共同开发了源库，允许任何人复制模型结果，微调他们自己的模型或集成一个检索系统。”"还一起开发了文档和社区流程."

除了培训基础设施，Together 还与其他研究机构合作，包括 LAION(帮助开发了稳定扩散)和技术专家 Huu Nguyen 的on record来创建模型的培训数据集。该数据集被称为开放式教学通才数据集，包含超过 4000 万个问题和答案、后续问题的示例，更多的是为了“教”模型如何响应不同的指令(例如，“为一篇关于内战的历史论文写一个大纲”)。

为了征求反馈，Together 发布了一个演示，任何人都可以使用它与 OpenChatKit 模型进行交互。

“关键的动机是让任何人都能使用 OpenChatKit 来改进模型，并创建更多特定任务的聊天模型，”Prakash 补充道。“虽然大型语言模型在回答一般问题方面表现出了令人印象深刻的能力，但当针对特定应用进行微调时，它们往往会获得更高的准确性。”

Prakash 表示，这些模型可以执行一系列任务，包括解决基本的高中数学问题，生成 Python 代码，编写故事和总结文档。那么它们经得起考验吗？以我的经验来看，这已经足够了——至少对于像写看似可信的求职信这样的基本事情来说是如此。

OpenChatKit 可以写求职信等。图片来源: OpenChatKit

但是有一个非常明确的界限。与 OpenChatKit 模型保持足够长的聊天时间，它们会开始遇到 ChatGPT 和其他最近的聊天机器人所表现出的相同问题，如鹦鹉学舌般重复错误信息。例如，我让 OpenChatKit 模型给出了一个关于地球是否是平的矛盾答案，以及一个关于谁赢得了 2020 年美国总统选举的彻头彻尾的错误声明。

OpenChatKit，回答一个关于 2020 年美国总统大选的问题(不正确)。图片来源: OpenChatKit

OpenChatKit 模型在其他不那么令人担忧的领域表现不佳，比如上下文切换。在谈话中改变话题经常会让他们感到困惑。他们也不特别擅长创造性写作和编码任务，有时会无休止地重复他们的回答。

Prakash 将其归咎于训练数据集，他指出这是一项正在进行的积极工作。“这是一个我们将继续改进的领域，我们已经设计了一个开放社区可以积极参与其中的流程，”他说，指的是演示。

OpenChatKit 的回答质量可能还有待提高。(平心而论，ChatGPT 并没有因为提示的不同而显著提高。)但是总的来说,在中庸方面是积极主动的——或者至少试图积极主动。

虽然 ChatGPT 的一些聊天机器人可能会被提示撰写有偏见或仇恨的文本，但由于它们的训练数据(其中一些来自有毒来源)，OpenChatKit 模型更难强制。我设法让他们写了一封钓鱼邮件，但他们不会被引诱到更有争议的领域，比如赞同大屠杀或证明为什么男性比女性更适合做首席执行官。

OpenChatKit 采用了一些节制，如这里所见。图片来源: OpenChatKit

不过，适度是 OpenChatKit 的一个可选特性——开发人员不需要使用它。根据 Prakash 的说法，虽然其中一个模型是为另一个更大的模型(为演示提供动力的模型)“专门设计为护栏”，但这个更大的模型默认情况下没有应用过滤。

这与 OpenAI、Anthropic 和其他人喜欢的自上而下的方法不同，这种方法涉及在 API 级别结合人工和自动的审核和过滤。普拉卡什认为，从长远来看，这种闭门不出的不透明可能比 OpenChatKit 缺乏强制过滤更有害。

“像许多两用技术一样，人工智能当然可以在恶意的背景下使用。对于开放的人工智能，或者通过 API 商业化的封闭系统来说都是如此，”Prakash 说。“我们的论点是，开放的研究社区越能审核、检查和改进生成性人工智能技术，我们作为一个社会就越有能力提出这些风险的解决方案。我们认为，在一个大型生成性人工智能模型的权力完全掌握在少数大型科技公司手中、无法审计、检查或理解的世界里，风险会更大。”

为了强调 Prakash 关于开放开发的观点，OpenChatKit 包括第二个训练数据集，名为 OIG-温和，旨在解决一系列聊天机器人温和的挑战，包括机器人采用过于激进或压抑的语气。(参见:必应聊天。)它被用来训练 OpenChatKit 中两个模型中较小的一个，Prakash 说，如果开发人员选择这样做，OIG 适度可以被用来创建其他检测和过滤有问题文本的模型。

“我们非常关心人工智能的安全，但我们认为，从长远来看，通过模糊来实现安全是一种糟糕的方法。普拉卡什说:“开放、透明的姿态被广泛接受为计算机安全和密码学世界的默认姿态，我们认为如果我们要构建安全的人工智能，透明度将是至关重要的。”“维基百科是一个很好的证明，证明了一个开放的社区可以成为大规模挑战审核任务的巨大解决方案。”

我不太确定。首先，维基百科并不完全是黄金标准——该网站的审核过程是出了名的不透明和地域化。然后，事实是开源系统经常被滥用(而且很快)。以生成图像的人工智能系统 Stable Diffusion 为例，在它发布的几天内，像 4chan 这样的社区就在使用这个模型——它还包括可选的调节工具——来创建著名演员的非合意色情深度假货。

OpenChatKit 的许可证明确禁止使用，如产生错误信息，促进仇恨言论，垃圾邮件和参与网络欺凌或骚扰。但是没有什么可以阻止恶意行为者忽略这些术语和调节工具。

预计到最坏的情况，一些研究人员已经开始对开放式聊天机器人发出警告。

追踪网上错误信息的 NewsGuard 公司在最近的研究中发现，新的聊天机器人，特别是 ChatGPT，可能会被提示撰写推进关于疫苗的有害健康声明的内容，模仿来自中国和俄罗斯的宣传和虚假信息，并呼应党派新闻机构的语气。根据这项研究，ChatGPT 在被要求根据错误和误导性的想法写回答时，大约 80%的时间都服从了。

针对 NewsGuard 的发现，OpenAI 在后端改进了 ChatGPT 的内容过滤器。当然，对于 OpenChatKit 这样的系统来说，这是不可能的，因为 OpenChatKit 将保持模型最新的责任推给了开发人员。

普拉卡什坚持自己的观点。

“许多应用程序需要定制和专业化，我们认为开源方法将更好地支持方法和应用程序的健康多样性，”他说。“开放式模型正在变得越来越好，我们希望看到它们的采用率大幅上升。”
