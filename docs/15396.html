不要让开发者在 230 条款的辩论中落后

原文：https://web.archive.org/web/https://techcrunch.com/2023/02/26/dont-leave-developers-behind-in-the-section-230-debate/?scid=_pKQ2mg0Ic

谢莉·麦金利撰稿人

谢莉·麦金利是 GitHub 的首席法律官，GitHub 是超过 1 亿软件开发者的家园。

上周标志着美国最高法院第一次审查 1996 年通信体面法案的第 230 条。在Gonzalez 诉谷歌案的口头辩论中，提出了关于平台责任和病毒内容风险的重要问题。

随着法院努力解决这些问题，这是一个反思的机会，首先是为什么创建 230，它如何促进创新，以及如果 230 内的保护范围缩小，我们都会失去什么。

被杰夫·科瑟夫戏称为“创造互联网的 26 个词”的 230 条款为托管第三方内容的平台建立了一个责任屏障。在互联网的初期，230 为初创公司和企业家的繁荣创造了有利的法律条件，巩固了美国在软件领域的世界领先地位。

虽然今天的技术格局与 90 年代初的互联网有着显著的不同，但 230 条款背后的推理今天仍然适用。法律的架构为创新创造了条件，也可能冷却创新。

在针对大型社交媒体平台的巨大影响力的争论中，似乎迷失了对 230 条款如何支持更广泛的在线生态系统，特别是软件开发商的赞赏。开发人员是我们网络世界的核心，也是为全球挑战创造解决方案的最前沿，他们致力于使支撑我们数字基础设施的软件更加安全可靠。

决策者应该认识到开发者的关键作用，并努力支持他们，而不是扼杀创新。

开发者依靠 230 在 GitHub 等平台上进行合作，并建立和运营重新思考社交媒体的新平台。缩小 230 保护范围可能会产生深远的影响，将法律不确定性引入软件开发商、初创公司和平台的重要工作中，这些工作为他们提供了实现愿景的工具。当政策制定者考虑如何解决中介责任的新领域时，将开发者置于决定互联网未来的中心是至关重要的。

软件开发商为美国的经济竞争力和创新做出了巨大贡献，也是平台政策的重要利益相关者。GitHub 的平台上有 1700 万美国开发者，比其他任何国家都多。仅他们的开源活动每年就为美国经济贡献超过 1000 亿美元。

这些开发人员维护着我们日常生活中不可见但必不可少的软件基础设施。几乎所有的软件——97%——都包含开源组件，这些组件通常在 GitHub 上开发和维护。

作为 GitHub 的首席法律官，GitHub 是一个由 1 亿多软件开发人员组成的代码协作全球社区，我亲身体会到保持 230 完整的重要性。虽然 GitHub 与通用社交媒体平台相去甚远，但 GitHub 依赖 230 保护来托管第三方内容并进行善意的内容审核。

当一个平台拥有超过 3.3 亿个软件仓库时，这一点尤为重要。由于中介责任保护，GitHub 能够在保持平台健康的同时发展。GitHub 有一个强大的、开发者优先的内容审核方法，以保持我们平台的安全、健康和包容性，同时根据代码协作的独特环境定制我们的方法，在这种环境下，单个项目的失败可能会对数千个或更多软件项目产生重大的下游影响。

当谈到冈萨雷斯诉谷歌案的具体细节时，该案要求法院考虑 230 条款的责任保护是否应该包括算法推荐的第三方内容，有利于原告的裁决可能会给开发者带来意想不到的后果。推荐算法在整个软件开发过程中以无数种不同于通用社交媒体平台的方式使用。

GitHub 对的贡献微软在案例中的 amicus brief 概述了我们的担忧:GitHub 上由算法驱动的推荐被用来连接具有相似兴趣的用户，让他们找到相关的软件项目，甚至被用来推荐改进代码和修复软件漏洞的方法。GitHub 的 CodeQL 就是这样一个例子，它是一个语义代码分析引擎，允许开发人员发现开源代码中的漏洞和错误。

开发人员正在使用 GitHub 维护开源项目，这些项目采用算法建议来阻止仇恨言论和删除恶意代码。法院决定缩小第 230 条的范围，排除对推荐算法的保护，这可能会迅速吸引各种有社会价值的服务，包括维护软件供应链质量和安全的工具。

冈萨雷斯诉谷歌一案中的一项裁决试图撤销对社交媒体平台的保护，这有可能影响更广泛的群体。在法院审理此案的准备阶段，大量法庭之友简报强调了其深远的影响:从非营利组织(维基媒体基金会)到社区内容审核( Reddit 和 Reddit 版主)以及小企业和初创公司(引擎)。

虽然缩小 230 的呼吁主要集中在检查大型技术上，但这样做会无意中抑制竞争和创新，同时为下一代开发者和新兴提供商创造更多的进入壁垒。

这些担忧并不夸张:在“法律如何造就硅谷”一书中，阿努帕姆·钱德尔考察了美国法律体系如何为互联网创业创造有利条件，并与欧洲和亚洲形成对比，在欧洲，“对侵犯版权和严格隐私保护的担忧阻碍了互联网创业”，在亚洲，“亚洲网络企业不仅面临版权和隐私限制，还面临严格的中介责任规则。”

缩小 230 不仅会损害美国的全球竞争力；这将阻碍美国国内的技术进步。虽然 GitHub 已经从初创阶段走了很长一段路，但我们致力于创造公平的竞争环境，让任何人、任何地方都可以成为开发者。

在我们等待法院对冈萨雷斯诉谷歌一案的判决之际，需要注意的是，无论案件的结果如何，肯定会有更多的努力来缩小 230 的范围，无论他们是针对算法推荐、人工智能还是其他创新。尽管这些新技术对中介责任的未来提出了重要问题，但政策制定者必须努力规划一条前进的道路，创造一个支持开发者、创业公司、小企业和非营利组织的法律环境，为互联网如此多的社会有益部分提供动力。

关心减少有害内容的政策制定者可以看看开发者是如何在内容节制方面领先的。开发者使用 GitHub 开发有价值的软件项目，包括反映政策制定者对平台上算法透明度的呼吁的开源内容审核算法，如 2022 年的算法问责法案和算法公正和在线平台透明法案。

包括 Twitter 、 Bumble 和 Wikimedia 在内的平台已经使用 GitHub 分享算法的源代码，这些算法分别标记错误信息、过滤淫秽图像和阻止垃圾邮件。开源正在刺激内容调节方面的创新，同时为社区参与、监督和透明度提供了新的模式。

当我们遇到中介责任的新领域时，决策者应该认识到开发商的关键作用，并努力支持——而不是扼杀——创新。
