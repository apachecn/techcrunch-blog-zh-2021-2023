OpenAI 将 ChatGPT 连接到互联网| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/23/openai-connects-chatgpt-to-the-internet/

OpenAI 的病毒式人工智能聊天机器人 ChatGPT 现在可以在某些情况下浏览互联网。

OpenAI 今天发布了 ChatGPT 的插件，通过授权它访问第三方知识源和数据库，包括网络，扩展了机器人的功能。OpenAI 面向 ChatGPT 用户和等待名单上的开发者提供 alpha 版本，open ai 表示，在推出更大规模和 API 访问之前，它将首先优先考虑一小部分开发者和订户加入其高级 ChatGPT Plus 计划。

毫无疑问，最吸引人的插件是 OpenAI 的第一方网络浏览插件，它允许 ChatGPT 从网络中提取数据，以回答向它提出的各种问题。(此前，ChatGPT 的知识仅限于 2021 年 9 月前后之前的日期、事件和人物。)该插件使用 Bing 搜索 API 从网络上检索内容，并在 ChatGPT 的回复中引用其来源，显示它在制作答案时访问过的任何网站。

OpenAI 自己的研究发现，一个可以上网的聊天机器人前景堪忧。人工智能初创公司在 2021 年建立的一个实验系统，名为 WebGPT ，有时会引用不可靠的来源，并受到激励，从它预计用户会发现令人信服的网站中挑选数据——即使这些来源客观上不是最强的。Meta 已经解散的 BlenderBot 3.0 也可以访问网络，并且很快脱离了轨道，当被提示某些文本时，钻研阴谋论和攻击性内容。

图片来源: OpenAI

当然，实时网络不如静态训练数据集精确，也不像静态训练数据集那样被过滤。像 Google 和 Bing 这样的搜索引擎使用他们自己的安全机制来减少不可靠内容出现在搜索结果顶部的机会，但是这些结果是可以被欺骗的。它们也不一定代表整个网络。正如《纽约客》中的一篇文章指出的，谷歌的算法优先考虑使用加密、移动支持和模式标记等现代网络技术的网站。结果，许多拥有高质量内容的网站在混乱中迷失了方向。

这使得搜索引擎在数据方面拥有很大的权力，这些数据可能会告知网络连接语言模型的答案。人们发现，谷歌在搜索中优先考虑自己的服务，例如，用谷歌地点的数据回答一个旅游查询，而不是像猫途鹰那样更丰富、更社会化的来源。与此同时，算法式的搜索方法为坏演员打开了大门。2020 年，Pinterest 利用谷歌图片搜索算法的一个怪癖，在谷歌图片搜索中展示更多内容，据《纽约客》报道。

OpenAI 承认，基于网络的 ChatGPT 可能会执行所有类型的不良行为，如发送欺诈和垃圾邮件，绕过安全限制，以及通常“增加欺诈、误导或虐待他人的坏人的能力”。但该公司还表示，内部和外部红队通知它“实施了几项保护措施”以防止这种情况发生。时间会证明它们是否足够。

除了 web 插件之外，OpenAI 还为 ChatGPT 发布了一个代码解释器，它在 中为聊天机器人提供了一个工作的 Python 解释器，一个沙箱、防火墙环境以及磁盘空间。支持上传文件到 ChatGPT，下载结果；OpenAI 表示，它对解决数学问题、进行数据分析和可视化以及在格式之间转换文件特别有用。

许多早期合作者为 ChatGPT 构建了插件，以加入 OpenAI 自己的插件，包括 Expedia、FiscalNote、Instacart、Kayak、Klarna、Milo、OpenTable、Shopify、Slack、Speak、Wolfram 和 Zapier。

它们很大程度上是不言自明的。例如，OpenTable 插件允许 chatbot 在餐馆中搜索可用的预订，而 Instacart 插件允许 ChatGPT 从当地商店下单。到目前为止，Zapier 是最具可扩展性的，它可以与 Google Sheets、Trello 和 Gmail 等应用程序连接，以触发一系列生产力任务。

为了促进新插件的创建，OpenAI 开源了一个“检索”插件，使 ChatGPT 能够通过用自然语言提问来访问来自文件、笔记、电子邮件或公共文档等数据源的文档片段。

“我们正在努力开发插件，并将它们带给更广泛的受众，”OpenAI 在一篇博文中写道。“我们有很多东西要学，在大家的帮助下，我们希望能造出既有用又安全的东西。”

插件是 ChatGPT 发展时间线上一个奇怪的附加部分。ChatGPT 曾经局限于其训练数据中的信息，现在有了插件，突然变得更有能力——而且法律风险可能更小。一些专家指责 OpenAI 从 ChatGPT 接受培训的未经许可的工作中获利；ChatGPT 的数据集包含各种各样的公共网站。但是插件允许公司保留对其数据的完全控制权，从而潜在地解决了这个问题。
