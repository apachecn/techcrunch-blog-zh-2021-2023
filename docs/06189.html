Workrise 修复泄露用户个人信息的| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/05/workrise-api-personal-information/

劳动力管理独角兽 Workrise 修复了一个暴露的 API，该 API 泄漏了一些用户的个人信息。

这家总部位于德克萨斯州奥斯汀的初创公司，之前被称为 RigUp，成立于 2014 年，是石油和天然气行业按需提供技术劳动力的市场。该公司于 2021 年 2 月更名为 Workrise，以适应更广泛的能源行业，如太阳能、建筑和国防。到 2021 年 5 月，Workrise 表示，它已经以 29 亿美元的估值筹集了 3 亿美元。但上个月， Workrise 宣布裁员，据报道，在中流行支点失败后，该公司的 600 名员工中有数百人受到影响。

现在，一位名叫 Rzlr 的安全研究人员告诉 TechCrunch，他们发现了一个暴露的 Workrise API，允许任何人直接从 Workrise 服务器上检索分包商的个人信息，而无需密码。

API 能够返回分包商工作的姓名、电子邮件地址和一些雇佣细节，以及为分包商提供证明人的姓名和电子邮件地址，比如他们以前的同事和经理。

简单来说，API 允许两个东西在互联网上相互交谈，比如智能手机应用程序、需要与服务器通信的 Peloton 自行车或门锁。在这种情况下，可以使用 web 浏览器通过插入与分包商的审查相对应的唯一的四位数用户 ID 来查询未经身份验证的 API。但是用户 id 是连续的，允许任何人只需通过改变一个数字的用户 ID 来访问另一个分包商的信息，这是一个常见的安全缺陷，称为不安全的直接对象引用漏洞——尽管 Rzlr 表示，并不是每个数字都返回有效的响应。

TechCrunch 看到的几个暴露的记录早在 2019 年就创建了，并标记为“草稿”。

Rzlr 表示，在对 1000 条记录的有限测试中，他们发现了超过 920 条带有姓名和电子邮件地址的记录。Rzlr 表示，API 没有限制可以下载的数据量，他们警告说这可能会带来刮擦风险。

分享给 TechCrunch 的一张截图显示，这些数据很容易被抓取。

TechCrunch 给首席执行官宣勇和首席运营官·迈克·维特发了电子邮件，他们没有回复，但不久之后，该 API 不再公开，而是受到登录页面的保护。在一封电子邮件回复中，Workrise 的安全副总裁埃里克·墨菲(Eric Murphy)告诉 TechCrunch:“默认情况下，用户保留公共档案，”墨菲说。如果 Workrise 确定任何不打算公开的活动用户数据被暴露，Workrise 计划直接通知这些用户

Rzlr 表示，他们在 4 月 22 日联系了几个 Workrise 的电子邮件地址——包括墨菲和该公司的主要安全电子邮件地址——关于暴露的 API。当被问及为什么在 TechCrunch 联系该公司之前两周没有保护 API 时，墨菲说，该研究人员的电子邮件被标记为垃圾邮件。

Workrise 还修复了第二个 API 问题，该问题允许任何人获取用户的推荐代码，然后可以使用该代码查询 API，以获取邀请其他人加入该网站的用户的姓名、电子邮件地址、电话号码和推荐支付金额。

当被问及该公司是否对其系统进行了安全审计时，墨菲表示，该公司已经接受了“多次”第三方审计，但拒绝透露据称执行这些审计的公司的名称。
