信任时间:网络安全客户提出的问题以及如何回答

罗斯·哈勒留克撰稿人

More posts by this contributor

信任从根本上说是一种安全感、熟悉感和“一切都会好起来”的保证信念是在我们解决我们的疑惑和问题时建立起来的，这些疑问和问题让我们怀疑当我们需要时是否可以依靠某人或某物。

网络安全中的供应商选择过程并没有太大的不同。不同的是人们问的问题和他们希望找到的线索。

这家公司想解决什么问题？

当人们了解一种新工具时，他们问的第一个问题是:“这家公司试图解决什么问题？”

许多创业公司就是在这第一步失败的。不管是有意还是无意，网络安全营销很少让人容易理解产品做了什么，同样重要的是，它没有做什么。

在极少数情况下，当一家公司清楚透明地表明自己的立场时，我们会看到安全从业者被打动。

行动项目

你应该让人们容易理解产品在安全堆栈中的位置，它能做什么和不能做什么。使访问你的帮助中心、技术和 API 文档以及其他材料变得容易，这样人们可以快速建立你的产品的心智模型。

真的解决了吗？

事实上，有人试图解决一个问题并不意味着他们真的在解决它。关于社交证明的重要性，有很多要说的，但困难的是，安全团队通常不想公开他们正在使用什么解决方案以及它们如何适应他们的环境，因为对手可以使用这些信息来实现他们的目标。

在早期建立信任的最好方法是从你的产品的开源版本开始，这样潜在的买家就可以检查了。

然而，还有其他方法来提供真实的用户反馈，并证明该公司确实做到了它所说的:

行动项目

收集客户评价，并使其易于获取。如果你没有任何付费客户，问问你的设计伙伴是否愿意公开成为产品的用户。

确保你的证明是真实可信的，提供报价的人准备好被潜在客户随机提问。一个好的安全团队会进行尽职调查，并与其他客户交谈，尤其是在初创公司尚未建立声誉的情况下。

拥有一个易于访问的在线社区，人们可以在那里提问、交谈和直接发送信息。安全专业人员经常给他们的同行发邮件，以获得一些关于供应商的未经过滤的反馈。

这家公司一年后还会存在吗？

尝试和实施新的安全工具可能会变得非常昂贵，而且组织并不打算每年更换他们的安全堆栈。同时，绝大多数网络安全公司都是创业公司，创业公司无时无刻不在来来去去。
