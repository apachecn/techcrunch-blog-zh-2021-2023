Vercel 简化了预览部署的协作

流行的前端开发平台 Vercel 现在允许你使用一个类似 Figma 的高度可视化的评论系统对预览部署进行直接的评论(T2)。该公司自 9 月份以来一直在测试这项功能，现在所有用户都可以使用这项功能。

正如 Vercel 的联合创始人兼首席执行官 Guillermo Rauch 告诉我的那样，这里的想法是在公司现有的开发基础设施上允许更容易的协作，同时扩大可以参与这个过程的人数。

图片来源: Vercel

“这背后的大愿景是，我们希望为前端开发做 Figma 和 Google Docs 所做的事情，在人们现有的工作方式基础上实现这种多人协作，无论是文档还是图像文件，”他说。

Vercel 最早的创新之一是，每当有 pull 请求时，Vercel 都会给你一个 URL，显示前端应用程序的当前状态。这个新的评论系统建立在这个基础之上，通过向这些预览添加一个新的 UI 元素，允许您添加评论和查看现有的评论。因为这些预览直接来自 Vercel，该公司可以在本地集成它，而不需要浏览器扩展(尽管也有一个 Chrome 扩展来实现一个额外的功能，允许你在评论中添加截图)。

图片来源: Vercel

在很大程度上，新的评论系统非常简单。2022 年，评论系统将会出现表情符号反应和所有其他功能，例如，包括一个收集最新评论的收件箱。正如 Vercel 首席技术官 Malte Ubl 告诉我的那样，该公司甚至一度启用了在页面上涂鸦的功能。这并没有成为最终产品，但它可能会成为未来的版本。

Ubl 注意到该工具的 beta 测试人员倾向于大量使用它。“你没有必要每天都想以这种方式回顾一些事情。这不是开发人员的工作方式，”他说。“可能会有一些根本不是图形化的变化。那么你就不需要这个产品，你的拉动式请求审核也完全没问题。但随后，公司开始发布，我们在统计数据中看到它们从这里或那里的评论线程增加到每天 100 条。”

新的评论功能今天向所有 Vercel 用户推出。
