拥抱脸和服务现在推出 BigCode，一个开源代码生成人工智能系统的项目 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/09/26/hugging-face-and-servicenow-launch-bigcode-a-project-to-open-source-code-generating-ai-systems/amp/

图片来源: Getty Images

像 DeepMind 的 AlphaCode、亚马逊的 CodeWhisperer 和 OpenAI 的 Codex 这样的代码生成系统，为 GitHub 的 Copilot 服务提供了动力，提供了一个诱人的视角，让人们看到今天人工智能在计算机编程领域的可能性。但到目前为止，只有少数这样的人工智能系统已经免费向公众开放并开源——这反映了建造它们的公司的商业动机。

为了改变这一点，人工智能初创公司拥抱脸和 service now Research(service now 的 R&D 分部)今天推出了 BigCode ，这是一个新项目，旨在以“开放和负责任”的方式开发“最先进”的人工智能系统代码。目标是最终发布一个足够大的数据集来训练一个代码生成系统，然后使用 ServiceNow 的内部显卡集群来创建一个原型——一个 150 亿参数的模型，比 Codex(120 亿参数)大，但比 AlphaCode(约 414 亿参数)小。在机器学习中，参数是人工智能系统从历史训练数据中学习的部分，本质上定义了系统在某个问题上的技能，如生成代码。

组织者表示，受拥抱脸 BigScience 努力开源高度复杂的文本生成系统的启发，BigCode 将向任何具有专业人工智能研究背景并能为该项目投入时间的人开放。申请表今天下午开始发放。

“一般来说，我们希望申请人隶属于一个研究机构(无论是学术界还是工业界)，并致力于编码应用程序的[大型语言模型]的技术/伦理/法律方面，”ServiceNow 在一篇博客文章中写道。“一旦[代码生成系统]得到训练，我们将评估它的能力……我们将努力使评估更容易、更广泛，以便我们能够更多地了解[系统]的能力。”

在合作开发代码生成系统的过程中，BigCode 正在寻求解决围绕人工智能代码生成实践产生的一些争议，特别是关于合理使用的争议。该系统将在许可证下开源，允许开发人员在特定条款和条件下重用它。非营利软件自由保护协会和其他组织批评 GitHub 和 OpenAI 使用公共源代码来训练和赚钱，这些代码并不都是许可的。Codex 可以通过 OpenAI 的付费 API 获得，而 GitHub 最近开始对 Copilot 的访问收费。对于 GitHub 和 OpenAI 来说，他们继续声称 Codex 和 Copilot 没有违反任何许可条款。

BigCode 的组织者表示，他们将努力确保只有来自拥有许可许可证的存储库的文件才能进入上述训练数据集。他们说，在这一过程中，他们将努力建立“负责任的”人工智能实践，用于培训和共享所有类型的代码生成系统，在做出政策声明之前征求相关利益攸关方的反馈。

ServiceNow 和 Hugging Face 没有提供该项目何时完成的时间表。但他们预计，未来几个月，它将探索几种代码生成形式，包括从代码片段和自然语言描述中自动完成和合成代码的系统，以及跨广泛领域、任务和编程语言工作的系统。

假设伦理、技术和法律问题有一天得到解决，人工智能驱动的编码工具可以大幅削减开发成本，同时允许编码人员专注于更具创造性的任务。根据剑桥大学的一项研究,至少一半的开发人员的精力花在了调试上，而不是积极地编程，这估计每年花费软件行业 3120 亿美元。
