传统上，公司依靠数据屏蔽来保护数据隐私。基本想法是从每个记录中删除所有个人身份信息(PII)。然而，许多引人注目的事件表明，即使是理应解除身份的数据也可能泄露消费者隐私。

1996 年，一名麻省理工学院的研究人员通过将健康记录与公共选民登记数据进行匹配，在一个据称被屏蔽的数据集中发现了时任马萨诸塞州州长的健康记录。2006 年，UT Austin 的研究人员在一个所谓的匿名数据集中重新识别了数千人观看的电影，网飞通过结合 IMDB 的数据将其公之于众。

在 2022 年《自然》杂志的一篇文章中，研究人员使用人工智能对一个据称匿名的数据集中超过一半的手机记录进行指纹识别和重新识别。这些例子都强调了攻击者如何利用“辅助”信息来重新识别可能被屏蔽的数据。

这些失败导致了差分隐私。不同于共享数据，公司会共享结合了随机噪声的数据处理结果。设置噪声级别是为了使输出不会告诉潜在的攻击者关于目标的任何有统计意义的信息:相同的输出可能来自有目标的数据库，也可能来自完全相同但没有目标的数据库。共享的数据处理结果不会泄露任何人的信息，因此保护了每个人的隐私。

要实现差分隐私，不应该从零开始，因为任何实现错误对于隐私保证都可能是灾难性的。

在早期，实施差别隐私是一个重大挑战。第一批应用程序主要来自拥有大型数据科学和工程团队的组织，如苹果、谷歌或微软。随着技术越来越成熟，成本越来越低，所有拥有现代数据基础架构的组织如何在实际应用中利用差异隐私？

差异隐私适用于聚合和行级数据

当分析师无法访问数据时，通常使用差别隐私来生成差别隐私集合。敏感数据可通过 API 访问，该 API 仅输出保护隐私的噪声结果。该 API 可以对整个数据集执行聚合，从简单的 SQL 查询到复杂的机器学习训练任务。

利用个人数据和不同隐私保证的典型设置。图片来源: Sarus

这种设置的一个缺点是，与数据屏蔽技术不同，分析师不再通过查看单个记录来“感受数据”减轻这种限制的一种方法是提供差别私有合成数据，其中数据所有者产生模仿原始数据集的统计属性的假数据。
