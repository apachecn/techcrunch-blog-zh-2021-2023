敏感的美国军事电子邮件泄露到网上

政府云电子邮件服务器在没有密码的情况下连接到互联网

美国国防部周一保护了一台被暴露的服务器，该服务器在过去两周内将美国军方的内部电子邮件泄露到了公开的互联网上。

被曝光的服务器托管在微软为国防部客户提供的 Azure 政府云上，该公司使用的服务器与其他商业客户在物理上是分开的，因此可以用来共享敏感但非机密的政府数据。被暴露的服务器是一个内部邮箱系统的一部分，该系统存储了大约 3tb 的内部军事电子邮件，其中许多邮件与美国特种作战司令部(USSOCOM)有关，USSOCOM 是美国负责执行特种军事行动的军事单位。

但是一个错误的配置使得服务器没有密码，使得互联网上的任何人只需要知道邮箱的 IP 地址，就可以通过网络浏览器访问敏感的邮箱数据。

阿努拉格·森，一位以发现无意中在网上公布的敏感数据而闻名的诚信安全研究员，在周末发现了被暴露的服务器，并向 TechCrunch 提供了详细信息，以便我们能够提醒美国政府。

服务器上塞满了多年前的军方内部电子邮件，其中一些包含敏感的个人信息。其中一份被曝光的文件包括一份完整的 SF-86 问卷，由寻求安全许可的联邦雇员填写，其中包含高度敏感的个人和健康信息，以便在他们获准处理机密信息之前对他们进行审查。这些人员问卷包含大量对外国对手很有价值的安全许可持有者的背景信息。2015 年，疑似中国黑客窃取了数百万份政府雇员的敏感背景调查文件，这些雇员在美国人事管理办公室的数据泄露事件中寻求安全许可。

TechCrunch 看到的有限数据似乎都不是机密的，这与 USSOCOM 的民用网络一致，因为从互联网上无法访问机密网络。

Shodan 是一个搜索引擎，它在网上搜索暴露的系统和数据库，根据 shod an 上的列表，邮箱服务器在 2 月 8 日首次被检测到溢出数据。尚不清楚邮箱数据是如何暴露在公共互联网上的，但很可能是由于人为错误造成的配置错误。

TechCrunch 在美国假日周末的周日上午联系了 USSOCOM，但被曝光的服务器直到周一下午才得到保护。当通过电子邮件联系时，一名五角大楼高级官员证实，他们已经将被暴露的服务器的详细信息传递给了 USSOCOM。不久之后，服务器就无法访问了。

美国战略司令部发言人肯·麦格劳(Ken McGraw)周二在一封电子邮件中表示，周一开始的调查正在进行中。“我们可以确认在这一点上没有人入侵美国特种作战司令部的信息系统，”麦格劳说。

不知道除了 Sen 之外，是否有人在两周的时间内发现了暴露的数据，因为云服务器可以从互联网上访问。TechCrunch 询问国防部是否有技术能力，如日志，来检测数据库中任何不当访问或数据泄露的证据，但发言人没有说。
