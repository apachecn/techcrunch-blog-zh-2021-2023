苹果最新的 iPadOS 16 测试版将 Stage Manager 扩展到了较旧的 iPad Pro 机型

苹果最新的 iPadOS 16 开发者测试版为旧款 iPad Pro 机型带来了 Stage Manager，因为不再需要 M1 芯片。Stage Manager 提供更简单的多任务处理和应用程序之间的切换，此前仅兼容 M1 驱动的 iPad Air 以及去年发布的 11 英寸和 12.9 英寸 iPad Pro 机型。现在，该功能也将出现在第三代和第四代 iPad Pro 机型上。这条新闻是由 Engadget 首先报道的。

但是，较旧的 iPad 型号无法将其显示器扩展到外部型号，这意味着 Stage Manager 只能在 iPad 的显示器上工作。

此外，Stage Manager 的外部显示支持将推迟到今年晚些时候的软件更新中。鉴于一些人分享了他们对 Stage Manager 外部显示功能的担忧，苹果可能需要更多时间来微调该功能。

苹果向 TechCrunch 提供了以下关于这些变化的声明:

我们引入了 Stage Manager 作为一种全新的多任务处理方式，在 iPad 显示屏和独立的外部显示屏上显示重叠的、可调整大小的窗口，能够同时在屏幕上运行多达八个实时应用程序。提供这种多显示器支持只有在 M1 的 iPads 的全部功能下才有可能。拥有 iPad Pro 第三代和第四代的客户对能够在他们的 iPad 上体验 Stage Manager 表现出了浓厚的兴趣。作为回应，我们的团队努力寻找一种方法来为这些系统提供单屏幕版本，同时支持 iPad 屏幕上多达四个实时应用程序。

在今年晚些时候的软件更新中，将提供对 M1 ipad 上舞台管理器的外部显示支持。

苹果决定向更多 iPads 开放 Stage Manager 是一个显著的变化，因为该公司此前曾表示，该功能需要 M1 芯片。软件工程的苹果 SVP Craig fede righi在 6 月告诉 TechCrunch，内存的可用性导致苹果将 Stage Manager 限制在 M1 ipad 上。Federighi 还指出，M1 支持以前的 iPads 所不支持的连接。

上个月，苹果向注册开发者设备推出了 iPadOS 16.1 测试版，这标志着与标准发布 cadence 的突破，自 2019 年首次发布以来，标准发布 cadence 一直将平板电脑操作系统与其智能手机操作系统 iOS 捆绑在一起。这意味着苹果决定跳过秋季发布的 iPadOS 16.0，直接进入 16.1。很可能苹果直接升级到 16.1 的原因是由于今天透露的变化。
