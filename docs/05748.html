关于如何赢回开发者的信任，Twitter 有一个新颖的想法:它将允许他们建立在平台本身上，然后在关键时刻直接向 Twitter 上的用户推广他们的工具和服务。

该公司今天推出了一项实验，这可能是近年来最重要的努力之一——当然，除了推出重建的 Twitter API——旨在展示其与开发者合作的意图，并更好地响应他们的需求。通过新的测试，Twitter 表示，当最终用户可能需要时，它将直接在其平台上推广精选的开发者应用。

这项实验将从小规模开始，重点突出 Twitter 最近推出的现成服务“ Twitter 工具箱”集合中的应用，特别是在开发者构建的安全工具领域，包括阻止党、保镖和温和派。测试版将只显示给选定的用户。

当 Twitter 用户通过 web 应用程序阻止或屏蔽使用 Twitter 内置工具的人时，他们会看到一个新的非侵入式提示，提示各种第三方服务，这些服务能够提供更高级别的安全和保护。

Twitter first 在二月份推出了 Twitter 工具箱。

目前，工具箱是一个在线中心，由 Twitter 的开发者社区建立，面向公众，提供随时可用的自助式应用和服务。除了这三个安全工具，今天的工具箱还包括调度应用 Chirr 应用和Buffer；线程阅读器 app；以及测量工具 ilo ， Blackmagic.so ， Direcon Inc 。，跟随器工作和 Tweepsmap 。

Twitter 表示，在其平台上推出工具箱应用的想法来自于与开发者社区的讨论。开发者告诉该公司，他们希望改善用户在 Twitter 上的体验，更重要的是，他们希望分销，以便人们了解他们的产品。

“这是我们与开发者合作，真正实现这些需求的第一步，”Twitter 开发者平台产品负责人 Amir Shevat 解释道，他在 Twitch、Slack、谷歌和微软担任类似职务后，通过 2021 年的重组收购来到该公司。

“(开发者)需要用户，我们希望在合适的时间为他们提供合适的用户——既给 Twitter 用户带来价值，他们目前需要这些开发者的创新，也给这些开发者提供分销渠道，”他说。

图片来源:推特

通过只在用户意图最强烈时显示提示——在这种情况下，当用户已经屏蔽或静音某个账户时——Twitter 预计用户会表现出与开发者的应用程序互动的意愿增加。对于 Block Party 这样的开发商来说，目前通过口碑推荐获得用户，在 Twitter 上获得用户的能力本身可能会对其业务产生巨大影响。

Block Party 创始人兼首席执行官 Tracy Chou 表示:“我们非常期待有机会接触到真正从我们的工作中受益的用户。”。“通过这种方式，我们可以更直接地接触到需要这种工具的人。”

保镖公司的创始人兼首席执行官查尔斯·科恩已经预计，随着新测试的结果，他的业务将大幅增长。

“我们目前估计 25%安装保镖的用户来自 Twitter 工具箱网站，我们预计这一数字将在未来几天内上升到 50%，”他说。“我们也对这项新实验感到兴奋，它将为 Twitter 用户带来保镖，这些用户需要即时、免费、实时、可定制的高质量保护，以抵御社交媒体上针对他们的有毒内容。”

Twitter 向开发者伸出橄榄枝

尽管 Twitter 多年来一直以更开放的身份运营其开发者社区，但它仍有许多历史要克服，以完全重置其开发者关系。修复其声誉将是一项不小的努力。

早些年，Twitter 对开发者的态度是杂乱无章的，不断变化的。2010 年，它在举办了第一次开发者大会 Chirp ，期望在未来的几年里，它将继续与那些使用其 API 和开发者工具的开发者直接合作。第二年，这个活动被取消了，也没有回来。

2012 年，Twitter 推翻了开发第三方 Twitter 客户端的开发者，几年后，它在 2014 年收购合作伙伴的竞争对手 Gnip 后，欺骗了自己的合作伙伴，后者同意转售 Twitter 的 firehose 数据——未经过滤的完整推文流及其元数据。

此外，正如一些人所说，Twitter 并不总是诚实地解释其 API 变化背后的原因。Twitter 早期员工 Evan Henshaw-Plath(@乌合之众)最近的 Twitter 帖子解释说，Twitter 限制第三方客户端的决定实际上是由于来自 UberMedia 的竞争威胁，UberMedia 一直在购买客户端，目的是将用户转移到它自己的微博服务。

Henshaw-Plath 感叹道，“Twitter 认为开放的生态系统将被占领，所以他们关闭了生态系统以拯救公司。我希望他们与应用程序开发人员合作，诚实地面对对公司的攻击……”

今天，Twitter 正试图做到这一点。

“我认为我们需要赢得信任。我认为我们需要透明。Shevat 说:“我认为我们需要在开放的环境中进行建设。

他解释说，如今，Twitter 希望与开发者合作，开发他们成功所需的工具。

例如，在重建的 API 中，该公司专注于加入早期版本中缺失的功能，如对话线程、投票结果、固定推文、垃圾邮件过滤、流过滤等。它还开放了自己的平台，减轻了入职负担，增加了更多的免费访问，并取消了一些早期的政策限制(包括那些影响第三方应用的限制)。

到 2021 年 11 月，该公司表示基于 API v1.1 构建的所有现有应用的 90%可以在 v2 上得到完全支持，同时还可以访问其的关键新功能。

“Twitter 的 API v2 显示了对开发者需求和反馈的真正倾听，所有这些都在朝着正确的方向发展，”科恩说。

周也同意。“…与早期更混乱的时期相比，似乎确实发生了变化，Twitter 的人们现在似乎真的看到了拥有这个开发者生态系统的价值，并真的想投资于此——因为他们看到，围绕适度和安全等问题，让他们的用户有更多这些选择是一件好事。”

但是协作和交流只是服务开发者社区的一部分。帮助应用程序被发现并产生收入也是关键因素。

“随着 Twitter 上的界面越来越多，我们希望创造更好的方式来发现(开发者的应用)，”谢瓦特说。“为了在平台上取得成功，开发者需要互动界面……他们需要被发现，这就是我们在接下来的几个实验中试图做的，”他补充道。他们需要通过创新获得金钱上的成功。"

他指出，这种策略不仅对开发者有利，对从这些第三方工具和服务中受益的最终用户也有好处。

“这基本上是我们正在推出的东西背后的基本假设，”谢瓦特说。

推动这一战略的另一个因素是，Twitter 开始认识到，即使加快了产品开发的步伐，它也无法构建用户需要的一切。

与此同时，它通过蓝天项目的去中心化努力只会推动对开发者主导的创新的更多需求。举一个当前的例子，对算法选择和内容过滤或审核的用户设置级别的需求正在增长。埃隆·马斯克对 Twitter 的出价部分是想要一套不同的节制规则。

像 Block Party 这样的应用程序，可以过滤掉不想要的提及，并阻止巨魔，最终可以帮助解决这些问题。

Chou 解释说:“考虑 Block Party 的一种方式是，就通知和人们何时标记你而言，它是一种不同的算法。

“默认算法是，任何时候有人给你加标签，你都会看到提及。安装了类似 Block Party 的新算法——即使用户不会认为我在选择自己的算法——是一种不同的体验。她说:“仅仅因为有人攻击了我，并不意味着我一定要立即看到它。”“我可以选择不看它。所以现在有一种不同的算法来控制提及的体验。”

虽然 Block Party 的重点是过滤提及，但在 Twitter 产品的其他领域，用户可能希望更多地定制他们的体验，而不是使用平台默认设置。

其中一个领域涉及创造者。Shevat 指出，新的 API 为 Twitter 的 Super Follow creator 订阅产品添加了端点，该产品允许用户通过应用内购买订阅喜欢的账户，以接收独家内容。有了这个 API，开发者可以开发一个超级关注应用，为关注者和订阅者提供不同的内容。或者，它可以过滤最终用户体验，以关注来自用户超级关注订阅的付费内容。

当用户与超级关注创建者接触或第一次订阅时，这样的应用程序可以在 Twitter 中推广。他还想象开发人员如何在 Twitter 空间内构建——他建议想象一下空间中的 Wordle 或者甚至在 Twitter 时间线上推广他们的应用。或许，如果有人发布了一首歌曲的链接，你可以用你最喜欢的流媒体应用程序来播放它。

很容易想象其他 Twitter 工具箱应用程序也在 Twitter 平台的不同地方进行类似的推广——例如，当用户寻找批量发布推文或检查他们的分析时。

“我现在认为 Twitter 是老式的诺基亚手机……它是一部好手机。但如果你记得的话，它上面唯一的应用程序是 Snake，”谢瓦特说。我认为 Twitter 的未来就像 iPhone，你获得的价值实际上是通过开发者的创新。"

Twitter 自己的“应用商店”

Shevat 指出，现在有一个专门的团队专注于解决应用程序发现问题，解决方案包括工具箱和最新的 Twitter 测试。随着 Twitter 在 12 月增加了对 OAuth 2.0 的支持，这些应用和服务可能会与 Twitter 平台进行更深入的集成，他将此描述为 Twitter 及其应用之间未来更紧密、无缝集成的第一步。

也可以考虑其他模式——比如 Twitter 的“应用商店”

当被问及这一选择时，谢瓦特说:“我可以肯定地认为这是我们未来的可能性之一。”。“从我的经验来看，作为一个纯粹的产品人，我想尝试看看哪里是最好的方式，因为……这个世界充满了成功的市场，但也充满了没有人想去的旧市场的墓地，”他继续说道。

“所以我需要确保用户发现价值——也许我可以在产品中体现出来，而不是在一个单独的市场中。也许我可以在产品中创建一个展示开发人员创新的部分。”

“但我们肯定会解决发现的问题，”他说。
