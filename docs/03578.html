AWS 将其本地区域迷你数据中心带到 32 个新城市 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/16/aws-brings-its-local-zone-mini-data-centers-to-32-new-cities/

延迟对于许多工作负载来说至关重要，但大型云提供商通常会将他们的主要数据中心建在电力便宜、当地税收优惠高的地方。然而，最近几年，我们已经看到了类似 AWS 的 Local Zones 这样的项目的新变化。这些小型数据中心靠近主要的人口中心，为游戏、视频流或机器学习推理等需要低延迟连接的应用程序提供核心云功能。

今天，在去年的 re:Invent 会议上，AWS 宣布将最初的 16 个区域扩大到 48 个。

这些新区将位于 26 个国家的城市:阿姆斯特丹、雅典、奥克兰、曼谷、孟加拉鲁鲁、柏林、波哥大、布里斯班、布鲁塞尔、布宜诺斯艾利斯、钦奈、哥本哈根、德里、河内、赫尔辛基、约翰内斯堡、加尔各答、利马、里斯本、马尼拉、慕尼黑、内罗毕、奥斯陆、珀斯、布拉格、克雷塔罗、里约热内卢、圣地亚哥、多伦多、温哥华、维也纳和华沙。到目前为止，本地区域仅在美国可用

承诺是，开发者将能够为他们在这些城市的用户的应用程序提供一位数的毫秒级性能。

AWS 基础设施服务副总裁 Prasad Kalyanaraman 在今天的公告中表示:“云的边缘正在扩展，现在几乎在任何地方都可以使用。“成千上万使用美国 AWS 本地区域的 AWS 客户能够优化专为其行业和客户用例设计的低延迟应用。随着我们第一批 16 个本地区域的成功，我们正在向世界各地更多的客户扩展，这些客户要求这些相同的功能，以将云服务的优势推向新的领域。AWS 本地区域现在将在全球 30 多个新位置提供，为客户提供强大的新功能，让全球数亿最终用户在几毫秒内利用云服务。”
