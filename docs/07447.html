GitHub 的人工智能编程助手 Copilot 现在已经上市

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/21/copilot-githubs-ai-powered-programming-assistant-is-now-generally-available/amp/

图片来源: GitHub

去年 6 月，微软旗下的 GitHub 和 OpenAI 推出了 Copilot ，这是一项为微软 Visual Studio 等开发环境中的整行代码提供建议的服务。Copilot 是一个可下载的扩展，由一个名为 Codex 的人工智能模型提供支持，该模型在数十亿行公共代码上进行训练，以根据现有代码的上下文建议额外的代码行和功能。Copilot 还可以利用其知识库和当前上下文，提出一种方法或解决方案，以响应开发人员想要完成的内容的描述(例如，“Say hello world”)。

Copilot 以前只在技术预览版中提供。但是在发出信号该工具将于今年夏天全面上市后，GitHub 今天宣布 Copilot 现在对所有开发者开放。如前所述，它将对学生和“经过验证的”开源贡献者免费——从从社区和参加 GitHub Education 计划的学生中选出的大约 60，000 名开发人员开始。

GitHub 称在预览阶段有 120 万人注册。据该公司称，Copilot 现在建议使用 40%的新编写的代码——高于今年早些时候的 35%。

GitHub 产品副总裁 Ryan J. Salva 通过电子邮件告诉 TechCrunch，“在过去的一年里，我们继续迭代和测试工作流程，以帮助推动 Copilot 的‘魔力’”。“我们不仅使用预览版来了解人们如何使用 GitHub Copilot 还安全地扩展服务。”

有了 Copilot，开发人员可以循环浏览 Python、JavaScript、TypeScript、Ruby、Go 和许多其他编程语言的建议，并接受、拒绝或手动编辑它们。Copilot 适应开发人员所做的编辑，将特定的编码风格与自动填充样板文件或重复的代码模式相匹配，并推荐与实现代码相匹配的单元测试。

除了 Visual Studio 代码，Copilot 扩展还可用于 Noevim 和 JetBrains，或者在 GitHub Codespaces 上的云中。

Copilot 的一个新功能是 Copilot Explain，它可以将代码翻译成自然语言描述。这是一个研究项目，目标是帮助开发新手或那些不熟悉代码库的人。

萨尔瓦说:“虽然很明显 Copilot 有助于开发人员更快地完成任务，但我们仍在继续探索超越这一点的更新，帮助开发人员保持流畅，专注于更令人满意的工作，并在节省时间的同时节省精神能量。”。“作为我们观察到的影响的一个例子，分享我们正在进行的一项研究的早期结果是值得的。在实验中，我们要求开发人员编写一个 HTTP 服务器——一半使用 Copilot ，一半不使用。初步数据表明，开发人员在使用副驾驶时不仅更有可能完成任务，而且他们完成任务的时间也减少了大约一半。”

由于人工智能模型的复杂性，副驾驶仍然是一个不完美的系统。GitHub 表示，它已经实现了过滤器，以阻止以标准格式显示的电子邮件和攻击性词语，并且它正在建立一个过滤器，以帮助检测和抑制从公共存储库中重复出现的代码。但该公司承认，Copilot 可能会产生不安全的编码模式、错误和对过时 API 的引用，或者反映其训练数据中不完美代码的习惯用法。

“这只是人工智能驱动的开发工具的开始，所以在未来几个月和几年内，看看开发人员如何使用 Copilot 将是令人兴奋的——同时，我们如何推进产品，”Salva 继续说道。
