Checkout.com 推出虚拟和实体信用卡发行

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/22/checkoutcom-launches-virtual-and-physical-card-issuing/

金融科技初创公司Checkout.com以其支付处理服务而闻名，但该公司今天推出了一款新产品:其客户现在可以为自己的客户创建支付卡。

该公司已经测试 Checkout.com 发行有一段时间了，通过这项新服务已经制作了数百万张卡片。Checkout.com 既支持实体卡，也支持虚拟卡，虚拟卡可以多次使用，也可以设置为在第一次付款后禁用。

例如， Jow ，一个为你创建食材清单的膳食计划应用，一直在使用 Checkout.com 发行来创建一次性虚拟卡，可以用来在合作公司购买食品杂货。

Checkout.com 表示，与其他金融科技公司相比，其发行的产品不太依赖与第三方公司的集成。这应该给它更多的灵活性，在飞行中生成新卡。

这也意味着 Checkout.com 卡可以与公司服务的其他部分携手合作。例如，客户可以轻松地创建自定义规则来自动化资金移动，以便卡可以有特定金额的现金。

发行也代表着金融科技公司的商机。当有人用卡支付时，卡交易费用在商户的银行、卡组织(例如 Visa 或 Mastercard)和发卡机构(在这种情况下是 Checkout.com)之间分摊。Checkout.com 将与自己的客户分享其那部分交易费。

“随着在线旅游、市场和数字银行等行业使用支付作为客户金融生活的核心，信用卡发行和嵌入式金融在过去几年出现了爆炸式增长。该公司首席采购官梅龙·科尔贝奇在一份声明中说:“Checkout.com 发行建立在开放、灵活的 API 基础上，这意味着企业可以创建专门的卡计划，增强现金流，并开启新的收入机会。”。

那么谁会使用 Checkout.com 发行呢？像 DoorDash 或 Uber Eats 这样的按需送货公司依靠像 Marqeta 这样的公司发行的卡，所以他们的送货伙伴可以购买杂货或食品，而不需要使用他们自己的个人资金。

许多行业也在利用发卡在各种商家之间进行即时支付。只要商家接受卡支付，卡就可以充当支付接口。例子包括旅行社、保险公司、费用管理平台等等。
