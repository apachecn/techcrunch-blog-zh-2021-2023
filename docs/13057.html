MergeStat 利用开源和 SQL 将“运营分析”引入软件工程

原文：https://web.archive.org/web/https://techcrunch.com/2022/12/06/powered-by-open-source-and-sql-mergestat-brings-operational-analytics-to-software-engineering/

一家新的开源初创公司正在着手帮助软件开发团队从他们的代码库中收集更深入的见解，使用 SQL 查询他们在软件构建过程中使用的所有数据源。

众所周知，初创公司 merge stat(T1)一直未被注意，但由于计划在现有的 T2 开源项目(T3)基础上推出一款商业产品，该公司今天宣布了 120 万美元的预种子资金，并就未来几个月的发展方向给出了一些见解。

就上下文而言，MergeStat 的起源可以追溯到 2020 年中期，当时第一次向提交了一个名为 Gitqlite 的项目，这实质上是一个实验，将 SQLite 和 Git 结合在一起，使得在代码库中查询历史数据变得更容易。

“当时，我对探索源代码的历史非常感兴趣，以了解我工作过的传统代码库，”MergeStat 的创始人兼首席执行官帕特里克·德维沃向 TechCrunch 解释道。“关于代码库的某些特性或部分，Git 历史是否可以用来确定最合适的联系人？作为一种方式来确定某些代码领域的“专家”，并围绕谁负责源代码的哪些部分提供聚合上下文？同样，它是否能覆盖那些依赖于不再从事某个项目的人的高风险区域？”

本质上，这一切都是关于深入到代码历史中去的——这包括查询基本的元素，比如提交历史，以及通过“Git 责备”命令显示作者元数据，但是它的意图是远远超出这一点，使开发人员能够利用 SQL 来询问关于代码本身的问题。

“运营分析”

快进到 2021 年 4 月，商业合并统计公司正式诞生，DeVivo 继续从云巨人 Equinix 吸引 Josue Lopez 担任首席运营官(首席运营官)以及正式的联合创始人。

“这让我们走到了今天的地步，我们的任务是支持软件工程团队的运营分析，”DeVivo 说。“如果它涉及到构建或运输软件，我们希望能够用 SQL 进行查询。”

本质上，任何使用 PostgreSQL 的工具——包括大多数商业智能(BI)和数据可视化工具——都可以使用 MergeStat。该平台本身包括一个管理界面和一个 PostgreSQL 数据库，MergeStat 将来自各种软件开发生命周期(SDLC)源的数据同步到主 PostgreSQL 数据库中。用户可以在 MergeStat 的应用程序中查询这些数据，或者将其连接到第三方工具，如 Grafana、Tableau 或 Superset。

但是 MergeStat 可能支持哪些类型的用例呢？嗯，它的核心是从可能分布在不同代码库和开发团队的信息中获取洞察力。例如，如果一家大型企业的经理想知道有多少团队——以及哪些团队——采用了新工具，或者有多少代码库使用了特定版本的编程语言或库，他们可以使用 MergeStat 来询问这些。或者，他们可能希望提取所有第三方依赖项或配置文件值，MergeStat 在这方面也可以提供帮助。

如果一家公司正在进行一个大型迁移项目，或者如果他们正在计算他们的潜在攻击面，在特定的依赖关系中存在已知的漏洞，了解这些问题的答案是至关重要的。

MergeStat 正在运行。 图片来源: MergeStat

其他潜在的用例包括审计和合规性，以便公司可以遵循适当的程序和最佳实践作为监管框架的一部分。例如，作为 SOC 2 审计的一部分，服务提供商可能需要证明他们正在正确管理其客户的数据，MergeStat 可用于收集和呈现这一证据，显示谁访问了特定文件或谁修改了什么代码。

竞争格局

这里值得指出的是，已经有可能获得这些问题的答案，但这通常涉及一个涉及多个屏幕和工具的手动过程，并将文本复制到电子表格中，这可能是一个资源密集型过程。MergeStat 允许工程师通过 SQL 提问，并通过 BI 工具在仪表板、报告和警报中查看答案，从而实现了大部分自动化。

“MergeStat 可以在团队进行正常工作时连续回答这些问题——merge stat 访问底层数据的变化以反映更新的状态，”DeVivo 补充道。

通过 MergeStat 获得的拉取请求(PR)数据示例。 图像来源: MergeStat

还有许多 SaaS 工具至少实现了 MergeStat 承诺的一部分。例如，工程度量被覆盖，比如 linear ab或者水母，而代码搜索是 Sourcegraph 和 GitHub 本身的一个核心组件。在审核与合规方面，有 Drata 、 Vanta 和 Laika ，与 GitHub 集成用于取证。

虽然这些都带来了价值，但 MergeStat 打赌，许多工程领导不希望围绕“速度”或“生产率”等主观概念预先构建“固定”的度量和图表。MergeStat 认为，许多人更愿意在软件开发生命周期中访问底层数据，并灵活地以与其特定组织和用例相关的方式进行查询。

“每个组织都不同，我们相信给他们工具来处理他们的数据，设计更具体的问题，会带来更好的结果，”DeVivo 说。我们将自己定位为一个数据基础设施产品，并相信给予“较低级别的”对构建和运输软件所涉及的数据的访问权通常对工程组织实施软件是有用的。”

当然，开源也是 MergeStat 灵活性承诺的一大部分。它让公司能够完全控制他们的数据和部署，同时他们能够以他们认为合适的方式对其进行分割，如果他们愿意的话，可以在本地笔记本电脑上进行，以便在全面投入之前解决所有问题。

下一步是什么

虽然 MergeStat 目前仍然是一个开源项目，但该公司目前正在开发一个托管云产品和一个以企业为中心的化身，可以自托管或部署在客户选择的任何云上。这其中很大一部分将围绕最近宣布的“PostgreSQL 方法”进行，该方法涉及将数据同步到 Postgres 数据库中，以便为进一步的下游查询提供动力。

在商业发布的准备阶段，MergeStat 表示，它已经在早期测试中与“许多公司”合作，包括 Equinix Metal 的团队，DeVivo 表示，该团队目前正在使用跨 800 个存储库的自托管 MergeStat 实例。

MergeStat 的 pre-seed 轮由 OSS Capital 领投，Caffeinated Capital 和一系列天使投资者参与其中。
