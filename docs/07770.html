谷歌推出高级 API 安全

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/30/google-launches-advanced-api-security-to-protect-apis-from-growing-threats/

谷歌推出高级 API 安全，保护 API 免受日益增长的威胁

谷歌今天宣布了高级 API 安全的预览，这是一个面向谷歌云的新产品，旨在检测与 API 相关的安全威胁。基于 Apigee，谷歌的 API 管理平台，该公司表示，客户可以从今天开始请求访问。

“应用编程接口”的缩写，API 是计算机之间或计算机程序之间的连接。API 的使用正在上升，一项调查发现 2021 年超过 61.6%的开发者比 2020 年更依赖 API。但是他们也越来越成为攻击的目标。根据网络安全供应商 Imperva 委托的 2018 年报告，三分之二的组织正在向公众和合作伙伴公开不安全的 API。

高级 API 安全专门负责两项任务:识别 API 错误配置和检测僵尸程序。该服务定期评估受管 API，并在检测到配置问题时提供建议的操作，它使用预配置的规则来提供识别 API 流量中恶意僵尸程序的方法。每个规则代表来自单个 IP 地址的不同类型的异常流量；如果 API 流量模式符合任何规则，高级 API 安全会将其报告为 bot。

“错误配置的 API 是 API 安全事故的主要原因之一。谷歌云产品负责人 Vikas Ananda 在声明发布前与 TechCrunch 分享的一篇博客文章中说:“虽然识别和解决 API 错误配置是许多组织的首要任务，但配置管理过程非常耗时，需要大量资源。“高级 API 安全性使 API 团队更容易识别不符合安全标准的 API 代理。。。。此外，高级 API 安全性通过识别成功产生 HTTP 200 OK 成功状态响应代码的僵尸程序，加快了识别数据泄露的过程。”

随着高级 API 安全的推出，谷歌显然正在寻求加强 Apigee 下的安全产品，它于 2016 年以超过 5 亿美元的价格收购了 API gee。但该公司也在应对 API 安全领域日益激烈的竞争。提供专注于 API 的网络安全产品的初创公司包括 Salt Security 、 Noname Security 和 Neosec 。许多成熟的供应商近年来也扩展了他们的产品，包括 Barracuda、Akamai、42Crunch、Traceable、Ping Identity 和 Signal Sciences。

三月份，Cloudflare 推出了一款旨在提升 API 安全性的新网关。并且在 5 月，Imperva 收购了 API 安全公司 CloudVector。

虽然这些产品的性能如何还没有定论，但是 API 带来的攻击的威胁是真实存在的。在过去的几个月里，像 Peloton、Parler 甚至 LinkedIn 这样的公司已经成为 API 驱动的攻击的受害者。他们不是唯一的。根据 Cloudentity 最近的研究，44%的公司遇到过“实质性的” API 授权问题，涉及内部和面向外部的 API 的隐私、数据泄漏和对象属性暴露。
