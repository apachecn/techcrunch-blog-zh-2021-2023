移动运营商与微软的 AWS 合作推出了开放网关，这是一套类似 Twilio 的 API，用于开发网络服务| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2023/02/26/mobile-carriers-team-up-with-aws-microsoft-to-launch-open-gateway-a-set-of-twilio-like-apis-to-tap-network-services/

API 是技术世界如何运作的构建模块:API 调用用于将应用程序相互集成，如今构成了全球互联网流量的大部分。现在，电信运营商，通常被排除在科技的发展之外，但现在正在寻找更多的方法来赚钱，比如 5G，想参与进来。

今天，代表全球主要移动运营商的协会 GSMA 宣布了一项与 21 家运营商的新计划，名为“开放网关”(Open Gateway)，这是一个在运营商网络中提供通用的、基于开放源代码的 API 的框架，开发者可以访问和使用各种移动网络服务，如位置或身份验证和运营商计费，而这些服务在以前集成和使用起来会更复杂或更昂贵(如果不是不可能的话)。该计划旨在能够在沉浸式混合现实体验和 web3 应用等应用中使用 API 启动更多开发，这反过来将为移动运营商带来更多 5G 业务。

随着 GSMA 大型行业展会的开幕，位于巴塞罗纳的 MWC、亚马逊的 AWS 和微软的 Azure 被命名为首批两家与运营商合作向开发者提供 API 的大型云提供商。

最初签约开放网关的运营商有:América Móvil、美国电话电报公司、Axiata、Bharti Airtel、中国移动、德国电信、e& Group、KDDI、KT、Liberty Global、MTN、Orange、Singtel、Swisscom、STC、Telefónica、Telenor、Telstra、TIM、威瑞森和沃达丰。这些运营商已经签署了一份谅解备忘录，计划通过 CAMARA 来构建和使用这些 API，这是一个由 Linux 基金会和 GSMA 共同开发的开源项目，旨在帮助开发者获得“增强的”网络能力。

运营商已经在新的网络技术上投资了数十亿美元，但是他们在这些网络上并没有太多的使用。这一举措部分是由他们试图在自己身上启动活动推动的。

“GSMA 开放网关将使单点接入超宽带网络成为可能，并为沉浸式技术和 web3 提供催化剂，使它们能够发挥潜力并达到临界质量，”GSMA 主席、西班牙电信公司首席执行官兼董事长何塞·玛丽亚·Á·阿尔瓦雷斯-帕莱特·洛佩斯说。“电信公司在开发一个连接所有人和所有事物的全球平台方面已经取得了长足的进步。现在，通过联合开放网络 API 和应用互操作性的漫游概念，移动运营商和云服务将真正整合，创造一个充满机遇的新世界。电信运营商和云提供商之间的合作在这个新的数字生态系统中至关重要。”

关于我们可能会看到哪些服务首先推出的细节还没有给出。Open Gateway 最初推出了针对八种服务的 API 规范:SIM 交换(eSIMs，以便更容易地更换运营商；“按需质量”；设备状态(让用户知道他们是连接到本地网络还是漫游网络)；数字验证；边缘站点选择和路由；号码验证(SMS 2FA)；承运人计费或结帐；和设备位置(当服务需要验证位置时)。它说，今年将增加更多的 API。也许有些名字也会被修改。例如，SIM 交换已经有了一个更加邪恶的含义。在 MWC 的活动中，将会有很多展示 API 使用方法的演示。

考虑到移动运营商的历史以及它们所扮演的角色，尤其是自智能手机使用热潮以来，这一消息是一个有趣的发展。

移动运营商长期以来一直担心成为“比特管道”——一种以越来越有竞争力的价格(也就是低利润)出售服务的商品。当智能手机和应用程序出现时，这似乎是不可避免的命运。新的内容是通过移动网络(而不是网络本身)传递的。由手机制造商(而非运营商)控制的应用商店分发这些内容，并收取费用。像苹果和谷歌这样的公司越来越多地决定如何提供网络服务。

然而，运营商认为他们可以通过自己的优势避免比特管道的降级:在获得语音和数据等基本服务时，他们与用户有着主要的关系，他们的移动网络在基本供应的基础上为他们内置了许多不同的功能，他们认为这将使他们有机会建立自己的应用商店等。

然而，各种各样的努力取得了很大的成功(T2 和 T3)。现在，通过为第三方创建 API 以更轻松地使用这些工具来开放这些移动网络的转变可以有两种方式:革命性的—电信公司一直非常谨慎地保护他们的网络，现在他们意识到以更现代的方式思考这一点是有价值的，以便为行业带来更多的发展和进步，通过构建在任何地方都可以工作的 API，为开发人员提供一个不那么分散、更具可扩展性的选择，这与人们在旅行时可以在网络之间轻松漫游的方式一致。或者，这可以被解读为一种失败:如果他们无法击败科技巨头，就加入他们。

亚马逊和微软一直在与运营商建立更长期的关系，部分是为了在多年被视为威胁后与他们建立更多的信任。(两家公司过去都向电信公司展示过自己的竞争形象，尽管微软在这方面有更长更复杂的历史，它曾灾难性地收购了诺基亚。)上周，作为今天新闻的先兆， AWS 宣布了一系列新产品，与运营商合作建设和运营移动网络；昨晚，微软为电信公司发布了自己的新服务。

对于 GSMA 的活动来说，可以预见的是，技术合作伙伴们对这笔交易大加赞赏，并对其潜力大加赞赏。

微软董事长兼首席执行官塞特亚·纳德拉在一份声明中表示:“在微软，我们专注于与我们的运营商合作伙伴一起，将分布式计算结构从云端扩展到边缘。“我们期待将 GSMA 开放网关计划引入微软 Azure，为开发者提供支持，并帮助运营商实现 5G 投资的价值。”

AWS 电信业务首席技术专家 Ishwar Parulkar 将此处采取的步骤比作不是由电信公司而是由科技公司启动的下一步进展，特别是 Twilio 多年前通过其开创性的 API 在开发人员中访问 SMS 功能。

“这是关于网络即服务，”他在一次采访中说。早在 2019 年，AWS 就一直致力于 5G 和云服务，并且“其中一个想法(即使在那时)是，如果我们可以在其中添加 API，我们就可以改变体验。”用了将近四年，但那个好像在这里。

就像如今的支付、定位服务和其他服务一样，无论运营商做什么，总会有软件解决方案，所以这是一个谁能首先做到的问题，也是开发者和最终用户最容易采用的问题。

5G consultants Nonvoice 的创始人西蒙·白金汉(Simon Buckingham)在接受采访时表示:“他们知道有很多工作要做，也不一定能成功，但他们必须尝试。”“对于像边缘计算这样的东西，如果你可以开始对服务质量收费，对于已经在基础设施上花费了数百万美元的电信公司，如果你可以为此收取额外费用，这可能会改变游戏规则。问题是，他们是否能够比超级缩放者做得更快、更好，”他这样称呼世界上的科技巨头和初创公司，“还是谷歌、微软和其他公司会抢先一步？”
