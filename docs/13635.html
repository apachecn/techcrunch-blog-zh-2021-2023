简化调试，降低嵌入式系统开发的复杂性

尼古拉斯·拉博撰稿人

Nicolas Rabault

是 Luos 的联合创始人兼首席执行官，Luos 是一家开源初创公司，帮助用户开发可扩展的 edge 和嵌入式分布式软件。

与嵌入式系统开发相关的复杂性正在迅速增加。例如，据估计，在过去的十年里，汽车行业软件项目的平均复杂性增加了 300%。

今天，每一个硬件都是由软件驱动的，大多数硬件都是由多个运行同步应用程序的电子板组成的。设备的功能越来越多，但是增加功能意味着增加开发和调试的复杂度。剑桥大学的一份报告发现，开发人员花费高达 50%的编程时间进行调试。

幸运的是，有一些实用的方法可以降低调试嵌入式系统的复杂性。让我们来看看。

越早越好

只有拥有正确的信息，调试才会有效。

bug 会在产品的整个生命周期中出现:在开发、测试和现场。稍后解决一个 bug 会使成本增加 15 倍，并导致用户沮丧，此外还会带来与更新生产中的嵌入式设备相关的挑战。

然而，在产品开发的早期阶段识别 bug 将允许您跟踪它们，同时根据严重性对它们进行优先级排序。这将允许您在生命周期的后期引入其他依赖项和变量之前进行调试，这使得解决 bug 更容易，成本更低。

管理版本

为了正确地复制一个 bug，您必须能够让设备处于与 bug 第一次出现时完全相同的状态。对于嵌入式设备，当出现问题时，有三个不同的变量需要考虑:

  1. 软件版本。这是每个特性的版本。这适用于您构建的代码以及潜在的依赖项，如导入的库。
  2. 棋盘版。具体来说，就是板子的设计。随着元件的添加/移除或移动，电路板设计也在不断变化。
