Safari 16 终于增加了一个自从 OmniWeb 倒闭以来我一直缺少的功能 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/09/13/ive-missed-this-feature-since-omniweb-was-shuttered-safari-16-just-brought-it-back/

今天，苹果发布了 Safari 16，这是在 Ventura 之前首次发布的一个重要版本。浏览器更新主要关注用户看不到的东西，比如安全性和性能。但是有一个新的面向用户的功能已经在我的愿望清单上近十年了——侧边栏标签。

当然，侧边栏标签并不是一个新想法。微软的 Edge 提供了开箱即用的功能，Chrome 和 Firefox 都有支持该功能的扩展， Safari 也曾经支持过。但是向我介绍这一概念的浏览器是 OmniWeb ，它的实现如此优雅，对我来说，它真的毁了其他浏览器。

20 年前，OmniWeb 有一个侧边栏(界面构建器中的“抽屉”)，忠实地呈现打开的网页的缩略图。它们在后台更新，可以通过拖动来重新排序。当标签太多的时候，你可以把它们折叠成更小的纯文本按钮。你只需点击两次就可以刷新整个堆栈，只需浏览缩略图就可以看到哪些页面已经更新。对于本世纪初痴迷于网络的人来说，这是超级用户的梦想。

图片来源:蒂姆·德·钱特/TechCrunch

对于这些人来说，垂直标签确实是最好的选择。一段时间以来，电脑屏幕的宽度一直大于高度，在窗口的一侧放置标签可以更好地利用空间，让用户可以查看网页的更多内容。此外，当标签数开始以十为单位计数时，垂直列表更容易导航，这是我经常遇到的情况。(截图显示了 OmniWeb 6 中展开和折叠的标签是如何出现的，omni web 6 取消了不推荐使用的 drawer UI 元素，支持侧边栏。)

OmniWeb 可以说是 Mac OS X 的第一款浏览器。在 Internet Explorer 与 Mac OS X 开发者预览版 4 捆绑在一起之前，无畏的测试人员可以使用 OmniWeb 来满足他们的浏览需求。该应用最初是为 OS X 的前身 NeXTSTEP 开发的，1995 年推出了测试版。在 OS X 开发者预览版发布后不久，OmniWeb 的开发者 OmniGroup 移植了这款浏览器。

这款应用是你能得到的最纯粹的 Mac OS X 体验。它是用 Cocoa 写的，Cocoa 是当时新的编程语言，代表了与经典 Mac OS 的彻底决裂。界面元素是可点击的 Aqua 主题，图像和文本使用新操作系统的合成器 Quartz 渲染。图像明亮，文字清晰流畅。哦，它不是由微软制造的，而是一家独立商店，有着生产可靠的 NeXTSTEP 和 Mac OS 软件的悠久历史。对于像我这样的苹果迷来说，这是另一个强大的卖点。

在 Mac OS X 公开发布后的几年里，OmniWeb 和 Internet Explorer 几乎是网页浏览仅有的两种选择。然后微软放弃了 Mac 版 IE，苹果决定进入游戏，于 2003 年 1 月发布了 Safari。

基于开源 KHTML 渲染引擎，Safari 快速灵活，但是它严重缺乏我所期望的强大功能。它有标签，但我发现他们笨重。它还缺少工作区、工具栏搜索定制、同步书签和内容过滤(带 regex ！)，等等。这些年来，我已经习惯了它们，我发现这是不可能改变的。

谢天谢地，有了 OmniWeb 4.5，OmniGroup 决定改用 WebCore，Safari 就是基于 WebCore 的。这给了 OmniWeb 一个新的生命，让它或多或少地保持相关性，直到 21 世纪初。

2009 年，OmniGroup 决定不能继续向 OmniWeb 投入资源，omni web 最初是一款付费应用，后来过渡到免费。Chrome 正在挤进来，而大多数 Mac 用户只是坚持使用电脑自带的 Safari。OmniGroup 已经开始开发另一个主要版本，6.0，虽然它的今天仍在更新作为一个热情的项目，但它对大多数人来说并不是一个真正可行的日常浏览器。在很大程度上，OmniWeb 已经死了。

当我意识到不祥之兆时，我尝试了一系列不同的浏览器，包括 Chrome 和 Firefox，但我对自己的用户体验有点挑剔(如果你说不出来的话)，这两种浏览器都不符合我的预期。在某个时候，我切换到 Safari，依靠一系列的技巧，试图将一些最受欢迎的功能带在身边。它运行良好，但已经不一样了。

直到今天。我觉得我的浏览体验又开始类似于早期的 Mac OS X。去年，标签组开始帮助我驯服 Safari 窗口溢出，垂直标签应该会有进一步的帮助，将标签管理集中在一个地方。在版本 16 中，Safari 仍然不完美——我仍然希望每个页面都有缩略图预览，如果能关闭现在冗余的水平标签栏就太好了——但它比过去几年的任何时候都更接近理想状态。
