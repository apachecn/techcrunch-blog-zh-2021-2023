AWS 宣布数字主权承诺| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/28/aws-announces-digital-sovereignty-pledge/

就在感恩节后在拉斯维加斯举行的年度再发明庆典上，AWS 昨晚宣布其“AWS 数字主权承诺”——在你点击离开之前，让我指出这绝对比平淡无奇的名字所暗示的更重要。随着全球各国纷纷立法规定企业如何以及在哪里保存本地用户的数据，大型云要么必须提供有吸引力的解决方案，要么面临让客户迁移到本地云的风险。微软和谷歌也提供数据治理工具，但在将数字主权作为云战略的核心支柱方面，它们都没有像 AWS 走得那么远。

AWS 负责销售、营销和全球服务的高级副总裁 Matt Garman 指出，让客户控制他们的数据一直是 AWS 的优先事项，但随着法律要求的不断变化和发展，管理所有这些变得越来越复杂。

“在世界上的许多地方，如欧洲，数字主权政策正在迅速发展。客户正面临着难以置信的复杂性，在过去的 18 个月中，许多人告诉我们，他们担心他们将不得不在 AWS 的全部功能和功能有限的主权云解决方案之间做出选择，这可能会妨碍他们的创新、转型和发展能力。他写道:“我们坚信，客户不应该做出这样的选择。

这一承诺的想法是告诉这些客户，AWS 完全致力于在其云中建立一套主权控制和功能。其中一些功能已经存在，包括在 AWS 控制塔中，而其他功能显然仍在开发中。随着 re:Invent 的到来以及之前的这份声明，我们很可能会在本周晚些时候听到更多关于它的消息(或者明年——AWS PR 以神秘的方式工作)。

虽然工具仍在开发中，但承诺已经完成。这里的想法很简单，AWS 承诺确保客户始终能够完全控制其数据在 AWS 中的位置，对数据的访问方式进行可验证的控制，并能够在任何地方对数据进行加密，无论是在传输中、静止还是在内存中。

AWS 还承诺使其云能够抵御网络中断和自然灾害。当然，这并不是什么新鲜事，在某种程度上，该公司在誓言书中做出的其他承诺也不是什么新鲜事。但事实是，AWS 清楚地说明了这一点，表明该公司将此视为一个机会，在争夺利润丰厚的公共部门合同时，将自己与其他云提供商区分开来。但它也清楚地将 it 视为威胁(T2)，因为这些客户越来越多地期待本地云解决方案能够帮助他们应对这些数据主权挑战。

毕竟，大多数公共部门机构不必担心为全球市场提供服务，容器和 Kubernetes 的出现使得转移工作负载变得更加容易。
