凭借 1800 万美元的新资金，EngFlow 希望加快您的构建速度| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/11/15/with-18m-in-new-funding-engflow-wants-to-speed-up-your-builds/amp/

图片来源: EngFlow

早在 2015 年，谷歌就推出了其内部自动化工具的开源端口，用于构建和测试代码。这项服务被称为 Bazel (因为内部工具被称为 Blaze)，它使用 Python 方言来帮助开发者编写规则和宏。如今，从 Adobe 到 Databricks、Dropbox、LinkedIn 和 Redfin 等公司都在使用这种工具，因此，当谷歌 Bazel 的首席开发人员 Ulf Adams 和谷歌 Bazel 的企业客户主管兼 BazelCon 的联合创始人 Helen Altshuler 为他们以 Bazel 为中心的初创公司 EngFlow 筹集新资金时，这可能值得关注。

正如该团队今天宣布的那样，EngFlow 已经从 Tiger Global、Firstminute Capital 和 Andreessen Horowitz 获得了 1800 万美元的 A 轮融资，这些公司还领导了该公司的370 万美元的种子轮。蟑螂实验室首席执行官斯潘塞·金博尔、GitHub 联合创始人汤姆·普雷斯顿-沃纳和雪花首席财务官迈克·斯卡佩里，以及特使创作者马特·克莱恩和 GitHub 工程副总裁雷切尔·波特温也参加了这一轮。

图片制作者名单: EngFlow

EngFlow 将自己描述为“构建加速公司”，帮助其企业客户更有效地构建和测试他们的源代码，支持 Bazel、Chromium 和 Android 平台。

“当我们在谷歌谈论 Bazel 的使用时，一个重要的因素是谷歌已经将 Bazel 与许多开发者服务集成在一起，”亚当斯说。“因此，不仅仅是构建工具，还有远程执行、用户界面、用于分析测试覆盖数据的覆盖运行器、与部署的集成等等。我们与 Bazel 用户交谈了四年，他们正在寻找这些东西。有时候我们说谷歌是一片雪花，谷歌很特别，但是我们经常看到其他公司也想做类似的事情。所以我们认为这是一个机会。”

Altshuler 还强调，越来越多的企业正在创建平台工程团队，旨在将更为固执己见的 CI/CD 方法带到他们的团队中。“这通常是巴泽尔尝试让它为自己的特定公司工作，然后向工程师推介的受众。她指出:“这些平台工程团队是我们的主要客户，但在每个平台工程团队的背后，都有成百上千甚至上万的工程师因此受到了 Bazel 的影响。希望他们不必是 Bazel 的专家，也不要知道所有的内部情况——Bazel 应该只为他们工作。”

形象信用: EngFlow

团队注意到许多新客户正在从 Gradle 或 CMake 迁移到这项服务，因此 EngFlow 最近还雇佣了开源 Gazelle 工具的开发人员杰伊·康罗德来迁移到 Bazel(他也曾是谷歌的员工)也许并不奇怪。

值得强调的是，虽然 Bazel 是这里的一个主要重点，该平台还支持 Chromium 和 Android 平台构建。比如凭借基于 Chromium 的浏览器，Brave 就是一个客户。对 Brave 来说，EngFlow 的分布式构建系统将构建时间从两个小时缩短到了 15 分钟。不过，Bazel 仍然是本文的主要关注点，团队今天还发布了其Bazel involution Analyzer，这是一款新的开源工具，允许开发人员更深入地了解他们的 Bazel 概要文件并优化其构建。

“构建是构建软件最关键的方面之一。安德森霍洛维茨基金的普通合伙人马丁卡萨多表示:“传统上，公司要想做出正确的决策，需要耗费大量时间和成本。”“EngFlow 是改变这一切的领先公司。凭借在 Bazel 的深厚根基和良好的构建，他们共同打造了一款我们有生以来第一次见到的解决方案，它能够应对最复杂的代码基础和大型基础架构环境，并显著节省开发时间和成本。这些不仅仅是文字，EngFlow 在许多垂直市场都拥有非凡的客户吸引力，远远超过我们现阶段通常看到的水平。我们很高兴成为你们的投资者，随着 EngFlow 继续成为领先的建筑公司，我们将加倍努力。”

鉴于采用 Google 开源工具并将其引入企业的流行(你好，Kubernetes)，EngFlow 不是该领域唯一的初创公司也就不足为奇了。例如，YC 支持的 BuildBuddy，T2 也提供了一个以 Bazel 为中心的构建系统。同时，资金雄厚的 Gradle Enterprise 为 Gradle、Maven 和 Bazel 提供支持，作为其企业构建工具的一部分。
