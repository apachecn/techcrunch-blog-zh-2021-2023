一个简单的安全漏洞如何成为大学校园的“万能钥匙”

原文：https://web.archive.org/web/https://techcrunch.com/2022/03/03/cbord-university-digital-locks/

当 Erik Johnson 无法让他的大学的移动学生证应用程序可靠工作时，他试图找到一个变通办法。

这款应用相当重要，因为它让他和大学里的其他学生可以支付餐费，参加活动，甚至打开宿舍、实验室和校园其他设施的门。这款应用名为 GET Mobile，由 CBORD 开发，CBORD 是一家为医院和大学提供门禁和支付系统的技术公司。但约翰逊和许多沮丧地离开该应用程序一星评价的人表示，该应用程序很慢，加载时间太长。一定有更好的方法。

因此，通过分析该应用程序在打开宿舍门的同时的网络数据，约翰逊找到了一种方法来复制网络请求，并通过使用他的 iPhone 上的一个快捷按钮来解锁房门。要使它工作，快捷方式必须首先发送他的精确位置和开门请求，否则他的门打不开。约翰逊表示，作为一项安全措施，学生必须亲自靠近才能使用该应用程序解锁，这被视为一项旨在防止校园意外开门的措施。

成功了，但为什么要止步于此呢？如果他可以在不需要应用程序的情况下打开一扇门，他还能复制什么其他任务呢？

约翰逊不需要到处寻求帮助。CBORD 发布了一个通过其 API 可用的命令列表，可以使用学生的凭证来控制，比如他的凭证。(一个 API 允许两个东西通过互联网相互交流，在这个例子中是一个移动应用程序和一个存储学生数据的大学服务器。)

但是他很快发现了一个问题:API 没有检查学生的证书是否有效。这意味着约翰逊或互联网上的任何人都可以与 API 通信，并接管另一个学生的帐户，而不必知道他们的密码。约翰逊说，API 只检查学生的唯一 id，但警告说，这些有时与大学颁发的学生用户名或学生 ID 号相同，一些学校在其在线学生目录上公开列出，因此不能被视为秘密。

约翰逊将密码漏洞描述为他的大学的“万能钥匙”——至少是打开由 CBORD 控制的大门的钥匙。至于需要靠近一扇门才能开锁，约翰逊说，这个 bug 允许他欺骗 API，让它认为自己实际存在——只需发送回锁本身的近似坐标。

由于这个漏洞是在 API 中发现的，约翰逊说这个漏洞可能会影响其他大学，尽管他没有检查自己是否正确，担心这会超出他的帐户访问权限。相反，他想办法向 CBORD 报告漏洞，但在其网站上找不到专门的安全电子邮件。他拨打了电话支持热线来披露漏洞，但一名支持代表说他们没有安全联系人，并被告知通过他的学校报告漏洞。

假设这个漏洞很容易被利用，如果还没有被利用的话，约翰逊要求 TechCrunch 与 CBORD 分享这个漏洞的细节。

在我们于 2 月 12 日联系该公司后不久，该漏洞就被解决了。在一封电子邮件中，CBORD 首席信息官 Josh Elder 证实，该漏洞现已修复，会话密钥已失效，有效地关闭了任何剩余的未经验证的 API 访问。埃尔德说，CBORD 的客户得到了通知，但埃尔德拒绝与 TechCrunch 分享通信内容。一位安全主管告诉 TechCrunch，他们没有收到任何来自 CBORD 的关于该漏洞的通知，他所在的组织也是 CBORD 的客户。目前还不清楚 CBORD 是否计划通知用户和账户持有人——包括像约翰逊这样的学生。

埃尔德没有质疑约翰逊的发现，但当被问及该公司是否存储日志或有能力检测其 API 的恶意利用时，他拒绝进一步置评。在我们要求与公司发言人交谈以回答我们的进一步问题后，TechCrunch 没有得到回复。

这不是 CBORD 第一次不得不修复可能远程解锁门的漏洞。Wired 在 2009 年报道称，可以拦截一个车门解锁命令并猜测下一个序列号，从而不再需要身份证。
