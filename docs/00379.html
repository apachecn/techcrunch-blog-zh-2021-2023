微软 Azure 希望让在其平台上处理数据变得更容易

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/02/microsoft-azure-wants-to-make-working-with-data-on-its-platform-easier/

在今天的 Ignite 大会上，微软宣布了其 Azure 数据平台的一系列新功能。其中包括其古老的 SQL Server 系列 2022 年版的首次预览，以及 Azure Synapse 分析平台的更新和新工具，这些工具可以更轻松地将 Cassandra 工作负载迁移到云。但这些更新的核心是该平台各种服务之间更深层次的整合——这不是巧合。

正如微软 Azure Data 公司副总裁 Rohan Kumar 告诉我的那样，云中过多的数据服务已经成为客户的一个痛点。“每样东西都有专门设计的服务。选择你最喜欢的任务，你想做的数据，有一个服务，”他说。“我们从客户那里清楚而响亮地听到，这变得非常具有挑战性。首先，有太多的选择，事情几乎每天都在变化。然后，许多客户的痛苦确实来自于将这些东西拼凑在一起。”

从用户的角度来看，他们必须管理支持各种应用程序的运营数据库，但也有基于机器学习的分析、商业智能和预测服务，通常由数据仓库和数据湖提供支持。此外，所有这些数据(通常包括敏感信息)也必须进行治理，以确保符合各种法规，所有这些数据也可能分布在多个云和内部数据中心。

借助 Synapse Analytics、Cosmos DB、Azure Data Lake、view 和其他工具，微软为所有这些用例提供了一个解决方案，但它现在明确专注于将它们捆绑到更容易管理的系统中。

正如 Kumar 所指出的，该团队相信它能够在其数据产品组合中提供一流的服务。“但是，如果你真正看看我们是如何与众不同的——微软的数据[产品]和微软本身的独特之处在于，事实上我们拥有所有这些支柱[……]。对我们来说，这种整合与成为每个支柱领域的最佳产品一样重要，”他说。

当然，说起来容易做起来难，但是 Ignite 今天的一些更新显示了微软的发展方向。例如，SQL Server 2022 现在处于预览阶段，其特点是与 Synapse Link 集成，微软的工具用于将他们的数据库用于分析和事务性工作负载，以及 Azure 权限用于更轻松的数据治理。它还将支持 Azure SQL 托管实例，以实现更深入的云集成和灾难恢复。

库马尔指出，Synapse Link 将成为 Azure 数据集成故事的关键部分。“你可以想象需要多少技术创新。我们必须优化源和目标。因此，Synapse(分析)针对这一点进行了优化，Cosmos DB 针对这一点进行了优化，假以时日，Azure SQL 和其他(数据)存储也将针对这一点进行优化，”他说。

除了这些新功能——以及对 Azure 权限的持续关注，以帮助企业进行数据治理 Azure Data 团队还引入了许多其他新功能。支持将 Apache Cassandra 数据迁移到 Cosmos DB。Azure Advisor 正在获得新的成本控制功能。Azure SQL 托管实例还有其他更新。它是微软 Ignite，所以像往常一样，新功能比你能总结的还要多。但是比这些单独的特性更重要的是重新关注各种产品之间的集成。
