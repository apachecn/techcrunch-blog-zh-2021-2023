英国隐私监管机构保持沉默，因为谷歌对其 Topics API 未能改革广告跟踪的批评不予理睬

原文：https://web.archive.org/web/https://techcrunch.com/2023/01/17/privacy-sandbox-topics-api-criticism/

上周晚些时候，有消息称，谷歌打算无视万维网联盟(W3C)的呼吁，该联盟是一个致力于指导网络标准发展的国际机构，它呼吁重新考虑主题 API:谷歌所谓的隐私沙箱提案中的一个关键广告定位组件，旨在发展 Chrome 支持的广告技术堆栈。

Topics 指的是沙盒提案的广告定位组件，它基于通过浏览器跟踪网络用户的兴趣。

W3C 技术架构组(TAG) 在谷歌去年 3 月要求对 Topics API 进行“早期设计审查”后，提出了一系列担忧—上周写道，其“初步观点”是谷歌提议的 Topics API 未能保护用户免受“不必要的跟踪和分析”，并维持“网络上不适当的监控”的现状。

“我们不希望看到它继续下去，”艾米盖伊补充说，代表标签评论。

标签的采取并不是第一次对主题的悲观评估。浏览器引擎开发商 WebKit 和 Mozilla 最近也对谷歌的做法表示反对——前者警告网络上预先存在的隐私缺陷被用作“新规范和提案中隐私缺陷的借口”,后者认为主题“更有可能降低信息对广告商的有用性，而不是为隐私提供有意义的保护”

如果浏览器之间对主题的支持有限，那么网络用户体验就会出现碎片化的风险——这可能会导致网站试图阻止使用非 Chromium 浏览器的访问者——是该标签关注的另一个问题。

尽管网络基础设施领域对谷歌的做法越来越反对，但英国的隐私监管机构似乎对袖手旁观感到满意，并让谷歌继续进行一项提议，即 W3C 的技术专家警告这种隐私入侵(以及用户代理和透明度失败)有可能持续下去，这种入侵多年来一直使广告技术行业陷入监管(和声誉)困境。在这种情况下，信息委员会办公室(ICO)是一个关键的监督机构，因为在英国竞争和市场管理局(CMA)加入的重大反垄断干预后，它积极参与评估沙盒遵守数据保护法的情况。

当被问及是否担心话题对隐私的影响时，包括根据标签的评估，ICO 花了几天时间考虑这个问题，然后拒绝置评。

监管机构告诉我们，它将继续与谷歌和中国市场监管局合作——这是谷歌去年向竞争监管机构所做承诺的一部分。ICO 的发言人还指出了前英国信息专员就该主题发布的一份 2021 意见(哈！)提出了一系列针对广告技术行业的“原则”和“建议”，包括规定用户可以选择接收广告，而不需要对个人数据进行任何跟踪、分析或处理。发言人表示，该法案列出了目前与这些建议相关的“总体预期”。

但是，对于 W3C 标签对主题的详细批评，ICO 没有做出更令人满意的回应。

与此同时，谷歌发言人证实，谷歌已经向监管机构汇报了相关事宜。在回答关于标签的问题时，该公司还告诉我们:

虽然我们赞赏 TAG 的意见，但我们不同意他们认为主题保持现状的描述。谷歌致力于主题，因为这是对第三方 cookies 的重大隐私改进，我们正在向前迈进。

Topics 支持基于兴趣的广告，使网络保持免费和开放，与第三方 cookies 相比，显著提高了隐私性。在没有可行替代方案的情况下删除第三方 cookies 会伤害出版商，并可能导致更糟糕的方法，如秘密跟踪。许多公司正在积极测试主题和沙盒 API，我们致力于提供工具来提高隐私和支持网络。

此外，谷歌产品管理高级总监品冠周五在推特上(继媒体报道该标签的担忧后)在推特上发布了一份声明的观点的线程版本(在声明中，黄还声称用户可以“轻松控制分享什么主题或关闭它”)——最后规定这家广告技术巨头“100%致力于将这些 API 作为更私人的互联网的基石。”

所以，TL；博士，谷歌不是用来打开话题的。

它在一年前宣布了沙盒的这一组成部分——取代了一个备受批评的早期基于兴趣的广告定位提议，称为 FLoCs(又名联合学习群组)，该提议提议将具有可比兴趣的用户分组到可定位的桶中。

FLoCs 很快就被攻击为一个可怕的想法，批评者认为它可能会放大现有的广告技术问题，如歧视和掠夺性目标。因此，谷歌在消灭絮凝物方面可能没有太多选择——但这样做为它提供了一种方法，通过让公司看起来有所回应，将对其声称的亲隐私广告进化项目的公关头痛变成快速胜利。

问题是，对于谷歌声称的“先进”广告技术提供“更私人的互联网”来说，快速堆积的话题评论看起来也不太好。

根据 Topics 的提议，Chrome(或基于 Chrome 的浏览器)跟踪用户的网络活动，并根据他们在网上看的内容为他们分配兴趣，然后可以与调用 Topics API 的实体共享，以便向他们投放广告。

有一些限制——比如可以分配多少个主题、共享多少个主题、主题存储多长时间等等——但是，从根本上说，该提案要求用户的网络活动受到浏览器的监视，然后浏览器与请求数据的网站共享它推断出的兴趣分类的片段。

正如 TAG 的评估所说，这对于网络用户来说并不是 100%清楚的(并且是可控制的):

提议的主题 API 将浏览器置于与任何可以调用 API 的站点共享来自用户浏览历史的用户信息的位置。这是通过这样一种方式实现的，即用户无法精细控制要显示什么，在什么上下文中，或者向哪一方显示。用户似乎也很难理解到底发生了什么；数据是在幕后收集和发送的，非常不透明。这违背了增强用户控制的原则，我们认为对于任何声称是网络用户代理的软件来说，这都是不恰当的行为。

…

给予网络用户对浏览器设置的访问权，以配置哪些主题可以被观察和发送，以及从/向哪些方发送，将是对此类 API 的必要补充，并且在某种程度上有助于恢复用户的代理，但是这绝不是足够的。人们可能会以他们意想不到的方式变得脆弱，而且毫无察觉。不能指望人们完全理解分类法中的每个可能的主题，因为它与他们的个人情况有关，也不能指望人们完全理解与网站和广告商共享这些数据的即时或连锁效应，也不能指望他们随着个人或全球情况的变化而不断修改他们的浏览器设置。

还有一个风险是，调用 API 的网站能够通过使用其他形式的跟踪(如设备指纹识别)来“丰富”主题收集的每个用户的兴趣数据，从而以跟踪和分析一贯采用的同样具有腐蚀性、反网络用户的方式剥夺网络用户的隐私。

虽然谷歌已经表示，种族或性别等“敏感”类别不能通过主题处理转化为有针对性的兴趣，但这并不能阻止广告商识别他们可以用来瞄准受保护特征的代理类别，正如使用现有的基于跟踪的广告瞄准工具所发生的那样(例如，参见脸书的“种族亲和力”广告瞄准——这导致早在 2016 年就出现了歧视性广告的警告，禁止具有受保护特征的人查看工作或住房广告)。

(标签再次提到了这种风险——进一步指出:“对于一个话题是否‘敏感’没有二元评估。这可能因背景、相关人员的情况以及同一个人随着时间的推移而发生的变化而异。”)

愤世嫉俗者可能会说，关于絮凝物的争议，以及谷歌相当迅速地放弃它，为该公司提供了有用的掩护，以推动主题成为更受欢迎的替代品——而没有吸引与一项提案相同水平的细粒度审查，毕竟，该提案旨在跟踪网络用户——考虑到所有的注意力都已经花在絮凝物上(以及一些监管粉末花在反垄断隐私沙盒的考虑上)。

就像谈判一样，第一个要求可能很离谱——不是因为期望得到清单上的所有东西，而是作为一种扭曲期望的方式，以便在以后获得尽可能多的东西。

谷歌建立一个新的(它声称)“更好地保护隐私”的广告技术栈的高度技术性计划早在 2020 年就正式宣布了——当时它制定了策略，反对在 Chrome 中支持第三方跟踪 cookies，因为竞争对手的浏览器更早采取了反跟踪行动。但该提议遭到了出版商和营销商的大量批评，他们担心这将进一步巩固谷歌在在线广告领域的主导地位。这反过来又吸引了反垄断监管机构的一系列监管审查和摩擦，导致最初的迁移时间表出现一些延迟。

英国在这方面处于领先地位，其 CMA 在不到一年前从科技巨头那里获得了一系列承诺——关于它将如何开发替代的 adtech 堆栈以及何时可以应用任何开关。

这些承诺主要是为了确保谷歌从行业中获得反馈，以解决任何竞争问题。但 CMA 和 ICO 也宣布共同致力于这一监督——考虑到广告定位方式的任何改变对网络用户隐私的明显影响。这意味着，如果网络用户不想继续被“相关广告”的名义所欺骗，竞争和隐私监管机构需要密切合作。

然而，ICO 的广告技术问题是一个棘手的问题。

这是因为它在历史上未能对当代 adtech 系统违反隐私法的行为采取执法行动。因此，尽管监管机构允许侵犯隐私的广告技术公司继续非法处理网络用户的数据，ICO 对谷歌从一开始就标榜为支持隐私的现状的观点可能看起来有点“不着边际”。

结果是，ICO 在如何主动监管谷歌沙盒提议的细节方面陷入了困境。这当然正中谷歌的下怀——因为唯一一个积极关注这种事情的隐私监管机构被迫袖手旁观(或充其量玩弄其拇指),让谷歌塑造话题的叙事并忽视知情的批评——所以你可以说谷歌是以自己的不作为摩擦监管机构的脸。因此，坚定不移地谈论“向前迈进”，对“第三方 cookies 的重大隐私改善。”

“提高”当然是相对的。因此，对于用户来说，现实是，当涉及到决定你将从其人员跟踪业务中获得多少增量隐私时，仍然是谷歌在驾驶座上。向国际奥委会抱怨是没有意义的。
