亚马逊 Lex 为聊天机器人大楼带来自动化通话记录审核

原文：https://web.archive.org/web/http://techcrunch.com/2021/12/01/amazon-lex-brings-automated-call-transcript-review-to-chatbot-building/

亚马逊 Lex 为聊天机器人大楼带来自动化通话记录审查

今天，在拉斯维加斯举行的 AWS re:Invent 上，该公司在预览版中宣布了亚马逊 Lex 自动聊天机器人设计器，这是一个新功能，通过将自动化水平提高到一个新的水平，简化了聊天机器人的培训和设计过程。

“我们很高兴宣布亚马逊 Lex 自动聊天机器人设计器，这是一种新功能，可以将机器人设计从数周缩短到几个小时，”亚马逊人工智能副总裁 Swami Sivasubramanian 在今天的人工智能和机器学习主题演讲中告诉观众。

它通过利用由深度学习技术支持的高级自然语言理解来实现这一点。事实上，他说开发者现在只需点击几下鼠标，就可以创建一个使用历史通话记录设计的基础聊天机器人。

“[亚马逊 Lex]自动聊天机器人设计者通常可以在几个小时内分析 10，000 行文字记录，以确定诸如“提出新索赔”或“检查索赔状态”等意图它确保了这些意图被很好地分开，并且它们之间没有重叠，消除了对试错方法的需要，”他解释道。

他说，如果没有这种自动化，这将是一项高度手工化和乏味的开发任务。“聊天机器人的组织设计非常复杂，需要人工操作，而且容易出错。它需要理解口语和人类互动的细微差别，如果没有这种特定的专业知识，开发人员需要花费数百个小时来梳理所有历史调用记录，以找到常见的用户请求和解决这个问题所需的信息。”

当你想到人工智能的一个常见用例时，你肯定会想到聊天机器人。它们可能是为内部使用而设计的，用于回答有关如何订购新电脑或如何为您的新生儿购买公司健康保险的问题，或者它们可以作为客户服务前端来收集重要信息并回答简单的问题，同时将更复杂的问题传递给人工客户服务代理。

许多创业公司已经推出了更容易创建更准确的聊天机器人，但对于亚马逊这样的公司来说，这也是一个容易实现的结果，其客户可能正在该平台上寻找一个解决方案，以配合他们的其他人工智能和机器学习项目。

亚马逊 Lex 自动聊天生成器从今天开始提供预览。开发人员可以在预览阶段免费使用这一功能，但一旦正式发布，将根据该工具分析脚本和识别意图所需的时间收费。
