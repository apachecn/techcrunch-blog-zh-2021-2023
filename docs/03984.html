Signadot 向开发者承诺更快的反馈循环——TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/23/signadot-promises-developers-faster-feedback-loops/amp/

Signadot ，一家旨在通过让开发者更容易测试他们的代码来简化基于微服务的应用程序的开发过程的初创公司，今天宣布它已经筹集了 400 万美元的种子资金。这轮融资由红点风险投资公司(Redpoint Ventures)牵头，Y Combinator(该公司是【2020 年冬季批次的一部分)以及一批天使投资人参与，其中包括前 Heroku 首席执行官亚当·格罗斯(Adam Gross)、前 GitHub 首席技术官杰森·华纳(Jason Warner)、GitLab 的塞巴斯蒂安·帕尔(Sébastien Pahl)和 LaunchDarkly 联合创始人兼首席执行官约翰·科杜马尔(John Kodumal)。

该公司今天还宣布，其服务现已成为公开测试版。

Signadot 的核心是一个基于 Kubernetes 的平台，它为开发人员提供了一个类似生产的环境，以便在代码进入登台环境之前对其进行测试。这允许开发人员在共享的阶段环境中测试他们的代码，而不会对其他开发人员产生任何潜在的影响。例如，所有这些环境(Signadot 称之为“沙箱”)都可以在几秒钟内针对每个拉取请求或提交进行加速和减速，从而显著加快反馈循环。

该公司由 Arjun Iyer(首席执行官)和 Anirudh Ramanathan(首席技术官)创建，前者曾建立 AppDynamics 的数据科学平台，后者曾是谷歌的软件工程师，从事 Kubernetes 的工作。

Signadot 联合创始人兼首席执行官 Arjun Iyer 和首席技术官 Anirudh Ramanathan。图片来源:签名

“对我来说，这个故事实际上是从 AppDynamics 开始的，”Iyer 告诉我。“我在那里管理着一个大型工程团队，我们构建了云原生的一切，并最终将一切转移到微服务和 Kubernetes。我见证了整个转变。我意识到我的开发人员花了很多时间才发现问题。他们只能在生产前发现问题，甚至在最坏的情况下，在生产中发现问题。[……]开发人员经历了漫长的反馈循环，并且很晚才发现问题，这真的损害了我们快速发布软件的能力。”

图片致谢:签名

但与此同时，开发人员不希望花费大量时间构建和拆除测试环境，因此 Signadot 的承诺是，该服务将允许他们在隔离的环境中快速运行针对真实服务的集成测试。Iyer 还指出，这里的部分想法是确保生产环境不是开发者无法触及的围墙花园。相反，Signadot 希望为开发人员带来一个非常类似生产的环境。

这些沙盒环境也是 Signadot 定价模型的基础。客户将根据每月使用和创建的沙盒数量支付订阅费。

在私有 alpha 阶段，Signadot 与许多非常大的客户合作，以确保其服务可以扩展。不过，这项服务应该对小公司同样有用，因为很少有公司会喜欢较慢的开发过程。

与现阶段的大多数公司一样，Signadot 计划利用新的资金来组建团队和扩展平台。

“众所周知，软件应用的未来完全在云端，”红点风险投资公司的合伙人托马斯·东古兹说。“随着云原生变得越来越普遍，对它的需求也在增长，开发人员将需要他们所能获得的所有帮助来简化他们的过程，并在不牺牲质量的情况下快速构建软件。Arjun 和 Anirudh 与 Signadot 一起创造了完美的解决方案，我想不出还有谁比他们更有资格解决这个问题。我毫不怀疑这只是公司旅程的开始。”
