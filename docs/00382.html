微软推出 Azure Container Apps，一种新的无服务器容器服务

原文：https://web.archive.org/web/https://techcrunch.com/2021/11/02/microsoft-launches-azure-container-apps-a-new-serverless-container-service/amp/

图片来源:放大镜/ Getty Images

在 Ignite 大会上，微软今天宣布了 Azure Container Apps 的预览版，这是一种新的完全托管的无服务器容器服务，补充了该公司现有的容器基础设施服务，如 Azure Kubernetes 服务(AKS)。微软指出，Azure Container Apps 是专门为微服务打造的，能够根据 HTTP 流量、事件或长期运行的后台作业快速扩展。

在许多方面，它可能最像 AWS App Runner ，这是亚马逊无服务器容器服务的一个小舰队，App Runner 也专门专注于微服务。与此同时，谷歌也提供一套以容器为中心的服务，包括 Cloud Run ，它是一个运行基于容器的应用程序的无服务器平台。

微软表示，有了 Azure Container 应用程序，开发者将能够用他们选择的语言和框架构建他们的应用程序，然后在这项新服务的帮助下部署它。基础设施本身位于开源项目之上，如微软自己的 Dapr 应用运行时，其扩展技术由 Kubernetes 事件驱动自动扩展 (KEDA)提供支持，该项目由微软、Red Hat 和 Codit 以及 Vexxhost 和 Snyk 支持。

“我认为 Azure 容器应用程序的体验更像是 AKS 的通行证式体验，也就是你的 IaaS，”微软 Azure Edge 和平台公司副总裁 Roanne Sones 解释道。“在这个世界上，客户从 IaaS 开始，最终，如果他们可以直接转向 PaaS，他们可能会这样做。因为为什么不呢？我所要做的就是考虑我的应用程序或者使用它——这已经为我完成了。这就是我对爬梯的看法。如果 AKS 是你为运行符合 CNCF 标准的 Kubernetes 服务的客户提供的底层基础设施服务，那么 Azure Container Apps 位于其上，它抽象出更多的基础设施，所以你不必进入那个级别的设计和生命周期管理。”

毫无疑问，所有主要的云提供商现在都提供各种各样的无服务器容器服务，既面向需要完全控制其部署的复杂运营团队，也面向希望其他人为他们处理所有这一切的公司。尽管有太多的工具来简化这一过程，但毕竟管理 Kubernetes 集群仍然是基础设施团队的全职工作。容器的承诺一直是能够根据需要轻松地扩大和缩小服务，并使开发人员不必考虑他们的代码将在其上运行的基础设施。对于许多只想将他们的服务投入生产的团队来说，无服务器是一个不错的选择。
