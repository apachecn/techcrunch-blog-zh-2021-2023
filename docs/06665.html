DigitalOcean 推出基于 Nimbella 的无服务器产品

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/24/digitalocean-launches-serverless-product-based-on-last-years-nimbella-acquisition/

DigitalOcean 在去年收购 Nimbella 的基础上推出无服务器产品

当 DigitialOcean 去年收购 Nimbella】时，你知道它打算更深入地研究无服务器技术。果然，该公司今天宣布了一项基于 Nimbella 技术的新功能，名为数字海洋功能。

DigitalOcean 以低于三大巨头的价格点提供云基础设施服务，这使其受到开发者的欢迎。DigitalOcean 的首席产品官 Gabe Monroy 表示，客户一直要求更多的无服务器功能。

“DigitalOcean 去年收购了 Nimbella，并一直致力于为我们的平台添加无服务器和无服务器功能。这是我们的客户提出的最重要的问题之一，我认为推动这一点的一个因素是开发人员非常希望直接从代码转向云，而完全不用担心基础设施，”Monroy 解释道。

无服务器计算使开发人员无需提供任何基础设施就能编写程序。当然，服务器仍然存在，但是云提供商提供运行该功能所需的任何资源，仅此而已。

该公司今天宣布的功能特性为开发人员提供了一个开发无服务器功能的编程框架。Monroy 说他们从 Jamstack APIs 开始，因为这是开发人员开始函数编程的简单方法。“无服务器编程可能有点难以进入，因为事件驱动的编程模型可能很复杂，”他说。

“幸运的是，API 和 HTTP 用例也是事件。这使得 API 成为一个非常好的起点，因为开发人员不需要学习很多新的编程结构就可以高效地使用无服务器，这正是我们从 API 开始的原因。”

但是 Monroy 说，这不是一个非此即彼的命题，你必须用无服务器功能来编程整个应用程序。“开发人员可以在我们的平台上运行容器中运行的 Django 应用程序或 Ruby on Rails 应用程序，然后在该应用程序旁边运行一些面向功能的 API，连接到他们需要的相同数据存储。”

自动化供应的一个问题是，你必须关注成本，但 Monroy 说，他们在设计计费时考虑到了这一点。“从定价的角度来看，我们试图通过提供更简单的定价模型来实现差异化，这种模型不包括请求或调用。所以人们最终会更容易计算。”

从今天开始，DigitalOcean 功能将在所有地区全面推出。
