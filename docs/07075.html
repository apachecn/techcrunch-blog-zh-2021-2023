GitHub 于 2011 年推出了软件开发环境 Atom

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/08/github-sunsets-atom-the-software-dev-environment-it-launched-in-2011/amp/

图片来源: GitHub

GitHub 今天宣布将终止 Atom，该公司在 2011 年推出的用于软件开发的文本编辑器。在一篇博客文章中，GitHub 表示，它将在 2022 年 12 月 15 日对 Atom 仓库和 Atom 组织中剩余的所有其他仓库进行归档。

Atom 充当了电子框架的基础，为成千上万的应用铺平了道路，包括微软 Visual Studio Code、Slack、GitHub 自己的 GitHub 桌面。但是 GitHub 声称，随着新工具的出现，Atom 社区的参与度已经下降。除了维护和安全更新之外，Atom 本身在过去几个月中没有出现重大的特性开发。

GitHub 在今天早上发布的一篇博客中写道:“当我们在 2011 年推出 Atom 时，我们开始为开发人员提供一个可深度定制但易于使用的文本编辑器——一个让更多人可以构建软件的编辑器。”。“可靠性、安全性和性能是 GitHub 的核心，为了更好地服务于开发人员社区，我们正在对 Atom 进行归档，以优先考虑支持未来软件开发的技术。”

图片来源: GitHub

GitHub 表示，它将重新关注微软 Visual Studio Code (VS Code)和 GitHub Codespaces，这是一个云驱动的开发环境。“我们认识到社区仍在使用 Atom，并希望承认迁移到替代解决方案需要时间和精力，”该公司继续说道。

奇怪的是，2015 年推出的 VS Code 在某种程度上是对 Atom 的回答。微软在 2018 年收购 GitHub，将 Atom 和 VS 代码置于一个屋檐下，蚕食了后者。但是 VS Code 的受欢迎程度持续增长。根据 Stack Overflow 的 2021 年开发者调查，只有 13%的开发者使用 Atom 作为他们的主要环境；71%的人使用 VS 代码。

这不一定是 Atom 的末日。一旦归档，代码将可供开发人员检查和构建。该项目的核心贡献者之一 Max Brunsfeld 正在领导一项名为 Zed 的精神继承者的工作，该项目将于本周在 private alpha 上发布。
