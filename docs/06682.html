微软用新功能扩展 Azure OpenAI 服务

原文：https://web.archive.org/web/https://techcrunch.com/2022/05/24/microsoft-expands-azure-openai-service-with-fine-tuning-features-and-more/

去年年底，微软宣布推出 Azure OpenAI 服务，这是一个完全托管的、以企业为中心的产品，旨在让企业访问人工智能实验室 OpenAI 的技术，并增加治理功能。最初作为 Azure 认知服务的一部分，该服务允许通过 Azure 平台访问 OpenAI 的 API，用于语言翻译和文本自动完成等应用程序。

现在，在 Build 的同时，在 OpenAI 新创新的推动下，微软正在扩展 Azure OpenAI 服务，以支持更多的用例。该服务处于有限访问预览阶段，这意味着想要使用它的客户可以申请预付费访问。此前，Azure OpenAI 服务只能通过邀请获得。

微软人工智能平台公司副总裁埃里克·博伊德(Eric Boyd)在接受 TechCrunch 电话采访时表示:“我们希望帮助客户从实验阶段转向将这项技术作为其业务的核心部分，以实现真正的增长。”。

完全管理的模型

由 OpenAI 构建的支持 Azure OpenAI 服务的模型，包括 GPT-3 ，可以定制来执行任务，从将自然语言翻译成代码到生成问题的答案。GPT-3 自 2020 年年中以来一直通过 OpenAI 的 API 公开提供。但是 Azure OpenAI 服务在 API 没有的模型之上添加了企业定制的层，包括更大的扩展能力、私有网络和访问管理。

例如，出于合规性原因，公司可以使用 Azure OpenAI 服务在特定地理区域运行模型，或者集中管理 API 端点并使用客户提供的密钥进行加密。Azure OpenAI 服务表面上还让现有 Azure 客户的账单更方便，因为它在单一账单下对模型使用收费，而不是在 OpenAI API 下单独收费。

Boyd 说，虽然客户以前可以使用他们自己的数据中的例子来微调 Azure OpenAI 服务中的模型，但微调功能正在随着服务的更新而扩展。“你可以使用标准的 OpenAI，在真实结果上获得更高的效率和效果，”他补充道。“我们(从 OpenAI)带来了全套模型……这样公司就不必从零开始才能获得真正准确的结果。”

这些改进部分归功于 InstructGPT，这是 OpenAI 最近开发的一系列基于 GPT 3 的模型，它们不太可能生成有问题的文本，同时更接近用户的意图。由于人工智能系统的开发方式，OpenAI 开发的早期版本的模型被发现会产生有毒的文本，如将“淘气”或“吮吸”等词放在女性代词附近，将“伊斯兰”放在“恐怖主义”等词附近。虽然 InstructGPT 并不完美，但它在 OpenAI 内部测试中的表现足以让该公司将其作为 OpenAI API 中的默认语言生成模型。

Azure OpenAI 服务中的另一个新模型集合，称为嵌入模型，在文本相似性、文本搜索和代码搜索等任务上表现良好。文本相似性捕获文本片段之间的语义相似性，而文本和代码搜索在文件中查找信息以满足一组标准。

Boyd 说，Azure OpenAI 服务的客户 CarMax 使用这些模型来整理和总结其广阔市场上的汽车评论。摘要突出了关于个人汽车的热门话题，提出了具体问题的“权威答案”。

与此同时，总部位于新西兰的农村供应合作农场利用 Azure OpenAI 服务来总结客户互动，将他们分为“中性”、“负面”或“正面”，并提取关键词。Farmlands 还在为其电子商务商店测试聊天机器人和人工智能生成的产品描述，两者都由 Azure OpenAI 服务提供支持。

除了增强的微调和新模型，Azure OpenAI 服务现在还提供对 Codex 的访问，它可以根据自然语言提示生成代码。(Codex 于去年 8 月在 OpenAI API 中推出。)Codex 可用于编程任务，包括跨 Python、JavaScript、Go、Perl、PHP、Ruby、Swift 和 TypeScript 等语言的编译、解释代码和重构代码。

Boyd 说，他见过客户使用 Codex 与《我的世界》API 直接对话的例子，以及其他应用程序。

最后，Azure OpenAI 服务有一个新的“负责任的人工智能”系统，旨在过滤掉与性、暴力、仇恨和自残有关的内容。该系统试图检测模型用户的滥用或伤害模式，当发现时，会提示专门的微软团队与客户合作，调查并在必要时阻止滥用。随着新形式的仇恨言论和诋毁言论的出现，该团队还负责更新内容过滤器。

“Azure OpenAI Services 中新的负责任的 AI 系统包括自动内容过滤，为客户提供更高质量的内容。对于担心产生有害内容的 API，默认情况下它是打开的。客户不能选择退出，”微软发言人通过电子邮件澄清道。“内容过滤符合微软管理 Azure OpenAI 服务使用的内容政策。如今，内容过滤器屏蔽四类内容:性/成人、暴力、仇恨和自残。这些过滤器中的每一个都基于专门为该主题开发的机器学习模型。为了帮助确保 Azure OpenAI 服务用于预期目的，负责的 AI 系统还会检查可能与系统滥用有关的内容，如虚假信息生成。这些过滤器是根据 Bing 威胁情报开发的，生成的警报会发送给一个人工审查团队进行调查。”

有限访问

根据 Boyd 的说法，Azure OpenAI 服务的接受度很高，有一些财富 500 强公司在使用这项服务。当被问及更广泛的可用性时，Boyd 说，在可预见的未来，Azure OpenAI 服务将仍然是有限的。

Boyd 说，只有计划实现“定义良好的”用例的公司才能注册这项服务，他们需要证明他们将以“可靠”的方式使用这些模型。微软正在提供用户体验设计指南和模式，以及描述 Azure OpenAI 服务的限制、预期用途和特征的“透明说明”。

“设计指南基于我们的 HAX 工具包，但经过定制，为生成式人工智能应用提供了更具体的指导，”发言人继续说道。“Azure OpenAI 服务的透明度说明将很快发布。它将类似于 Azure 认知服务的其他透明说明，在这里找到。透明笔记是微软将我们的人工智能原则付诸实践的更广泛努力的一部分。”

展望未来，Boyd 表示，微软将继续与 OpenAI 密切合作，开发并找出实验室技术商业化的方法。微软与 OpenAI 保持着密切的关系，已经在 2020 年向该实验室投资了 10 亿美元，并独家授权了 GPT-3。微软在 Azure 中维护一台“人工智能超级计算机”，OpenAI 用它来训练其模型，该公司已经将 GPT-3 和 Codex 集成到它的几项服务中，包括 Power Apps 和 GitHub 的 Copilot 。

Boyd 说:“我们肯定会和 OpenAI 谈，OpenAI 也和客户保持联系。”
