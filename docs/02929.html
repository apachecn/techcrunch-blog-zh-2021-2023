GitHub 推出赞助商专用库 TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/02/02/github-introduces-sponsor-only-repositories/

几年前，GitHub 引入了赞助，允许任何人向开源开发者直接捐款。今天，推出了赞助商专用库，也就是说，只有赞助商才能访问的私人库，这让这个概念更进了一步。

在某些方面，这非常符合当前的趋势。毕竟，Twitch、SubStack 和类似的平台也通过向赞助商提供额外的东西来激励订阅。然而，在开源世界中，这并不是一件真正的事情，有些人可能认为这与其整体精神背道而驰。

GitHub 表示，这里的想法是让资助者在项目正在建设时尽早访问项目，或者访问该公司所谓的“赞助商软件”，即只为赞助商访问项目。该公司指出，开发者也可以使用这些存储库来主持与赞助商的讨论。为了给开发者一些灵活性，他们可以将特定的存储库附加到不同的赞助层。

图片来源: GitHub

除了赞助商专用库，GitHub 还对程序做了一些其他的小改动。例如，开发者现在可以设置最低的自定义赞助金额，并且他们可以为每个赞助层级编写自定义的欢迎信息。该公司还向赞助商支持的存储库添加了新的行动呼吁，以提高该计划的可见性。

GitHub 的赞助商产品负责人 Jessica Lord 在今天的公告中写道:“敬请关注未来的工作，以改善 GitHub 上的发现体验，使社区更容易探索依赖关系并决定支持谁，并帮助使用赞助商的维护者增加他们的受众、社区和整体资金。”。
