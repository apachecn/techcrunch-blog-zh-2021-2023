谷歌云允许开发者访问其基础模型

原文：https://web.archive.org/web/https://techcrunch.com/2023/03/14/google-cloud-gives-developers-access-to-its-foundation-models/

谷歌云今天宣布为其生产力工具推出一系列新的人工智能功能，但该公司今天也为开发者推出了一系列新的 API 和工具，同样有趣——如果不是更有趣的话。除了通过 API 向开发人员提供大型语言模型，谷歌今天还推出了 MakerSuite，这是一款基于浏览器的新工具，将使开发人员更容易在谷歌的基础模型上构建人工智能应用程序。

谷歌还将为 Vertex AI 带来对生成式人工智能的支持，这是其构建和部署 ML 模型的平台，并推出了其生成式人工智能应用构建器，这是一种新的服务，将帮助开发者提供机器人、聊天界面、数字助理和定制搜索引擎。

这是我们第一次采用新的生成式人工智能模型，并通过 API 让开发者社区直接访问它们。谷歌的托马斯·库里安

就像其在 Workspace 中的新人工智能工具一样，这些新功能将首先向有限的开发人员推出，并将只在谷歌的北美数据中心提供。一些已经在使用这些工具的客户包括丰田、梅奥诊所和德意志银行。

这里有很多东西要解开，所以让我们从头开始。PaLM API 是今天发布的核心。虽然谷歌长期致力于 PaLM 模型，但该公司将 PaLM API 描述为谷歌访问其大型语言模型的门户。“这是我们第一次采用新的生成式人工智能模型，并通过 API 让开发者社区直接访问它们，”谷歌云首席执行官托马斯·库里安解释道。

Kurian 将新的 API 描述为“开发者开始用生成式人工智能构建的一种极其平易近人的方式。”他指出，新的 API 将使开发人员能够访问这些基础模型，但也允许开发人员用自己的数据调整和增强这些模型，并用合成数据增强他们的数据集，“以负责任和安全地构建应用程序，使用谷歌的基础设施扩展应用程序的服务或推理，并生成最先进的嵌入。”

图片来源:谷歌

谷歌表示，从今天开始，它将推出一款“在尺寸和功能方面有效的型号”，并将很快增加其他型号和尺寸。为什么 Google 没有为这个 API 使用一个比“PaLM”更通用的名称是任何人都无法猜测的，但是该公司确实通过这个 API 为多轮对话和单轮通用用例(如文本摘要和分类)提供了 PaLM 模型。一位公司发言人告诉我，谷歌选择 PaLM 作为第一个版本，“因为它特别适合聊天和文本用例。”其他车型的可能候选人是 LaMDA 和 MUM。

对于不想钻研 API 的开发者，Google 推出了低代码 MakerSuite 服务。这项服务也将只对可信的测试人员开放，并将向这些开发人员提供两种模型:PaLM chat-bison-001 和 PaLM text-bison-001。PaLM chat 将成为构建聊天风格的多回合应用程序的工具，而 PaLM text 则是用于单回合输入/输出场景。

这里的想法是让开发人员给工具提供一些例子，告诉它他们在寻找什么样的结果，然后测试这些例子，并以代码形式提供它们，但谷歌没有提供关于这项服务在实践中具体如何工作的细节。

图片来源:谷歌

不过，该公司确实花了很多时间谈论 Vertex AI 和生成式 AI 应用程序生成器。对于 Vertex AI，谷歌一直采取平台的方式，为开发者提供端到端的服务，以构建他们的 AI 模型和应用。现在，这包括访问生成文本和图像的基础模型。Kurian 说，随着时间的推移，音频和视频也会加入进来。“这里的想法是让开发人员给模型提供一些例子，然后快速测试这些例子，并使它们作为代码可用，”该公司解释说。

图片来源:谷歌

生成式人工智能应用生成器是一个全新的服务。它将允许开发人员基于自己的数据构建人工智能驱动的聊天界面和数字助理。“Generative AI Application Builder 是一个快速的应用程序开发环境，旨在允许业务用户-不一定只是开发人员-而是允许业务用户与开发人员协同工作，以利用搜索、对话体验和基础模型的力量，同时尊重企业控制，”Kurian 解释道。

为了做到这一点，谷歌将其基础模型与其企业搜索功能和对话人工智能相结合，以建立单回合和多回合对话。Kurian 指出，这可以用来检索信息，但也可以——通过正确地连接到公司的 API——来进行交易。他强调，用户将获得对生成流程的控制权。他们可以选择让大型语言模型控制这个流程，或者使用一个更确定的流程(可能在客户服务场景中)，这样就不会有模型偏离轨道的风险。

在整个声明中，谷歌强调，公司的训练数据将始终保持隐私，不会用于训练更广泛的模型。这里的重点显然也是业务用户，他们希望用自己的数据扩充模型和/或针对他们的用例进行调整。

图片来源:谷歌

我们今天没有看到的一件事是谷歌最知名的车型 LaMDA 的公开发布。有趣的是，谷歌将 PaLM 模型作为这些服务的基础。它最早于一年前宣布掌。当时，该公司指出，负责该模型的谷歌研究团队正在寻求建立一个可以“在高效的同时跨领域和任务通用化”的模型。它拥有 5400 亿个参数，比 OpenAI 拥有 1750 亿个参数的 GPT3 大得多。

这在实践中意味着什么，尤其是 GPT 3.5 已经上市，GPT4 可能很快就会推出，还有待观察，尽管一年前，谷歌表示，PaLM 通常在数学问题上优于 GPT 3，这是大型语言模型不一定擅长的。虽然这不是谷歌在这些当前产品中使用 PaLM 的重点，但当时该公司还表示，PaLM 已经“在单一模型中表现出跨编码任务和自然语言任务的强大性能，尽管它在预训练数据集中只有 5%的代码。”
