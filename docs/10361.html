OpenAI 开源 Whisper，一个多语言语音识别系统| TechCrunch

原文：https://web.archive.org/web/https://techcrunch.com/2022/09/21/openai-open-sources-whisper-a-multilingual-speech-recognition-system/amp/

图片来源: OpenAI

语音识别在人工智能和机器学习中仍然是一个具有挑战性的问题。作为解决这个问题的一步，OpenAI today 开源了自动语音识别系统 Whisper，该公司声称该系统能够在多种语言中实现“强大的”转录，以及从这些语言翻译成英语。

无数组织已经开发了非常强大的语音识别系统，这些系统位于谷歌、亚马逊和 Meta 等科技巨头的软件和服务的核心。但 OpenAI 表示，Whisper 的不同之处在于，它接受了从网络上收集的 68 万小时多语言和“多任务”数据的训练，从而提高了对独特口音、背景噪音和技术术语的识别能力。

“[耳语]模型的主要预期用户是研究当前模型的鲁棒性、泛化、能力、偏差和约束的人工智能研究人员。然而，Whisper 也有可能成为开发人员的自动语音识别解决方案，特别是英语语音识别，”OpenAI 在 GitHub repo 中写道。“[这些模型]在大约 10 种语言中显示出很强的 ASR 结果。如果在语音活动检测、说话人分类或说话人双音化等特定任务上进行微调，它们可能会显示出额外的功能，但尚未在这些领域进行稳健的评估。”

Whisper 有其局限性，尤其是在文本预测领域。由于该系统是在大量“嘈杂”数据的基础上训练的，OpenAI 警告说，Whisper 可能会在其转录中包含实际上没有说出的单词——可能是因为它既试图预测音频中的下一个单词，又试图转录音频本身。此外，Whisper 在不同语言之间的表现并不相同，当涉及到在训练数据中没有得到很好代表的语言时，错误率更高。

不幸的是，最后一点对语音识别领域来说并不新鲜。偏见长期以来一直困扰着最好的系统，2020 年斯坦福大学的一项研究发现，亚马逊、苹果、谷歌、IBM 和微软的系统在白人用户比黑人用户的情况下犯的错误要少得多，约为 19%。

尽管如此，OpenAI 认为 Whisper 的转录功能正被用来改善现有的辅助工具。

该公司在 GitHub 上继续说道:“虽然 Whisper 模型不能用于即时转录，但它们的速度和大小表明，其他人可能能够在它们的基础上建立允许近实时语音识别和翻译的应用程序。”“基于 Whisper 模型构建的有益应用的实际价值表明，这些模型的不同性能可能具有实际的经济意义……[W]我们希望该技术将主要用于有益的目的，使自动语音识别技术更容易使用可以使更多的参与者建立强大的监控技术或扩大现有的监控工作，因为速度和准确性允许对大量音频通信进行可负担的自动转录和翻译。”

Whisper 的发布并不一定预示着 OpenAI 的未来计划。虽然越来越关注像 DALL-E 2 和GPT 3这样的商业努力，但该公司正在追求几个纯理论的研究思路，包括通过观察视频学习的人工智能系统。
