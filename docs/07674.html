Flagright 提供基于使用定价的交易监控

原文：https://web.archive.org/web/https://techcrunch.com/2022/06/28/flagright-offers-transaction-monitoring-with-usage-based-pricing/

初创公司面临许多挑战，但其中一个最大的挑战是必须应对每个司法管辖区不同的监管要求。尤其是在财务法规遵从性方面，实施解决方案并不容易，也不便宜。埃森哲最近的一项调查发现，90%的公司预计未来两年内不断变化的业务、监管和客户需求将使其合规成本增加 30%。

巴兰·厄扎坎告诉 TechCrunch，另一个障碍是创始人并不总能轻易获得信息，这导致了知识缺口。厄扎坎是 Flagright 的创始人，这是一家旨在防止洗钱或恐怖融资等金融犯罪的初创公司，其产品以 API 为先。

“我曾经在立陶宛的一家金融科技初创公司负责产品团队，我的职责之一就是合规和欺诈防范，”厄扎坎在接受电子邮件采访时表示。“我花了大约 15 个月的时间寻找一个实时解决方案，与现有提供商的合作经历非常糟糕。有些太贵了，我们的启动预算很少。有些没有直观用户界面的良好操作工具。有些人非常注重业务，在 API 和开发人员经验方面非常不成熟。。。总的来说，他们都专注于大型企业交易，没有一个人真正关心初创公司。”

厄扎坎对现有财务合规软件不太满意的体验促使他和 Madhu G. Nadig 共同推出了 Flagright。此前，Nadig 是 Palantir 印度办事处的首批雇员之一，并在 Amazon Web Services 构建无代码自动化部署产品。

Flagright 提供交易监控技术，支持基于风险的阈值和客户细分。从一个控制台，公司可以根据客户档案为不同的规则设置阈值和行动，以做出“基于风险的”决策，并且在理论上最大限度地减少安全团队所需的工作。

“我们正从教育的角度来看待我们的无代码控制台，用户可以利用预先配置的模板，而不必聘请合规专家，”厄扎坎说。“我们提供许多资源来帮助初创公司起步，并以交钥匙解决方案的形式提供平台。你只需整合，点击“激活”，它就能工作。”

Flagright 的产品由在个人客户数据“竖井”中训练的人工智能模型支撑。厄扎坎说，他和 Nadig 做出了有意识的决定，不使用一个客户的数据来训练用于其他客户的模型。Flagright 还选择不采用完全自动化的交易监控解决方案，而是将人工智能与“人类输入”相结合——即调查和外部犯罪预防工作。

“我们的客户群告诉我们，人工智能本身对日常运营弊大于利，”厄扎坎说。“合规需要可解释性和一定程度的机械监督。最终，董事会和高管都要承担责任。”

Flagright 的定价是基于使用情况的，这是一个迄今为止运行良好的模式。在与 Unit21、Sardine 和 Sentinels 争夺市场份额的竞争中，一家小公司 Flagright(总共有 9 名员工)声称，它已经成功收购了 8 家企业客户。都是创业公司。

“疫情并不是金融犯罪预防业务的负面因素，因为包括数字支付在内的在线交易已经大幅增长。这推动了对反洗钱、反恐怖融资和反欺诈解决方案的更高需求，”厄扎坎说。“Flagright 在支出方面相当自律。我们目前没有任何生存担忧，而是专注于将我们的产品组合推向更高水平。”

根据厄扎坎的说法，Flagright 已经筹集了总计 320 万美元的风险资本，其中大部分将用于推出客户风险评分、金融科技许可证自动化、制裁筛选和“加密筛选”功能。Y Combinator 和 Moonfire 是这家初创公司的投资者之一。
